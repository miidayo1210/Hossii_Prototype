## 2. 解決したい課題・背景

- 多拠点に分かれた社員同士のコミュニケーションが希薄
- Slackなどのテキストチャットとは異なる「やわらかい場」が社内にない
- 社内環境をもっとクリエイティブ・文化的にしたい
- 社員の気持ちや「今日のひとこと」が見えない

---

## 3. 体験ゴール

### 参加者（社員）にとって

- 業務連絡とは別の、「人としての気持ち」を置ける場がある
- 別拠点の同僚の日常・感情が見えて、親近感が生まれる
- 投稿することで、自分が会社という場に貢献している感覚が生まれる

### クライアント・主催者（人事部）にとって

- 社内の雰囲気・エンゲージメントが可視化される
- 拠点を超えたゆるやかな連帯感が育まれる
- 会議やSlackでは見えない社員の感情・状態が把握できる

### Hossiiとして検証したいこと

- 社内コミュニケーションというクローズドな環境でのHossii利用
- 「業務ツール」ではなく「場の文化」として定着するか
- 多拠点を一つのスペースでつなぐ設計が機能するか

## 4. ベーススペースからの差分

> ベーススペース: [plaza-space.md](../spaces/plaza-space.md)

### 追加・変更する機能

- **多拠点対応**: 投稿に拠点タグ（東京/大阪/つくばなど）を付けられる設計
- **部署フィルター（将来）**: 全社 / 部署別 / 拠点別で表示を切り替えられる
- **社内限定公開**: 社外からアクセスできない認証付きスペース
- **クリエイティブ表現の強調**: 写真・お絵描き投稿を積極的に推奨

### 使わない機能

- ゲスト入室フロー: 社員のみ（メール認証 or 会社ドメイン制限）
- 公開ボードモード: 社内クローズド

### Feature Flag 設定

| Flag Key | ON/OFF | 理由 |
|----------|--------|------|
| `commentPost` | ON | 一言・ひとこと |
| `emotionPost` | ON | 今日の気持ち |
| `photoPost` | ON | 仕事の景色・アイデアメモ |
| `numberPost` | OFF（検討） | 気分スコアとして将来検討 |
| `random_recall_enabled` | ON | 過去の社員の声が蘇る |
| `public_board_mode` | OFF | 社内限定 |
| `comments_thumbnail` | ON | 写真表示 |

### カスタム感情セット

- `fun`（楽しい）・`inspire`（アイデア）・`joy`（うれしい）・`think`（気になる）・`wow`（びっくり）
- 社内向けにポジティブ・クリエイティブ寄りのセットを優先
- 「つかれた」などネガティブ感情を含めるかは社風・運用方針に依る

## 6. 運用フロー

### 事前準備

- [ ] 社員向けのスペース説明・オンボーディング資料作成
- [ ] 全社員へのURL・QRコード配布（HR経由）
- [ ] 社内ポスター・デジタルサイネージでの告知

### 当日 / 運用中

- HR担当者が月1〜2回、投稿内容をサマリーして経営層に報告（任意）
- 季節・社内イベントに合わせた「お題投稿」を促進
- 週1回程度、面白い投稿をピックアップしてSlackで紹介（任意）

### 終了後・アーカイブ

- 半期ごとに「社内の気持ち変化レポート」を作成

## 7. 成功指標（KPI）

| 指標 | 目標 | 計測方法 |
|------|------|---------|
| 社員参加率 | 全社員の50%以上が1回以上投稿 | authorId 種類数 / 社員数 |
| 月間アクティブ投稿者 | 参加者の30%以上が月1回以上投稿 | 月次 authorId |
| 拠点横断の交流 | 複数拠点の投稿が混在している | 拠点タグ別投稿数 |
| 社内エンゲージメントスコア | サーベイで「社内の雰囲気が明るくなった」の肯定率向上 | 定期アンケート |
