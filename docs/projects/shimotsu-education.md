## 2. 解決したい課題・背景

- 教育プログラムの参加者が「自分がどう変化しているか」を実感しにくい
- セッションごとの気づき・学びが散逸し、後から振り返れない
- アウトプットが最終発表だけになりがちで、プロセスが見えない
- ファシリテーターが参加者の変化を把握するのに時間がかかる

---

## 3. 体験ゴール

### 参加者にとって

- プログラム開始から終了まで、自分の変化・成長がログとして残る
- セッションごとの気づきを積み重ね、後から「自分の旅」として振り返れる
- 過去の自分の投稿が現れることで成長実感が生まれる（セレンディピティ）

### クライアント・主催者にとって

- 参加者一人ひとりの変化・成長の軌跡が可視化される
- セッションの効果測定・プログラム改善に使えるデータが蓄積される
- 最終発表の代わりに、またはその準備として活用できる

### Hossiiとして検証したいこと

- 継続型（数週間〜数ヶ月）でのHossii利用の継続率
- 「過去の自分との出会い」がモチベーション向上につながるか
- ファシリテーターが参加者の変化を把握するツールとして機能するか

## 4. ベーススペースからの差分

> ベーススペース: [workshop-space.md](../spaces/workshop-space.md) + [introspection-space.md](../spaces/introspection-space.md)

### 追加・変更する機能

- **セッションタグ**: 「第1回」「第3回」など、どのセッションの投稿かを識別できるタグ
- **成長ログビュー**: 参加者個人の時系列変化を見せるビュー
- **ファシリテーター向けダッシュボード**: 全参加者の投稿・変化を俯瞰できる画面（将来）
- **ランダム想起**: 数週間前の自分の投稿が現れ、成長を実感させる

### 使わない機能

- ZINE出力: 最終発表のタイミングで活用を検討（未定）
- 公開ボードモード: 参加者限定のクローズドスペース

### Feature Flag 設定

| Flag Key | ON/OFF | 理由 |
|----------|--------|------|
| `commentPost` | ON | 気づき・学びのテキスト |
| `emotionPost` | ON | そのセッションでの感情 |
| `photoPost` | ON | ワークの成果物・模造紙の写真 |
| `numberPost` | ON | 変化度・理解度のセルフスコア（1-10） |
| `random_recall_enabled` | ON | 過去セッションの自分との出会い |
| `public_board_mode` | OFF | 参加者限定 |

### カスタム感情セット

- `inspire`（閃いた）・`think`（気になる）・`wow`（驚いた）・`moved`（ぐっときた）・`fun`（楽しい）
- 教育・成長文脈に合わせた語感（「閃いた」→「気づいた」など検討）

## 6. 運用フロー

### 事前準備

- [ ] プログラム開始前にスペース作成
- [ ] 参加者にQRコード・URLを配布
- [ ] 各セッション開始時の「投稿のお作法」を説明

### 当日 / 運用中

- 各セッションの冒頭・終盤に投稿タイムを設ける
- ファシリテーターが投稿内容を次のセッション設計に反映
- 参加者自身も定期的にマイログで振り返る

### 終了後・アーカイブ

- プログラム終了後、全ログをエクスポート
- 参加者に個人ログを渡す（成長の証として）
- ファシリテーターへのレポート共有

## 7. 成功指標（KPI）

| 指標 | 目標 | 計測方法 |
|------|------|---------|
| 全セッション参加率 | 80%以上が全セッションで投稿 | セッション別 authorId |
| 投稿数 / 人 / セッション | 3件以上 | 平均 |
| 成長実感スコア | 「自分の変化を実感できた」の肯定率70%以上 | 最終アンケート |
| 継続投稿率 | プログラム後半でも投稿が落ちない | 週次投稿数推移 |
