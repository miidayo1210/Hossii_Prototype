## 2. 解決したい課題・背景

- 筑波山を訪れた来訪者の気持ちやアクティビティが散逸している
- 「行ったよ」「よかった」という体験が記録・共有されない
- 来訪者同士のつながりや場の空気の可視化が難しい

---

## 3. 体験ゴール

### 参加者にとって

- QRをスキャンするだけで、訪問の記念として気持ちを残せる
- 他の来訪者の投稿が見えて「同じように感動した人がいる」と感じられる
- 筑波山・施設への愛着が深まる

### クライアント・主催者にとって

- 来訪者の声・感情が可視化される
- 訪問者の体験の質・満足度が把握できる
- 再訪の動機づけになるコンテンツが自然に蓄積される

### Hossiiとして検証したいこと

- 観光地・宿泊施設という場所で広場スペースが機能するか
- 屋外・移動中でも気軽に投稿できるか（UXの耐性テスト）

---

## 4. ベーススペースからの差分

> ベーススペース: [plaza-space.md](../spaces/plaza-space.md)

### 追加・変更する機能

- **アクティビティログ**: 「登山した」「ロープウェイに乗った」「温泉入った」などのアクション選択肢（数値投稿 or カスタム感情として実装検討）
- **場所タグ**: どこで投稿したか（山頂 / 施設内 / ロープウェイなど）のタグ候補

### Feature Flag 設定

| Flag Key | ON/OFF | 理由 |
|----------|--------|------|
| `commentPost` | ON | 一言コメント |
| `emotionPost` | ON | 気持ちタグ |
| `photoPost` | ON | 景色・体験の写真 |
| `numberPost` | OFF | 不要（アクティビティは感情タグで代替） |
| `random_recall_enabled` | ON | 過去の来訪者の声がランダムに現れる |
| `public_board_mode` | ON | QRから誰でも見える |

### カスタム感情セット

- `wow`（すごい）・`moved`（感動）・`fun`（楽しい）・`joy`（うれしい）・`think`（考えた）を優先
- 筑波山の体験（雄大さ・自然・癒し）に合わせた絵文字・ラベルのカスタマイズを検討

## 6. 運用フロー

- [ ] スペース作成・設定（感情セット・背景・スペース名）

### 当日 / 運用中

- 基本無人運用
- 月次でクライアントにレポート共有
- 季節・イベントに合わせたスペース設定の更新

### 終了後・アーカイブ

## 7. 成功指標（KPI）

| 指標 | 目標 | 計測方法 |
|------|------|---------|
| 月間投稿数 | 200件以上 | hossiis テーブル |

| 再訪投稿率 | 常連投稿者が10人以上 | authorId の月次重複 |
| クライアント満足度 | 継続利用の意向あり | ヒアリング |
