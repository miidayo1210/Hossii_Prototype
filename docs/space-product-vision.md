# Hossii Space Product Vision

**Version:** 0.1
**Owner:** Mii Uchino
**Status:** Draft (Iterative Development)

---

## 1. 概要

Hossiiは「スペースを売る」プロダクトである。

スペースとは、特定の目的に特化した入力・可視化・蓄積・展開の構造を持つ **"体験パッケージ"** である。

単なるログツールではなく、

- 入力しやすく
- 触って気持ちよく
- 変化が見え
- 持ち帰れる成果物が生まれる

体験設計を中核とする。

---

## 2. 基本構造（4レイヤー）

Hossiiスペースは以下4レイヤーで構成される。

### 2.1 Input Layer（入力）

人が自然に入力できる形式：

- テキスト（コメント・気づき）
- 画像（現場・ポストイット・模造紙）
- 音声（内省・ブレスト）
- 気持ちタグ
- 行動ログ（スタンプ・数値）

> 設計思想：小さな入力が積み重なり、後から意味を持つ。

### 2.2 Render Layer（見える化）

- ボード型表示（Miro的）
- 時系列表示
- ランダム想起
- 絵馬・バッジ・カード化
- 可動式UI（ふきだし等）

> 設計思想：「整理」ではなく「体験化」。

### 2.3 Archive Layer（蓄積・成果物）

- ZINE出力（PDF）
- 成長ログ可視化
- アクティビティ統計
- 変化の通知

> 設計思想：何かを"持ち帰れる"。

### 2.4 Expand Layer（展開）

- SNS共有
- 外部公開レベル制御
- すれ違い通信（将来）
- 他スペースとの連携

> 設計思想：スペースは閉じない。

---

## 3. スペース商品思想

Hossiiは機能を売らない。**用途特化スペースを売る。**

スペースは以下要素の組み合わせで構成される：

| 要素 | 内容 |
|------|------|
| Input パターン | どのような入力を受け付けるか |
| Render パターン | どのように見せるか |
| Archive パターン | 何を蓄積・出力するか |
| Feature Flag 構成 | どの機能を有効にするか |

---

## 4. 優先開発スペース

### 4.1 ワークショップスペース（Priority 1）

#### 目的

ワークショップのプロセスをデジタル化し、最後にZINEとして出力できる。

#### レイヤー構成

| レイヤー | 主な要素 |
|----------|----------|
| Input | 写真（ポストイット・模造紙）、コメント、感情タグ |
| Render | ボード型表示、グルーピング、タイムライン |
| Archive | PDF ZINE出力、参加者別ログ |

#### 成功指標

- 模造紙不要
- 発言が増える
- 終了後すぐ成果物共有可能

---

### 4.2 広場スペース（店舗・公共空間用）（Priority 2）

#### 目的

誰でも自由に書き込める、**"デジタル掲示板"型コミュニケーション空間**。

#### レイヤー構成

| レイヤー | 主な要素 |
|----------|----------|
| Input | 一言コメント、気持ち、画像 |
| Render | ふきだし型UI、ランダム表示、混ざり合うビジュアル |
| Archive | 月次まとめ、来訪者数可視化 |

#### 成功指標

- 書き込みハードルが低い
- 空間に愛着が生まれる
- 再訪が起きる

## 5. Feature Flagとの接続

各スペースはFeature Flagセットで構成される。

例：

| Flag Key | 説明 |
|----------|------|
| `comments_thumbnail` | コメント一覧でのサムネイル表示 |
| `zine_export_enabled` | ZINE（PDF）出力機能 |
| `random_recall_enabled` | ランダム想起表示 |
| `public_board_mode` | 公開ボードモード（広場スペース向け） |

将来的には：

```
Space Template = Feature Flag プリセット
```

として管理する。Feature Flagの詳細な実装については [feature-flags.md](./feature-flags.md) を参照。

## 7. セレンディピティ設計

Hossiiのコアコンセプト：**スペースに置いたものが、予期しない形で自分や誰かと出会う。**

スペースは「整理された情報庫」ではなく、「セレンディピティが起きる場所」として設計する。

### 何が出会いを生むか

| 出会いの種類 | 説明 | 実現手段 |
|------------|------|---------|
| 過去の自分との出会い | かつて置いた気持ち・アイデアが不意に現れる | ランダム想起・日付検索 |
| 誰かとの出会い | 同じ感情を持つ他者の投稿が目に触れる | 感情タグ・スペース共有 |
| アイデアとの出会い | 忘れていた気づきが別の文脈で繋がる | タグ・キーワード・ランダム表示 |
| 場との出会い | 蓄積された景色が場の記憶になる | Archive Layer・ランダム想起 |

### 設計上の注意

- 「整理させない」 — ユーザーに分類・タグ付けを強制しない。置くだけでいい
- 「忘れさせて、また出会わせる」 — 投稿後にすぐ消えるのではなく、どこかで再浮上する
- 「偶然性を設計する」 — ランダム想起・時間差表示・サジェストを積極的に使う

---

## 8. 設計原則

1. **小さな入力が未来で意味を持つ** — 記録の敷居を下げ、蓄積に価値を生む
2. **整理よりも体験** — ユーザーが「使いたい」と思えるUXを優先する
3. **成果物がある** — 体験が何かに変換されて手元に残る
4. **触って気持ちいい** — インタラクションの質が継続を生む
5. **文化を作れる** — スペースが場の空気・習慣を育てる
6. **セレンディピティを設計する** — 置いたものが予期しない形で戻ってくる
