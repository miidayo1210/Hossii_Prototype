## 1. 概要

| 項目 | 内容 |
|------|------|
| スペース名 | 広場スペース |
| 一言説明 | 誰でも自由に書き込める、デジタル掲示板型コミュニケーション空間 |
| 目的 | 店舗・公共空間・常設イベントに置くことで、来訪者の声を蓄積し場に愛着を生む |
| カテゴリ | 常設系 |

---

## 2. ユーザー

| 役割 | 説明 |
|------|------|
| 主ユーザー（通りすがり参加者） | QRコードをスキャンしてその場で1投稿する。継続利用は想定しない |
| 副ユーザー（常連参加者） | 繰り返し訪れて投稿する。場の変化を楽しむ |
| 運営・管理者 | 店舗オーナー・施設担当者。スペースを設置・管理し、月次レポートを確認する |

---

## 3. 利用シーン

| 項目 | 内容 |
|------|------|
| いつ | 常時（営業時間中） |
| どこで | 店舗入口・レジ横・待合室・公共掲示板 |
| 何人規模 | 日5〜200人（来訪者数に依存） |
| 利用デバイス | 参加者: スマートフォン（QRコードからアクセス） |
| 投影画面 | 店内モニター・サイネージにスペースビューを表示（任意） |

## 4. 体験価値

### 参加者に何が起きるか

- 書き込みのハードルが低く、通りすがりでも参加できる
- 自分の投稿が即座に「場の景色」の一部になる
- 他の人の声が見えて、場への愛着・安心感が生まれる

### 運営に何が残るか

- 来訪者の声・気持ちが自動で蓄積される
- 月次の感情分布・来訪者数のレポートが得られる
- 場の変化（盛り上がり時間帯・人気の感情タグ）が可視化される

### Hossiiとして何を検証したいか

- 常設スペースで再訪・再投稿が起きるか
- 投稿の参加ハードル（ニックネーム入力）が障壁にならないか
- サイネージ表示が来訪者の投稿を誘発するか

## 5. アクション設計

| 入力種別 | 説明 | 必須/任意 |
|---------|------|----------|
| 気持ちタグ | wow / joy / fun / empathy / moved など（場に合わせてカスタム） | 任意 |
| テキストコメント | 一言コメント（200文字以内） | 任意（いずれか1つ必須） |
| 画像 | 任意（店舗の写真・思い出） | 任意 |
| 数値 | 評価・満足度（任意） | 任意 |
| 音声（Listen） | 未使用 | - |

| 項目 | 内容 |
|------|------|
| 操作の重さ | 1タップ（気持ちのみ）が理想。最短5秒で参加完了 |
| 宛先 | スペース全体（全員に見える） |
| 匿名/記名 | ゲスト参加（ニックネーム入力省略できるオプションを検討） |

> **設計上の重要ポイント:** ニックネーム入力の手間が参加の壁になりうる。
> 初回アクセス時にGuestEntryScreenを簡素化する、またはスキップオプションを設けることを検討。

## 6. 表現設計（Render）

### メイン画面での表示

- バブル型で投稿がふわふわと漂う景色
- 感情タグが大きく、絵文字が目立つ
- 混ざり合うビジュアルで「賑わい」を演出

### サブ画面 / スクリーン表示

- 店内モニター・サイネージに流れる景色
- 「今日の場の雰囲気」サマリー表示（感情タグの割合・投稿数）

### 履歴・ログ表示

- ランダム想起表示（過去の投稿が不意に現れる）
- 月次サマリーカード

### 蓄積・成果物（Archive）

- **月次まとめカード**: 来訪者数・感情分布・印象的な一言をカード化
- **来訪者統計**: 投稿数・感情分布の時系列グラフ

|----------|--------|------|

## 10. 運用仕様

| 項目 | 内容 |
|------|------|
| 誰が起動するか | 店舗オーナー・施設担当者（Admin）が一度設定したら基本放置 |
| 当日の操作 | 原則なし。QRコードを貼るだけで運用できることが理想 |
| 管理画面の必要性 | 軽量に（月次レポート確認・スペース設定変更程度） |
| スペースの有効期間 | 常設（無期限） |
| 障害時の代替方法 | QRコードを外す・スペース一時停止 |

## 11. KPI

| 指標 | 目標値 | 計測方法 |
|------|--------|---------|
| 参加率（来訪者のうち投稿した割合） | 10%以上 | 投稿数 / 推定来訪者数 |
| 再訪率 | 常連参加者の30%以上が再投稿 | authorId の重複率 |
| 投稿継続日数 | 30日間ゼロ投稿の日なし | 日次投稿数 |
| 会話発生数 | サイネージ経由で会話が生まれる | 定性ヒアリング |
| 場への愛着スコア | 定性評価（好き・また来たい） | ユーザーインタビュー |

## 12. Coreとの差分

### 新しく必要な機能

- **ランダム想起表示** (`random_recall_enabled`): 過去の投稿が不意に蘇る表示
- **公開ボードモード** (`public_board_mode`): URLを知らない人でも閲覧可能
- **月次まとめ生成**: 月ごとの感情分布・来訪者数をカード化して出力
- **ゲスト入室の簡素化**: ニックネーム入力を最短化（任意化・スキップ対応）

### 不要なCore機能

- Listen モード（自動音声投稿）: 常設環境では不要
- セッション管理: 不要（常設）
- ZINE出力: 不要

### カスタムコンポーネント

- `MonthlySummaryCard`: 月次サマリーカード表示
- `RandomRecallOverlay`: 過去の投稿をランダムに浮かび上がらせるオーバーレイ
- `SignageView`: サイネージ向け大画面表示（スペースビューの派生）

### 特別な運用上の制約

- 管理者が長期間ログインしなくても動き続ける（無人運用が前提）
- スペースが「育つ」感覚が大事。古い投稿も消えずに残る設計

## 13. 空間状態の遷移

| フェーズ | 状態 | 表現 |
|---------|------|------|
| 設置直後 | 投稿ゼロ。場が静か | バブルなし。「最初の一言を置いてね」 |
| 序盤 | 初期参加者が投稿し始める | バブルが数個、ゆっくり漂う |
| 活性期 | 毎日投稿が積み重なる | バブルが増え、場が賑やか |
| 成熟期 | 常連参加者が増え、場に文化が生まれる | 過去投稿がランダムに現れ、余韻を演出 |
| 記念日・特別な日 | 投稿が一時的に増える | 急激な盛り上がりが可視化される |
