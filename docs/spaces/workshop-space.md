## 1. 概要

| 項目 | 内容 |
|------|------|
| スペース名 | ワークショップスペース |
| 一言説明 | ワークショップのプロセスをデジタル化し、ZINEとして持ち帰れる |
| 目的 | 模造紙・ポストイットを不要にし、参加者の気づきをリアルタイムで共有・蓄積する |
| カテゴリ | イベント系 |

---

## 2. ユーザー

| 役割 | 説明 |
|------|------|
| 主ユーザー（参加者） | ワークショップ参加者。スマートフォンから投稿・共感する |
| 副ユーザー（ファシリテーター） | 場を進行する人。スペースの開始・終了・画面切替を行う |
| 運営・管理者 | ワークショップ主催者。スペース作成・設定・結果ダウンロードを行う |

---

## 3. 利用シーン

| 項目 | 内容 |
|------|------|
| いつ | ワークショップ当日（2〜6時間程度） |
| どこで | 会議室・イベント会場・オンライン |
| 何人規模 | 5〜50人 |
| 利用デバイス | 参加者: スマートフォン（QRコードからアクセス） / ファシリ: タブレット・PC |
| 投影画面 | プロジェクター・大型ディスプレイにスペースビューを表示 |

## 4. 体験価値

### 参加者に何が起きるか

- 自分の気づきやアイデアを即座に場に出せる
- 他の参加者の投稿が見えて共感・刺激が生まれる
- 「言えなかったこと」がテキスト・画像で残る

### 運営に何が残るか

- 当日の参加者の感情ログ・発言ログが自動で蓄積される
- セッション終了後、すぐにZINE（PDF）として成果物を共有できる
- 参加者別・時系列での振り返りができる

### Hossiiとして何を検証したいか

- 模造紙の代替として機能するか（物理アウトプットがデジタルで再現できるか）
- ファシリテーターなしでも場が動くか
- ZINEの出力品質と参加者の満足度

## 5. アクション設計

| 入力種別 | 説明 | 必須/任意 |
|---------|------|----------|
| 気持ちタグ | wow / empathy / inspire / think / laugh / joy / moved / fun | 任意 |
| テキストコメント | 気づき・アイデア・感想（200文字以内） | 任意（いずれか1つ必須） |

| 数値 | 未使用 | - |

| 項目 | 内容 |
|------|------|
| 操作の重さ | 1タップ（気持ちのみ）または短文入力 |
| 宛先 | スペース全体（全員に見える） |
| 匿名/記名 | ニックネーム（ゲスト参加可、ニックネーム必須） |

## 6. 表現設計（Render）

### メイン画面での表示

- バブル型またはボード型で投稿をリアルタイム表示
- 画像投稿はサムネイルつきで表示
- 感情タグは絵文字で強調表示

### サブ画面 / スクリーン表示

- プロジェクター投影用のスペースビュー（`#screen`）
- セッションの経過に合わせて「今の場の雰囲気」が可視化される

### 履歴・ログ表示

- コメント一覧（`#comments`）でタイムライン表示
- 画像一覧でポストイットを俯瞰できる

### 蓄積・成果物（Archive）

- **PDF ZINE出力**（優先度高）: セッションの投稿・画像・感情ログをA5またはA4のPDFに自動レイアウト
- 参加者数・投稿数・感情分布のサマリー

## 10. 運用仕様

| 項目 | 内容 |
|------|------|
| 誰が起動するか | 主催者（Admin）がスペースを事前作成。当日QRコードを参加者に配布 |
| 当日の操作 | ファシリテーターがセッション切替・スクリーン投影を操作 |
| 管理画面の必要性 | あり（スペース設定・Feature Flag・結果ダウンロード） |
| スペースの有効期間 | イベント単位（当日のみ〜数日間） |
| 障害時の代替方法 | 物理ポストイット + 後日写真アップロード |

## 11. KPI

| 指標 | 目標値 | 計測方法 |
|------|--------|---------|
| 参加率 | 参加者の70%以上が1投稿以上 | hossiis テーブルの authorId 種類数 |
| 投稿数 / 人 | 3件以上 | 平均投稿数 |
| 画像投稿率 | 参加者の30%以上 | imageUrl あり投稿数 |
| ZINE出力率 | 主催者の100%がダウンロード | ダウンロードイベント（将来） |
| 模造紙使用率 | 0（理想） | 定性ヒアリング |

## 12. Coreとの差分

### 新しく必要な機能

- **ZINE（PDF）出力**: 投稿・画像・感情分布をPDFにレイアウト
- **セッション管理**: 1スペース内で複数セッション（午前/午後など）を切り替える
- **グループ表示**: チーム別に投稿を色分け・フィルタリング

### 不要なCore機能

- Listen モード（自動音声投稿）: ワークショップでは不要（ただしファシリ端末では任意利用可）
- スタンプカード: 参加エンゲージメントとして残してもよい

### カスタムコンポーネント

- `ZineExportButton`: ZINE出力ボタン（スペース設定 or コメント一覧画面に追加）
- `SessionSwitcher`: セッション切替UI（管理画面）
- `GroupFilter`: グループ別フィルタ

### 特別な運用上の制約

- イベント終了後にスペースを「アーカイブ」状態にする機能が必要（投稿不可・閲覧のみ）

## 13. 空間状態の遷移

| フェーズ | 状態 | 表現 |
|---------|------|------|
| 開始前 | 場が静か。参加者が入室中 | バブルが少なく、静かな星空 |
| 序盤 | アイスブレイクで投稿が増え始める | バブルが増え、絵文字が動く |
| メインセッション | 活発に気づきが投稿される | 画面が賑やかになる。画像がボードを彩る |
| 振り返りタイム | 投稿が落ち着き、参加者が過去ログを見る | スライドショーや一覧表示に切替 |
| 終了・成果物 | ZINEが出力され持ち帰れる | セッションサマリーを全体投影 |
