# ナビゲーション・UI 構成 仕様

> **分類:** `[Core]` 画面遷移・ナビゲーション構造
> **関連:** [core-concepts.md](../../core/core-concepts.md) §5（画面一覧）

> 最終更新: 2026-06-02

---

## 概要

Hossii はスマートフォンと PC の両方で使用されることを想定しており、
画面幅によってナビゲーションの表示形式を切り替えるレスポンシブ対応を採用する。

ブレークポイントは `769px` を境界とする。

### グローバルなクリックフィードバック

アプリ全体で、主要なクリック時に **短い紫の星** がクリック位置付近に表示される（軽いフィードバック）。`prefers-reduced-motion: reduce` では無効。仕様の詳細は [56_クリックフィードバックとフローティングパネル](./56_クリックフィードバックとフローティングパネル.md) を参照。

## ナビゲーション構成

### PC（横幅 769px 以上）

画面右上に固定表示されるフローティングメニュー（`TopRightMenu`）を使用する。

| ボタン | 動作 | 説明 |
|---|---|---|
| スペース | `#screen` へ遷移（`#screen` 以外からのみ表示 or アクティブ非表示） | スペース画面に戻る |
| 投稿する | **クイック投稿パネルを開閉**（`#screen` 中）。他画面からは `#screen` へ戻してから開く | `#post` フルページへは遷移しない |
| ログ一覧 | `#comments` 画面へ遷移 | 投稿・コメント一覧へ |
| アカウント | `#account` 画面へ遷移 | ニックネーム設定・アカウント情報へ |

- ドロップダウンメニューは使用しない（シンプル構成）
- 現在表示中の画面に対応するボタンはアクティブ表示する
- **「投稿する」は全レスポンシブ共通でクイック投稿パネルを使う**（[51_簡単投稿](./51_簡単投稿.md)）

#### ヘッダー（`TopBar`）

- スペース名のみを表示する
- ナビゲーションボタンは持たない（役割を TopRightMenu に集約）

---

### スマートフォン（横幅 768px 以下）

画面下部に固定表示されるタブバー（`BottomNavBar`）を使用する。

| タブ | アイコン | 動作 |
|---|---|---|
| スペース | House | `#screen` 画面（スペース） |
| 投稿 | PlusCircle | **`#screen` 中:** クイック投稿パネルを開閉（[76](./76_スマホ投稿ナビとクイック投稿.md)）。**他画面:** `#screen` へ戻してから開く |
| ログ | ScrollText | `#comments` 画面（[62](./62_スマホのスペース画面.md) と整合） |
| アカウント | User | `#account` 画面 |

- **投稿タブ:** `#screen` 表示中は **画面遷移せず** クイック投稿パネルを開閉。他画面からは `#screen` へ戻してから開く。詳細は [76](./76_スマホ投稿ナビとクイック投稿.md)
- **ログタブ:** **常に** `#comments` へ遷移
- ラベルはすべて日本語で表示する
- iOS の安全エリア（ノッチ）を考慮した下部パディングを設定する
- フローティングパネル類はナビバーに被らないよう [56](./56_クリックフィードバックとフローティングパネル.md) の下端余白に従う

## 廃止: 右端キューブドック（`SpacePanelCubeDock`）

> **❌ 廃止方針（2026-06-02）**

以前は PC の `#screen` 右端に「投稿 / ログ / 音声」の 3 キューブを並べていたが、
以下の理由により **廃止**する。

- 存在に気づきにくく、初見ユーザーに機能が伝わらない
- 「投稿する」を TopRightMenu から開けるようにすることで代替できる
- 音声・ログはダブルクリック / トリプルクリック / 左バーで操作できる

**代替手段:**

| 旧キューブ機能 | 代替 |
|--------------|------|
| 投稿キューブ | TopRightMenu「投稿する」・背景ダブルクリック |
| ログキューブ | 背景トリプルクリック（PC）・下ナビ「ログ」（モバイル） |
| 音声キューブ | 左バーのマイクボタン |

`SpacePanelCubeDock.tsx` / `SpacePanelCubeDock.module.css` は **削除対象**。

## アカウント画面（`AccountScreen`）

アカウントボタンまたはアカウントタブから遷移するページ。
以下の情報を1ページにまとめて表示する（タブ切替はなし）。

### セクション構成

#### 1. アカウント情報

ログイン済みの場合:
- アバターアイコン
- ニックネーム（デフォルトニックネーム）
- メールアドレス
- ログアウトボタン

ゲスト・未ログインの場合:
- ゲスト表示
- ログイン・アカウント登録への導線ボタン

#### 2. このスペースでのニックネーム

- 現在アクティブなスペースのニックネームを編集・保存できる
- スペース名を表示して「どのスペースか」を明示する
- 未設定の場合はデフォルトニックネームが使われる旨を案内する

#### 3. デフォルトニックネーム

- すべてのスペースで共通のデフォルト表示名を編集・保存できる
- スペース別ニックネームが設定されている場合はそちらが優先されることを案内する

## ナビゲーション統一（admin / 一般共通）

`isAdmin` の有無にかかわらず、`BottomNavBar` は全ユーザー共通の4タブを表示する。

| タブ | アイコン | 遷移先 |
|---|---|---|
| スペース | House | screen 画面 |
| 投稿 | PlusCircle | クイック投稿パネル（`#screen` 中）/ `#screen` 遷移後に開く |
| ログ | ScrollText | comments 画面 |
| アカウント | User | account 画面 |
