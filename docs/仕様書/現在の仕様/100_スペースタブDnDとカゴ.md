## 1. 背景・目的

スペース内タブ（`SpacePane`）が増えると、タブバーが横スクロールだらけになり **一覧性が落ちる**。

| ニーズ | 概要 |
|--------|------|
| **ブラウザタブのような DnD** | タブを掴んで左右に動かし、離した位置で並び替え |
| **カゴ 🧺 で整理** | あまり見ないタブをカゴに「入れる」。バーはすっきり、必要なときだけカゴを開いて中身を見る |
| **差分小さめ** | 既存 `sort_order` / `SpacePaneBar` / 設定画面の並び替え API を最大限再利用 |

**93 §7.3** では設定画面の **上下ボタン** のみ確定済み。本仕様は **スペース画面タブバー上の DnD** と **カゴ UI** を追加する（設定画面の上下ボタンは **併存**）。

---

## 2. 用語

| 用語 | 意味 |
|------|------|
| **タブバー** | `SpacePaneBar` — スペースタイトル下の横並びタブ |
| **バー上タブ** | 通常表示されるタブ（`tabBar.group = 'bar'`）。比喩: **デスクトップ上のファイル** |
| **カゴ** | 折りたたみコンテナ。**100B** では 1 スペース **1 つ**（🧺 固定）。**100C** 以降は **複数・名称付きフォルダ** へ拡張 |
| **カゴ内タブ** | カゴ / フォルダに入れたタブ（100B: `group = 'basket'`）。比喩: **フォルダ内のファイル** |
| **タブフォルダ** | 100C 構想の正式名称。ユーザーが名付けたカゴ（例: 「2025春」「アーカイブ」） |
| **DnD** | Drag and Drop（掴む → 移動 → 離す） |

## 4. 差分見積もり（実装可否）

**結論: Phase 100A は小さめ。100B も migration なしで中規模まで抑えられる。**

| 領域 | 100A（DnD 並び替え） | 100B（+ カゴ） |
|------|---------------------|----------------|
| **新規ファイル** | 0〜1（DnD ヘルパー） | +1（カゴ UI 断片） |
| **主な変更** | `SpacePaneBar.tsx` / `.module.css`、`SpaceScreen.tsx` | 同上 + settings 読み書き |
| **既存再利用** | `computeReorderUpdates`、`applySpacePaneSortOrders` | 100A の DnD 基盤 |
| **DB / migration** | 不要 | 不要（JSON） |
| **概算行数** | **+120〜200 行** | **+100〜180 行**（100A 上に累積） |

### 4.1 小さく保つための技術選択

| 判断 | 採用案 | 理由 |
|------|--------|------|
| DnD ライブラリ | **ネイティブ HTML5 Drag and Drop** | 依存追加なし。タブ数 ≤20 で十分 |
| カゴの永続化 | **`space_panes.settings.tabBar`**（所属）+ **`spaces.tab_folders`**（フォルダ定義） | pane 所属は JSONB 既存列。フォルダ名・一覧はスペース正本 |
| カゴの開閉状態 | **`localStorage`（閲覧者ごと）** | UI 状態のみ。サーバー不要 |
| 並び順 | 既存 **`sort_order`** | バー・カゴ **共通の全順序**。カゴは「グループ」だけ分離 |

## 6. Phase 100A — タブバー DnD 並び替え

### 6.1 ユーザーストーリー

> 管理者として、ブラウザのタブのようにタブを掴んで左右に動かし、離した位置で順序を確定したい。

### 6.2 操作

```text
[ メイン ] [ 今日の問い ] [ アイデア ] [ ＋ ]
     ↑ 掴んで ←→ 移動 → 離す
```

| 項目 | 仕様 |
|------|------|
| ドラッグ開始 | タブを **200ms 長押し** または **即ドラッグ**（PC は即ドラグ推奨） |
| ドラッグ中 | タブが半透明 + 挿入位置に **縦線インジケータ** |
| ドロップ | 離した位置で `sort_order` を再計算 → **即 API 保存** |
| デフォルトタブ | **並び替え可**（93 §7.5）。**カゴへは不可**（100B） |
| ＋ボタン | ドラッグ不可 |
| 失敗時 | toast「並び替えに失敗しました」+ `reloadPanesAndSyncActive` |

### 6.3 保存

```typescript
// 既存そのまま
const updates = computeReorderUpdates(allVisiblePanes, draggedId, targetIndex);
await applySpacePaneSortOrders(updates);
await reloadPanesAndSyncActive();
```

- 対象リスト: **`visiblePanes` を `sort_order` 順**（カゴ導入前は全タブ）
- 100B 導入後: **バー上タブ同士**、**カゴ内タブ同士**、**バー↔カゴ** で別ルール（§7）

### 6.4 アクセシビリティ（100A）

| 項目 | 初版 |
|------|------|
| キーボード並び替え | **対象外**（設定画面の上下ボタンで代替） |
| `aria-grabbed` | ドラッグ中のタブに付与 |

## 7. Phase 100B — カゴ 🧺

### 7.1 ユーザーストーリー

> タブが多すぎるので、あまり使わないタブを **カゴに入れて** バーをすっきりさせたい。  
> カゴをタップすると **右にギュンと伸びて** 中のタブが見える。もう一度タップで閉じる。

### 7.2 ワイヤー

**閉じている（既定）**

```text
[ メイン ] [ 今日の問い ] [ 🧺 3 ] [ ＋ ]
                              ↑ タップで開く
```

**開いている**

```text
[ メイン ] [ 今日の問い ] [ 🧺 3 │ アイデア │ 振り返り │ メモ ] [ ＋ ]
                              └─ 右方向へスライド展開（同一行）
```

| 項目 | 仕様 |
|------|------|
| 位置 | **＋ボタンの左**（93 の追加ボタン並びを維持） |
| ラベル | 🧺 + **件数**（例 `3`）。0 件でも 🧺 は表示（ドロップゾーンとして） |
| 開閉 | **クリック / タップ** でトグル。`aria-expanded` |
| 展開方向 | カゴチップの **右側** にインラインでタブが並ぶ（`max-width` + 横スクロール可） |
| アニメ | `width` または `transform: scaleX` + `opacity`、**220ms ease-out**（`prefers-reduced-motion` 時は即時） |
| 閉じる | カゴ再タップ、またはバー外クリック（任意） |

### 7.3 DnD — カゴへの入出し

| 操作 | 結果 |
|------|------|
| バー上タブ → カゴ 🧺 上にドロップ | `settings.tabBar.group = 'basket'` |
| カゴ内タブ → バー領域にドロップ | `group = 'bar'` |
| カゴ内 ↔ カゴ内 | `sort_order` のみ更新 |
| デフォルトタブ → カゴ | **拒否**（ドロップ不可カーソル） |
| カゴ → 非表示 | 設定画面の非表示と同様（93）。カゴから自動で `bar` に戻す |

**「掴んで離す」テンション:** バー上タブと同じ DnD。カゴチップは **ドロップターゲット**（ホバーでハイライト）。

### 7.4 データモデル

`space_panes.settings` JSON を拡張（**migration 不要**）。

```typescript
// SpacePaneSettingsOverride に追加
type SpacePaneTabBarSettings = {
  /** 省略時 'bar' */
  group?: 'bar' | 'basket';
};

type SpacePaneSettingsOverride = {
  postFields?: ...;
  posting?: ...;
  tabBar?: SpacePaneTabBarSettings;
};
```

| フィールド | 意味 |
|-----------|------|
| `tabBar.group = 'bar'` | タブバーに通常表示（省略 = bar） |
| `tabBar.group = 'basket'` | カゴ内。バーには 🧺 件数にのみ反映 |

**並び順:** 引き続き **`sort_order` 単一列**。表示時に `group` でフィルタ。

```typescript
barPanes   = visiblePanes.filter(p => resolveTabBarGroup(p) === 'bar')
basketPanes = visiblePanes.filter(p => resolveTabBarGroup(p) === 'basket')
// いずれも sort_order 昇順
```

**API:** 既存 `updateSpacePane(id, { settings: merge... })` で保存。

### 7.5 カゴ開閉の永続化

| 状態 | 保存先 |
|------|--------|
| 開閉 | `localStorage` キー `hossii.tabBasketOpen:{spaceId}` = `'1' \| '0'` |
| 理由 | 閲覧者ごとの UI 好み。サーバー負荷なし |

### 7.6 カゴ内タブの選択

- カゴが **開いている** とき、中のタブは通常タブと同様 **クリックで Pane 切替**
- カゴが **閉じている** とき、🧺 クリックは **開閉のみ**（中タブは選べない）
- アクティブ Pane がカゴ内のみにある場合: バーに **「🧺 内: {タブ名}」** の薄いインジケータ（任意・100B 必須ではない）

## 15. Phase 100C — 複数タブフォルダ（構想・未実装）

> **きっかけ:** カゴも名付けたり、複数個作れたらいい。**PC のフォルダとファイル** のように整理したい。

### 15.1 比喩（メンタルモデル）

| PC | Hossii タブバー |
|----|----------------|
| デスクトップ上のファイル | **バー上タブ**（いま使うタブ） |
| フォルダ | **タブフォルダ**（100C）。名称・アイコン付き |
| フォルダにファイルを入れる | タブを DnD でフォルダへ |
| フォルダを開く | フォルダチップをタップ → 右に中身が展開（100B と同じテンション） |
| フォルダ名の変更 | 管理者が設定 or インライン編集 |
| 新規フォルダ | ＋ の近くに「フォルダ追加」 |

**100B との関係:** いまの 🧺 1 つは、100C 移行時に **「未分類」** または **「カゴ」** という名前の **デフォルトフォルダ 1 件** に読み替える。

### 15.2 ユーザーストーリー

> 管理者として、「2025春」「卒業生向け」「アーカイブ」のように **フォルダを複数作り**、タブを目的別に入れたい。  
> 参加者はフォルダ名を見て、必要なタブだけ開いて選びたい。

### 15.3 ワイヤー（イメージ）

**フォルダが複数（閉じている）**

```text
[ メイン ] [ 今週 ] [ 📁 2025春 3 ] [ 📁 アーカイブ 5 ] [ ＋ ] [ 📁+ ]
                                              ↑新規フォルダ
```

**1 フォルダを開いている**

```text
[ メイン ] [ 今週 ] [ 📁 2025春 3 │ 振り返り │ 遠足 │ 文化祭 ] [ 📁 アーカイブ 5 ] [ ＋ ]
                              └─ 右へ展開（100B カゴと同 UI パターン）
```

| 項目 | 100C 案 |
|------|---------|
| フォルダ表示 | **名称** + **件数**。アイコンは省略可（📁 既定、絵文字選択は任意） |
| 同時に開くフォルダ | **1 つ**（100B 踏襲。バーが溢れない） |
| 空フォルダ | 表示する（ドロップ先として。100B の 0 件 🧺 と同様） |
| 名称 | 最大 **20 文字** 程度（93 タブ名 30 字より短めで OK） |
| 削除 | 中身があるフォルダは **確認ダイアログ** → 中タブは **バーへ戻す** or **未分類へ** |

### 15.4 DnD（100C で追加）

| 操作 | 結果 |
|------|------|
| バー上タブ → フォルダ A | `tabBar.folderId = A` |
| バー上タブ → フォルダ B | `folderId = B`（フォルダ間移動） |
| フォルダ内タブ → バー | `group = 'bar'`, `folderId` 削除 |
| フォルダ内 ↔ フォルダ内（同一フォルダ） | `sort_order` 更新 |
| デフォルトタブ → フォルダ | **拒否**（100B 同様） |
| フォルダチップの並び替え | 管理者 DnD（任意・後回し可） |

### 15.5 データモデル（案）

100B の `group: 'basket'` を **フォルダ所属** に一般化する。

**スペース側 — フォルダ一覧（正本）**

```typescript
// spaces.tab_folders jsonb（2026-07-04 〜）
type TabFolder = {
  id: string;
  name: string;
  sortOrder: number;
};

// Space 型: tabFolders?: TabFolder[]
```

**旧実装（〜2026-07-03）:** `localStorage` キー `hossii.tabFolders:{spaceId}`。初回表示時に DB が空なら **自動移行** して legacy キーを削除。

**フォルダ定義が無いが pane が folderId を参照している場合:** UI は **`フォルダ`** という名称のプレースホルダチップを合成表示（タブが非表示にならない）。

**Pane 側 — 所属**

```typescript
type SpacePaneTabBarSettings = {
  /** 'bar' = バー表示。'folder' = フォルダ内（100B の 'basket' を置換） */
  group?: 'bar' | 'folder';
  /** group === 'folder' のとき必須 */
  folderId?: string;
};
```

**100B → 100C 移行（想定）**

| 100B 現行 | 100C 移行後 |
|-----------|-------------|
| `group: 'basket'` | `group: 'folder'`, `folderId: '<default-folder-id>'` |
| 単一 🧺 | デフォルトフォルダ名「カゴ」または「未分類」 |
| `hossii.tabBasketOpen:{spaceId}` | `hossii.tabFolderOpen:{spaceId}:{folderId}` |

### 15.6 UI 入口（案）

| 入口 | 内容 |
|------|------|
| タブバー | フォルダチップ列 + **📁+**（フォルダ新規） |
| 設定画面「タブ管理」 | フォルダ一覧・名称編集・削除（PaneManagementTab 拡張） |
| フォルダ名変更 | 長押し / 右クリックメニュー / 設定画面（実装時に決定） |

### 15.7 100C のスコープ案

| 含める | 含めない（さらに将来） |

| フォルダ CRUD（作成・改名・削除） | フォルダの入れ子 |
| 複数フォルダ + DnD 所属 | フォルダごと背景色 |
| 100B カゴ UI の一般化 | 参加者によるフォルダ作成 |
| デフォルトフォルダへの自動移行 | フォルダ単位の権限 |

### 15.8 受け入れ条件（100C · 草案）

| # | 条件 |
|---|------|
| C1 | 管理者がフォルダを 2 つ以上作成し、それぞれに名称を付けられる |
| C2 | タブをフォルダ A / B に DnD で振り分けできる |
| C3 | フォルダをタップすると 100B と同様右方向に展開し、中タブを選択できる |
| C4 | 100B で `group: 'basket'` だったタブが、移行後もデフォルトフォルダに表示される |
| C5 | デフォルトタブはフォルダに入れられない |
| C6 | PC / スマホで **同一スペース** のフォルダ名・件数・タブ所属が一致する（フォルダ開閉を除く） |

### 15.9 実装済み（2026-06-30）

| ステップ | 実装内容 |
|--------|---------|
| データ | `pane.settings.tabBar = { group: 'folder', folderId }` — 既存 JSONB 利用でマイグレ不要 |
| 型拡張 | `SpacePaneTabBarSettings.group` に `'folder'` 追加 + `folderId?: string` |
| ストレージ | `spaces.tab_folders`（Supabase）/ `Space.tabFolders`（デモ localStorage）+ `tabFolderStorage.ts` ヘルパー |
| 端末同期 | フォルダ定義・タブ所属とも **別端末で一致**。開閉状態のみ localStorage |
| ロジック | `resolvePaneFolderId` / `splitPanesByFolders` / `buildTabFolderPatch` / `resolveEffectiveTabFolders` |
| UI | `SpacePaneBar` — 複数 `FolderCluster` ループ、インライン命名、削除、`📁＋` 追加ボタン |
| 後方互換 | `group: 'basket'` → デフォルトフォルダ (`id='default'`) として自動解釈 |

## 13. リスク・注意

| リスク | 対策 |
|--------|------|
| HTML5 DnD と横スクロールの競合 | モバイルは長押し閾値。`e.dataTransfer` で paneId のみ渡す |
| カゴ展開でタブバーがはみ出す | 展開部分のみ `overflow-x: auto`、最大幅 `min(50vw, 320px)` |
| `settings` マージ漏れ | `mergePaneSettingsOverride` 経由で **必ず** patch |
| Realtime 他管理者との同時編集 | 初版は **最後の保存 wins**。競合時は reload（93 と同様） |

## 16. ローカル dev とデプロイの連動確認

### 16.1 前提

| 保存対象 | Supabase あり（同一 DB） | デモモード（`VITE_SUPABASE_*` 未設定） |
|----------|--------------------------|----------------------------------------|
| タブ一覧（名前・件数・並び） | `space_panes` テーブル | `localStorage` `hossii_demo_space_panes:*` |
| タブのフォルダ所属 | `space_panes.settings.tabBar` | 同上 |
| フォルダ定義（名称・並び） | `spaces.tab_folders` | 旧: `hossii.tabFolders:*`（自動移行あり） |
| フォルダ開閉 | `localStorage`（端末ごと） | 同左 |

**local の `npm run dev` とデプロイ版が連動する条件:**

1. **両方**に `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` が設定されている
2. **同じ Supabase プロジェクト**を指している（ホスト名が一致）
3. DB migration（`spaces.tab_folders` 列など）が **デプロイ DB に適用済み**

### 16.2 コンソールでの確認（管理者）

スペース画面を開くと、管理者ログイン時に `[Hossii tab sync]` がコンソールに出力される。

再確認:

```javascript
__hossiiTabSyncCheck()
```

**local と deploy で比較する項目:**

| フィールド | 一致すべき値 |
|------------|--------------|
| `supabaseConfigured` | 両方 `true` |
| `supabaseHost` | **同一**（例: `xxxx.supabase.co`） |
| `paneStorage` | `supabase:space_panes` |
| `folderStorage` | `supabase:spaces.tab_folders` |
| `visiblePaneCount` / `folderCount` | 同一スペースなら一致 |

`supabaseConfigured: false` の場合 → **デモモード**。タブはデプロイと **絶対に連動しない**。

### 16.3 よくある不一致

| 症状 | 原因 | 対処 |
|------|------|------|
| local にだけタブがある | local がデモモード | `.env.local` に Supabase 環境変数を設定 |
| ホスト名が違う | dev / prod で別 Supabase | 意図的なら OK。同じにしたいなら env を揃える |
| フォルダ名だけずれる | 旧 localStorage 未移行 | 一度 PC でスペースを開く（自動移行）または `tab_folders` migration 適用 |
| deploy だけタブが少ない | migration 未適用 / RLS | Supabase migration と管理者権限を確認 |
