## 1. 目的

スペース上の投稿を、**投稿の新しさ（時系列）** に応じて空間の奥へ蓄積されていくように見せる。

| 表現 | 意図 |
|------|------|
| 新しい投稿 | 手前に見える（既存 z-index）＋ 現行サイズ（1.00）を維持 |
| 古い投稿 | 奥に見える（既存 z-index）＋ 段階的に小さく |
| 時間経過 | 投稿が空間の奥へ沈んでいくメタファー |

**本機能が解決しないこと:** 投稿の「重要度」「いいね数」「管理者評価」と新しさの混同。新しさは **表示上の奥行き** のみに使う。

**初版の範囲（確定）:** 奥行き表現は **外形サイズ（星ドットのみ）** と **既存 z-index** に限定する。opacity・影・blur・新規アニメーションは追加しない。

**現行との関係:** [87](./87_スペース表示パフォーマンス最適化.md) の **index ベース z-index** と **animation tier**（0–9: full, 10–29: light, 30+: none）は **初版では変更しない**。サイズだけを `timelineDepthScaleFromIndex` で追加する。

## 3. 新旧判定の基準（確定）

| 項目 | 仕様 |
|------|------|
| **主キー** | `filteredHossiis` の **描画 index**（0 = 最新） |
| **ソート** | `createdAt` 降順（`runDisplayPipeline` と同じ） |
| **表示上限** | `displayLimit` スライス後の配列に対する index（DB 全体の順位ではない） |
| **タグ絞り込み後** | 絞り込み後配列で index を **再計算**（相対的新しさ） |
| **新規投稿** | 先頭に追加され、既存投稿の index が +1 されることは **許容**（段階的に奥へ） |
| **ピン留め** | ソート順は変えない。サイズは通常の時系列を適用（§12） |

### ページング・表示件数（確定・現行実装に合わせた記述）

| 操作 | 既存投稿の index |
|------|------------------|
| `displayLimit` を **増減** | 表示中の投稿は **基本的に変わらない**（より古い投稿が末尾に追加 or 末尾から消える） |
| unlimited **追加読み込み** | 末尾 append のみ。既存 index **不変** |
| **新規投稿**のリアルタイム追加 | 先頭挿入により既存がシフト → **許容** |
| **タグ絞り込み**変更 | 絞り込み後で index 再計算 → tier が変わる可能性あり **許容** |

### 将来案

- `createdAt` からの **経過日数** ベースの段階
- **全投稿中の絶対順位**（fetch 完了前提）

### 懸念（未決・コード変更は別途）

`runDisplayPipeline` は `createdAt` のみでソートするが、fetch / merge 側の `sortHossiisNewestFirst` は `createdAt` + **`id` 二次キー** を使う。**同一 `createdAt` 時の順序がずれる可能性**がある。

時系列サイズ実装前、または別コミットで `sortHossiisNewestFirst` への統一を検討する（Phase 0）。

## 4. サイズ段階（確定・初期実装値）

初版は **4 段階**。最新投稿は現行サイズ **1.00 を維持**し、古い投稿だけ段階的に小さくする。**1.00 を超える拡大は行わない**（既存配置の衝突を増やさない）。

| 描画 index | 段階 | depth scale（初期実装値） |
|----------|------|---------------------------|
| 0–9 | L | **1.00** |
| 10–29 | M | **0.94** |
| 30–59 | S | **0.88** |
| 60+ | XS | **0.82** |

> 上記数値は **初期実装値**。Phase 2 の実機確認後に調整可能。

### 純関数（確定）

z-index・animation・opacity には触れず、**サイズ係数だけ**を返す。

```ts
// 配置案: src/core/utils/timelineDepthScale.ts（または animationLevel.ts 隣接）
timelineDepthScaleFromIndex(index: number): number
```

`resolvePostDepth()` のような一括統合関数は **初版では採用しない**。

## 5. 星ドットへの適用（確定）

| 項目 | 初版 |
|------|------|
| **視覚的外形** | `timelineDepthScaleFromIndex(index)` を `starDot` に適用（例: CSS 変数 `--depth-scale` + `transform: scale()`） |

| 要素 | ルール |
|------|--------|

| **星ドット見た目** | depth scale に応じて変化してよい |
| **視覚下限（目安）** | モバイル 14px / PC 22px 相当を下回らないよう実装時にクランプ検討 |

## 7. z-index と animation tier（確定・初版は変更なし）

初版では以下を **変更しない**。

| 関数 | 役割 |
|------|------|
| `displayStackZFromIndex` | random / star の重ね順 |
| `defaultAnimationLevelByIndex` | animation tier 境界 |
| `resolveAnimationLevel` | ホバー・選択・プレビュー等の昇格 |

インタラクション昇格（現行踏襲）:

- ホバー: `+10`
- 選択 / active: `+20`〜`+30`
- recent highlight / プレビュー: 既存ルール

**ordered モード:** 初版対象外。z は現行どおり `orderedGridIndex + 1`（変更なし）。

## 8. opacity・影・blur（初版対象外 → 将来案）

初版では以下を **追加しない**。

- opacity の時系列変化
- 影の時系列変化
- 彩度の変化
- blur による奥行き
- 新しいアニメーション

**将来案（Phase 4）:** opacity 段階。モバイル **背景ピーク**（一括 `opacity: 0.35`）との優先順位を整理してから導入。

**現行参考:** `bubbleAnimNone` は animation 停止用の背景不透明化であり、年齢ベース opacity ではない。

## 18. 懸念点

1. **index の不安定性（タグ絞り込み）** — 許容済み。タグ変更で tier が変わる
2. **読みづらさ** — 古い星が小さすぎないか実機確認（Phase 2）
3. **操作領域** — 見た目は小さくても 44px ヒットボックス必須
4. **recent highlight との二重強調** — §9 未決
5. **ソートタイブレーク不一致** — §3 懸念。Phase 0 で対応検討
6. **新規投稿による index シフト** — 許容済み
7. **重要性との混同** — ヘルプ等で「新しいほど手前」と明示（将来）
8. **全件読み込み後** — 最古が XS（0.82）で止まり、それ以上の「奥」は表現できない
9. **ピン留めの古い投稿** — 初版は小さく見える。Phase 4 で昇格検討
