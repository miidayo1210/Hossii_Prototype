# 110 アカウントマージ将来仕様（未実装）

> **ステータス:** 将来仕様のみ。Phase 6 では自動 merge を実装しない。  
> **branch:** `feat/community-platform-phase2-6`（development 検証）。main / production 未反映。

## 背景

管理者発行参加 ID（101）と既存メールアカウントが別 `auth.users` として共存する。Phase 6 では明示的な連携（オプトインの membership 作成）のみ行い、**同一人物の自動統合は行わない**。

## 将来に必要な要素

1. **identity 確認** — 本人が同一人物であることの確認手順（メール・管理者承認等）。
2. **authorship 移行** — `hossii_authorships` を正本 ID へ再紐付け。欠損・競合時のポリシー。
3. **membership 統合** — `community_memberships` / `space_memberships` の重複解消。status の優先順位。
4. **rollback** — 誤マージ時の復旧手順と監査証跡。
5. **audit** — 実行者・対象・移行前後の ID マッピング。
6. **duplicate email** — 既存メールと synthetic email の衝突処理。
7. **conflict resolution** — 両アカウントに active membership がある場合のルール。

## 非目標（Phase 6）

- 自動 merge RPC / UI
- メールアドレスの強制上書き
- 投稿・MyLogs の自動再割当（手動以外）
