## 1. 背景と目的

### 1.1 背景（現行）

Hossii はスペース画面の中心キャラクターとして `HossiiLive` で描画されているが、利用者への **目的に沿った案内** を管理者が設定する手段は限られている。

| 現行の案内系 | 用途 | 限界 |
|-------------|------|------|
| `welcomeMessage` | 入室・ニックネーム設定時の歓迎 | スペース利用中の継続的な案内には使われない |
| `description` | スペース名横の一行説明（常時 pill） | Hossii の声かけではない |
| `IDLE_SPEECH`（ハードコード） | 12〜20 秒間隔の自発セリフ | 管理者がスペース目的に合わせて変更できない |
| 投稿読み上げ（F07） | ランダム投稿の読み上げ | 案内・促進メッセージではない |

### 1.2 目的（確定）

スペースの目的や過ごし方に合わせ、Hossii が **ガイド吹き出し（guideBubble）** で次を行えるようにする。

- 案内（このスペースで何をする場所か）
- 問いかけ（振り返り・対話のきっかけ）
- 投稿促進（タグ利用・記録の促し）

Hossii を単なる表示キャラクターではなく、**スペースの案内役**として機能させる。

### 1.3 非目的（将来）

- AI による文章自動生成
- 利用者ごとのパーソナライズ表示
- 吹き出しタップで投稿フォームを開く（Phase 1）
- メッセージごとの CTA・タグ選択・Pane 遷移（将来 Todo）

## 2. 用語

| 用語 | 定義 |
|------|------|
| **スペース Hossii** | `HossiiLive` が描画する中心キャラクター。本機能の話者 |
| **guideBubble** | 本機能のガイド用吹き出し。新しい独立表示チャネル |
| **idleBubble** | 既存の自発セリフ吹き出し（`HossiiLive` `.idleBubble`） |
| **投稿吹き出し** | キャンバス上の投稿 UI（`Tree.tsx` / `Bubble`）。別レイヤー |
| **目的別パッケージ** | システム固定のメッセージセット。管理者は **選択のみ**（本文編集不可） |
| **手動メッセージ** | 管理者がスペース独自に登録する文言（Phase 2 以降） |
| **話し方** | パッケージのみ / 手動のみ / 併用の設定モード |
| **Hossiiのひとこと** | 管理設定 UI のセクション名（利用者には表示しない） |

## 4. 現行実装の整理

### 4.1 Hossii 描画（現行）

| ファイル | 役割 |

**描画順（現行）:** 投稿吹き出し → `MyHossiiLayer` → `HossiiLive` → 装飾

### 4.2 HossiiLive の吹き出し種別（現行）

`HossiiLive.tsx` は次の吹き出しを **相互排他** で表示する（優先: 投稿反応 > タップ > brain > idle）。

| CSS クラス | 用途 | 管理者設定 |
|-----------|------|-----------|
| `.bubble` | 投稿感情リアクション | なし |
| `.longBubble` | タップ反応（`TAP_LINES`） | なし |
| `.brainBubble` | `useHossiiBrain`（voice ON 時） | なし |
| `.idleBubble` | 自発セリフ（`IDLE_SPEECH` / F07 読み上げ） | なし |

**自発セリフタイマー（現行・コード準拠）:** 12〜20 秒間隔で `scheduleIdleSpeech`。`readingEnabled` かつ投稿あり時 50% で投稿読み上げ。

### 4.3 既存案内フィールド（現行）

| `description` | `spaces.description` | `BasicInfoTab` | スペース名横 pill（91） |

### 4.4 設定画面（現行）

| ファイル | 役割 |

## 5. 既存機能との関係

### 5.1 welcomeMessage との分離（確定）

| 観点 | welcomeMessage | Hossiiガイド吹き出し |
|------|----------------|---------------------|
| 目的 | 入室・ニックネーム設定時の歓迎 | スペース利用中の活動案内 |
| 保存先 | `spaces.welcome_message` | `space_settings.hossii_guide`（案・18章） |
| 管理 UI | `BasicInfoTab` | `CharacterTab`「Hossiiのひとこと」 |
| 表示タイミング | 入室モーダル表示時 | スペース初期表示完了後（16章） |
| 話者表現 | モーダル内吹き出し | `HossiiLive` の guideBubble |

**確定:** 保存先・表示タイミング・管理 UI を **共有しない**。

### 5.2 description との関係（確定）

`description` は常時表示の一行説明（91）。guideBubble は Hossii の **声かけ** であり、置き換え・統合しない。

### 5.3 HossiiLive 内の吹き出し優先（確定＋未確定の切り分け）

`HossiiLive` コンテナ内のキャラ付属吹き出しは、**同時に 1 種類のみ** 表示する（現行の相互排他を拡張）。

| 種別 | 現行/新規 | Phase 1 |
|------|----------|---------|
| `.bubble`（投稿感情リアクション） | 現行 | 優先順位は **未確定（B2）** |
| `.guideBubble` | **新規** | Phase 1 で追加 |
| `.longBubble`（`TAP_LINES`） | 現行 | タップ時。ガイド再表示との関係は **未確定（B1）** |
| `.brainBubble` | 現行 | voice ON 時。guide 表示中は出さない（確定） |
| `.idleBubble` | 現行 | guide 表示中は **抑止**（確定） |

**確定:**

- guideBubble は idleBubble・brainBubble と **同時表示しない**
- guide 表示中は idle タイマーは動いてよいが、**idleBubble の描画は抑止**
- guide を閉じた後、idleBubble は **通常どおり復帰**

**未確定（B2）:** 投稿反応 `.bubble` 発生時に guideBubble をどうするか（26章 U5）。**Phase 1 暫定実装仮定**は 5.7 を参照。

### 5.4 TAP_LINES（longBubble）との関係（未確定）

Hossii タップ時の既存 `.longBubble` と、ガイド再表示の関係は **B1 確定まで実装しない**（17章・26章 U1〜U3）。Phase 1 では **タップによるガイド再表示は含めない**。

### 5.5 投稿吹き出し（確定）

キャンバス上の投稿 UI（`Tree.tsx` / `Bubble`、29・30）は **別レイヤー**。guideBubble と **同時表示してよい**。投稿操作を guide が覆って妨げないこと（24章）。

### 5.6 マイHossii 吹き出し（確定）

`MyHossiiLayer` の参加者吹き出し（103）は別レイヤー・別ライフサイクル。guideBubble と **同時表示してよい**。本機能のスコープ外。

### 5.7 Phase 1 暫定実装仮定（B2 確定前のみ・確定仕様ではない）

B2 未確定の間、実装を進める場合の **仮置き**:

- 投稿反応 `.bubble` 発生時は guideBubble を **一時的に非表示**（反応優先）
- 反応終了後、guide は **自動では復帰しない**（閉じた扱いにはしない。再表示はページ再読み込み後の初回表示のみ）
- B2 確定後に本節を更新または削除する

### 11.3 並べ替えの提案（暫定）

**現行 UI との整合:** `CharacterTab` のカスタム感情リスト等は登録順表示が主で、DnD 並べ替えは必須ではない。手動メッセージも Phase 2 は **登録順（追加順）** で表示・ランダム対象とする。**並べ替え（DnD / 上下ボタン）は将来 Todo** とする。

**理由:** Phase 2 の管理 UI をユーザ案の `[編集][削除][＋追加]` リストで完結でき、最小差分で実装できる。

### 11.4 文字数・件数（暫定・実装時確定可）

| 項目 | 候補値 | 根拠 |
|------|--------|------|
| 1 メッセージ最大文字数 | **80 文字** | `welcomeMessage` 100 文字より短め。吹き出し 1〜2 行想定 |
| 手動メッセージ最大件数 | **20 件** | 管理 UI の一覧性。ランダムプールとして十分 |
| 最小文字数 | **1 文字**（空白のみ不可） | 空文字保存禁止 |

### 11.5 テキスト安全表示（確定）

**禁止文字リストは設けない。** 次の方針で安全に表示する。

| 方針 | 内容 |
|------|------|
| 保存時 | 前後空白を trim。空文字は保存拒否 |
| 表示時 | HTML として解釈しない（React テキストノード） |
| 改行 | 入力は **1 行**（`<textarea>` 不使用、`<input>` または単行制限）。保存データに `\n` を含めない |
| 長文 | CSS で `overflow-wrap: break-word`、最大行数で省略（例: 3 行 + `…`） |
| 絵文字 | 許可（既存 Hossii 吹き出しと同様） |
| 重複 | **同一文言の重複登録を許可**（ランダムプールに同文が複数入る）。重複警告 UI は将来 Todo |

## 13. ランダム選択ルール（確定）

### 13.1 候補プールの構築（Phase 別）

| Phase | プールの構成 |
|-------|-------------|
| 1 | `resolvePackageMessages(packageKey)` の結果のみ。`customMessages` は無視 |
| 2 | `mode === 'custom'` 時は trim 済み有効手動メッセージのみ |
| 3 | `mode === 'mixed'` 時はパッケージ + 有効手動を結合 |

**有効メッセージの定義:** trim 後に 1 文字以上ある文字列のみ。空文字・空白のみはプールから除外。

### 13.2 選択アルゴリズム（確定・再現可能）

```
pool = buildPool(settings)   // 13.1
if pool.length === 0:
  return null                // 表示しない。エラー UI なし
if pool.length === 1:
  return pool[0]             // 常にこの 1 件
return pool[randomIndex]     // randomIndex は [0, pool.length) の一様乱数
```

| 入力状態 | 出力 |
|----------|------|
| `enabled: false` | 選択しない |
| 不正 `packageKey` | `pool = []` → 表示しない（10.5） |
| 有効候補 0 件 | 表示しない |
| 有効候補 1 件 | その 1 件 |
| 有効候補 n 件（n≥2） | 均等ランダム 1 件 |

**確定:** 選択結果はメモリ上のみ。投稿データ・DB・localStorage へ **保存しない**。

### 13.3 連続同一表示（暫定）

Phase 1 では **必須としない**。Phase 2 で任意 Todo: 直前表示と同一なら再抽選（最大 3 回）。管理画面「別の言葉を試す」はプレビュー用であり、利用者側の連続回避とは無関係。

## 16. 閉じる操作と保持（確定）

### 16.1 閉じる（確定）

利用者は guideBubble を閉じられる。閉じたら、**同一 `SpaceScreen` インスタンスが生存している間**、当該 `spaceId` の自動初回表示を止める。

### 16.2 保持範囲（確定）

| 観点 | 方針 |
|------|------|
| 保持単位 | `spaceId` ごと |
| 保持期間 | **ブラウザタブ内・JS メモリが生存している間**（`SpaceScreen` の state / ref） |
| Pane 切替 | 再表示しない（同一 `SpaceScreen`） |
| フルページ再読み込み | 保持は **消える**。初回表示を **してよい** |
| sessionStorage | **使わない**（再読み込み後も残り続け、15.2 と矛盾するため） |
| localStorage | **使わない** |
| ユーザーアカウント / DB | **保存しない** |

### 16.3 実装方式の比較と推奨（確定）

| 方式 | 再読み込み後 | Pane 切替 | 最小差分 |
|------|-------------|-----------|----------|
| **React state / ref（`SpaceScreen`）** | 保持クリア → 再表示可 ✅ | 同一インスタンスなら保持 ✅ | **推奨** |
| メモリ上の `Set<spaceId>`（モジュールスコープ） | クリア ✅ | 保持 ✅ | テスト時に注意 |
| sessionStorage | **残る ❌**（15.2 と矛盾） | 保持 ✅ | Phase 1 では不採用 |
| localStorage | 長期保持 ❌ | — | 不採用 |

**確定（Phase 1）:** `SpaceScreen` 内の `useRef<Set<string>>` または `useState` で `dismissedGuideSpaceIds` を管理する。

- 閉じる操作: `spaceId` を Set に追加
- 初回表示判定: Set に無ければ表示候補
- `SpaceScreen` アンマウントまたはフルリロードで Set は消える

**注意（現行）:** Pane 切替で `SpaceScreen` がアンマウントされる実装に将来変わった場合は、保持方式を再検討する（将来 Todo）。
