## 概要

コミュニティ管理者は自分のコミュニティのスペース管理画面のみを操作できる。  
Hossii スーパー管理者（運営）は全コミュニティを一覧表示し、各コミュニティのスペース管理画面へ遷移できる。

### 現状の問題点

- `SpacesScreen` のヘッダーにコミュニティ名が表示されておらず、どのコミュニティの管理画面か不明
- スーパー管理者がコミュニティ一覧を俯瞰して各コミュニティのスペース管理画面に遷移する画面が存在しない

## 修正 A: コミュニティ名をスペース管理画面に表示

> **実装ステータス:** 未実装

### 概要

`SpacesScreen` のヘッダーに「`{コミュニティ名}` のスペース管理」と表示し、ログイン中のコミュニティ管理者が自分のコミュニティのスペースを管理していることを明示する。

### 対象コンポーネント

- `src/components/SpacesScreen/SpacesScreen.tsx`

### UI 変更

```
【現状】
✨ Hossii   スペース管理              👤 アカウント

【変更後】
✨ Hossii   {communityName} のスペース管理   👤 アカウント
```

- `currentUser.communityName` を使用する
- `communityName` が取得できない場合は「スペース管理」にフォールバック

### データソース

`AuthContext` の `currentUser.communityName`（すでに取得済み）を参照するだけで実装可能。新たな API コールは不要。

```ts
// SpacesScreen.tsx
const { currentUser } = useAuth();
const pageTitle = currentUser?.communityName
  ? `${currentUser.communityName} のスペース管理`
  : 'スペース管理';
```

## 修正 B: スーパー管理者向けコミュニティ一覧画面

> **実装ステータス:** 未実装

### 概要

Hossii 運営（スーパー管理者）がログインすると、全コミュニティの一覧が表示される。コミュニティをクリックすると、そのコミュニティのスペース管理画面（`SpacesScreen`）に遷移する。

### スーパー管理者の定義

Supabase の `auth.users` テーブルで `app_metadata.role = "super_admin"` のユーザー。  
または別の識別子（メールアドレスのドメインなど）で判定する方法も検討する。

### ナビゲーションフロー

```
スーパー管理者ログイン
  → CommunitiesScreen（コミュニティ一覧）
      ├── コミュニティカード一覧（全コミュニティ）
      │     ├── コミュニティ名
      │     ├── ステータスバッジ（pending / approved / rejected）
      │     ├── スペース数
      │     └── [スペースを管理] ボタン
      └── コミュニティをクリック
            → SpacesScreen（選択したコミュニティの communityId でフィルタリング）
                  └── 戻るボタン → CommunitiesScreen へ戻る

コミュニティ管理者ログイン
  → SpacesScreen（自分の communityId でフィルタリング、従来通り）
```

### 新規ルート

| ルート名 | 画面 | 説明 |
|---|---|---|
| `communities` | `CommunitiesScreen` | スーパー管理者専用のコミュニティ一覧 |

`useRouter.ts` に `'communities'` を追加する。

### CommunitiesScreen の UI

| 要素 | 説明 |
|---|---|
| ヘッダー | 「✨ Hossii  コミュニティ管理」 + アカウントドロップダウン |
| コミュニティカード | コミュニティ名・ステータス・登録日時・スペース数 |
| ステータスバッジ | `pending`: 🟡 審査中 / `approved`: 🟢 承認済み / `rejected`: 🔴 却下 |
| スペースを管理ボタン | クリックで当該コミュニティの `SpacesScreen` へ遷移 |
| 空の状態 | 「登録済みコミュニティはありません」 |

### 必要な API 追加

`communitiesApi.ts` に全コミュニティ取得関数を追加する。

```ts
// src/core/utils/communitiesApi.ts に追加
export async function fetchAllCommunities(): Promise<Community[]> {
  // スーパー管理者のみ呼び出し可（RLS でガード）
}
```

Supabase 側の RLS:
```sql
-- スーパー管理者のみ全コミュニティを SELECT 可
CREATE POLICY "communities_select_super_admin" ON communities
  FOR SELECT USING (
    auth.jwt() ->> 'role' = 'super_admin'
    OR (auth.jwt()->>'app_metadata')::jsonb ->> 'role' = 'super_admin'
  );
```

### SpacesScreen への communityId 注入

現状の `SpacesScreen` は `currentUser.communityId` を `HossiiProvider` 経由で使用している。  
スーパー管理者が別のコミュニティのスペースを管理する場合は、選択したコミュニティの `communityId` を渡す必要がある。

対応方針: `SpacesScreen` に `overrideCommunityId?: string` prop を追加し、スーパー管理者が選択したコミュニティIDを注入できるようにする。

```ts
// SpacesScreen.tsx
type SpacesScreenProps = {
  overrideCommunityId?: string; // スーパー管理者が選択したコミュニティID
};
```

または、React Context / URL パラメータ（`#spaces?communityId=xxx`）で渡す方法も検討する。

### `SpacesScreen` の「戻るボタン」制御

| ユーザー種別 | 「戻る」の遷移先 |
|---|---|
| コミュニティ管理者 | なし（`SpacesScreen` がトップ） |
| スーパー管理者 | `CommunitiesScreen` へ戻る |
