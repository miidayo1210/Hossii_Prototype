# スペース管理画面でのスペース作成

## 概要

スペース管理画面（`SpacesScreen`）からスペースを作成・管理する機能。

## 現在の実装状況

### スペースの作成

- スペース名とスペースURL（スラッグ）を入力してスペースを作成できる
- スラッグは 3〜40 文字の半角英小文字・数字・ハイフンのみ使用可能
- 「自動生成」ボタンで 8 文字のランダムスラッグを生成できる
- スラッグの重複チェックあり

### スペースの永続化

- **localStorage**（`hossii.spaces`）に保存されるため、リロードしてもスペースは消えない
- Supabase が設定されている場合は DB にも同期される
  - エラー時に空配列が返っても、既存の localStorage データを上書きしないよう保護済み（`SET_SPACES` ガード）

### スペース URL でのアクセス

- `/s/[spaceURL]` または `/c/[communitySlug]/s/[spaceURL]` の形式で直接アクセス可能
- Supabase 未設定時は localStorage のスペースを即座に検索するため、ページ読み込み後すぐアクセスできる
- Supabase 設定時は読み込み完了後に検索（`spacesLoadedFromSupabase` フラグで制御）

### スペース URL の変更

- `SpacesScreen` のスラッグ編集、または `ShareTab`（スペース設定 > URL）から変更可能
- **変更後も「スペースが見つかりません」が表示されないよう修正済み**
  - `App.tsx` で `initialSlugHandledRef` を導入し、初回 URL 解決後はスペース状態変化による再トリガーをブロック

## 修正済みバグ

| # | 症状 | 修正内容 |
|---|------|----------|
| 1 | URLを開くと「スペースが見つかりません」と表示される | `initialSlugHandledRef` で初回ナビゲーション済み後は再解決しないよう修正（`App.tsx`） |
| 2 | リロードするとスペースが消える | localStorage 永続化済み。Supabase エラー時の空配列による上書きもガード済み（`useHossiiStore.tsx`） |
| 3 | IDを変更してもURLに反映されない（URLが古いままの状態での「not found」発生） | バグ1と同根。`initialSlugHandledRef` により修正済み |
