# スペース管理 - 各スペースの設定 - 背景設定

## 概要

背景設定は以下の2か所から操作できる。どちらから変更しても同じデータ（`Space.background` / `Space.savedBackgroundImages`）を読み書きするため、完全に連動する。

| 操作場所 | アクセス方法 |
|---|---|
| スペース設定画面 > 背景タブ | スペース設定 > サイドバー「背景設定」 |
| スペース管理一覧 > カード | カード上の「背景を変更」から直接操作 |

## 背景の種類

背景は「色」「パターン」「テーマ」「画像」の4カテゴリから選択できる（タブ切り替えUI）。

### 色

単色のカラープリセットから選択する。

| キー | ラベル | カラーコード |
|---|---|---|
| sky | 空色 | #EAF4FF |
| space | スペース色 | #E8F5E9 |
| sunset | 夕焼け | #FFF3E0 |
| lavender | ラベンダー | #F3E5F5 |
| mint | ミント | #E0F2F1 |
| peach | ピーチ | #FFE0E6 |
| cream | クリーム | #FFFEF0 |
| ocean | 海色 | #E1F5FE |

### パターン

柄パターンのプリセットから選択する。デフォルトは「霧（mist）」。

| キー | ラベル |
|---|---|
| mist | 霧 |
| dots | ドット |
| grid | グリッド |
| waves | 波 |
| stars | 星空 |

### テーマ

宇宙系のグラデーションテーマ。CSS グラデーションで実装されておりリロードでも消えない。

| キー | ラベル | 説明 |
|---|---|---|
| nebula | ネビュラ | 星雲のような幻想的な背景 |
| galaxy | ギャラクシー | 銀河のような渦巻き背景 |
| starfield | スターフィールド | 無数の星が輝く背景 |

### 画像

Zoom のビデオ背景設定に近いギャラリーUI。

#### ギャラリー操作

- **追加ボタン（`+`）**: 画像ファイルを選択してアップロード → ギャラリーに追加される
- **サムネイルをクリック**: その画像を現在の背景に設定（✓ チェックマークで現在選択中を表示）
- **`×` ボタン**: ギャラリーから削除（Supabase Storage からも削除される）
- 枚数カウンター（`N / 3 枚`）を表示

#### 保存上限

現在 **3枚** まで保存可能（`MAX_BACKGROUND_IMAGES = 3`）。
将来的に上限を増やす場合はこの定数を変更するだけでよい。

#### 永続化

| 状態 | source 値 | 永続化 |
|---|---|---|
| Supabase 設定済み | `cloud` | リロード後も保持 |
| Supabase 未設定 | `temp` | リロードで消える（ギャラリーには追加されない） |

#### ストレージパス

```
hossii-images バケット / backgrounds/{spaceId}/{timestamp}.jpg
```

## データ型

```ts
type SpaceBackground =
  | { kind: 'color'; value: string }                        // 例: "#EAF4FF"
  | { kind: 'pattern'; value: PatternKey }                  // 例: "mist", "nebula"
  | { kind: 'image'; value: string; source: ImageSource };  // temp / preset / cloud

// Space 型の関連フィールド
background?: SpaceBackground;          // 現在選択中の背景
savedBackgroundImages?: string[];      // 保存済み画像URLリスト（最大 MAX_BACKGROUND_IMAGES 枚）
```

## DB スキーマ（spaces テーブル）

| カラム | 型 | 説明 |
|---|---|---|
| `background` | jsonb | 現在の背景設定 |
| `saved_background_images` | jsonb | 保存済み背景画像URLの配列 |

## 今後の追加予定

- 保存上限の拡張（`MAX_BACKGROUND_IMAGES` を変更するだけ）
- プリセット画像の追加（`/bg/` ディレクトリへの画像追加）
