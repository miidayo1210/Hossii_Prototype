
# Hossii タグ機能まわりの現状整理と修正方針仕様書

> **ステータス: 解決済み（2026-03-09）**
> フェーズ1（`spaces.preset_tags` 保存の安定化）完了。

---

## 目的
Hossii で以下を実現したい。

1. スペースごとに「使えるタグ候補」を登録できる
2. 投稿時に、そのタグをワンタップで付与できる
3. ログ一覧でタグによるフィルタリングができる

---

## 調査で判明した原因

### 根本原因: RLS による UPDATE のサイレントブロック

`spaces` テーブルの UPDATE ポリシーは `auth.uid()` を使う条件だったが、
開発中は Supabase クライアントが未認証（`auth.uid() = NULL`）で動いていた。

結果として:
- PATCH リクエストは正しく飛んでいる（`preset_tags` も含まれている）
- Supabase が 204 を返す
- しかし 0 rows updated で DB は更新されない

### 確認したこと

| 項目 | 結果 |
|------|------|
| PATCH リクエストの送信 | ✅ 正常（DevTools で確認） |
| リクエストの保存先テーブル | ✅ `spaces` で正しい |
| WHERE 条件 (`id = '...'`) | ✅ 対象 row は存在する |
| `auth.uid()` | ❌ NULL（未認証） |
| RLS UPDATE ポリシー | ❌ `auth.uid()` 前提のため実質ブロック |

---

## 実施した修正

### 1. Supabase に開発用 RLS ポリシーを追加

```sql
-- 開発用: 本番前に必ず DROP すること
CREATE POLICY "dev_spaces_update_all" ON spaces
  FOR UPDATE USING (true);
```

これにより、認証なしでも `spaces` テーブルを UPDATE できるようになった。

### 2. `updateSpaceInDb` にデバッグ機能を追加（`src/core/utils/spacesApi.ts`）

- `supabase.auth.getUser()` で実行時の認証ユーザーをコンソール出力
- `.select()` を追加して更新結果を受け取る
- 0件更新の場合は `console.warn` でログ出力
- エラー時は `console.error` してから `throw`

### 3. 型コメントで責務を整理

- `space.ts`: `presetTags` を「スペースのタグ候補（管理者登録）」として明記
- `index.ts`: `Hossii` 型にフェーズ2で追加予定の `tags` フィールドをコメントアウト形式で記載
