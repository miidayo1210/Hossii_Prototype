# Hossii タグ機能まわりの現状整理と修正方針仕様書

> **ステータス: Development 実装完了（2026-07-24 更新）**
> フェーズ1〜3（Development）完了。
> Production への migration 適用・型揃えは未実施。

---

## 目的
Hossii で以下を実現したい。

1. スペースごとに「使えるタグ候補」を登録できる
2. 投稿時に、そのタグをワンタップで付与できる
3. ログ一覧でタグによるフィルタリングができる

---

## 調査で判明した原因

### 根本原因: RLS による UPDATE のサイレントブロック

`spaces` テーブルの UPDATE ポリシーは `auth.uid()` を使う条件だったが、
開発中は Supabase クライアントが未認証（`auth.uid() = NULL`）で動いていた。

結果として:
- PATCH リクエストは正しく飛んでいる（`preset_tags` も含まれている）
- Supabase が 204 を返す
- しかし 0 rows updated で DB は更新されない

### 確認したこと

| 項目 | 結果 |
|------|------|
| PATCH リクエストの送信 | ✅ 正常（DevTools で確認） |
| リクエストの保存先テーブル | ✅ `spaces` で正しい |
| WHERE 条件 (`id = '...'`) | ✅ 対象 row は存在する |
| `auth.uid()` | ❌ NULL（未認証） |
| RLS UPDATE ポリシー | ❌ `auth.uid()` 前提のため実質ブロック |

---

## 実施した修正

### 1. Supabase に開発用 RLS ポリシーを追加

```sql
-- 開発用: 本番前に必ず DROP すること
CREATE POLICY "dev_spaces_update_all" ON spaces
  FOR UPDATE USING (true);
```

これにより、認証なしでも `spaces` テーブルを UPDATE できるようになった。

### 2. `updateSpaceInDb` にデバッグ機能を追加（`src/core/utils/spacesApi.ts`）

- `supabase.auth.getUser()` で実行時の認証ユーザーをコンソール出力
- `.select()` を追加して更新結果を受け取る
- 0件更新の場合は `console.warn` でログ出力
- エラー時は `console.error` してから `throw`

### 3. 型コメントで責務を整理

- `space.ts`: `presetTags` を「スペースのタグ候補（管理者登録）」として明記
- `index.ts`: `Hossii.tags` をプリセットタグ用フィールドとして定義（DB は `hossiis.tags text[]`）

### 4. `hossiis.tags` カラムの正式追加（2026-07-24）

正本 migration:

`supabase/migrations/20260724200000_add_hossiis_tags.sql`

```sql
ALTER TABLE public.hossiis
  ADD COLUMN IF NOT EXISTS tags text[] DEFAULT '{}';
```

- `hashtags` と同じく `text[] DEFAULT '{}'`（NULL 許容）
- Development に適用済み
- Production には未適用（手動 jsonb が残っている可能性あり）

### 5. 投稿 INSERT 永続化・表示・絞り込み（2026-07-24）

| 項目 | 内容 |
|------|------|
| INSERT | `hossiisApi.hossiiToInsertRow` が `tags` を送信 |
| 復元 | `rowToHossii` が `tags` をマッピング（SELECT `*`） |
| 吹き出し | `Tree.tsx` で `tags`∪`hashtags`（重複除去）。`full` 等は時刻+タグ、`bubble` はタグがあるときだけタグ行 |
| ログ / モデレーション | 表示済み UI ＋ 和集合フィルタ（空 `tags[]` でも hashtags を維持） |

## 今後の対応

### 本番化の前にやること

1. 開発用ポリシーを削除する

```sql
DROP POLICY "dev_spaces_update_all" ON spaces;
```

2. 正しい Auth/RLS 設計に戻す（Auth 設計が確定したタイミングで）

3. Production に手動追加済みの `hossiis.tags`（調査時点では jsonb）がある場合、
   正式型 `text[]` への揃えを別タスクで実施する（**本タスクでは Production 未適用**）
