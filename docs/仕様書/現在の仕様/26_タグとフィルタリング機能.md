## 概要

スペースごとに「デフォルトタグ」を管理者が登録できる。
投稿者はそのタグを押すだけで付与でき（自由入力も従来通り可）、
コメント一覧・ログ一覧からタグを指定して投稿をフィルタリングできる。

### `tags` と `hashtags` の役割

| フィールド | 役割 | DB |
|-----------|------|-----|
| `tags` | 管理者が `presetTags` で用意し、投稿時にワンタップ選択したプリセットタグ | `hossiis.tags`（正本型: **`text[]`**。migration / Development） |
| `hashtags` | 自由入力、または本文中の `#タグ` 抽出 | `hossiis.hashtags`（`text[]`） |

両者はデータ上は常に分離する。表示・フィルタでは和集合を使ってよいが、ログ／モデレーションのチップでは見た目を分け、同一文字列の重複表示は避ける。

## T01 スペース設定：デフォルトタグ管理

### 対象コンポーネント

`SpaceSettingsScreen` > **タグ設定** タブ（`TagsTab.tsx`）

### UX フロー

```
管理者
  → SpaceSettingsScreen > タグ設定タブ
  → テキスト入力 + 「追加」ボタン でタグを追加
  → 登録済みタグ一覧（チップ形式）+ 各チップに「×」削除ボタン
  → 「保存」ボタンで確定 → Supabase の spaces テーブルへ反映
```

### 制約

| 項目 | 値 |
|------|----|
| タグ名の最大文字数 | 20 文字 |
| タグ登録上限 | 20 件 / スペース |
| `#` プレフィックス | 入力時に自動付与（入力は `質問` でも `#質問` でも可） |
| 重複チェック | 同名タグの重複登録を不可（大文字小文字を区別しない） |

## T02 投稿画面：デフォルトタグのワンタップ選択

### 概要

`Space.presetTags` が 1 件以上存在する場合、投稿フォームに「タグを選ぶ」セクションを表示。
タップするだけでタグを付与できる。自由入力（`hashtags` フィールドへの手入力）と併用可能。

### データ保存先の分離

| フィールド | 保存内容 | DB / 永続化 |
|-----------|---------|------------|
| `Hossii.tags` | プリセットから選んだタグ | `hossiis.tags text[]`（migration + INSERT 永続化済み） |
| `Hossii.hashtags` | 自由入力タグ / 本文中の `#タグ` 抽出 | `hossiis.hashtags text[]`（永続化済み） |

投稿時は `selectedPresetTags` → `Hossii.tags`、自由入力 / 本文抽出 → `Hossii.hashtags` に分離して保存する。

### parseHashtags のバグ修正（2026-03-10）

- 全角 `＃` を半角 `#` に正規化してから抽出（`text.replace(/＃/g, '#')`）
- `handleHashtagKeyDown` に `e.isComposing` チェックを追加し、日本語 IME 変換中の誤確定を防止
- `addHashtagFromInput` で `＃` / `#` 両方を先頭から除去

## T03 モデレーションタブ：タグ表示・フィルタリング

### 対象コンポーネント

`SpaceSettingsScreen` > **モデレーション** タブ（`ModerationTab.tsx`）

### 実装済みフィルター

| フィルター | 実装状況 |
|-----------|---------|
| 表示状態（全件 / 表示中 / 非表示） | ✅ 実装済み |
| キーワード検索（メッセージ・投稿者名） | ✅ 実装済み |
| 期間（from〜to） | ✅ 実装済み |
| タグ（ドロップダウン、複数選択 OR 条件） | ✅ 実装済み |

### タグ表示

各投稿行のメッセージ下にタグチップを表示する。

- **紫チップ**: `hossii.tags`（プリセット選択タグ）
- **グレーチップ**: `hossii.hashtags`（自由入力タグ）。`tags` と同一文字列は出さない（重複回避）
- `tags` / `hashtags` がどちらも空（null または `[]`）のときはチップ行自体を出さない

### タグフィルター候補

`presetTags` ＋ 投稿に実際についている `tags` / `hashtags` を収集してドロップダウンに表示。
タグが1件以上ある場合のみフィルターバーを表示。

### フィルターロジック

複数タグ選択時は **OR 条件**。`tags` と `hashtags` の和集合で判定する
（`tags` が DB default の空配列 `[]` でも `hashtags` を落とさない）。

```ts
const raw = selectedTags.map((t) => t.replace(/^#/, ''));
const searchIn = [...(h.tags ?? []), ...(h.hashtags ?? [])];
const hasTag = raw.some((t) => searchIn.includes(t));
```

## T04 コメント一覧：タグ表示・フィルタリング

### 対象コンポーネント

`LogListBody.tsx`（コメント一覧 / ログ一覧）

### タグ表示

各カードのメタ情報（時刻・いいね）の上にタグチップを表示する。

- **紫チップ**: `hossii.tags`（プリセット選択タグ）
- **グレーチップ**: `hossii.hashtags`（自由入力タグ）。`tags` と同一文字列は出さない（重複回避）
- `tags` / `hashtags` がどちらも空（null または `[]`）のときはチップ行自体を出さない

### タグフィルター UI

既存の「コメント / 気持ち」FilterBar の下に「タグで絞り込む」行を追加。

```
[コメント ☑] [気持ち ☑]
[#質問 ×] [#感想 ×] [タグで絞り込む ▼]
```

- ドロップダウンから選択 → 選択済みチップとして追加
- チップの「×」で解除
- `presetTags` ＋ 投稿の `tags` / `hashtags` を候補として収集
- タグが1件も存在しない場合はフィルター行を非表示

### フィルターロジック

T03 と同じ OR 条件（`tags` / `hashtags` の和集合）。

## T05 スペース画面：タグ絞り込み

> **✅ 実装済み** — 詳細は [84_スペース表示モード拡張](./84_スペース表示モード拡張.md)

### 概要

`displayHossiis`（期間・件数・viewMode 適用後）を、単一タグでさらに絞り込む。`tags` と `hashtags` の和集合で OR 一致（T04 と同じ）。

| デバイス | 操作 |
|----------|------|

### フィルタ後の影響

- バブル / 星の描画集合
- 投稿数バッジ
- 星プレビューローテ
- `ordered` 格子の件数再計算
- `byAuthor` の作者グループ集合

### 星モードでのタグ表示（プレビュー）

`presentationMode === 'stars'`（およびモバイルの星描画）では、星プレビュー／ホバープレビューに限りタグを出す。

- 対象: `tags` ∪ `hashtags`（同一文字列は1件、preset 優先）
- 最大2件、超過分は `+N`
- タグ0件のときは行・余白を出さない
- 色分け: プリセット＝紫 / 自由入力＝グレー（ログ一覧と同様）
- 操作ボタンにはしない（視覚のみ）

実装: `StarView.tsx` / `StarHoverPreview.tsx`（`hossiiDisplayTags.ts` / `HossiiDisplayTagRow.tsx`）

## 既知の注意事項・制約

### Supabase RLS（開発中の一時対応）

`spaces` テーブルの UPDATE は RLS によって `auth.uid()` が必要だが、
開発中は認証なし（`auth.uid() = NULL`）でも動作させるため、
以下の一時ポリシーを Supabase に追加済み：

```sql
-- 開発用: 本番前に必ず DROP すること
CREATE POLICY "dev_spaces_update_all" ON spaces
  FOR UPDATE USING (true);
```

**本番化の際は必ずこのポリシーを削除し、正しい Auth/RLS 設計に戻すこと。**

### `hossiis.tags` カラム

正本型は **`text[] DEFAULT '{}'`**（NULL 許容）。追加 migration:

`supabase/migrations/20260724200000_add_hossiis_tags.sql`

Production の既存 `jsonb` を正本へ揃える変換 migration（**Production 未適用**）:

`supabase/migrations/20260724210000_convert_hossiis_tags_jsonb_to_text_array.sql`

### タグ判定の和集合

フィルタ・スペース絞り込み・吹き出しフッターは `[...(h.tags ?? []), ...(h.hashtags ?? [])]` の和集合を使う。
`tags` が DB default の `[]` でも自由入力 `hashtags` を落とさない。

ログ／モデレーションのチップ表示はプリセット（紫）と自由入力（グレー）を分け、
同一文字列が両方にある場合はプリセット側のみ表示して重複を避ける。

### `viewMode === 'bubble'` でのタグ表示

コンパクト表示では時刻付きフッター全体は出さない（[80](./80_吹き出しレイアウトv2.md)）。
タグがある投稿に限り、フッター相当の位置に **タグ行のみ** を出す最小対応とする。
タグ 0 件の投稿には行を出さない（余分な空白なし）。

### Production との型差分（変換 migration 準備済み・**未適用**）

| 環境 | `hossiis.tags` | 備考 |
|------|----------------|------|
| 正本 | `text[] DEFAULT '{}'`（NULL 許容） | `20260724200000` |
| Development | `text[]` | 追加済み。変換 migration は no-op 検証済み |
| Production | **`jsonb DEFAULT '[]'::jsonb`**（NULL 許容） | read-only 調査済み。**未変換・未適用** |

**Production read-only 調査（件数は型安全性確認用。内容は非開示）:**

| 項目 | 結果 |
|------|------|
| 型 / nullable / default | `jsonb` / YES / `'[]'::jsonb` |
| 総行 | 約 890 |
| `NULL` | 多数（約 700） |
| 空配列 `[]` | 約 180 |
| 非空配列 | 7（いずれも配列・要素は string） |
| 非配列 jsonb | **0** |
| 非 string 配列要素 | **0** |

**変換 migration:** `20260724210000_convert_hossiis_tags_jsonb_to_text_array.sql`

- jsonb のときだけ適用前検査（非配列 / 非 string 要素 → EXCEPTION）
- 安全なら `text[]` へ変換（NULL 維持、`[]`→`{}`、順序保持）
- 既に `text[]` なら no-op
- 検証: `scripts/verify-hossiis-tags-jsonb-to-text-array.py`

**Production 未適用。** 適用は明示許可後のみ。

**適用後の受入項目（予定）:**

1. `information_schema` で `tags` が `text[]` / DEFAULT `'{}'` / nullable
2. 変換前後の NULL・空配列・非空件数が整合
