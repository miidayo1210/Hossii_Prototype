## 概要

スペースごとに「デフォルトタグ」を管理者が登録できる。
投稿者はそのタグを押すだけで付与でき（自由入力も従来通り可）、
コメント一覧・ログ一覧からタグを指定して投稿をフィルタリングできる。

---

| 機能ID | 概要 | 対象ユーザー |
|--------|------|------------|

## T01 スペース設定：デフォルトタグ管理

### 対象コンポーネント

`SpaceSettingsScreen` > **タグ設定** タブ（`TagsTab.tsx`）

### UX フロー

```
管理者
  → SpaceSettingsScreen > タグ設定タブ
  → テキスト入力 + 「追加」ボタン でタグを追加
  → 登録済みタグ一覧（チップ形式）+ 各チップに「×」削除ボタン
  → 「保存」ボタンで確定 → Supabase の spaces テーブルへ反映
```

### 制約

| 項目 | 値 |
|------|----|
| タグ名の最大文字数 | 20 文字 |
| タグ登録上限 | 20 件 / スペース |
| `#` プレフィックス | 入力時に自動付与（入力は `質問` でも `#質問` でも可） |
| 重複チェック | 同名タグの重複登録を不可（大文字小文字を区別しない） |

## T02 投稿画面：デフォルトタグのワンタップ選択

### 概要

`Space.presetTags` が 1 件以上存在する場合、投稿フォームに「タグを選ぶ」セクションを表示。
タップするだけでタグを付与できる。自由入力（`hashtags` フィールドへの手入力）と併用可能。

### データ保存先の分離（フェーズ2実装済み）

| フィールド | 保存内容 |
|-----------|---------|
| `Hossii.tags` | プリセットから選んだタグ（`hossiis.tags jsonb`） |
| `Hossii.hashtags` | 自由入力タグ / 本文中の `#タグ` 抽出（`hossiis.hashtags text[]`） |

### parseHashtags のバグ修正（2026-03-10）

- 全角 `＃` を半角 `#` に正規化してから抽出（`text.replace(/＃/g, '#')`）
- `handleHashtagKeyDown` に `e.isComposing` チェックを追加し、日本語 IME 変換中の誤確定を防止
- `addHashtagFromInput` で `＃` / `#` 両方を先頭から除去

## T03 モデレーションタブ：タグ表示・フィルタリング

### 対象コンポーネント

`SpaceSettingsScreen` > **モデレーション** タブ（`ModerationTab.tsx`）

### 実装済みフィルター

| フィルター | 実装状況 |
|-----------|---------|
| 表示状態（全件 / 表示中 / 非表示） | ✅ 実装済み |
| キーワード検索（メッセージ・投稿者名） | ✅ 実装済み |
| 期間（from〜to） | ✅ 実装済み |
| タグ（ドロップダウン、複数選択 OR 条件） | ✅ 実装済み |

### タグ表示

各投稿行のメッセージ下にタグチップを表示する。

- **紫チップ**: `hossii.tags`（プリセット選択タグ）
- **グレーチップ**: `hossii.hashtags`（自由入力タグ）

### タグフィルター候補

`presetTags` ＋ 投稿に実際についている `tags` / `hashtags` を収集してドロップダウンに表示。
タグが1件以上ある場合のみフィルターバーを表示。

### フィルターロジック

複数タグ選択時は **OR 条件**。`h.tags` を優先し、なければ `h.hashtags` にフォールバック（移行期対応）。

```ts
const raw = selectedTags.map((t) => t.replace(/^#/, ''));
const searchIn = h.tags ?? h.hashtags;
const hasTag = raw.some((t) => searchIn?.includes(t));
```

## T04 コメント一覧：タグ表示・フィルタリング

### 対象コンポーネント

`CommentsScreen.tsx`（コメント一覧 / ログ一覧）

### タグ表示

各カードのメタ情報（時刻・いいね）の上にタグチップを表示する。

- **紫チップ**: `hossii.tags`（プリセット選択タグ）
- **グレーチップ**: `hossii.hashtags`（自由入力タグ）

### タグフィルター UI

既存の「コメント / 気持ち」FilterBar の下に「タグで絞り込む」行を追加。

```
[コメント ☑] [気持ち ☑]
[#質問 ×] [#感想 ×] [タグで絞り込む ▼]
```

- ドロップダウンから選択 → 選択済みチップとして追加
- チップの「×」で解除
- `presetTags` ＋ 投稿の `tags` / `hashtags` を候補として収集
- タグが1件も存在しない場合はフィルター行を非表示

### フィルターロジック

T03 と同じ OR 条件（`h.tags ?? h.hashtags` でフォールバック）。

## データ型

### `Hossii` 型（`src/core/types/index.ts`）

```ts
hashtags?: string[]; // F09: 自由入力タグ / 本文抽出タグ
tags?: string[];     // T02: プリセットから選択したタグ（フェーズ2で追加）
```

### `Space` 型（`src/core/types/space.ts`）

```ts
presetTags?: string[]; // T01: スペースのデフォルトタグ候補（例: ["#質問", "#感想"]）
```

### DB スキーマ

```sql
ALTER TABLE spaces   ADD COLUMN preset_tags jsonb DEFAULT '[]';
ALTER TABLE hossiis  ADD COLUMN tags        jsonb DEFAULT '[]';
```

## 既知の注意事項・制約

### Supabase RLS（開発中の一時対応）

`spaces` テーブルの UPDATE は RLS によって `auth.uid()` が必要だが、
開発中は認証なし（`auth.uid() = NULL`）でも動作させるため、
以下の一時ポリシーを Supabase に追加済み：

```sql
-- 開発用: 本番前に必ず DROP すること
CREATE POLICY "dev_spaces_update_all" ON spaces
  FOR UPDATE USING (true);
```

**本番化の際は必ずこのポリシーを削除し、正しい Auth/RLS 設計に戻すこと。**

### `hossiis.tags` カラムの追加（手動実行が必要）

```sql
ALTER TABLE hossiis ADD COLUMN tags jsonb DEFAULT '[]';
```

Supabase コンソールで実行すること。

### 移行期フォールバック

`hossiis.tags` カラム追加前の旧データは `tags` が `null` になる。
フィルタ・表示ロジックは `h.tags ?? h.hashtags` でフォールバックしているため後方互換を維持している。
