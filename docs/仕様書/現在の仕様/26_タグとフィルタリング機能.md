## 概要

スペースごとに「デフォルトタグ」を管理者が登録できるようにする。
投稿者はそのタグを押すだけで付与でき（自由入力も従来通り可）、
ログ一覧からタグを指定して投稿をフィルタリングできる。

---

## 機能スコープ

| 機能ID | 概要 | 対象ユーザー |
|--------|------|------------|
| T01 | スペース設定でデフォルトタグを登録・編集・削除 | 管理者 |

| T03 | ログ一覧（ModerationTab）でタグフィルタリング | 管理者 |

## T01 スペース設定：デフォルトタグ管理

### 対象コンポーネント

`SpaceSettingsScreen` > **タグ設定** タブ（新設）

### UX フロー

```
管理者
  → SpaceSettingsScreen > タグ設定タブ
  → テキスト入力 + 「追加」ボタン でタグを追加
  → 登録済みタグ一覧（チップ形式）+ 各チップに「×」削除ボタン
  → 「保存」ボタンで確定 → Supabase の spaces テーブルへ反映
```

### UI コンポーネント

| 要素 | 説明 |
|------|------|
| タグ入力フィールド | テキスト入力。Enter / 「追加」ボタンで確定 |
| 登録済みタグ一覧 | チップ（pill）形式で表示。各チップに「×」ボタン |
| 保存ボタン | 変更を Supabase に保存 |
| ガイドテキスト | 例: 「投稿時にワンタップで選択できるタグを登録できます」 |

### 制約

| 項目 | 値 |
|------|----|
| タグ名の最大文字数 | 20 文字 |
| タグ登録上限 | 20 件 / スペース |
| `#` プレフィックス | 入力時に自動付与（入力は `質問` でも `#質問` でも可） |
| 重複チェック | 同名タグの重複登録を不可（大文字小文字を区別しない） |

## T02 投稿画面：デフォルトタグのワンタップ選択

### 概要

スペースにデフォルトタグが登録されている場合、投稿フォームに
「タグを選ぶ」セクションを表示し、タップするだけでタグを付与できる。
自由入力（従来の `hashtags` フィールドへの手入力）と併用可能。

### 表示条件

`Space.presetTags` が 1 件以上存在する場合のみセクションを表示する。

### UI

```
[タグを選ぶ]
  #質問  #感想  #提案  #告知    ← 押すとトグル選択（複数選択可）
```

- 選択中のチップは強調スタイル（背景色変化）
- 選択済みタグをもう一度タップ → 選択解除
- 自由入力タグとデフォルトタグは送信時にマージして `hashtags` フィールドへ格納

### データ変更なし

既存の `Hossii.hashtags: string[]` をそのまま使用する。新フィールドの追加は不要。

## T03 ログ一覧：タグフィルタリング

### 対象コンポーネント

`SpaceSettingsScreen` > **モデレーション** タブ（`ModerationTab.tsx`）

### 現状の既存フィルター

| フィルター | 実装状況 |
|-----------|---------|
| 表示状態（全件 / 表示中 / 非表示） | ✅ 実装済み |
| キーワード検索（メッセージ・投稿者名） | ✅ 実装済み |
| 期間（from〜to） | ✅ 実装済み |
| **タグ** | **未実装（今回追加）** |

### 追加する UI

既存の検索フォームに「タグ」フィルターを追加する。

```
[フィルターバー]
  表示状態: [全件] [表示中] [非表示]
  キーワード: [____________]
  期間: [from____] 〜 [to____]
  タグ: [#質問 ×] [#感想 ×]  ← 新規追加
         [+ タグを選ぶ ▼]
```

- 「+ タグを選ぶ」をクリック → スペースの `presetTags` 一覧がドロップダウン表示
- 選択したタグはフィルターチップとして追加（複数選択可）
- チップの「×」でそのタグのフィルター解除
- タグが設定されていない場合（`presetTags` が空）はこのセクション自体を非表示

### フィルターロジック

複数タグを選択した場合は **OR 条件**（いずれかのタグを含む投稿を表示）。

```ts
// ModerationTab.tsx の filtered ロジックに追加
if (selectedTags.length > 0) {
  const hasTag = selectedTags.some(tag =>
    h.hashtags?.includes(tag)
  );
  if (!hasTag) return false;
}
```

## 既知の注意事項・制約

### Supabase RLS（開発中の一時対応）

`spaces` テーブルの UPDATE は RLS によって `auth.uid()` が必要だが、
開発中は認証なし（`auth.uid() = NULL`）でも動作させるため、
以下の一時ポリシーを Supabase に追加済み：

```sql
-- 開発用: 本番前に必ず DROP すること
CREATE POLICY "dev_spaces_update_all" ON spaces
  FOR UPDATE USING (true);
```

**本番化の際は必ずこのポリシーを削除し、正しい Auth/RLS 設計に戻すこと。**

### `updateSpaceInDb` のデバッグログ

`src/core/utils/spacesApi.ts` の `updateSpaceInDb` に以下を追加済み：

- 実行時の認証ユーザー ID をコンソール出力（`[auth] updateSpaceInDb 実行時のユーザー:`）
- `.select()` で更新結果を取得
- 0件更新の場合は `console.warn` で検知
