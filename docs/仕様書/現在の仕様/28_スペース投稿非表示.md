# スペース投稿 非表示機能

## 概要

管理者がスペース上で「非表示」にした投稿（Hossii）を、**すべての画面から一貫して非表示**にする。

---

## 現在の実装状況

### データモデル（`hossiis` テーブル）

| カラム | 型 | 説明 |
|--------|-----|------|
| `is_hidden` | `boolean` | 非表示フラグ。デフォルト `false` |
| `hidden_at` | `timestamptz` | 非表示にした日時 |
| `hidden_by` | `uuid` | 非表示にした管理者の `userId` |

対応する TypeScript 型（`src/core/types/index.ts`）:
```ts
isHidden?: boolean;
hiddenAt?: Date;
hiddenBy?: string; // 管理者の userId
```

### 非表示操作（管理者のみ）

| 操作場所 | 方法 |
|----------|------|
| スペース画面（SpaceScreen） | 吹き出し選択 → 「🚫 非表示」ボタン |
| モデレーションタブ（SpaceSettingsScreen） | 投稿一覧 → 「非表示」ボタン |

操作は楽観的更新（即時反映）＋ Supabase への永続化。`moderation_logs` テーブルに `hide` / `restore` + `admin_id` で監査ログを記録。

### 各画面の対応状況

| 画面 | `isHidden` フィルタ | 状態 |
|------|-------------------|------|
| **スペース画面**（SpaceScreen） | ✅ あり（`displayHossiis` で除外） | 実装済み |
| **ログ一覧**（CommentsScreen） | ❌ なし（全件表示） | **未実装** |
| **スライドショー**（SlideshowView） | 未確認 | 要確認 |
| **モデレーションタブ** | — 意図的に全件表示（管理者用） | 正常 |

---

## 要件

### 非表示投稿の扱い

- 一般ユーザー・ゲストには**全画面で非表示**にする
- 管理者にも**通常画面では非表示**にする（モデレーションタブのみ全件確認可能）

### 対象画面

1. **ログ一覧（CommentsScreen）** — 非表示投稿を除外する（**優先対応**）
2. **スライドショー（SlideshowView）** — 非表示投稿を除外する（要確認・対応）

---

## 実装方針

### ログ一覧（CommentsScreen）

`CommentsScreen.tsx` が呼び出す `getActiveSpaceHossiis()` は現在フィルタなし:

```ts
// src/core/hooks/useHossiiStore.tsx（現状）
const getActiveSpaceHossiis = useCallback(() => {
  return state.hossiis.filter((h) => h.spaceId === state.activeSpaceId);
}, [state.hossiis, state.activeSpaceId]);
```

#### 対応案A: `CommentsScreen` 側でフィルタ

```ts
// CommentsScreen.tsx
const hossiis = getActiveSpaceHossiis().filter((h) => !h.isHidden);
```

メリット: 変更範囲が最小、既存の他コンポーネントに影響なし

#### 対応案B: `getActiveSpaceHossiis()` 自体でフィルタ

```ts
// useHossiiStore.tsx
const getActiveSpaceHossiis = useCallback(() => {
  return state.hossiis.filter(
    (h) => h.spaceId === state.activeSpaceId && !h.isHidden
  );
}, [state.hossiis, state.activeSpaceId]);
```

メリット: 一元管理  
デメリット: 呼び出し元が複数ある場合に意図しない影響が出る可能性あり（モデレーションタブは別取得なので問題なし）

**推奨: 対応案A**（CommentsScreen 側でフィルタ。影響範囲が明確）

---

## 確認済みのフロー

```
管理者が「非表示」を押す
  → hideHossii(id, adminId) が呼ばれる
  → dispatch('HIDE_HOSSII') → store の isHidden: true に更新
  → hideHossiiInDb(id, adminId) → Supabase の is_hidden: true に更新
  → insertModerationLog(..., action: 'hide') → 監査ログ記録

↓

スペース画面: displayHossiis フィルタで除外 ✅
ログ一覧: フィルタなしで表示されてしまう ❌ ← 本チケットの対応箇所
```
