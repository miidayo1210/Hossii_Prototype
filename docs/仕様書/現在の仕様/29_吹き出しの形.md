## 概要

吹き出し（Hossii カード）の**形状**と**色**のカスタマイズに関する仕様。  
形状には「コメ枠」形式の透過 PNG を使用し、フレームをオーバーレイ・背景色をフィルとして 3 レイヤーで表現する。

## 設計方針

### レンダリングモデル

PNG は「コメ枠」形式（枠は不透明・内部は透過・外部は透過）のため、**3 レイヤー構成**で表現する。

```
┌─────────────────────────────────┐
│  ① 色レイヤー（background-color） │  ← bubble div の background-color（フィル）
│  ② PNG フレーム（z-index: 1）   │  ← position: absolute の <img>（枠描画）
│  ③ テキスト（z-index: 2）       │  ← .bubbleInnerShaped（最前面）
└─────────────────────────────────┘
```

#### 実装アプローチ（PNG オーバーレイ方式）

> **注意:** `mask-image` を親要素に適用すると PNG の透明内部で子要素（テキスト・絵文字）も
> 透明にマスクされてしまうため不採用。代わりに PNG を `<img>` タグで重ねる。

```tsx
{/* フィル: bubble div の background-color */}
{bubbleShapePng && (
  <img
    src={bubbleShapePng}
    className={styles.bubbleShapeOverlay}  // position: absolute; inset: 0; z-index: 1
    alt=""
    draggable={false}
  />
)}
<div className={`${styles.bubbleInner} ${bubbleShapePng ? styles.bubbleInnerShaped : ''}`}>
  {/* z-index: 2 でテキスト・絵文字を PNG の上に表示 */}
</div>
```

バブルのサイズは **190×190 px 固定**（`.bubbleCustomShape`）にして PNG の縦横比に合わせる。スケール（`F05`）は引き続き有効。

## 機能仕様

### B01: Feature Flag `bubble_shapes_extended`

| 項目 | 内容 |
|---|---|
| Flag キー | `bubble_shapes_extended` |
| デフォルト | `false`（OFF） |
| 管理場所 | スペース設定 > Feature Flags タブ |

- **OFF（デフォルト）:** 吹き出しはデフォルト形状のみ。形状選択 UI は非表示。
- **ON:** 投稿画面・スペース設定に形状選択 UI が表示され、`bubble-shapes/` 内の PNG を選択できる。

---

### B02: 投稿ごとの形状選択（PostScreen）

`bubble_shapes_extended` が ON のとき、投稿画面に形状ピッカーを追加する。

#### UI 仕様

- 色選択ピッカー（既存 `F01: bubbleColor`）の隣か下に形状ピッカーを配置する。
- 形状ピッカーは横スクロール可能なサムネイル一覧。
- 各サムネイルは PNG を縮小表示（32×32 px 程度）。
- 「デフォルト」選択肢（形状なし）を先頭に置く。
- 選択中の形状はハイライト枠で示す。

#### データ

`Hossii` 型に `bubbleShapePng` フィールドを追加する。

```ts
// index.ts の Hossii 型に追加
bubbleShapePng?: string;  // 例: "/assets/bubble-shapes/Hossiiコメ枠ハート.png"
```

```ts
// AddHossiiInput にも追加
bubbleShapePng?: string;
```

#### 優先順位

投稿ごとの形状（`Hossii.bubbleShapePng`）> スペースのデフォルト形状（`Space.bubbleShapePng`）> デフォルト（形状なし）

---

### B03: スペースデフォルト形状（SpaceSettings）

`bubble_shapes_extended` が ON のとき、スペース設定（General タブ または専用タブ）に「デフォルト吹き出し形状」セクションを追加する。

- スペース内の全投稿に適用されるデフォルト形状を選べる。
- 投稿者が個別に形状を選んだ場合はそちらを優先する（B02 の優先順位参照）。
- 既存の `Space.bubbleShapePng` フィールドをそのまま使用する。

---

### B04: カスタム形状の追加（管理者運用）

カスタム形状を追加するには PNG ファイルを `public/assets/bubble-shapes/` に格納する。

- ファイル形式: **透過 PNG 推奨**
- アウトラインの線幅: 視認できる太さ（4px 以上推奨）
- 推奨サイズ: 200×200 px 以上（正方形ベース）
- ファイルを追加後、アプリ側のシェイプ一覧定義（定数）に追記する。

## 実装メモ

### Feature Flag フォールバック

`bubble_shapes_extended` は Supabase 未設定または RLS で書き込み不可の場合、`localStorage` にフォールバック保存される（`featureFlagsApi.ts`）。読み込み時も `localStorage` オーバーライドを最優先でマージする。ページリロード後も設定が保持される。

### CSS 設計ポイント

- `.bubbleCustomShape` は `overflow: hidden` を**設定しない**（カラーパレット・リサイズハンドルがクリップされるため）
- `.bubbleSelectedShape` の `overflow: visible` は `!important` を付けない（競合を避けるため）
- フレーム PNG の外側に背景色がはみ出す可能性があるが、190×190 px 正方形バブルと「コメ枠」デザイン上ほぼ目立たない
