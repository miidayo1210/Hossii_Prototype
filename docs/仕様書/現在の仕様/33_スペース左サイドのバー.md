## 2. ボタン構成と表示区分

### 2-1. 主要ボタン（バー上に常時表示）

| ボタン | アイコン | 説明 | PC | モバイル |
|--------|---------|------|-----|---------|
| 大画面表示 | Maximize2 / Minimize2 | スペース表示領域を最大化 | あり | あり |
| Hossii 表示 | Eye / EyeOff | Hossii キャラの表示切り替え | あり | あり |
| マイク | Mic / MicOff | 音声認識 ON/OFF | あり | あり |
| 読み上げ | Volume2 / VolumeX | 投稿の読み上げ ON/OFF。**既定 OFF**。初回 ON は [08 §F07.2](./08_スペース（HOME）の仕様書.md) の `VoiceConsentModal` | あり | あり |
| 表示倍率 | ZoomIn + % | 75% / 100% / 125% / 150% を循環 | あり | **なし**（バー上） |
| 投稿数バッジ | Hash | 右上に「N 件」を出す ON/OFF | あり | **なし**（[61](./61_スペース表示中投稿数表示.md)） |
| QR コード | QrCode | QR パネルの表示切替 | あり | **なし** |
| 島ワープ | 🏝 | 隣スペースへワープ（隣人ありのみ） | あり | あり |
| **`...` オーバーフロー** | `···` | 二次コントロールを展開するトグル | あり | あり |

### 2-2. オーバーフロー内（`...` 展開後）

| グループ | 内容 | PC | モバイル |
|---------|------|-----|---------|
| 表示モード | 📝 フル / 💬 バブル / 🖼 画像 / 🎞 スライドショー | あり | あり |
| 並び | 🔀 ランダム / ↔️ 投稿順 / 👥 投稿者まとめ | あり | あり |
| 期間 | 1日 / 1週 / 1月 / 全期 | あり | あり |
| 件数 | 50 / 100 / 150 / ∞ | あり | あり |

> **2026-06-27:** 投稿数 ON/OFF・ログ一覧・音声入力パネルは `...` 内から **削除**（PC 投稿数はバー上 `#` のみ。ログは下ナビ `#comments` / トリプルクリック等。音声はバー上マイク・Listen インジケータ）。

### 2-3. 表示設定のデフォルト（PC・モバイル共通）

初回利用時（localStorage 未設定）の `...` 内各項目:

| 項目 | デフォルト | 保存キー |
|------|-----------|---------|
| 件数 | **50** | `hossii.displayLimit` |
| 期間 | **全期間**（`all`） | `hossii.displayPeriod` |
| 表示 | **フル表示**（`full`） | `hossii.viewMode` |
| 並び | **ランダム**（`random`） | `hossii.layoutMode` |
| PC 星/カスタム | **星モード**（`stars`） | `hossii.presentationMode` |

モバイル縦は presentationMode に関わらず常に StarView。

## 3. `...` オーバーフローパネルの挙動

| 項目 | 仕様 |
|------|------|
| 開閉 | `...` ボタンクリックでトグル。開いているときボタンはアクティブスタイル |
| 表示位置（PC・スマホ横） | ボタンの **右側** にポップアウト（`position: absolute; left: calc(100% + 10px)`） |
| 表示位置（スマホ縦・横バー） | `...` ボタンの **真下**（`getBoundingClientRect` 基準、`top: bottom + 8px`） |
| スマホ（768px 以下）の描画 | **`document.body` へ Portal**。`position: fixed`、`z-index: 1100`。左バー（`z-index: 85`）のスタッキングコンテキスト外に出し、クリップ・隠れを防ぐ |
| スタイル | 左バー本体と同系（ガラスモーフィズム・角丸） |
| 閉じる操作 | パネル外タップ（`pointerdown`、**開いた直後の同一タップを避けるため 1 フレーム遅延**してリスナー登録）。パネル内タップでは閉じない |
| 永続化 | 開閉状態は **永続化しない**（ページ再読み込みで閉じる） |
| スライドショー中 | バー全体と同様に非表示 |

## 4. スマホ縦の横長バー仕様（確定）

```
position: fixed
top: max(52px, env(safe-area-inset-top) + 44px)
left: max(8px, env(safe-area-inset-left))
right: unset           ← 全幅に伸ばさない
width: fit-content     ← コンテンツ幅に収縮
max-width: min(70vw, 100vw - 安全域)  ← 画面から外れない程度に上限
flex-direction: row
overflow-x: auto       ← アイテムが多い場合の横スクロール
overflow-y: visible    ← 縦方向クリップは Portal 側で回避
border-radius: 1.25rem（全辺丸く）
```

**`...` ボタン:** `overflowWrapper` に `position: sticky; right: 0` を付与し、横スクロール時も右端に見えるようにする。

### 4-1. モバイルでの表示制御（CSS）

Tooltip ラッパーは `display: contents` のため、**直接子セレクタ**（`.controlBar > *:not(.mobileVisible)`）ではボタンごと消えてしまう。

| 方式 | 内容 |
|------|------|
| **採用（現行）** | `.controlButton:not(.mobileVisible) { display: none }` で PC 専用ボタンのみ非表示 |
| 区切り線 | モバイルでは `.controlBar > .divider { display: none }` |
| 各主要ボタン | ボタン要素に `mobileVisible` クラスを付与（Tooltip の外側ではなく **button 側**） |

## 5. ツールチップ

各ボタンにホバー説明ツールチップを付与する。詳細は [79_ツールチップ](./79_ツールチップ.md)。

| バー向き | ツールチップ出現方向 |
|---------|-----------------|
| 縦バー（PC・スマホ横） | 右側 |
| 横バー（スマホ縦） | 下側（79 の将来案。現行 `Tooltip` は右側固定） |

### 5-1. 読み上げ（🔊）ボタンの同意フロー

マイク（Listen）と同様、**初回 ON 時のみ** 同意モーダルを挟む（[08 §F07.2](./08_スペース（HOME）の仕様書.md)）。

| 操作 | 挙動 |
|------|------|
| OFF → ON（未同意） | `VoiceConsentModal` を表示。キャンセルなら OFF のまま |
| OFF → ON（同意済み） | 即 `voiceEnabled = true` |
| ON → OFF | 即 OFF。同意状態は維持 |
