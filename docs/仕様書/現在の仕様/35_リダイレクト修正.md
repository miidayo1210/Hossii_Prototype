# 35 リダイレクト修正

## 問題

`https://hossiiprototype.vercel.app/admin/login` を直接入力しても、`https://hossiiprototype.vercel.app/#spaces` が開いてしまい、管理ログイン画面が表示されなかった。

## 原因

### 原因1: Google OAuth の redirectTo がルートURL固定だった

`loginWithGoogle` が `redirectTo: window.location.origin`（= `https://hossiiprototype.vercel.app`）を使っていたため、Googleログイン後にルート `/` に戻ってしまっていた。その結果 `appRoute` が `'default'` になり、管理ログイン画面がスキップされてアプリが通常フローで起動していた。

### 原因2: ログイン済み管理者向けの中間画面があった

管理者がすでにログイン済みの状態で `/admin/login` を開くと「すでにログインしています」という中間画面が表示され、手動でボタンを押さないと管理画面へ進めなかった。これが「ログイン画面が出てこない」「`/#spaces` に飛んでしまう」という混乱を招いていた。

## 対応

### `AuthContext.tsx` の修正

Google OAuth の `redirectTo` を現在のパス（`window.location.pathname`）に変更。

```ts
// 修正前
options: { redirectTo: window.location.origin }

// 修正後
const redirectTo = `${window.location.origin}${window.location.pathname}`;
options: { redirectTo }
```

`/admin/login` からGoogleログインすると、認証後に `/admin/login` へ戻ってくる。

### `App.tsx` の修正

「すでにログインしています」中間画面を廃止し、`useEffect` で自動遷移するように変更。

- `/admin/login` を開いた時点で管理者ログイン済みであれば、自動的に管理画面（`#spaces` または `#communities`）へ遷移する
- 未ログインの場合は従来通り `AdminLoginScreen`（ログインフォーム）を表示する
- 認証解決中（`isResolvingAuth`）は暗背景のローディング画面を表示してフラッシュを防止する

## 修正後の動作

| 状況 | 修正前 | 修正後 |
|------|--------|--------|
| `/admin/login` を直接入力（未ログイン） | ログインフォーム表示 | ログインフォーム表示 |
| `/admin/login` を直接入力（ログイン済み） | 「すでにログインしています」画面 → ボタン押下で `#spaces` | 自動で管理画面へ遷移 |
| Googleログイン後 | ルート `/` に戻り管理ログイン画面が出ない | `/admin/login` に戻り自動で管理画面へ遷移 |

## 備考

`https://hossiiprototype.vercel.app/#spaces` および `https://hossiiprototype.vercel.app/#community` は直接URLとして使用しない。

- `#spaces` は管理者ログイン後にアプリ内部から遷移するルート
- `#community` は無効なルート（正しくは `#communities`）
- これらのURLを直接入力しても意図した画面は表示されない
