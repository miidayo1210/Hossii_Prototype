# 41: 個人ユーザー登録

## 背景・現状

現在のユーザー種別は以下の2種類のみ。

| 種別 | 登録内容 | 画面 |
|------|----------|------|
| 管理者（コミュニティ） | コミュニティ名・メアド・パスワード | 管理者向け登録・ログイン画面 |
| 一般ユーザー（ゲスト） | ニックネームのみ | 参加画面 |

一般ユーザーはゲストとして参加する場合、ニックネームだけで即座に参加できる。

## 目標

- ゲスト参加の手軽さ（ニックネームのみ）は**そのまま維持**する
- 参加画面にログイン・新規会員登録の案内を並べて表示する
- 個人ユーザーアカウントを持てるようにし、継続的に投稿できる仕組みを作る

## 現状の参加画面

```
✨ Hossii
「SAPテスト」に参加する

ゲストとして参加
ニックネームだけで参加できます

または

アカウントでログイン
過去ログの閲覧が可能になります
```

## 修正内容

### 参加画面の変更

- 現状の「ゲストとして参加」「アカウントでログイン」に加えて、**新規会員登録** の導線を追加する
- 3択を並べて表示する（ゲスト参加 / ログイン / 新規登録）

```
✨ Hossii
「SAPテスト」に参加する

ゲストとして参加
ニックネームだけで参加できます

または

アカウントでログイン
過去ログの閲覧が可能になります

または

新規会員登録
アカウントを作成して参加する
```

### 個人ユーザー登録フォーム

新規登録時の入力項目：

| 項目 | 必須 / 任意 |
|------|-------------|
| ユーザー名（ニックネーム可） | 必須 |
| メールアドレス | 必須 |
| パスワード | 必須 |
| 生年月日 | 任意 |
| 性別 | 任意 |

- その他のプロフィール情報は、登録後に「アカウント」画面から編集・追加できる

### 登録後の状態

- アカウントを持つ個人ユーザーとして投稿が可能になる
- ログイン済みの場合は過去の投稿ログを閲覧できる

## 備考

- 管理者（コミュニティ）のアカウントとは別の個人ユーザーアカウントとして扱う
- Supabase Auth を利用した認証基盤への統合を想定
