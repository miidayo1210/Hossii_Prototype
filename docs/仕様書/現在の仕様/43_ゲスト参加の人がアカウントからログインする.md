# 43: ゲスト参加中のアカウントログイン・新規登録

## 概要

スペース URL からゲストとして参加した後、スペース内の「アカウント」画面からログインまたは新規登録できる。認証完了後は元のスペースへ自動でリダイレクトされる。

---

## 修正1: ゲスト中の AccountScreen にログイン・登録ボタン

### 対象ファイル

- `src/components/AccountScreen/AccountScreen.tsx`
- `src/components/AccountScreen/AccountScreen.module.css`
- `src/App.tsx`

### UI

ゲスト（未ログイン）状態で「アカウント」タブを開いたとき：

```
ゲストとして参加中
アカウントを作成すると、複数の端末で同じ情報を使えます。

[アカウントでログイン]   ← 白・枠線ボタン (.loginButton)
[新規会員登録]           ← 紫グラデーションボタン (.signUpButton)
```

### 実装内容

**`AccountScreen.tsx`**
- `onLoginRequested?: () => void` と `onSignUpRequested?: () => void` を Props として受け取る
- `currentUser === null`（ゲスト）のときにボタンを表示する

**`App.tsx`**
- `handleGuestAuthRequested(mode: 'login' | 'signup')` 関数を追加
  - 現在のアクティブスペースの `spaceURL` を `pendingLoginSlug` にセット
  - `pendingAuthMode` を指定モードにセット
- `<AccountScreen onLoginRequested onSignUpRequested>` に上記ハンドラを渡す
- `LoginScreen` の表示条件を `!currentUser && pendingLoginSlug` に変更（従来は `!isGuestMode` 条件があり、ゲスト入室中は表示されなかった）

### ログイン完了後のフロー

認証成功 → `currentUser` がセット → App.tsx の `useEffect` が `pendingLoginSlug` を検知 → `window.location.href = /s/[slug]` でスペースへリダイレクト

## 修正2: GuestEntryScreen のボタンレイアウト整理

### 対象ファイル

- `src/components/Auth/GuestEntryScreen.tsx`
- `src/components/Auth/GuestEntryScreen.module.css`

### 変更前

```
[ゲストとして参加]
ニックネームだけで参加できます
───── または ─────
[アカウントでログイン]
過去ログの閲覧が可能になります
───── または ─────
[新規会員登録]
アカウントを作成して参加する
```

### 変更後

```
[ゲストとして参加]     ← primaryButton（紫グラデーション）
[アカウントでログイン]  ← secondaryButton（白・枠線）
新規登録の方はこちら    ← テキストリンク (.signUpLink)
```

### 実装内容

- 「または」区切り（`.divider`）と説明文（`.guestNote` / `.loginNote`）を JSX から削除
- 「新規会員登録」ボタンを `.signUpLink` クラスのテキストリンクスタイルに変更（アンダーライン付き小文字、紫色）
- `.buttons` の `gap` を `0` → `0.75rem` に変更してボタン間に適切な余白を確保

## 修正3: ニックネーム入力時の Hossii キャラサイズ問題

### 対象ファイル

- `src/components/Auth/GuestEntryScreen.module.css`

### 原因

`idle_base.png` と `idle_smile.png` は画像内部の透明余白量が異なる。`object-fit: contain` で表示すると smile への切り替え時に余白分だけキャラが小さく見えた。

### 修正内容

`.characterImage` を `object-fit: contain` → `object-fit: cover` に変更。コンテナサイズ（72×72px・`border-radius: 50%`）は固定のまま、画像が常にコンテナいっぱいに表示されるため切り替え時のサイズ変化がなくなった。
