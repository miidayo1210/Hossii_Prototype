# 45 投稿の位置を定める機能

## 機能概要

投稿画面（`PostScreen`）に **3×3 の位置選択グリッド** を追加する。
投稿者はスペース画面のどのエリアに気持ちを置くかを「なんとなく」選択して投稿できる。
選択は任意であり、未選択の場合は従来通りランダム配置となる。

---

## エリア定義

サッカーのストラックアウトのように、縦3列・横3行の計9エリアで構成される。

```
┌──────────┬──────────┬──────────┐
│  上・左   │  上・中   │  上・右   │
├──────────┼──────────┼──────────┤
│  中・左   │  中・中   │  中・右   │
├──────────┼──────────┼──────────┤
│  下・左   │  下・中   │  下・右   │
└──────────┴──────────┴──────────┘
```

各エリアは `areaIndex`（0〜8）で管理し、左上から右下へ番号を付与する。

| areaIndex | 位置名 | positionX の範囲 | positionY の範囲 |
|-----------|--------|-----------------|-----------------|
| 0 | 上・左 | 0.05 〜 0.28 | 0.05 〜 0.28 |
| 1 | 上・中 | 0.38 〜 0.62 | 0.05 〜 0.28 |
| 2 | 上・右 | 0.72 〜 0.95 | 0.05 〜 0.28 |
| 3 | 中・左 | 0.05 〜 0.28 | 0.38 〜 0.62 |
| 4 | 中・中 | 0.38 〜 0.62 | 0.38 〜 0.62 |
| 5 | 中・右 | 0.72 〜 0.95 | 0.38 〜 0.62 |
| 6 | 下・左 | 0.05 〜 0.28 | 0.72 〜 0.95 |
| 7 | 下・中 | 0.38 〜 0.62 | 0.72 〜 0.95 |
| 8 | 下・右 | 0.72 〜 0.95 | 0.72 〜 0.95 |

実際の `positionX` / `positionY` は、各エリアの範囲内でランダムに決定する。
これにより、同エリアに複数の投稿が重なりすぎるのを緩和する。

---

## UI 仕様

### 配置

投稿フォームの写真セクションの下・送信ボタンの上に配置する。

### コンポーネント構成

```
[ 置く場所（任意） ]

┌───┬───┬───┐
│ □ │ □ │ □ │  ← 上段
├───┼───┼───┤
│ □ │ □ │ □ │  ← 中段
├───┼───┼───┤
│ □ │ □ │ □ │  ← 下段
└───┴───┴───┘
選択中: 上・左  ／  [ クリアする ]
```

### グリッドの動作

- **デフォルト**: 未選択（全エリアが非アクティブ）
- エリアをタップ／クリックすると選択状態になり、ハイライト表示される
- 選択中のエリアを再度タップするとクリアされる（未選択に戻る）
- 選択中のエリア名をグリッドの下に表示する（例: 「上・左」）
- 未選択の場合は「スペース全体にランダム配置」と表示する

---

## 投稿フロー

1. ユーザーがグリッドでエリアを選択する（任意）
2. 「気持ちを置く」ボタンを押す
3. 選択されたエリアに対応する `positionX` / `positionY` 範囲内でランダム座標を決定する
4. `addHossii` に `positionX`・`positionY`・`isPositionFixed: true` を渡す
5. スペース画面では、`isPositionFixed: true` の投稿を指定座標に表示する

未選択の場合は `isPositionFixed` を未設定のままとし、ランダム配置になる。

### 対象ファイル

- `src/core/types/index.ts`（`AddHossiiInput` への `positionX` / `positionY` / `isPositionFixed` 追加）

### 前提

`Hossii` 型にはすでに `positionX`・`positionY`・`isPositionFixed` が存在する（`src/core/types/index.ts` の54〜57行目）。
`AddHossiiInput` 側への追加のみ必要。

### 変更内容

#### 1. `AddHossiiInput` への追加（`src/core/types/index.ts`）

```ts
positionX?: number;        // 0〜1 の相対座標
positionY?: number;        // 0〜1 の相対座標
isPositionFixed?: boolean; // true のとき上記座標で固定配置
```

#### 2. `PostScreen.tsx` への追加

state の追加:

```ts
const [selectedArea, setSelectedArea] = useState<number | null>(null);
```

エリアから座標を計算するヘルパー関数:

```ts
const AREA_LABELS = ['上・左', '上・中', '上・右', '中・左', '中・中', '中・右', '下・左', '下・中', '下・右'];

function areaToPosition(areaIndex: number): { x: number; y: number } {
  const col = areaIndex % 3;
  const row = Math.floor(areaIndex / 3);
  const ranges = [
    { min: 0.05, max: 0.28 },
    { min: 0.38, max: 0.62 },
    { min: 0.72, max: 0.95 },
  ];
  const rx = ranges[col];
  const ry = ranges[row];
  const rand = (min: number, max: number) => min + Math.random() * (max - min);
  return { x: rand(rx.min, rx.max), y: rand(ry.min, ry.max) };
}
```

`handleSubmit` 内での座標の付与:

```ts
const positionProps =
  selectedArea !== null
    ? { positionX: areaToPosition(selectedArea).x, positionY: areaToPosition(selectedArea).y, isPositionFixed: true }
    : {};

addHossii({
  message: message.trim(),
  emotion: selectedEmotion ?? undefined,
  // ... 既存のフィールド
  ...positionProps,
});
```

投稿後のクリア処理に以下を追加:

```ts
setSelectedArea(null);
```
