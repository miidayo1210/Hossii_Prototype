# 45 投稿の位置を定める機能

## 機能概要

投稿画面（`PostScreen`）に **3×3 の位置選択グリッド** を追加する。
投稿者はスペース画面のどのエリアに気持ちを置くかを「なんとなく」選択して投稿できる。
選択は任意であり、未選択の場合は従来通りランダム配置となる。

本機能は **Feature Flag `position_selector`** によってスペースごとに有効/無効を切り替えられる。
デフォルトは **OFF**（無効）。管理者がスペース設定画面の「Feature Flags」タブから ON にすることで有効化される。

---

## Feature Flag

| キー | デフォルト | 説明 |
|------|-----------|------|
| `position_selector` | `false` | 投稿時の位置選択グリッドを表示する |

Flag が OFF のとき、位置グリッドは UI に表示されない。投稿は従来通りランダム配置になる。
既存投稿の `isPositionFixed: true` データはフラグに関わらず正しい座標に表示される（表示側は変更なし）。

---

## エリア定義

サッカーのストラックアウトのように、縦3列・横3行の計9エリアで構成される。

```
┌──────────┬──────────┬──────────┐
│  上・左   │  上・中   │  上・右   │
├──────────┼──────────┼──────────┤
│  中・左   │  中・中   │  中・右   │
├──────────┼──────────┼──────────┤
│  下・左   │  下・中   │  下・右   │
└──────────┴──────────┴──────────┘
```

各エリアは `areaIndex`（0〜8）で管理し、左上から右下へ番号を付与する。

| areaIndex | 位置名 | positionX の範囲 | positionY の範囲 |
|-----------|--------|-----------------|-----------------|
| 0 | 上・左 | 5 〜 28 | 5 〜 28 |
| 1 | 上・中 | 38 〜 62 | 5 〜 28 |
| 2 | 上・右 | 72 〜 95 | 5 〜 28 |
| 3 | 中・左 | 5 〜 28 | 38 〜 62 |
| 4 | 中・中 | 38 〜 62 | 38 〜 62 |
| 5 | 中・右 | 72 〜 95 | 38 〜 62 |
| 6 | 下・左 | 5 〜 28 | 72 〜 95 |
| 7 | 下・中 | 38 〜 62 | 72 〜 95 |
| 8 | 下・右 | 72 〜 95 | 72 〜 95 |

> **座標スケールについて**: `positionX` / `positionY` は **0〜100 のパーセント値** として保存・表示する。
> `Tree.tsx` では `left: ${pos.x}%` / `top: ${pos.y}%` としてそのまま CSS に渡される。

実際の `positionX` / `positionY` は、各エリアの範囲内でランダムに決定する。
これにより、同エリアに複数の投稿が重なりすぎるのを緩和する。

---

## UI 仕様

### 配置

投稿フォームの写真セクションの下・送信ボタンの上に配置する。
Feature Flag `position_selector` が OFF のとき、このセクション全体を非表示にする。

### コンポーネント構成

```
[ 置く場所（任意） ]

┌───┬───┬───┐
│ □ │ □ │ □ │  ← 上段
├───┼───┼───┤
│ □ │ □ │ □ │  ← 中段
├───┼───┼───┤
│ □ │ □ │ □ │  ← 下段
└───┴───┴───┘
選択中: 上・左  ／  [ クリアする ]
```

### グリッドの動作

- **デフォルト**: 未選択（全エリアが非アクティブ）
- エリアをタップ／クリックすると選択状態になり、ハイライト表示される
- 選択中のエリアを再度タップするとクリアされる（未選択に戻る）
- 選択中のエリア名をグリッドの下に表示する（例: 「上・左」）
- 未選択の場合は「スペース全体にランダム配置」と表示する

---

## 投稿フロー

1. ユーザーがグリッドでエリアを選択する（任意）
2. 「気持ちを置く」ボタンを押す
3. 選択されたエリアに対応する `positionX` / `positionY` 範囲内でランダム座標を決定する（0〜100 のパーセント値）
4. `addHossii` に `positionX`・`positionY`・`isPositionFixed: true` を渡す
5. スペース画面では、`isPositionFixed: true` の投稿を指定座標に表示する

未選択の場合は `isPositionFixed` を未設定のままとし、ランダム配置になる。

---

## 実装仕様

### 対象ファイル

- `src/core/utils/featureFlagsApi.ts`（`FeatureFlagKey` への `position_selector` 追加）
- `src/core/hooks/useFeatureFlags.ts`（`DEFAULT_FLAGS` への追加）
- `src/components/SpaceSettingsScreen/FeatureFlagsTab.tsx`（管理 UI への追加）
- `src/components/PostScreen/PostScreen.tsx`（グリッド UI を Flag で条件分岐）

### 前提

- `Hossii` 型にはすでに `positionX`・`positionY`・`isPositionFixed` が存在する（`src/core/types/index.ts` の54〜57行目）
- `AddHossiiInput` 型にも `positionX`・`positionY`・`isPositionFixed` は追加済み（`src/core/types/index.ts` の87〜89行目）
- `PostScreen.tsx` には `areaToPosition` 関数・`selectedArea` state・グリッド UI がすでに実装済み
- Supabase の `space_feature_flags` テーブルのスキーマ変更は不要（`key` カラムで任意のキーを保存できる）

### 変更内容

#### 1. `featureFlagsApi.ts` — キー追加

```ts
export type FeatureFlagKey =
  | 'comments_thumbnail'
  | 'likes_enabled'
  | 'random_recall_enabled'
  | 'public_board_mode'
  | 'zine_export_enabled'
  | 'bubble_shapes_extended'
  | 'position_selector';  // ← 追加
```

`buildDefaults()` にも追加:

```ts
position_selector: false,
```

#### 2. `useFeatureFlags.ts` — デフォルト値追加

```ts
const DEFAULT_FLAGS: FeatureFlags = {
  // ... 既存フラグ ...
  position_selector: false,  // ← 追加
};
```

#### 3. `FeatureFlagsTab.tsx` — 管理 UI 追加

`FLAG_LIST` 配列に以下を追加:

```ts
{
  key: 'position_selector',
  label: '投稿位置の指定',
  description: '投稿時に3×3グリッドでスペース内のエリアを指定できるようにします。',
},
```

#### 4. `PostScreen.tsx` — グリッドを Flag で条件分岐

現在の位置選択グリッドセクションを `featureFlags.position_selector` で囲む。
`bubble_shapes_extended` と全く同じパターンで実装する:

```tsx
{/* 位置選択グリッド（position_selector が ON の場合のみ） */}
{featureFlags.position_selector && (
  <div className={styles.section}>
    <div className={styles.label}>置く場所（任意）</div>
    {/* ... グリッド UI（既存の実装をそのまま囲む） */}
  </div>
)}
```

#### 座標計算の注意点

`areaToPosition` は **0〜100 のパーセント値** を返す（`* 100` 適用済み）:

```ts
const rand = (min: number, max: number) => (min + Math.random() * (max - min)) * 100;
return { x: rand(ranges[col].min, ranges[col].max), y: rand(ranges[row].min, ranges[row].max) };
```
