# 46 連続投稿を可能にするボタン

## 機能概要

投稿画面（`PostScreen`）の「気持ちを置く」送信ボタンの直下に、**「連続で気持ちを置く」チェックボックス**を追加する。

チェックを入れた状態で投稿すると、スペース画面へ遷移せずに投稿画面に留まり、続けて気持ちを置けるようにする。

---

## 現在の投稿フロー（変更前）

1. ユーザーが投稿内容を入力する
2. 「気持ちを置く」ボタンを押す
3. 約 800ms 後に `navigate('screen')` でスペース画面（`#screen`）へ遷移する

---

## 変更後の投稿フロー

| チェックボックスの状態 | 投稿後の挙動 |
|---|---|
| **OFF**（デフォルト） | 従来通り、800ms 後にスペース画面へ遷移する |
| **ON** | スペース画面へ遷移しない。入力フォームはリセットされ、そのまま投稿画面に留まる |

---

## UI 仕様

### 配置

「気持ちを置く」送信ボタンの**直下**に配置する。

### コンポーネント構成

```
[ 気持ちを置く ボタン ]

□ 連続で気持ちを置く
```

### チェックボックスの動作

- **デフォルト**: OFF（チェックなし）
- **状態の保持**: 投稿してもチェックの ON/OFF はリセットしない。ユーザーが明示的に変更するまで維持する

---

## 実装仕様

### 対象ファイル

`src/components/PostScreen/PostScreen.tsx`

### 変更内容

1. `useState<boolean>` で `continuousPost` state を追加する（初期値: `false`）
2. `handleSubmit` 内の `navigate('screen')` 呼び出しを `continuousPost` フラグで条件分岐する
3. 送信ボタンの直下にチェックボックスを追加する

#### 条件分岐のロジック

```ts
if (!continuousPost) {
  setTimeout(() => {
    navigate('screen');
  }, 800);
}
```

### 補足（クイック投稿パネルとの関係）

パネルモード（[51_簡単投稿](./51_簡単投稿.md)）では、連続投稿が ON のときは投稿後もパネルを閉じず、メッセージ欄が有効ならそこへフォーカスを戻す。OFF のときはトースト表示後にパネルを閉じる。判定は送信完了時点のチェック状態に合わせる（非同期処理中にチェックを変えた場合も含む）。
