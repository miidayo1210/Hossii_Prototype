# 51 簡単投稿（クイック投稿パネル）

## 機能概要

スペース画面上の空きスペース（ボタンやバブル以外の場所）を **ダブルクリック / ダブルタップ** すると、スペース画面を表示したまま投稿フォームがパネル形式で出現し、その場で気持ちを置けるようにする。

投稿はダブルクリックした座標位置に配置される（`isPositionFixed: true`）。

---

## デバイス別レイアウト

### PC（768px より広い画面）

スペース画面の**右側 1/4** にサイドパネルとしてスライドインする。

```
┌────────────────────┬──────────┐
│                    │          │
│   スペース画面      │ 投稿     │
│   （そのまま表示）  │ パネル   │
│                    │          │
└────────────────────┴──────────┘
         75%                25%
```

### モバイル（768px 以下）

スペース画面の**下半分**にボトムシートとしてスライドアップする。スクロールで投稿フォームの全体を確認できる。

```
┌──────────────────────────┐
│                          │
│     スペース画面          │
│    （上半分・そのまま）    │
├──────────────────────────┤  ← ここから下にシート
│                          │
│       投稿フォーム        │
│    （スクロール可能）      │
│                          │
└──────────────────────────┘
```

---

## 投稿位置の決定

ダブルクリック / ダブルタップした座標を、スペースコンテナに対する **パーセント値（0〜100）** に変換して `positionX` / `positionY` として使用する。

```
positionX = (clientX - containerRect.left) / containerRect.width  * 100
positionY = (clientY - containerRect.top)  / containerRect.height * 100
```

投稿は `isPositionFixed: true` で保存されるため、スペース画面上の指定座標に表示される。

---

## トリガー条件

- **対象**: スペース画面のコンテナ要素への `onDoubleClick`
- **除外**: バブル・ボタン・コントロールバー・装飾要素など、インタラクティブな要素上でのダブルクリックは除外する（各要素で `e.stopPropagation()` により自然にキャンセルされる）
- **パネルが既に開いている場合**: 再度ダブルクリックしても新たに開かない（無視する）

---

## UI 仕様

### パネルの表示内容

既存の `PostScreen` をパネルモードで流用する。パネルモードでは以下を **非表示** にする：

- `TopRightMenu`（右上メニュー）
- `HossiiMini` ヘッダー（Hossii キャラクターとセリフ）
- タイトル「気持ちを置く 🌸」（任意）

投稿フォーム本体（感情ボタン・メッセージ・送信ボタン等）はそのまま表示する。

### 閉じるボタン

パネル右上（または上端中央）に **「✕ 閉じる」** ボタンを配置する。

### 投稿後の挙動

パネルモードでは、投稿完了後に `navigate('screen')` は呼ばない。代わりに `onClose()` を呼んでパネルを閉じ、スペース画面はそのまま表示し続ける。

### `PostScreen` の props 追加

```tsx
type Props = {
  panelMode?: 'side' | 'bottom';
  initialPosition?: { x: number; y: number };
  onClose?: () => void;
};

export const PostScreen = ({ panelMode, initialPosition, onClose }: Props) => { ... }
```

| prop | 型 | 説明 |
|------|---|------|
| `panelMode` | `'side' \| 'bottom' \| undefined` | `'side'`=PC サイドパネル、`'bottom'`=モバイル ボトムシート、`undefined`=従来のフルスクリーン |
| `initialPosition` | `{ x: number; y: number } \| undefined` | ダブルクリック座標（0〜100 のパーセント値）。指定時は `positionX`/`positionY` の初期値として使用し `isPositionFixed: true` で投稿 |
| `onClose` | `() => void \| undefined` | パネルを閉じるコールバック。`panelMode` 時に「閉じる」ボタンと投稿完了後に呼ぶ |

### `handleSubmit` の分岐変更

```ts
// 変更前
if (!continuousPost) {
  setTimeout(() => navigate('screen'), 800);
}

// 変更後
if (!continuousPost) {
  if (panelMode) {
    onClose?.();
  } else {
    setTimeout(() => navigate('screen'), 800);
  }
}
```

### `SpaceScreen` でのパネル管理

```tsx
const [quickPostPos, setQuickPostPos] = useState<{ x: number; y: number } | null>(null);

const handleDoubleClick = useCallback((e: React.MouseEvent<HTMLDivElement>) => {
  if (quickPostPos) return; // 既に開いている場合は無視
  const rect = e.currentTarget.getBoundingClientRect();
  const x = ((e.clientX - rect.left) / rect.width) * 100;
  const y = ((e.clientY - rect.top) / rect.height) * 100;
  setQuickPostPos({ x, y });
}, [quickPostPos]);

// パネル描画
{quickPostPos && (
  <PostScreen
    panelMode={isMobile ? 'bottom' : 'side'}
    initialPosition={quickPostPos}
    onClose={() => setQuickPostPos(null)}
  />
)}
```

## 追加仕様（改善要件）

### A. ダブルクリックによるパネルのトグル開閉

#### 現状の動作

パネルが開いている状態でダブルクリックしても何も起きない（`if (quickPostPos) return;` で無視）。

#### 変更後の動作

| 状態 | ダブルクリック時の挙動 |
|------|----------------------|
| パネルが**閉じている** | パネルをその座標で開く（現行どおり） |
| パネルが**開いている** | パネルを閉じる（`quickPostPos` を null にリセット） |

パネルを閉じる際は **位置情報も同時にリセット**される（`quickPostPos` を null にするだけなので自動）。次にダブルクリックした場所が新しい投稿位置として使われる。

#### 実装変更箇所

`SpaceScreen.tsx` の `handleDoubleClick` のみ変更。

```tsx
// 変更前
const handleDoubleClick = useCallback((e: React.MouseEvent<HTMLDivElement>) => {
  if (quickPostPos) return; // 無視
  ...
}, [quickPostPos]);

// 変更後
const handleDoubleClick = useCallback((e: React.MouseEvent<HTMLDivElement>) => {
  if (quickPostPos) {
    setQuickPostPos(null); // トグル: 開いていれば閉じる
    return;
  }
  const rect = e.currentTarget.getBoundingClientRect();
  const x = ((e.clientX - rect.left) / rect.width) * 100;
  const y = ((e.clientY - rect.top) / rect.height) * 100;
  setQuickPostPos({ x, y });
}, [quickPostPos]);
```

#### 補足

- パネルを閉じる方向（右→外）のスライドアウトアニメーションも追加できるが、実装コストがかかる（閉じ中 state が必要）。初期実装は即座に消える形でよい。
- 「✕ 閉じる」ボタンのクリックと動作は統一される（どちらも `setQuickPostPos(null)`）。

### B. サイドパネルのデザイン改善（立体感・浮遊感・ヘッダー非重複）

#### B-1. 立体感・浮遊感

「白い紙がスペース画面の右側に浮いている」ような見た目にする。

| 属性 | 値 | 意図 |
|------|----|------|
| `border-radius` | `20px 0 0 20px` | 左側（接続面の反対側）のみ丸めて、浮いている感を出す |
| `background` | `linear-gradient(160deg, #fff 0%, #fdf6ff 100%)` | わずかに紫寄りのホワイト。Hossii の世界観に合わせる |
| `box-shadow` | `−10px 4px 40px rgba(139,92,246,0.18), −2px 0 12px rgba(0,0,0,0.06)` | 紫がかった影を左下方向に落とすことで「浮いている」奥行きを出す |
| `border-left` | `1px solid rgba(255,255,255,0.9)` | 光のハイライト（白線）で手前に出ている感を強調 |

```css
.sidePanelWrapper {
  position: absolute;
  top: 4rem;                        /* TopBar の高さ分下げる */
  right: 0;
  width: 25%;
  min-width: 300px;
  height: calc(100% - 4rem);        /* TopBar 分の高さを除く */
  overflow-y: auto;
  background: linear-gradient(160deg, #fff 0%, #fdf6ff 100%);
  border-radius: 20px 0 0 20px;
  border-left: 1px solid rgba(255, 255, 255, 0.9);
  box-shadow:
    -10px 4px 40px rgba(139, 92, 246, 0.18),
    -2px 0 12px rgba(0, 0, 0, 0.06);
  animation: slideInRight 0.25s cubic-bezier(0.34, 1.56, 0.64, 1);
  z-index: 100;
}
```

> `cubic-bezier(0.34, 1.56, 0.64, 1)` は軽くオーバーシュート（ぷっくり感）するイージング。

#### B-2. ナビゲーション・ヘッダーへの重複解消

現状の `.sidePanelWrapper` は `top: 0; height: 100%` で画面最上部から始まるため、PC の TopBar（`4rem`）に被る。

**修正:** `top: 4rem; height: calc(100% - 4rem)` にすることで TopBar の下端から始める。

モバイルの `.bottomSheetWrapper` は `position: fixed` かつ下から出るため、TopBar との重複は発生しない（対応不要）。

#### B-3. 縦方向のコンパクト化（パネルモード専用スタイル）

`PostScreen` の `panelMode` 時は以下を縮小する。

| 要素 | 通常モード | パネルモード |
|------|-----------|------------|
| `main` の padding | `1.5rem 1rem` | `0.75rem 0.75rem` |
| セクション間 margin | `1.5rem` | `0.75rem` |
| タイトル「気持ちを置く 🌸」 | 表示 | **非表示**（すでに実装済み対象外だが追加で非表示推奨） |
| `panelCloseBar` の padding | `0.5rem 0.75rem` | そのまま |

```css
/* PostScreen.module.css へ追加 */
.panelMain {
  padding: 0.75rem 0.75rem;
}

.panelSection {
  margin-bottom: 0.75rem;
}
```

`panelMode` 時は `<main className={panelMode ? styles.panelMain : styles.main}>` に切り替える。

### C. 「気持ちを置く」ボタンの立体感・ポヨンアニメーション

#### C-1. 立体感（ネオモーフィズム風 3D ボタン）

```css
.submitButton {
  /* 既存スタイルに追加 */
  position: relative;
  top: 0;
  box-shadow:
    0 5px 0 #7c3aed,           /* 下面の影（ボタン本体の厚み） */
    0 8px 16px rgba(124, 58, 237, 0.35); /* 浮遊感のある落ち影 */
  border-bottom: none;
  transition: top 0.08s ease, box-shadow 0.08s ease;
}
```

| 状態 | `top` | `box-shadow` |
|------|-------|-------------|
| 通常 | `0` | 5px の下面影 + 浮遊影 |
| ホバー | `0` | 影をやや大きく（`0 7px 20px`） |
| 押下中（`:active`） | `3px` | 下面影を `2px` に縮小 → 「沈んだ」感 |

#### C-2. ポヨンアニメーション

ボタンを離した瞬間（`:active` が外れた後）にバネで戻るような弾みをつける。React の `onClick` ハンドラで CSS クラスを付け外しする方法が安定している。

```css
@keyframes poyon {
  0%   { transform: scale(1); }
  25%  { transform: scale(0.91); }   /* 押し込まれる */
  55%  { transform: scale(1.07); }   /* 戻りすぎ（バネ） */
  75%  { transform: scale(0.97); }   /* 微調整 */
  100% { transform: scale(1); }      /* 定位置 */
}

.submitButtonPoyon {
  animation: poyon 0.35s cubic-bezier(0.36, 0.07, 0.19, 0.97);
}
```

**実装方法（PostScreen.tsx）:**

```tsx
const [poyonActive, setPoyonActive] = useState(false);

const handleSubmitClick = () => {
  setPoyonActive(true);
  handleSubmit();
};

const handleAnimationEnd = () => setPoyonActive(false);

// JSX
<button
  onClick={handleSubmitClick}
  onAnimationEnd={handleAnimationEnd}
  className={`${styles.submitButton} ${poyonActive ? styles.submitButtonPoyon : ''}`}
  ...
>
```

#### C-3. 閉じるボタンへの適用

`panelCloseButton` にも同様の立体感・ポヨンを適用する（スケールは控えめに `scale(0.95)` → `scale(1.03)` 程度）。
