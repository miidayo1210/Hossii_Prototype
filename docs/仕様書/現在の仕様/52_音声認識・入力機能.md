# 52 音声認識・入力機能

> 最終更新: 2026-04-12（71 スマホマイクへの参照を追加）

## 背景・課題

現状、音声入力をオンにするとほとんどの音が「笑い声」「😂」の気持ちとして自動投稿されてしまう。
感情検知による自動投稿は後回しにし、まず「音声による文字起こしと手動投稿」を優先的な機能として整備する。

---

## 設計方針

**現状のコードを最大限活かす。** 既存の `useSpeechRecognition` をベースに、「パネル表示」と「候補リスト」を追加する形にする。

### 変更の方向性

```
変更前: 音声認識 → バッファ → 自動で addHossii（感情ログ）
変更後: 音声認識 → パネルに全文表示 → ユーザーが候補を選んで addHossii（手動のみ）
```

**感情ログ（マイク音量からの笑い声・環境音の自動感情投稿）は停止中。** `SpaceScreen` では `useAudioListener` をマウントしない。

**スペースへの自動投稿（ことばログ）は廃止。** `useSpeechRecognition` に `onSpeechEvent` を渡さず、文字起こし用の `onFinalSegment` のみ使用する。投稿は候補の「スペースに置く」など、ユーザーの明示操作でのみ行う。

パネルは「手動確認して投稿する」UIとして、上記と並存させる。

**レイアウト**: 音声パネルは `FloatingPanelShell` でラップされ、**ドラッグ移動・エッジ/角リサイズ・位置とサイズの localStorage 永続化**が可能（[56_クリックフィードバックとフローティングパネル](./56_クリックフィードバックとフローティングパネル.md)）。初期位置・既定サイズは `floatingPanelStorage` のスピーチ用デフォルトを使用する。

## 既存リソースの整理（変更不要なもの）

| 既存リソース | 内容 | 活用方法 |
|---|---|---|
| `useSpeechRecognition.ts` | 音声認識・ノイズ除去・言語検出（`interimResults: true` 設定済み） | `interimText`・`onFinalSegment`。`onSpeechEvent` は省略可（自動投稿バッファを使わない） |
| `SpeechLevel` 型（`word` / `short` / `long`） | 粒度の3段階分類 | 候補抽出の粒度設定にそのまま再利用 |
| `classifySpeechLevel` 関数 | 文字数ベースで粒度判定（日本語: 10字以下=word, 30字以下=short） | 将来 `onSpeechEvent` を再度使う場合のバッファ用（現状は未使用） |
| `SpeechLevelSettings` / `listenStorage.ts` | 粒度設定の localStorage 保存 | パネル内「短め／長め」と `setSpeechLevels` で連動 |
| `listenMode` フラグ / `DisplayPrefsContext` | マイク ON/OFF | 音声認識の有効化・左バー・パネル内トグルと同期 |
| 🎙 Listening インジケーター（`SpaceScreen.tsx`） | `listenMode` ON 時に画面下部に表示 | タップでパネル開閉（マイクは ON のまま） |
| `SpacePanelCubeDock` | 右端のキューブ（投稿・ログ・**音声**） | **PC（`768px` 超）のみ表示**。「音声」キューブでパネルだけ開閉。モバイルでは左バーのマイク／Listening インジケーター等で代替（下記） |

## 機能仕様

### 1. 音声パネルの表示トリガー

パネルの表示は **`speechPanelOpen`（ローカル state）** で制御する。`listenMode` だけではパネルを閉じない（右キューブでパネルだけ開く動線のため）。

- **左バーまたは Hossii トグルで Listen ON** → `listenMode` が `true` になったとき `useEffect` で **パネルも開く**（追加タップ不要）
- **画面右端の「音声」キューブ**（投稿・ログの下）→ **PC のみ**。パネルの開閉のみ。閉じるときは蓄積テキストもリセット
- **「🎙 Listening」インジケーター**（`listenMode` ON 時のみ表示）→ タップでパネル表示／折りたたみ（マイクは ON のまま）
- パネル内ヘッダー右上の **🎤 ON / OFF** コンパクトボタンで `listenMode` を切り替え（未同意時は `ListenConsentModal`）。左バーのマイクと同じ同意フロー

主な動線は次のとおり。

1. **共通**: 左で Listen ON にする → パネルも開く → 文字起こし・候補から手動投稿  
2. **PC**: 右の「音声」キューブでパネルを開く → ヘッダーの 🎤 で ON → 同上  
3. **モバイル**: キューブは出ないため、**左バーのマイク**または **Listening インジケーター**でパネルと Listen を操作  

ダブルタップ起動は採用しない。

**モバイル**では音声パネルも [56](./56_クリックフィードバックとフローティングパネル.md) の **`bottomInset`** により、固定 BottomNavBar 直下に内容が潜まない。

スマホ特有の **縦横レイアウト・没入（大画面）・横持ち案内** の方針は [71_スマホマイク機能.md](./71_スマホマイク機能.md) と [62_スマホのスペース画面.md](./62_スマホのスペース画面.md) を参照。

---

### 2. 右パネルのUI構成

```
┌────────────────────────────────────┐
│ 🎙 音声テキスト    [🎤 ON]  ✕     │  ← 右上はコンパクトな ON/OFF（立体感・キューブ風）
│                                    │
│ （ここに話した内容が               │
│  リアルタイムで流れていく）        │
│  認識中のグレー／確定は黒          │
├────────────────────────────────────┤
│ 候補 …                             │
│ 粒度: ● 短め  ○ 長め             │
└────────────────────────────────────┘
```

- **ヘッダー右上**: `🎤` + `ON` または `OFF` のみ（「音声入力」の文言は付けない）。横幅はコンパクト。
- **✕** はその右。閉じると蓄積テキストをリセット。

---

### 3. リアルタイム全文表示エリア

`useSpeechRecognition` の **`interimText`** と確定コールバック **`onFinalSegment`** で実現。

- 認識中（未確定）はグレー、確定は黒で追記
- 最大 300 文字で古いものを切り捨て
- パネルを閉じると全文リセット。`listenMode` OFF 時も蓄積はクリア
- `listenMode` OFF のときは説明文のみ表示（文字起こしなし）

---

### 4. 候補リスト

全文テキストから、パネル下部の「短め／長め」に応じてセグメントを切り出す（実装は `SpeechPanel` 内の `extractCandidates`）。最大 5 件。

候補から「スペースに置く」で **`addHossii`**（`origin: 'manual'`）。**音声からの自動 `addHossii` は行わない。**

---

### 5. 粒度設定

パネル下部の「短め / 長め」で `setSpeechLevels` を更新（`DisplayPrefsContext`）。

## 投稿フロー（現状）

1. パネルを表示する（左で Listen ON、または右「音声」キューブなど）
2. 必要ならパネル内 **🎤 ON** でマイク開始（同意済み or モーダル）
3. 話すと全文エリアに流れ、候補が更新される
4. 候補の「スペースに置く」で投稿
5. ✕ または「音声」キューブで閉じると蓄積リセット

## バグ修正記録

### Bug-1: パネルに文字が流れない（旧: `speechLogEnabled` 連動）

**原因（旧）**: `enabled: listenMode && speechLogEnabled` などにより認識が止まっていた。

**現状**: `enabled: listenMode` のみ。`speechLogEnabled` と自動投稿用 UI は削除済み。

---

### Bug-2: パネルが表示されない（z-index・親の overflow）

**対応**: `SpeechPanel` を `z-index: 300` 付近で `createPortal(..., document.body)` し、クリップを回避。

---

### Bug-3: 泣き笑い（😂）の自動投稿（音量ベース）

**対応**: `useAudioListener` は `SpaceScreen` で未使用。感情ログストレージは常に無効化。

---

### Bug-4: パネル表示と `listenMode` の解耦

**症状（旧）**: `listenMode && speechPanelOpen` だけだと、キューブで先にパネルを開けない等。

**現状**: 表示は `speechPanelOpen`。`listenMode` が `true` になったらパネルを開く effect を維持。マイク OFF でもパネルだけ開いたままにできる。
