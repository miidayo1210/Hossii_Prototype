# 53 スペース投稿表示並び替え

> 最終更新: 2026-04-05

## 機能概要

スペース画面の左サイドバー（`LeftControlBar`）に、吹き出しの並び方を切り替えるトグルを追加する。

投稿が増えてくると吹き出しが重なって見えにくくなるため、整列モードに切り替えることで見やすくなる。

---

## できるようになったこと（実装済み）

- **PC のスペース画面**（スライドショー以外）で、左サイドバー「並び」の **🔀（ランダム）** と **↔️（投稿順）** を切り替えられる。
- **投稿順**では、格子の形（行×列の分割）はそのままに、**左上から右へ折り返すセルの詰め順**だけを **降順 / 昇順** で切り替えられる（「降順」「昇順」ボタンは `SpaceScreen` 上に配置。`hossii.orderedSortDirection` に保存）。**PC**: 固定 TopBar の直下・スペース右上（`top: calc(4rem + 10px)`、`z-index: 95`）。**モバイル**: 画面上部・共有ボタンの左。**降順（既定）**: 左上がいちばん新しい投稿。**昇順**: 左上がいちばん古い投稿。件数に応じて列数が変わる（例: 9件 → 3列）。
- **投稿順の重ね順**: 格子の **セル番号が大きいほど手前**（右下セルが最前面）。昇順にするとリストの DOM 順は変わるが、`orderedGridIndex` ベースの `z-index` で **常に右下が最前面**に統一する（下記「動作仕様」参照）。
- **ランダム**では、従来どおりインデックスごとの決定論的疑似ランダム配置に戻せる。
- 選択した並びは **`localStorage`（キー `hossii.layoutMode`）** に保存され、リロード後も維持される。
- **ランダム（🔀）**では、手動でドラッグして保存した位置（`isPositionFixed: true`）が **従来どおり優先**される。
- **投稿順（↔️）**では、保存座標に関係なく **常に格子の計算座標**で全件表示し、見た目を揃える。ドラッグで保存した値は DB には残るが、投稿順表示中は使わない。**ランダムに戻すと**再び保存座標が反映される。
- 投稿順では **各吹き出しの左上**をセル角に揃える（サイズ差は `transform-origin: top left` で拡大も左上基準）。モバイルの星も `anchor: topLeft` で左上基準に合わせる。
- **モバイル**では `LeftControlBar` は出ないが、同じ `layoutMode` が `StarView` の位置計算にも使われる（表示は星 UI）。

## 表示モード定義

| モード | 動作 |
|--------|------|
| `random`（デフォルト） | 現在の動作。`createBubblePosition(index)` による決定論的疑似ランダム配置。ドラッグで自由移動可能 |
| `ordered` | 投稿の新しい順（左上から右へ折り返しながら）に格子状に整列。切り替え後もドラッグで自由移動可能 |

## UI 仕様

### 配置

`LeftControlBar` の「表示モード」ボタングループ（📝/💬/🖼/🎞）の直下、区切り線の後に追加する。

**表示条件:** `LeftControlBar` と同じく、表示モードがスライドショー（`viewMode === 'slideshow'`）のときはサイドバーごと非表示のため、このトグルも表示されない。

### コンポーネント構成

```
─── 区切り線 ───
[ 並び ]
[ 🔀 ] [ ↔️ ]
  ↑ランダム  ↑投稿順
```

- `🔀`（ランダム）：デフォルト、選択中はハイライト
- `↔️`（投稿順）：選択中はハイライト

### 投稿順モード時の昇順・降順（`SpaceScreen`）

左サイドバーの **↔️** で投稿順にしたあと、スペース画面上に **「降順」「昇順」** ボタンが出る（配置は「できるようになったこと」の PC / モバイルの記述どおり）。`LeftControlBar` には含めない。

## 動作仕様

### ランダムモード（`random`）

- 既存の `createBubblePosition(index)` による疑似ランダム座標で表示
- `isPositionFixed: true` の吹き出しは保存された座標を優先表示（既存仕様どおり）

### 投稿順モード（`ordered`）

- `displayHossiis` は常に新しい順（先頭が最新）。格子の各セル位置は `createOrderedBubblePosition(cellIndex, total)` で決まる。**降順**: リストの `index` とセル番号が同じ（最新が左上）。**昇順**: セル番号を `total - 1 - index` で割り当て（最古が左上）。格子の列数・セルサイズの計算は変わらない
- 列数は表示件数の平方根を基準に動的決定（例: 9件 → 3列、16件 → 4列）
- **保存済み固定座標は表示に使わない**（常に `createOrderedBubblePosition` の結果）。全件が同じルールで整列する
- 各セルは表示エリア（横 8〜92%、縦 12〜78%）を行×列で等分割し、**吹き出しの左上**がそのセルの左上に来る（`Bubble` に `layoutAlignTopLeft`、`StarView` に `anchor: topLeft`）
- **重ね順（実装）**:
  - **セル番号** `orderedGridIndex` は常に **0＝左上セル … `n - 1`＝右下セル**（`displayHossiis` の並びとは独立）。**降順**（既定）: `orderedGridIndex = index`（先頭＝最新が左上）。**昇順**: `orderedGridIndex = n - 1 - index`（古い投稿が左上）。
  - 表示座標は `bubblePositions[orderedGridIndex]`（`bubblePositions[k]` はセル `k` の左上座標）。
  - **スタック**: `orderedStackZ = orderedGridIndex + 1` を `Tree` の `Bubble` と `StarView` に渡す。CSS 変数 `--bubble-stack` / `--star-stack` として `z-index` に使い、**ホバー・選択・ドラッグ中**は `calc(var(--bubble-stack) + offset)` でさらに上に乗せる（`SpaceScreen.module.css` / `StarView.module.css`）。
- 配置後もドラッグで自由移動可能（保存後、ランダムモードに戻すとその座標が反映される）

### ランダムモードと固定座標（②）

手動でドラッグした吹き出しは `isPositionFixed: true` + `positionX/Y` として Supabase に保存される。**ランダムモード**では保存座標を優先表示する。投稿順モードでは整列を優先して計算座標のみ使うが、DB の値は消えない。

```tsx
// SpaceScreen.tsx（概要）
const pos =
  layoutMode === 'random' &&
  hossii.isPositionFixed &&
  hossii.positionX != null &&
  hossii.positionY != null
    ? { x: hossii.positionX, y: hossii.positionY }
    : bubblePositions[index];
```

### 設定の永続化

`localStorage` に保存し、リロード後も設定を保持する（他の表示設定と同じパターン）。

- **キー:** `hossii.layoutMode`
- **値:** `random` | `ordered`（`DisplayPrefsContext` の `prefs.layoutMode` / `setLayoutMode`）
- **キー:** `hossii.orderedSortDirection`（投稿順モード時の整列方向）
- **値:** `desc` | `asc`（`prefs.orderedSortDirection` / `setOrderedSortDirection`）

### 各ファイルの変更概要

#### 1. `displayPrefsStorage.ts` — `LayoutMode` / `OrderedSortDirection` 追加

`ViewMode` と同じパターンで `LayoutMode` を追加。投稿順の昇降順は `OrderedSortDirection`（キー `hossii.orderedSortDirection`、既定 `desc`）。

```ts
export type LayoutMode = 'random' | 'ordered';
export type OrderedSortDirection = 'asc' | 'desc';

const LAYOUT_MODE_KEY = 'hossii.layoutMode';
const ORDERED_SORT_KEY = 'hossii.orderedSortDirection';
```

#### 2. `DisplayPrefsContext.tsx` — `layoutMode` / `orderedSortDirection` 組み込み

State 型・Action 型・reducer・Context 型・Provider 初期値に `layoutMode` / `setLayoutMode` と `orderedSortDirection` / `setOrderedSortDirection` を追加する（他の設定と同一パターン）。

#### 3. `bubblePosition.ts` — 順番配置関数を追加

```ts
export function createOrderedBubblePosition(index: number, total: number): { x: number; y: number } {
  const cols = Math.max(1, Math.ceil(Math.sqrt(total)));
  const rows = Math.ceil(total / cols);
  const col = index % cols;
  const row = Math.floor(index / cols);
  const cellW = 84 / cols;
  const cellH = 66 / Math.max(rows, 1);
  const x = 8 + col * cellW;   // セル左上（横は 8〜92% 相当）
  const y = 12 + row * cellH;  // セル左上（縦は 12〜78% 相当、createBubblePosition と同じ帯）
  return { x, y };
}
```

`total <= 0` のときは実装側で安全なフォールバック座標を返す（通常は空表示のため呼ばれない）。

#### 4. `SpaceScreen.tsx` — 格子座標・セル割当・重ね順・昇降順 UI

`layoutMode` / `orderedSortDirection` は `useDisplayPrefs()` から取得。

**`bubblePositions`（セル `0…n-1` の左上座標）:**

```tsx
const bubblePositions = useMemo(() => {
  if (layoutMode === 'ordered') {
    return displayHossiis.map((_, i) =>
      createOrderedBubblePosition(i, displayHossiis.length)
    );
  }
  return displayHossiis.map((_, i) => createBubblePosition(i));
}, [displayHossiis, layoutMode]);
```

**`orderedGridIndex` と表示座標・スタック（`displayHossiis.map` 内）:**

```tsx
const n = displayHossiis.length;
const orderedGridIndex =
  layoutMode === 'ordered' && orderedSortDirection === 'asc'
    ? n - 1 - index
    : index;

const pos =
  layoutMode === 'random' &&
  hossii.isPositionFixed &&
  hossii.positionX != null &&
  hossii.positionY != null
    ? { x: hossii.positionX, y: hossii.positionY }
    : layoutMode === 'ordered'
      ? bubblePositions[orderedGridIndex]
      : bubblePositions[index];

// Bubble / StarView へ（投稿順のときのみ）
orderedStackZ={layoutMode === 'ordered' ? orderedGridIndex + 1 : undefined}
```

**昇順 / 降順ボタン:** `layoutMode === 'ordered'` のとき表示。`setOrderedSortDirection('desc' | 'asc')` を呼ぶ。

#### 5. `Tree.tsx`（`Bubble`）— `--bubble-stack`

`orderedStackZ` が渡されたとき、`style` に `'--bubble-stack': orderedStackZ` を設定。`.bubbleLayoutGrid`（`SpaceScreen.module.css`）が `z-index: var(--bubble-stack)` を参照する。

#### 6. `StarView.tsx` — `--star-stack`

モバイルの星表示でも同じ `orderedStackZ` を渡し、`--star-stack` と `.starOrderedStack` で重ね順を統一。

#### 7. `LeftControlBar.tsx` — ランダム ↔ 投稿順

`VIEW_MODE_OPTIONS` と同様のボタングループで **🔀 / ↔️**。Props に `layoutMode` / `onLayoutModeChange`。昇降順ボタンはここではなく **`SpaceScreen`** 側。

## 将来対応（今回スコープ外）

### 配置保存機能（3スロットまで）

現状のランダムモードで誰かが整理した吹き出し配置を、名前付きで最大3枠まで保存・復元できる機能。

**実装が必要なもの:**
- 全バブルの現在座標をスナップショットとして保存する処理
- 最大3枠のスロット管理（localStorage か Supabase）
- 保存・復元のUI（スロット選択・上書き確認など）

**重くなるか？**

- localStorage 保存のみであれば件数が多くても軽い（座標データは小さい）
- Supabase 保存にする場合は保存操作のタイミング（手動保存ボタン方式）にすれば、リアルタイムの負荷は増えない
- 全件座標を都度 Supabase に同期する構成にすると重くなる可能性があるため、手動保存方式を推奨
