# 53 スペース投稿表示並び替え

## 機能概要

スペース画面の左サイドバー（`LeftControlBar`）に、吹き出しの並び方を切り替えるトグルを追加する。

投稿が増えてくると吹き出しが重なって見えにくくなるため、整列モードに切り替えることで見やすくなる。

---

## 実装スコープ

今回対応するのは以下の2点のみ。配置保存機能（３スロット）は将来対応とする。

| # | 内容 | 対応 |
|---|------|------|
| ① | モード切り替えトグル（ランダム ↔ 投稿順整列） | **今回実装** |
| ② | モード切り替え時に手動移動済みの位置を保持 | **今回実装（追加実装不要・既存仕様で対応済み）** |
| ③ | 配置保存機能（3スロットまで） | 将来対応 |

---

## 表示モード定義

| モード | 動作 |
|--------|------|
| `random`（デフォルト） | 現在の動作。`createBubblePosition(index)` による決定論的疑似ランダム配置。ドラッグで自由移動可能 |
| `ordered` | 投稿の新しい順（左上から右へ折り返しながら）に格子状に整列。切り替え後もドラッグで自由移動可能 |

---

## UI 仕様

### 配置

`LeftControlBar` の「表示モード」ボタングループ（📝/💬/🖼/🎞）の直下、区切り線の後に追加する。

### コンポーネント構成

```
─── 区切り線 ───
[ 並び ]
[ 🔀 ] [ ↔️ ]
  ↑ランダム  ↑投稿順
```

- `🔀`（ランダム）：デフォルト、選択中はハイライト
- `↔️`（投稿順）：選択中はハイライト

---

## 動作仕様

### ランダムモード（`random`）

- 既存の `createBubblePosition(index)` による疑似ランダム座標で表示
- `isPositionFixed: true` の吹き出しは保存された座標を優先表示（既存仕様どおり）

### 投稿順モード（`ordered`）

- 新しい順（`displayHossiis` の先頭 = 最新）に左上から右へ折り返しながら格子状に配置
- 列数は表示件数の平方根を基準に動的決定（例: 9件 → 3列、16件 → 4列）
- `isPositionFixed: true` の吹き出しは保存済み座標を優先表示（ランダムモード同様）
- 配置後もドラッグで自由移動可能

### モード切り替え時の位置保持（②）

**追加実装不要。** 手動でドラッグした吹き出しは `isPositionFixed: true` + `positionX/Y` としてSupabaseに保存されており、モードを切り替えても上書きされない。

```tsx
// SpaceScreen.tsx の既存ロジック（変更なし）
const pos = hossii.isPositionFixed && hossii.positionX != null && hossii.positionY != null
  ? { x: hossii.positionX, y: hossii.positionY }  // 保存済み座標を優先
  : bubblePositions[index];                         // モードに応じた計算座標
```

### 設定の永続化

`localStorage` に保存し、リロード後も設定を保持する（他の表示設定と同じパターン）。

### 各ファイルの変更概要

#### 1. `displayPrefsStorage.ts` — `LayoutMode` 追加

`ViewMode` と全く同じパターンで追加する。

```ts
export type LayoutMode = 'random' | 'ordered';

const LAYOUT_MODE_KEY = 'hossii.layoutMode';
const DEFAULT_LAYOUT_MODE: LayoutMode = 'random';

export function loadLayoutMode(): LayoutMode { ... }
export function saveLayoutMode(mode: LayoutMode): void { ... }
```

#### 2. `DisplayPrefsContext.tsx` — `layoutMode` 組み込み

State型・Action型・reducer・Context型・Provider初期値の5箇所に `layoutMode` / `setLayoutMode` を追加する（他の設定と同一パターン）。

#### 3. `bubblePosition.ts` — 順番配置関数を追加

```ts
export function createOrderedBubblePosition(index: number, total: number): { x: number; y: number } {
  const cols = Math.max(1, Math.ceil(Math.sqrt(total)));
  const rows = Math.ceil(total / cols);
  const col = index % cols;
  const row = Math.floor(index / cols);
  const x = 8 + (col / Math.max(cols - 1, 1)) * 84;
  const y = 15 + (row / Math.max(rows - 1, 1)) * 60;
  return { x, y };
}
```

#### 4. `SpaceScreen.tsx` — `bubblePositions` を `layoutMode` で分岐

```tsx
// 変更前
const bubblePositions = useMemo(() => {
  return displayHossiis.map((_, index) => createBubblePosition(index));
}, [displayHossiis]);

// 変更後
const bubblePositions = useMemo(() => {
  if (layoutMode === 'ordered') {
    return displayHossiis.map((_, i) => createOrderedBubblePosition(i, displayHossiis.length));
  }
  return displayHossiis.map((_, i) => createBubblePosition(i));
}, [displayHossiis, layoutMode]);
```

`layoutMode` は `useDisplayPrefs()` から取得する。

#### 5. `LeftControlBar.tsx` — トグルボタン追加

`VIEW_MODE_OPTIONS` と同じパターンで `LAYOUT_MODE_OPTIONS` を追加し、ボタングループとして表示する。Props に `layoutMode` / `onLayoutModeChange` を追加する。

## 将来対応（今回スコープ外）

### 配置保存機能（3スロットまで）

現状のランダムモードで誰かが整理した吹き出し配置を、名前付きで最大3枠まで保存・復元できる機能。

**実装が必要なもの:**
- 全バブルの現在座標をスナップショットとして保存する処理
- 最大3枠のスロット管理（localStorage か Supabase）
- 保存・復元のUI（スロット選択・上書き確認など）

**重くなるか？**

- localStorage 保存のみであれば件数が多くても軽い（座標データは小さい）
- Supabase 保存にする場合は保存操作のタイミング（手動保存ボタン方式）にすれば、リアルタイムの負荷は増えない
- 全件座標を都度 Supabase に同期する構成にすると重くなる可能性があるため、手動保存方式を推奨
