# 54 吹き出しの色・カラーパレット

## 概要

投稿画面（`PostScreen`）で吹き出しの背景色を選ぶ。従来の「固定 8 色 + デフォルト」に代わり、**4 つのテーマ別カラーパレット**から選ぶ。各テーマは **8 色のスウォッチ**に加え、**未指定のデフォルト**（グラデーションボタン）があり、実質 9 つの選択肢として操作できる。

## UI

- ラベル「吹き出しの色（任意）」の下に、**コンパクトなテーマ切替ボタンが 4 つ**（細いグラデーションの丸角）。
- その下に **デフォルト** スウォッチと、**現在のテーマの 8 色**スウォッチが並ぶ。
- テーマを切り替えると、そのテーマの 8 色が表示される。選択中の色が新テーマに含まれない場合は選択がクリアされ、未指定扱いになる。
- キーボードのフォーカス順は、テーマ 4 つ → デフォルト → 8 色（`role="toolbar"` 内）。

## 4 テーマ（表示順）

| 順 | id | 表示名 | 内容 |
|----|-----|--------|------|
| 1 | `mono` | モノトーン | 明〜暗のグレー／オフホワイト系 |
| 2 | `lavender` | ラベンダー | 紫〜ピンク〜白系 |
| 3 | `vivid` | ビビッド | **従来のカラフル 8 色**（従来 `BUBBLE_COLOR_PRESETS` と同じ） |
| 4 | `pastel` | パステル | 淡色 8 色（スペース上のバブル編集パレットと同系統） |

## 初期値（テーマとスウォッチ）

- **初回**（端末に下記のローカル保存がない場合）の既定は、従来どおり **テーマ `vivid`・具体色は未指定**（デフォルトスウォッチ）。
- **2 回目以降**は、前回投稿時に端末へ保存した **テーマ ID とスウォッチ**（未指定なら `null`）を読み込み、投稿画面を開いた時点で同じ選択状態から始める。

## 投稿送信後の UI

- 送信に成功したあとも、**テーマと具体色の選択はリセットしない**（連続で投稿しやすくする）。
- 送信成功のタイミングで、**現在のテーマ ID と具体色**を端末ローカルへ書き込む（次回オープン時の初期値に使う）。

## データ

### サーバーに載る投稿データ（従来どおり）

- 各投稿に保存されるのは **`bubbleColor`（1 色の HEX 文字列）または未指定**のみ。テーマ ID は Hossii レコードには含めない。

### 端末ローカル（投稿フォームの下書き用）

| 項目 | 内容 |
|------|------|
| キー | `hossii.lastPostBubbleColorPrefs` |
| 形式 | JSON 文字列。`paletteId`（4 テーマのいずれか）と `color`（HEX 文字列または未指定の `null`） |
| 読込・保存 API | `loadPostBubbleColorDraft` / `savePostBubbleColorDraft`（`src/core/utils/displayPrefsStorage.ts`） |
| 備考 | 無効な値や壊れた JSON は読込時に捨て、テーマは `vivid`・色は未指定にフォールバックする |

## 実装参照

- 定義: `src/core/utils/bubbleColorPalettes.ts`（`DEFAULT_BUBBLE_PALETTE_ID` は `'vivid'`）
- ローカル下書き: `src/core/utils/displayPrefsStorage.ts`（上記キー）
- UI: `src/components/PostScreen/PostScreen.tsx` / `PostScreen.module.css`
- スペース上のバブル色インライン編集: `src/components/SpaceScreen/Tree.tsx`（パステル 8 色は `BUBBLE_INLINE_EDIT_COLORS` と共通）
