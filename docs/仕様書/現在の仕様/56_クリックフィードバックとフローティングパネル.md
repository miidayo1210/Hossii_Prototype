# 56 クリックフィードバック（星）とフローティングパネル

> 最終更新: 2026-04-06

## 概要

アプリ全体で統一した **軽い操作フィードバック** と、Illustrator のパネルに近い **移動・リサイズ可能なフローティング UI** を提供する。

---

## 1. クリック時の星フィードバック（グローバル）

### 目的

主要操作が「クリック」で完結する体験に、**ほんの少しだけ**楽しさと応答感を足す。派手なエフェクトではなく、儚く短い演出に留める。

### 挙動

| 項目 | 内容 |
|------|------|
| トリガー | `document` に対する **primary click**（`button === 0`）。キャプチャフェーズで購読し、子要素で `stopPropagation` されていても発火する |
| 表示位置 | クリック座標（`clientX` / `clientY`）を中心に、小さな記号（✦ ✧ ⋆ 等）が**ごく近く**に短時間だけ広がって消える |
| 色 | **紫系**（テキスト色 `#a78bfa`、グローは `rgba(139, 92, 246, …)`） |
| アクセシビリティ | `prefers-reduced-motion: reduce` のときは **表示しない**（JS 側でスキップし、CSS でも非表示） |
| 除外 | キーボード等で座標が付かないクリック（`clientX === 0 && clientY === 0`）はスキップ |

### 実装

| ファイル | 役割 |
|---------|------|
| `src/components/GlobalClickStarBurst/GlobalClickStarBurst.tsx` | クリック購読 + portal 描画 |
| `src/components/GlobalClickStarBurst/GlobalClickStarBurst.module.css` | 粒子アニメーション・紫の見た目 |
| `src/App.tsx` | `AuthProvider` 直下で `<GlobalClickStarBurst />` をマウント（認証前後を問わずアプリに載る範囲で有効） |

### 関連

- いいねボタンのパーティクル（[31_いいねボタンUI](./31_いいねボタンUI.md)）とは別系統。あちらはボタン専用、こちらは **アプリ全体のクリック**。

## 2. フローティングパネルシェル（`FloatingPanelShell`）

### 目的

音声パネル・クイック投稿パネル・QR パネルなど、**画面に常に張り付かない**補助 UI を、ユーザーが好みの位置・大きさに調整できるようにする。

### 挙動

| 項目 | 内容 |
|------|------|
| 移動 | 枠の「空白」領域をドラッグ（ボタン・入力・`[data-no-drag]` 付き要素は除外） |
| リサイズ | 辺・角のヒット領域。横辺は幅のみ、縦辺は高さのみ、角は斜めに両方 |
| 永続化 | `localStorage` に矩形を保存（キーは呼び出し側の `storageKey`） |
| カーソル | 移動可能領域は `grab`、ドラッグ中は `grabbing`（ドラッグ中は `document.body` のカーソルも合わせる） |
| **モバイル下端余白**（`768px` 以下） | 固定の **BottomNavBar**（`z-index: 100`）とパネル内容が**重ならない**よう、`clampFloatingRect` に **`bottomInset`** を渡す。既定値は `floatingPanelStorage.ts` の **`MOBILE_BOTTOM_NAV_RESERVE_PX`（72px）** と `getFloatingPanelBottomInsetPx()`。ドラッグ・リサイズ・ウィンドウリサイズ時も同じ inset でクランプ |

### 実装

| ファイル | 役割 |
|---------|------|
| `src/components/FloatingPanelShell/FloatingPanelShell.tsx` | ドラッグ・リサイズ・永続化の共通実装（モバイル時は `getFloatingPanelBottomInsetPx()` を参照してクランプ） |
| `src/components/FloatingPanelShell/FloatingPanelShell.module.css` | 枠・リサイズハンドル |
| `src/components/FloatingPanelShell/floatingPanelDragContext.tsx` | 子がドラッグハンドル用 props を取得するコンテキスト |
| `src/components/FloatingPanelShell/floatingPanelHitTest.ts` | ドラッグ開始可否のヒットテスト |
| `src/core/utils/floatingPanelStorage.ts` | 矩形の読み書き・クランプ（`bottomInset` 対応）・パネル別デフォルト・モバイル下端予約 |

### 適用コンポーネント（例）

- `SpeechPanel`（[52_音声認識・入力機能](./52_音声認識・入力機能.md)）
- スペース画面のクイック投稿（[51_簡単投稿](./51_簡単投稿.md)）
- スペース画面のクイックログ一覧（[57_簡単にログ一覧](./57_簡単にログ一覧.md)）
- `QRCodePanel`（スペース画面の左バーから表示。設定画面の共有タブ `ShareTab` とは別 UI）

### 重なり順（スペース画面のクイックパネル）

クイック投稿パネル・クイックログパネルはいずれも `FloatingPanelShell` を使うが、`z-index` は **音声パネル 300 ＜ クイック投稿 310 ＜ クイックログ 320** とし、投稿とログを音声パネルより手前にし、ログをさらに手前にしている。

画面右端の **キューブドック**（[51](./51_簡単投稿.md) / [57](./57_簡単にログ一覧.md) / [52](./52_音声認識・入力機能.md)）は **400** とし、**PC（`768px` 超）のみ表示**。パネルが開いていてもトグル操作に届くようにしている。**モバイル**ではキューブドックを出さず、導線は [51](./51_簡単投稿.md)・[57](./57_簡単にログ一覧.md)・[52](./52_音声認識・入力機能.md) に記載のとおり。

**QR コードパネル**は `z-index: 200` とし、音声・クイック投稿・クイックログより **奥**に置く。表示のオンオフは左コントロールバーの QR ボタン。

## 3. QR コードフローティングパネル（スペース画面）

### 目的

投影や配布用に、**現在のスペースへ参加する URL** を QR 化して表示し、テキスト URL や QR 画像をその場でクリップボードへ取れるようにする。

### 表示条件

| 項目 | 内容 |
|------|------|
| 画面幅 | **パネル枠**は `769px` 超のみ表示。狭い幅では CSS により非表示 |
| 配置 | スペース画面 `SpaceScreen` 内。スライドショー中はトップバー・左バー・本パネルとも出さない |
| 表示切替 | 左コントロールバーの QR アイコンでオンオフ。表示中はボタンがアクティブ状態 |

### パネル内の操作

| 操作 | 内容 |
|------|------|
| リンクをコピー | ボタンで参加用 URL を**テキスト**としてクリップボードへ。成功時はボタン文言が一時的に変化 |
| QR 画像をコピー | QR コード枠にポインタを重ねると**右上にコピーアイコン**が表示され、そのボタンをクリックすると **PNG** としてクリップボードへ。`hover` が使えない環境ではコピーアイコンを常時表示しタップで同じ操作ができる |
| 移動・リサイズ | 他パネル同様 `FloatingPanelShell`。`storageKey` は `qr`。既定位置は `getDefaultQrRect`（縦位置にフローティングパネル用の下端リザーブを反映） |

### URL 生成

コミュニティ slug とスペース URL の有無に応じて `/c/.../s/...` 形式や `?space=` 形式へ切り替え。詳細は [03_スペースURL機能](./03_スペースURL機能.md) および実装 `src/components/Navigation/QRCodePanel.tsx`。

### 実装ファイル

| ファイル | 役割 |
|---------|------|
| `src/components/Navigation/QRCodePanel.tsx` | QR 表示・リンクコピー・画像コピー・URL 組み立て |
| `src/components/Navigation/QRCodePanel.module.css` | パネル枠・コピー UI。モバイル幅ではパネル非表示 |
