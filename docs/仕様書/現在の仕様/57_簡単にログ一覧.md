# 57 簡単にログ一覧（クイックログパネル）

## 機能概要

スペース画面上で、**クイック投稿パネル（[51_簡単投稿](./51_簡単投稿.md)）と同じ仕組み**により、画面遷移なしで **「ログ一覧」相当の UI** をフローティングパネルとして開く。

- **移動・リサイズ・位置/サイズの永続化**は [56_クリックフィードバックとフローティングパネル](./56_クリックフィードバックとフローティングパネル.md) の `FloatingPanelShell` と**同一の挙動・実装パターン**とする（`storageKey` のみ別キー）。
- 表示するデータは、メニューの「ログ一覧」で遷移する画面（`CommentsScreen`）と**同じスペース・同じ `Hossii` 一覧**を前提とする（アクティブスペースの `getActiveSpaceHossiis()` 相当）。

---

## 背景

- ダブルクリックで右側（PC）／下半分（モバイル）に **投稿用パネル**が開く仕様が既にある。
- 同様の操作感で **ログを一覧参照・絞り込み**したいニーズがある。
- 見た目は投稿パネルと区別しやすいよう、**配色を変える（例: 水色・シアン系）**ことを想定する。

---

## デバイス別レイアウト（投稿パネルと揃える）

クイック投稿（#51）と同様、初期矩形は `floatingPanelStorage` のデフォルトを踏襲する。

| 環境 | 初期イメージ | 備考 |
|------|-------------|------|
| **PC**（`768px` より広い） | 右側寄りの縦長パネル（幅およそ 25% 前後、`getDefaultQuickPostSideRect` と同系） | ユーザーが `FloatingPanelShell` で自由に移動・サイズ変更 |
| **モバイル**（`768px` 以下） | 下からのボトムシート型（高さおよそ `55vh` 前後、`getDefaultQuickLogBottomRect` → `getDefaultQuickPostBottomRect` と同系） | 下端は **BottomNavBar 分を除いた位置**に揃える（[56](./56_クリックフィードバックとフローティングパネル.md) の `bottomInset`）。`FloatingPanelShell` のクランプでナビに被らない |

※ 実際の数値は `floatingPanelStorage.ts` の `getDefaultQuickLogSideRect` / `getDefaultQuickLogBottomRect` と #51 と並べて定義（本仕様では「投稿パネルと対になる既定値」であることが重要）。

---

## パネル内コンテンツ（ログ一覧 UI）

### 目的

スペースを離れずに、**コメント／感情ログ等の一覧・フィルタ・ソート**ができること。

### データソース

- **自スペース表示中**: アクティブスペースの `Hossii` 一覧（`CommentsScreen` と同様に `getActiveSpaceHossiis()` を使用）。
- **訪問モード中**: 訪問先スペースの一覧（`visitingHossiis`。`fetchHossiis(visitingSpaceId)` で取得）。閲覧用。
- フィルタ・ソート・タグの UI は `LogListBody` に集約し、フル画面（`CommentsScreen`）とクイックログパネルで共有する。

### 最低限含める UI（フル画面ログ一覧との差分）

| 要素 | パネルモード | 備考 |
|------|-------------|------|
| フィルタバー（コメント／感情 等） | ✅ 含める | `FilterBar` + `filterStorage` と連携 |
| 並び替え（新着／いいね 等） | ✅ 含める | `CommentsScreen` と同様 |
| タグ絞り込み | ✅ 含める（スペースにタグがある場合） | 画面が狭いためドロップダウン UI は折りたたみ前提 |
| 一覧各行（本文・感情・いいね等） | ✅ 含める | スクロール可能なリスト |
| 管理者向け「非表示」 | ✅ 実装済み | `currentUser.isAdmin` のとき各行に表示。`hideHossii` と確認ダイアログ。訪問モード時はストア外データのため `onAfterAdminHide` で行を即時除去 |
| `TopRightMenu` 等のナビゲーション | ❌ 非表示 | パネル専用の閉じる操作のみ |
| 画像ライトボックス | ✅ 実装済み | `comments_thumbnail` フラグに連動 |

### 閉じる操作

- パネル上部に **「✕ 閉じる」**（投稿パネルと同様の位置・操作感）。
- **ESC キー**で閉じるかどうかは未決（#51 と同様に後追いでも可）。

## フローティングパネル（#56 との一致点）

| 項目 | 内容 |
|------|------|
| 移動 | 枠のドラッグ可能領域（`FloatingPanelShell` 既定。入力・ボタン・`[data-no-drag]` は除外） |
| リサイズ | 辺・角ハンドル |
| 永続化 | `localStorage`（実装キー: `logList.desktop` / `logList.mobile` — 投稿の `quickPost.*` とは**別キー**） |
| z-index | **音声パネル `300` ＜ クイック投稿 `310` ＜ クイックログ `320` ＜ 右側キューブドック `400`（PC のみ）**。キューブは常に手前に置き、パネルが開いていても折りたたみ操作に届く。**モバイルではキューブ非表示**のため 400 は該当なし |

---

## 見た目の差別化（配色）

投稿パネル（紫〜白のグラデーション等）と区別するため、パネル外枠（`FloatingPanelShell` に渡す `className`）で **水色〜シアン系**のトーンを適用する。

| 案 | 例 |
|----|-----|
| 背景 | `linear-gradient(160deg, #f0fdff 0%, #e0f7fa 100%)` など |
| 影・ボーダー | 紫ではなく `rgba(14, 165, 233, …)` や `rgba(6, 182, 212, …)` 系 |

※ 最終トークンはデザイン調整で変更可。**「投稿＝紫系／ログ＝水色系」**の対応が分かればよい。

## 開閉トリガー（ダブルクリックとの役割分担）

### 実装済みのトリガー（PC / モバイルで分岐）

| 操作 | 挙動 |
|------|------|
| **スペース背景のトリプルクリック**（`click` の `detail === 3`） | `quickLogOpen` をトグル。ダブルクリックで予約されたクイック投稿の遅延オープンはキャンセルされる（#51）。**モバイル（`768px` 以下）では無効**（下ナビに導線を寄せるため） |
| **左コントロールバー「ログ一覧」ボタン**（`ScrollText` アイコン） | 同上トグル。**PC のみ表示**（モバイルでは非表示） |
| **画面右端のキューブドック「ログ」**（水色系、`SpacePanelCubeDock`） | 同上トグル。**PC（`768px` 超）のみ表示**。`click` / `pointerdown` で `stopPropagation` し背景ジェスチャと分離 |
| **下部ナビゲーションの「ログ」**（`BottomNavBar`） | **スペース画面表示中**（`screen === 'screen'`）に押したときだけ、画面遷移せず **`quickLogOpen` をトグル**（`App.tsx` から `SpaceScreen` のハンドラを ref 登録）。**モバイルの主導線** |
| **訪問モード** | ログパネル・ログキューブ（PC）は**利用可**（訪問先のログ表示）。投稿側キューブのみ無効（#51） |

### モバイルについて

- **トリプルクリック**は使わない（上表）。
- **左バーのログボタン・右端キューブ**は出さない。**下部ナビの「ログ」**でクイックログを開閉する。
- フル画面のログ一覧（`CommentsScreen`）へ行きたい場合は、ナビで他のタブに移動したあと **「ログ」** を選ぶ、または実装上の遷移導線に従う（スペース中の「ログ」タップはオーバーレイ専用）。

### 参考（採用しなかった案）

- ダブルクリックだけで投稿／ログを交互に切り替える案は非推奨（#51 当初メモどおり）。
- 長押し・スワイプは未実装（将来の拡張余地）。

## 同時に両方開くか

| 方針 | 説明 |
|------|------|
| **推奨（初期）** | **投稿パネルとログパネルは同時に開いてよい**（別用途のため）。ただし画面が狭いので、開いたときに**重なり順**だけ統一する。 |
| 代替 | どちらか一方のみ（片方を開くと他方を閉じる）は、ユーザーが「ログを見ながら投稿」しづらくなるため、原則採用しない。 |

---

## 訪問モード（隣人スペース閲覧中）

- **クイック投稿**は開かない（訪問先へ投稿しない）。**クイックログ**は開き、訪問先スペースの `Hossii` を一覧表示する（閲覧）。
- 管理者がログ一覧から **非表示**にした場合、DB 更新に加えローカルの `visitingHossiis` から該当行を除去して即時反映する。
