# 58 仕様追随レビュー（依頼書・チェックリスト）

## この文書の目的

`main` との差分に含まれる実装が、次の仕様書で定義した**挙動・優先順位・非機能要件**と一致しているかを、レビュアーが漏れなく検証できるようにする。単なる「最近の変更を見てほしい」ではなく、**何をもって合格とするか**まで書いた依頼書兼チェックリストである。

対象ブランチは作業時点の **`main` から分岐したコミット列**（PR 化前提）。仕様の「変化した部分」は各ドキュメントの **実装済み／設計方針／z-index ルール** などに集約されているため、本書では仕様番号ごとに**確認観点**へ落とし込む。

## 横断チェック（先に確認）

次を満たすことを前提に、個別仕様の確認に進む。

- **`npm run lint` / `npm run build`** が通る（CI の Node バージョンとローカルが乖離していないかも意識）。
- **認証まわり**: ログイン／ゲスト入場後、スペース表示までの一連が破綻していない（大きな `AuthContext` 変更があれば初回セッションのフラッシュも見る）。
- **表示設定**: `DisplayPrefsContext` 分離後も、Listen・表示スケール・Hossii 表示トグルが期待どおり永続化・反映される（仕様 52・53 と干渉しやすい）。

## 51 簡単投稿 — 確認観点

**仕様上の変化の核**: スペース上に留まったまま投稿するパネル、ダブルクリック座標またはキューブ既定 `(50, 50)`、`isPositionFixed: true`、ログ（57）とのジェスチャ分担。

| # | 確認内容 | 合格の目安 |
|---|-----------|------------|
| 1 | ダブルクリックでパネルが開く／開いているときは閉じる | `SpaceScreen` の `quickPostPos` と `handleDoubleClick` が仕様どおり |
| 2 | 投稿オープンの **約 320ms 遅延** と、**トリプルクリックで遅延キャンセル** | `pendingQuickPostOpenRef` と `setTimeout(..., 320)` が一致 |
| 3 | キューブから開くときは中央既定、トグルで閉じる | `SpacePanelCubeDock`、訪問モードでは投稿キューブ無効 |
| 4 | スライドショー中はキューブ非表示 | `viewMode === 'slideshow'` と同条件 |
| 5 | パネル内 `PostScreen` | `panelMode`・`onClose`・`navigate('screen')` 非呼び出し（パネル時）、不要 UI 非表示 |
| 6 | `FloatingPanelShell` | `storageKey`・**z-index 310**（56・仕様 51 記載と整合） |
| 7 | 連続投稿（#46）とパネル | 仕様どおり「パネルは投稿後に閉じる」が優先されているか |

**主な実装ファイル**: `SpaceScreen.tsx`, `SpacePanelCubeDock.tsx`, `PostScreen.tsx`, `FloatingPanelShell/*`, `PostScreen.module.css`, `SpaceScreen.module.css`

## 52 音声認識・入力機能 — 確認観点

**仕様上の変化の核**: 感情ログの自動投稿をやめ、**文字起こし表示＋ユーザー明示操作での投稿**に寄せる。`useSpeechRecognition` の `onFinalSegment` 中心、パネルは `FloatingPanelShell`。

| # | 確認内容 | 合格の目安 |
|---|-----------|------------|
| 1 | スペースからの「音声だけで勝手に投稿」が起きない | `SpaceScreen` で旧 `useAudioListener` 経路が仕様どおり |
| 2 | Listen ON でパネルが開く／音声キューブはパネル開閉など | 仕様の動線 1・2 が再現できる |
| 3 | パネル閉じる・Listen OFF で蓄積リセット | 仕様「パネルを閉じると全文リセット」等 |
| 4 | 候補・粒度（短め／長め）・最大件数 | `SpeechPanel` 周りが仕様の上限・文言に沿う |
| 5 | フローティング化 | `floatingPanelStorage` のスピーチ用デフォルト、ドラッグ／リサイズ／永続化 |

**主な実装ファイル**: `useSpeechRecognition.ts`, `SpeechPanel` 系, `SpaceScreen.tsx`, `SpacePanelCubeDock.tsx`, `DisplayPrefsContext.tsx`

## 53 スペース投稿表示並び替え — 確認観点

**仕様上の変化の核**: `layoutMode`（ランダム／投稿順）、投稿順での昇降、`orderedGridIndex` に基づく **スタック用 z-index**、ランダム時のみ固定座標を尊重。

| # | 確認内容 | 合格の目安 |
|---|-----------|------------|
| 1 | 左バー「並び」🔀／↔️、`localStorage` キー `hossii.layoutMode` | リロード後も維持 |
| 2 | 投稿順＋降順／昇順ボタンの位置（PC／モバイル） | 仕様の `top`・モバイル配置の説明と一致 |
| 3 | 投稿順では格子座標のみ、ランダムに戻すと固定座標が再び効く | `SpaceScreen` の `pos` 算出分岐 |
| 4 | 重ね順 | `orderedStackZ` と CSS 変数が仕様の「セル番号大＝手前」に沿う |
| 5 | モバイル星表示 | `anchor: topLeft` 等、仕様の一言と矛盾しない |

**主な実装ファイル**: `SpaceScreen.tsx`, `LeftControlBar.tsx`, `Tree.tsx`, `StarView.tsx`, 関連 CSS

**仕様上の変化の核**: 4 テーマ × 8 色＋デフォルト、**初期テーマは `vivid`**、保存は **`bubbleColor` のみ**（テーマ ID は保存しない）。

| # | 確認内容 | 合格の目安 |
|---|-----------|------------|
| 1 | テーマ 4 ボタン＋スウォッチ、`DEFAULT_BUBBLE_PALETTE_ID === 'vivid'` | `bubbleColorPalettes.ts` |
| 2 | テーマ切替で選択色が新テーマに無い場合はクリア | 仕様の「未指定扱い」 |
| 3 | 投稿後もテーマは維持し、**色だけ**リセット | 連続投稿・クイック投稿の両方で確認すると安心 |
| 4 | キーボード順 | `role="toolbar"` のフォーカス順が仕様どおり |

**主な実装ファイル**: `bubbleColorPalettes.ts`, `PostScreen.tsx`, `PostScreen.module.css`, `Tree.tsx`（インライン編集との色集合の関係）

## 57 簡単にログ一覧 — 確認観点

**仕様上の変化の核**: 投稿パネルと同型のフローティングだが **別 `storageKey`**、水色系クローム、トリプル／左バー／ログキューブ、**訪問モードではログは可・投稿キューブのみ不可**。

| # | 確認内容 | 合格の目安 |
|---|-----------|------------|
| 1 | トリプルクリックでログパネルトグル、かつ 51 の遅延オープンをキャンセル | `SpaceScreen` の `click` `detail === 3` と 51 の連携 |
| 2 | `CommentsScreen` と同データ源（自スペース／訪問先） | `getActiveSpaceHossiis` / `visitingHossiis` |
| 3 | パネル内は `LogListBody` 共有、ナビ非表示 | フィルタ・ソート・管理者非表示・サムネフラグ |
| 4 | 投稿パネルと同時表示可、重なりは 56 のルール | z-index 320・配色クラス |
| 5 | 永続化キーが `quickPost.*` と分離 | `logList.desktop` / `logList.mobile` 等、仕様記載と一致 |

**主な実装ファイル**: `SpaceScreen.tsx`, `SpacePanelCubeDock.tsx`, `CommentsScreen` / ログ一覧共有コンポーネント, `SpaceScreen.module.css`（クイックログ用 chrome）
