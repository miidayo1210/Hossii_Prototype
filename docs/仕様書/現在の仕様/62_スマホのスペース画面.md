# 62 スマホのスペース画面

スペース画面（`#screen`）における、**左のコントロールバー**と**下ナビゲーション**の挙動を定義する。PC でも共通する部分は明記する。

> 最終更新: 2026-06-25（スペース説明 (i) ボタン）

---

## スペース説明（[91](./91_スペース説明表示.md)）

| 項目 | 内容 |
|------|------|
| 設定 | スペース管理 → 一般 →「スペース説明」（ウェルカムメッセージの下、最大 50 文字） |
| モバイル表示 | 左上スペース名 pill 横の `(i)` ボタン → タップでボトムシート |
| 左バー | **出さない**（PC 専用トグルは左バー、モバイルは pill 横のみ） |

---

## 用語・前提

- **ログ一覧ページ**: ハッシュ `#comments` の画面（`CommentsScreen`）。下ナビの「ログ」で遷移する先。**画面内で全体／私のログを切替**（[83](./83_ログ一覧の私のログ切替.md)）。
- **クイック投稿パネル**: スペース画面上に重ねて開く投稿フォーム（[51_簡単投稿](./51_簡単投稿.md)）。下ナビ「投稿」からも開く（[76](./76_スマホ投稿ナビとクイック投稿.md)）。**モバイルでは固定 UI**（縦=ボトムシート / 横=左スペース+右ドック）。ドラッグ・リサイズ不可。**下ナビ経路は座標固定せずランダム配置**（[85 §4.6](./85_スマホ吹き出し配置の中央寄せ調査.md)）。
- **クイックログパネル**: スペース上のフローティングログ（[57_簡単にログ一覧](./57_簡単にログ一覧.md)）。**下ナビからは開かない**（トリプルクリック等）。右端キューブドックは廃止（[06](./06_ナビゲーション・UI構成.md)）。

## 左のコントロールバー

### レイアウト（デバイス別）

| 向き | 配置 | 詳細 |
|------|------|------|
| **スマホ縦** | 画面 **左上・横長バー** | 16:9 背景のシャープ画像（`.bgImageSharp`）と **重ならない**。レターボックス／ぼかし帯上に載せる（[33](./33_スペース左サイドのバー.md)・[70](./70_背景画像サイズ.md)） |
| **スマホ横** | 画面 **左上・横長**（縦と同系） | [75_スマホ横持ちレイアウト](./75_スマホ横持ちレイアウト.md) |
| **PC** | 画面左中央・縦 | 従来どおり |

### 現状の課題（メモ）

- 大画面ボタンを押しても見た目がほとんど変わらない（特にモバイルでは Fullscreen API が効かないことがある）。
- **Listen・音声パネル**まわりは [71_スマホマイク機能.md](./71_スマホマイク機能.md) を参照。

### `...` オーバーフローメニュー（モバイル）

表示モード・並び・期間・件数など、バー上に載せきれない操作は **`...` ボタン**から開く（[33](./33_スペース左サイドのバー.md) §3）。

| 項目 | 内容 |
|------|------|
| 開き方 | 左上横長バー右端付近の `···` をタップ |
| 中身 | 表示モード / 並び / 期間 / 件数 / **タグで絞る** / **ログ一覧** / **音声入力パネル** |
| 描画 | 768px 以下では `document.body` へ Portal（バーの `overflow` や z-index に隠れない） |
| 閉じ方 | パネル外タップ |

投稿件数（`#`）・QR・表示倍率はモバイル左バーにも `...` 内にも **出さない**（PC のみ）。

### 大画面表示（あり）

**定義:** 「ブラウザの全画面」だけに依存せず、アプリ内の **レイアウトモード** でスペース表示領域を最大化する。

- **縦持ち:** スペース canvas を `100dvh` ベースで最大化。左バーは **横長・左上**（上記）。
- **横持ち:** PC のレスポンシブに近いレイアウト・余白（[75](./75_スマホ横持ちレイアウト.md)）。
- **Fullscreen API** は補助。失敗しても CSS 没入は有効。

### トグル一覧（モバイル左バーに出すもの）

| 項目 | モバイル左バー | 備考 |
|------|----------------|------|
| 大画面表示 | あり | 上記定義に従う |
| Hossii 表示 / 非表示 | あり | |
| 音声入力 | あり | |
| 音声出力（読み上げ） | あり | **既定 OFF**。初回 ON は `VoiceConsentModal`（[08 §F07.2](./08_スペース（HOME）の仕様書.md)） |
| 投稿件数表示の切り替え | **なし** | PC 左バーでは従来どおり可 |
| QR 表示の切り替え | **なし** | PC では従来どおり可 |
| **`...` オーバーフロー** | あり | 表示モード・並び・期間・件数・ログ・音声パネル（§上記） |

表示倍率・投稿数・QR はバー上に出さず、期間・件数・表示モード・**タグ絞り込み**等は **`...` 内**に格納する。

**タグ絞り込み（[84](./84_スペース表示モード拡張.md)）:** 星トグルはモバイル非表示。タグ選択中はスペース右上に `#タグ名` バッジのみ表示。

### 投稿者まとめモード（`layoutMode: byAuthor`）

[81_投稿者別まとめ表示モード](./81_投稿者別まとめ表示モード.md) 参照。

| 項目 | スマホ縦 | スマホ横 |
|------|----------|----------|
| 表示 | `StarView` 不使用。`AuthorClusterBubble` コンパクトカード | 同左（PC に近い密度） |
| 配置 | 左→右・折り返し格子（81 §5.4.1）。シャープ矩形内に `mapDisplayPos` | 同左 |
| 並び替え | スペース右上 **初投稿順 / 最新順 / 投稿数順**（81 §6.3） | 同左 |
| プレビューローテ | **無効**（個別投稿の星プレビューは出さない） | 無効 |
| タップ | 折りたたみ展開せず **ボトムシート**（`AuthorTimelineModal`）を開く | PC 同様に展開可 |

### 隣人システム「島」ボタン（あり）

- **表示条件**: 隣人が 1 件以上。自スペース・訪問中どちらでも表示。
- **操作**: 自スペース中はワープ、訪問中は自スペースへ戻る。
- 訪問中は `aria-label` / `title` を「自スペースに戻る」に切替。

## 下のナビゲーション

| タブ | 挙動 |
|------|------|
| スペース | `#screen` のまま |
| 投稿 | **`#screen` 中:** クイック投稿パネルを開閉（[76](./76_スマホ投稿ナビとクイック投稿.md)）。**他画面:** `#screen` へ戻してから開く。**`#post` への遷移はしない**（モバイル） |
| ログ | **常に** `#comments` へ遷移（**全体／私のログ切替**あり） |
| アカウント | `#account` へ遷移 |

スペース上でログをパネルで見る場合は背景 **トリプルクリック**（PC、[57](./57_簡単にログ一覧.md)）を使う。

### クイック投稿パネル（モバイル・開いているとき）

| 向き | レイアウト | 備考 |
|------|------------|------|
| **縦** | スペースの上に **固定ボトムシート**（下からスライドイン） | `mobilePostSheet`・下ナビは常に見える |
| **横** | 左: スペース / 右: 固定投稿ドック（幅 `clamp(240px, 36vw, 320px)`） | `containerMobilePostLandscapeSplit`・[75](./75_スマホ横持ちレイアウト.md) |

`FloatingPanelShell` は使わない。PC のみ従来のフローティングパネル（[51](./51_簡単投稿.md)）。

### 横持ち案内バナー（スマホ縦）

スマホ縦でスペース画面を表示中、**横向きにすると見やすい**旨を控えめに案内する（[71](./71_スマホマイク機能.md)）。

| 項目 | 内容 |
|------|------|
| 表示条件 | モバイル縦・スライドショー以外・未 dismiss |
| 非表示 | 横持ちにしたとき／✕ で閉じたとき（`localStorage` に保存） |
| 配置 | 画面上部中央・safe-area 内（16:9 領域・左バーと重なりにくい） |
| 見た目 | 紫〜ピンクグラデーションの丸ピル。📱 +「横向きにするともっと見やすいよ！」 |
| 実装 | `SpaceScreen` の `landscapeHintBanner` / `landscapeHintStorage.ts` |

## 空スペース

投稿 0 件でも **「まだ気持ちがありません」等の案内は出さない**（[08](./08_スペース（HOME）の仕様書.md) §0）。

---

## 訪問モード中の投稿ボタン

訪問中スペースでは投稿不可のため、「投稿」タブ押下時は **Toast 等で「訪問中は投稿できません」** を表示する（無言で何も起きない状態を解消）。

| 条件 | 挙動 |
|------|------|
| モバイル + 訪問中 + 「投稿」タブ | Toast「訪問中のスペースには投稿できません」を 2 秒表示。パネルは開かない |
| PC + 訪問中 + TopRightMenu「投稿する」 | 同上 |
