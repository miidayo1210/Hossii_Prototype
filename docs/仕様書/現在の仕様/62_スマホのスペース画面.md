# 62 スマホのスペース画面

スペース画面（`#screen`）における、**左のコントロールバー**と**下ナビゲーション**の挙動を定義する。PC でも共通する部分は明記する。

---

## 用語・前提

- **ログ一覧ページ**: ハッシュ `#comments` の画面（`CommentsScreen`）。投稿画面から下ナビの「ログ」で遷移する先と同一。
- **クイックログパネル**: スペース画面上に重ねて開くログ一覧（フローティング）。下ナビからは開かない（後述）。

---

## 左のコントロールバー

### 現状の課題（メモ）

- 大画面ボタンを押しても見た目がほとんど変わらない（特にモバイルでは Fullscreen API が効かないことがある）。
- モバイル左バーに、投稿件数・QR の切り替えが出ているが、本仕様では**出さない**。

### 大画面表示（あり）

**定義:** 「ブラウザの全画面」だけに依存せず、アプリ内の **レイアウトモード** でスペース表示領域を最大化する。YouTube の大画面のように、操作に対して確実に見た目が変わること。

- 縦持ち: スペース canvas を `100dvh` ベースで最大化し、余白・安全域を確保。左バーは折りたたみまたはオーバーレイでもよい。
- **横持ち（ランドスケープ）**時は、PC のレスポンシブ表示に近いレイアウト・余白に寄せる（`orientation: landscape` または `matchMedia` で切替）。
- **Fullscreen API**（`requestFullscreen`）は、対応ブラウザでは**補助的に**試みる。失敗しても上記レイアウトモードは必ず有効であること。

`screen.orientation.lock` など、端末・権限依存の強い API は必須としない。

### トグル一覧（モバイル左バーに出すもの）

| 項目 | モバイル左バー | 備考 |
|------|----------------|------|
| 大画面表示 | あり | 上記定義に従う |
| Hossii 表示 / 非表示 | あり | |
| 音声入力 | あり | |
| 音声出力（読み上げ） | あり | |
| 投稿件数表示の切り替え | **なし** | PC 左バーでは従来どおり可 |
| QR 表示の切り替え | **なし** | PC では従来どおり可 |

表示サイズ（％）、表示モード・並び・期間・件数など、**モバイルでは非表示**のまま（既存 CSS の `mobileVisible` 方針に合わせる）。

### 隣人システム「島」ボタン（あり）

- **表示条件**: 隣人が 1 件以上登録されているとき。**自スペース表示中・他スペース訪問中のどちらでも表示する**（訪問中に消さない）。スマホ・PC 共通。
- **操作**:
  - **自スペース表示中**: 従来どおり、ランダムな隣人スペースへワープ（訪問モードへ）。
  - **他スペース訪問中**: もう一度押したら **自スペースの表示に戻る**（`visiting` を解除）。ワープと戻るの切り替えはこの1ボタンで行う。
- **変更しない**: 画面上部の「このスペースを開く」「自スペースに戻る」などの既存 UI はそのまま残す。
- アクセシビリティ: 訪問中はボタンの `aria-label` / `title` を「自スペースに戻る」に切り替える。

## 下のナビゲーション

| タブ | 挙動 |
|------|------|
| スペース | そのまま（`#screen`） |
| 投稿 | そのまま（投稿画面ページ `#post`） |
| ログ | **常にログ一覧ページ（`#comments`）へ遷移**。スペース表示中でも、クイックログパネルを開かない。投稿画面から「ログ」と同じ遷移先に統一する。 |
| アカウント | アカウントページ（`#account`）へ遷移（後述の Coming soon はページ内で表現） |

スペース上でログ一覧をパネルで見たい場合は、**右側キューブの「ログ」**や既存のショートカットに任せる（下ナビの役割とは分離）。

---

## アカウントページ（ゲスト時）

- 画面遷移は上記どおりアカウントページへ行く。
- **ログイン**・**新規会員登録**は、現時点では利用不可として、**薄い表示・押せない（disabled）・「Coming soon」** の表記で示す。
- 将来オンにする場合は feature フラグ等で切り替え可能にしておく。

---

## 実装の推奨順（参考）

1. 下ナビ「ログ」の遷移統一（影響範囲が明確）
2. 島ボタンの常時表示と「訪問中は自スペースへ戻る」トグル
3. モバイル左バーから投稿件数・QR を出さない
4. アカウントの Coming soon 表示
5. 大画面レイアウト（アプリ内モード + 必要なら Fullscreen 補助）
