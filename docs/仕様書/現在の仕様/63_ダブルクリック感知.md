# 63 ダブルクリック感知（スペース背景とクイック投稿）

## 機能概要

スペース画面（`SpaceScreen`）において、**クイック投稿パネル**（[51_簡単投稿](./51_簡単投稿.md)）を開く **ダブルクリック** は、**スペースの「空き」背景上でのみ**有効とする。ボタン・バブル・ツールバーなど、**他の UI 要素上のダブルクリックでは投稿パネルを開かない**。

トリプルクリックによる **クイックログ**（[57_簡単にログ一覧](./57_簡単にログ一覧.md)）の開閉は、**同じく背景側の操作に限定する**ことで、ダブルクリックとの競合を抑える（下記「背景に限定する理由」参照）。

---

## 背景

### 過去の問題

- 画面**最外周のコンテナ**に `onDoubleClick` が付いていると、子要素（ボタン等）でのダブルクリックがバブリングし、**意図せずクイック投稿が開く**。
- **ダブルクリック（投稿）** と **トリプルクリック（ログ）** を同一の広い領域で兼ねると、`click` / `dblclick` / 遅延タイマーの組み合わせで、**両パネルが同時に開く**・**ログを閉じようとして投稿が開く**などの競合が起こり得る。

### 解決の方針（本仕様で採用）

**「投稿用ダブルクリックは、背景専用レイヤー（または data 属性で示した空き領域）にだけ付ける」**。

1. **背景専用レイヤー**  
   スペースの背景（パターン・色・画像）を表示する **下層** に、画面いっぱいのヒット領域を置き、その要素にのみ `onDoubleClick`（およびトリプル用の `onClick` で `detail === 3` の処理）を付ける。バブル・ナビ・バッジなどは **その上の層** に配置し、背景レイヤーにイベントが届かないようにする。

2. **`data` 属性による限定（代替・併用）**  
   背景を包む要素に `data-space-background` 等の属性を付与し、ハンドラ先頭で **`event.target` が当該背景（またはその子で「装飾として背景の一部」とみなす範囲）に限定されるか** を判定する。インタラクティブな子には別の `data-*` を付け、**そこを起点としたダブルクリックでは投稿を開かない**。

実装は (1) か (2)、またはレイアウト都合に応じた組み合わせとする。**目的は一貫して「空きのスペースをダブルクリックしたときだけ投稿」**である。

## 期待される挙動

| 操作の場所 | ダブルクリック | トリプルクリック（PC・背景のみ） |
|-----------|----------------|----------------------------------|
| **背景の空き**（バブルや UI の裏の「スペース」） | クイック投稿を開く（訪問モード等の既存制約は従来どおり） | クイックログの開閉（既存の `detail === 3` 相当） |
| **バブル（投稿）** | 投稿の選択・操作のみ。クイック投稿は開かない | バブル操作に専念。ログ用トリプルは発火させない |
| **ボタン・バッジ・ツールバー・キューブ等** | 各コントロールの既定動作のみ | 同上 |

※ モバイルではビューポートや実装により、クリック `detail` やタッチの扱いが異なる。コンテナ全体への無限定バインドは避け、**背景レイヤー／判定**と [57](./57_簡単にログ一覧.md) のナビ・ボトムバー経路の整合を取る。

## 実装メモ（コード側の目安）

- 対象: `src/components/SpaceScreen/SpaceScreen.tsx` の `handleDoubleClick` / `handleContainerClick` の**付け先**と、必要なら **座標計算の `getBoundingClientRect` の基準要素**（背景レイヤーに合わせる）。
- 右側キューブ（`SpacePanelCubeDock`）は既に `stopPropagation` があるが、**全 UI での個別対応に依存しない**よう、本仕様どおり **背景にイベントを集約**する。
- フローティングパネル（`FloatingPanelShell`）上の操作は、従来どおりコンテンツ優先。背景ダブルクリックと混ざらないよう、パネル外の背景操作が対象であることを維持する。
