# 64 ログイン・新規登録画面の UI（管理者・コミュニティ参加者）

## 機能概要

**管理者向け**（`/admin/login`）と**一般参加者向け**（ログイン／新規登録）の認証画面について、**見た目と動きの方針**を定義する。認証フローや権限の詳細は [05_管理者ログイン画面](./05_管理者ログイン画面.md) および [10_ログインユーザー管理設計](./10_ログインユーザー管理設計.md) を参照する。

本書の焦点は次のとおり。

- **実装済みの姿**と、**今後足し直す調整**（明るさ・キャラ・控えめな動き）を区別して書く。
- 原点の方針: 「目がチカチカしにくい」背景にしつつ、**フォーム周りとブランド表現**には適度な温かみを残す。

## 対象となる画面と実装

| 画面 | コンポーネント | 主な表示条件 |
|------|----------------|--------------|

## 現状のデザイン（第1回実装反映後）

### レイアウト全体（`.shell`）

- **ビューポート**: `min-height: 100vh` / `100dvh`、中央寄せ（flex）、横パディング `1rem`。
- **背景**: 放射グラデの重ね（下寄りの明るさ＋弱いグロー）。**星のスクロールは無し**（やや **暗め**なトーンのまま）。
- **オーバーフロー**: `hidden`。

### ガラス風カード（`.glassCard`）

- 最大幅 `420px`、半透明白＋`backdrop-filter: blur(16px)`、角丸・境界線・シャドウ。
- **入場**: `.cardEnter`（`prefers-reduced-motion` で無効化可）。

### タブ

- ピル型 2 ボタン、紫系アクティブ（`tabActive`）。
- コンテンツは **`key` + `.tabPanel`** で短いフェード。

### フォーム・ボタン

- 入力・送信・ソーシャルは従来どおり（ホバー時の軽い `transform` は `prefers-reduced-motion` で抑制）。

### 画面別のコンテンツ差分

**`LoginScreen`（参加者）**

- タイトル下: 「おかえりなさい」／「はじめまして」。
- タブ: ログイン／新規登録（アイコン付き）。
- 新規登録: ユーザー名必須、生年月日・性別は任意。
- 「または」のあと **Google・Facebook**。
- `onClose` 時のみ右上に閉じる。

**`AdminLoginScreen`（管理者）**

- タイトル下: 「管理者ログイン」／「コミュニティを登録申請」。
- **ログイン・登録ともメール＋パスワードのみ**（**Google ボタン・「または」区切りは置かない**。仕様・理由は [05_管理者ログイン画面](./05_管理者ログイン画面.md)）。
- 登録申請タブ: `registerNote`、コミュニティ名・メール・パスワード（8文字以上）。
- **フッター注意書き（「※ このページは管理者専用です」）は置かない**（カードを縦に短くし、[65 のトップページ](./65_トップページ.md) のガラスカードに近いコンパクトさを狙う。`AdminLoginScreen.module.css` の `.adminFormCard` 等でパディング・見出し／タブ下マージンを詰める）。
- **審査中／却下**: 上記と同じ **`authEntryShell` 背景＋ガラスカード** 内にメッセージと戻るボタン（星背景ではない）。

### 旧実装メモ（改修前）

- 多層 `starsMove` パララックスを撤去した経緯は「変更の経緯」参照。

## 課題と反応（プロダクト）

### 第1回で主に解消したこと

- **流れる細かい星**による視覚負担・チカチカ感。
- ログイン系画面同士の **背景・カードの統一**（シェル化）。

### 残っている要望（第2回改修の入力）

- **背景** → [65](./65_トップページ.md) と同じく **静止画スライドショー優先**（複数景色の **クロスフェード**。切替の既定は **約 5 秒**／フェード **約 1 秒**・`AuthEntryShell`）。プロモ動画背景は **採用しない**。画像の **枚数・解像度・トーン**は [65 の「背景用静止画の要件」](./65_トップページ.md#背景用静止画の要件制作側で用意) に従い、**制作側で素材を用意**する。
- **Hossii** → 景色の手前に **`HossiiLive`** を置き、**くるくる／ふわっとした動き**（CSS 等で補助可）。**`pointer-events: none`**、可読性優先。背景スライド 2 枚以上のとき **`emotion`／`motion` 画像プールからランダム差し替え**（[65](./65_トップページ.md)・`authEntryHossiiPool.ts` と同じ挙動）。ログイン／管理者も **トップと同じ `AuthEntryShell`** で揃う。
- **星パララックス** → 戻さない（旧 `starsMove` 系は不採用）。
- **`prefers-reduced-motion`** → スライド切り替え・キャラの装飾動きを **止める／極端に遅くする**（[65](./65_トップページ.md) と整合）。

## 改修方針（恒久）

### 1. 背景の動き

- **禁止に近い**: 細かいタイルを多層で **高速に見える差で流し続ける**パターン（旧 `starsMove` 系）。
- **許容**: ほぼ静止に近い範囲での **極弱・極遅**の変化、または **opacity のみ**のループ。

### 2. 動きの主役

- **カード入場・タブフェード**を主とする。背景は揺らさないか、ごく弱く。

### 3. アクセサリ（第2回）

- **背景**: **静止画スライドショー**（詳細・枚数・技術目安・**切替間隔（約 5 秒）・フェード（約 1 秒）**は [65](./65_トップページ.md) および `AuthEntryShell`）。カードの下に **スクリム／グラデ**を重ね、文字を読みやすくする。
- **Hossii**: スライドより手前・カードより奥。`HossiiLive`（`decorative`・`decorativeImageSrc`）＋ **ゆるい回転・浮遊**の CSS。**`pointer-events: none`**。スライド 2 枚以上で **`public/hossii/emotion/`・`motion/` プールからランダム**（初回表示は待機顔。詳細は [65](./65_トップページ.md)）。
- **`prefers-reduced-motion: reduce`**: スライド・装飾ループを **停止または短縮**（[MDN](https://developer.mozilla.org/ja/docs/Web/CSS/@media/prefers-reduced-motion) に準拠）。装飾 Hossii の **ランダム差し替えも行わず待機顔固定**。

### 4. スコープ

- 背景レイヤーは **`authEntryShell` または専用ラッパ**で [65](./65_トップページ.md) と **ログイン／管理者**で共通化するのが効率的。
- 静止画ファイルは **制作側が用意**し、リポジトリの所定パスに配置（ルールは実装時にコード／README で明示）。
- `App.tsx` の審査待ち／却下フルスクリーンの紫グラデ統一は別判断（必須ではない）。
