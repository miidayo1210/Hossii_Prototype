# 65 トップページ

## 名称

本プロダクトでは、**ルート URL**（例: `https://hossiiprototype.vercel.app/`）にアクセスしたときに表示される、**未ログイン・非ゲスト**向けの最初の画面を **トップページ** と呼ぶ。

- **実装上のコンポーネント名**: `StartScreen`（`src/components/StartScreen/StartScreen.tsx`）
- 「ホーム」単体はログイン後のメイン利用画面と混同しやすいため、仕様・ドキュメントでは **トップページ** を用いる。

---

## 機能概要

トップページは、ユーザーが **スペースに参加する入口** を示す。認証フローやスペースのデータモデルは [10_ログインユーザー管理設計](./10_ログインユーザー管理設計.md) 等を参照する。

### 表示条件（現状）

`App.tsx` において、**未認証**かつ**ゲストモードでない**ときに `StartScreen` を表示する。

```462:465:src/App.tsx
  // Show start screen if not authenticated (guest mode でない場合)
  if (!currentUser && !isGuestMode) {
    return <StartScreen />;
  }
```

## 変更の経緯

### 第1回実装（反映済み）

| 項目 | 内容 |
|------|------|
| 名称・役割 | 仕様上 **トップページ** と呼称統一。 |
| レイアウト | [64](./64_ログイン画面の使用.md) と同じ **`authEntryShell`**（`.shell` / `.glassCard` / `.cardEnter`）＋ `LoginScreen.module.css` のフォーム系で **ログイン画面とビジュアル統一**。 |
| コピー | 宇宙比喩をやめ、「**つながる空間**」寄りのサブコピー・リード文に変更。 |
| 参加導線 | **スペース ID または共有 URL** を入力 → `parseSpaceSlugFromInput` → `validateSpaceURL` → **`window.location.assign(`${origin}/s/${slug}`)`**（`App.tsx` のスラッグ解決と整合。`pushState` のみでは effect が再実行されないためフル遷移）。 |
| スペース作成 | 「**ログインしてスペースを作る**」は **Coming soon**（無効ボタン＋バッジ）。 |
| Hossii キャラ | 改修に伴い、旧実装にあった背景の **`HossiiLive` を一旦削除**（負荷・「動きはカード側」方針のため）。 |

### 第2回改修（反映済み）

プロダクトからのフィードバックとデザイン意向に基づき、次を **第2回** で実装した（`AuthEntryShell`・`public/auth-entry-bg/`・`authEntryBg.config.ts` を参照）。

| 項目 | 方針 |
|------|------|
| **背景の主手段** | **静止画のスライドショーを優先**する。プロモーション動画を薄く流す案は **採用しない（優先度はスライドショー）**。動画は将来オプションとして再検討してよい。 |
| **景色のイメージ** | 青空・広場・星空など **いろいろな景色**を、**クロスフェード**で切り替える（実装の既定は **約 5 秒** ごとに次画像へ、フェード **約 1 秒**。`AuthEntryShell` の定数で変更可）。全体の雰囲気は **『どうぶつの森』のタイトル画面のような優しさ**を参考にする（デザインのコピーではなく **トーン・明るさの参考**）。 |
| **Hossii** | 景色の **手前（カードより奥）** に `HossiiLive` を配置し、**くるくる／ふわっと踊る**感じを **CSS のゆるい回転・浮遊**などで補強。**`pointer-events: none`**、可読性を最優先。背景スライドが **2 枚以上**のときは、**クロスフェード完了後に表示が次画像に確定するたび**に `public/hossii/emotion/`（8 感情＋追加 PNG）と `public/hossii/motion/` から **1 枚をランダム**で差し替え（**最初の一枚目の表示中は**通常の待機顔 `idle`）。背景と **1:1 の対応は不要**（プールは `authEntryHossiiPool.ts`）。 |
| **星パララックス** | **戻さない**（旧 `starsMove` 系は不採用）。 |
| **アクセシビリティ** | 切替は **眩しすぎない程度**に抑え、必要なら実装の定数で間隔・フェードを調整する。**`prefers-reduced-motion: reduce`** ではスライド切り替えを **止め、1 枚固定**（トランジションなし）とし、装飾 Hossii も **待機顔のまま**（ランダム差し替えなし）。 |

目的は **「チカチカしない」「暗すぎない」「色々な景色の中で Hossii が楽しそうに見える」** の両立である。間隔・フェード時間は上記のとおり実装済みで、必要に応じて定数・CSS で調整する。

## 背景用静止画の要件（制作側で用意）

**画像はプロダクト／デザイン側（本仕様書の読者側）で用意する。** 実装は、リポジトリ内の所定パスに置かれたファイルを読み込む形にする想定（配置パス・ファイル名ルールはコード側で README またはコメントで明示してよい）。

### 枚数

| 目安 | 説明 |
|------|------|
| **推奨 4〜6 枚** | ループが長くなりすぎず、景色のバリエーションも出せる。 |
| **最低 3 枚** | スライドショーとして成立させる下限。 |
| **7 枚以上** | 可だが、1 周の時間と読み込み量が増えるため、必要なら **後から追加**でよい。 |

### どんな絵にするか（テーマ例）

すべてを満たす必要はなく、**雰囲気が揃っていればよい**。

- **青空・雲**（明るく開放的）
- **広場・芝生・木陰**（「空間」感・やさしい昼）
- **夕方・マジックアワー**（暖色、強すぎないコントラスト）
- **夜空・星**（宇宙ではなく **静かな夜**のイメージ。細かい点がチカチカしないトーン）
- **室内・窓辺・パステル調の抽象** など、Hossii の世界観に合う **柔らかい**シーン

**避けたい例**: 画面中央に強い模様・文字・顔がドンとある（**ガラスカード周辺の可読性**が落ちる）。

### トーン・構図

- **彩度は抑えめ〜中程度**、**コントラスト過多**にしない（参考: どうぶつの森系のやわらかさ）。
- **安全エリア**: 中央〜やや下は **フォームやタイトルが乗る**ため、そこだけ極端に busy にしないと読みやすい（四隅や周辺に「おいしさ」を寄せてもよい）。
- 全枚で **トーンをそろえる**（1 枚だけ写真実写で激しく、他がイラスト調、などのギャップは避ける）。

### 技術仕様（目安）

| 項目 | 推奨 |
|------|------|
| **比率** | 横長 **16:9** を基準（実装では `object-fit: cover` でトリミング前提） |
| **解像度** | **1920 × 1080** 程度で足りる（まずはこれで統一） |
| **形式** | **WebP** 優先。**JPEG** でも可。**PNG** はファイルサイズが大きくなりやすいので背景には非推奨 |
| **1 枚あたりの容量** | **約 200〜600 KB** を目安に圧縮（モバイル・初回表示を考慮） |
| **ファイル名** | 実装時に `01.webp` のような連番、または `sky.webp` / `plaza.webp` など **英小文字・ハイフン**で統一（コード側で一覧を配列管理） |

### 権利

- **自作・社内制作・ライセンス済みの依頼制作**のみ使用する。ストックの無断利用は不可。
