## 対応状況

本節はレビュー記録に対する**フォローアップ実装の結果**である。

| ID | 状態 | 実施内容の要約 |
|----|------|----------------|
| R1 | 対応済み | `AuthEntryShell` のインターバル内で `clearTimeout` / `cancelAnimationFrame` をクリーンアップし、`cancelled` ガードでアンマウント後の `setState` を抑止 |
| R2 | 対応済み | R1 と同一変更で、Strict Mode 下の二重マウント時もクリーンアップで後続コールバックを無効化 |
| U1 | 対応済み | `parseSpaceSlugFromInput` の `extractFromPath` で pathname の末尾スラッシュを正規化してからマッチ |
| N1 | 対応済み | 背景を WebP に変換し `AUTH_ENTRY_BG_IMAGES` を差し替え、大型 PNG をリポジトリから削除（1 枚あたり目安レンジに収まるサイズ） |
| N2 | 対応済み | スライド進行を `setBottomIdx` の updater 外のインターバルコールバックに移し、`bottomIdxRef` で現在インデックスを参照 |

**再オープン条件**: `AuthEntryShell` のタイマー定数・フェード CSS を変更するときは、クリーンアップと `FADE_MS` の整合を再度確認する。背景素材を差し替えるときは引き続き WebP 優先と容量目安に従う。

## 指摘一覧 重要度別

### 要対応候補 実装バグになりうる

| # | 観点 | 内容 | 合格の目安 |
|---|------|------|------------|
| R1 | ライフサイクル | `AuthEntryShell` のスライド切替で `setInterval` 内から `setTimeout` および二重 `requestAnimationFrame` を起動しているが、**アンマウント時にそれらをキャンセルしていない** | クリーンアップで `clearInterval` に加え、保留中の `clearTimeout` / `cancelAnimationFrame`、またはマウント済みフラグで `setState` を抑止する |
| R2 | React Strict Mode | 開発時の effect 二重実行と組み合わせると、上記の未キャンセルタイマーが**警告や不整合の原因**になりやすい | R1 の対策で同時に解消できることを確認する |

### UX 上バグに見えやすい

| # | 観点 | 内容 | 合格の目安 |
|---|------|------|------------|
| U1 | URL パース | 共有 URL の **pathname 末尾スラッシュ**（例: `/s/myspace/`）があると、正規表現がマッチせず入力全文がスラッグ扱いに近くなり、`validateSpaceURL` で弾かれる | マッチ前に末尾スラッシュを除去する、または正規表現で許容する。エラーメッセージがユーザーにとって理解しやすいこと |

### 非機能 パフォーマンス・運用

| # | 観点 | 内容 | 合格の目安 |
|---|------|------|------------|
| N1 | 背景画像容量 | `public/auth-entry-bg/` の PNG が **1 枚あたり数 MB 規模**で、初回表示・モバイルで負荷になりうる | 仕様 65 の推奨どおり WebP 化・圧縮、`AUTH_ENTRY_BG_IMAGES` の差し替え |
| N2 | 実装スタイル | `setBottomIdx` の関数型更新の内部で複数 `setState` とタイマーを起動しており、**React の推奨パターンから外れている** | インターバルコールバックを通常関数に切り出し、状態遷移を読みやすく保守しやすくする |

### 低リスク 記録のみ

| # | 観点 | 内容 |
|---|------|------|
| L1 | 装飾 Hossii | `decorative` 時は泳ぎ・タップ等は止めており、ビルドも通過。通常利用で致命傷は見込みにくい |
| L2 | トップ遷移 | `StartScreen` は `location.origin` 固定で `/s/...` に `assign` しており、任意ホストへのオープンリダイレクトにはなっていない |
| L3 | パス正規表現 | `parseSpaceSlugFromInput` の `/c/.../s/...` および `/s/...` は `App.tsx` の pathname 判定と揃えてある |
