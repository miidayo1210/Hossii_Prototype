# 68 スペース書き出し機能

## この文書の目的

スペース画面を **1 枚の画像／PDF として書き出し・ダウンロード**する機能の要件を定義する。レイヤーが多いスペース UI について、**写す範囲**と**写さない UI** を仕様上明確に分離し、形式・品質・権限・失敗時の扱いまで揃える。

**関連:** [10_ログインユーザー管理設計.md](./10_ログインユーザー管理設計.md)（コミュニティ／管理者） / [69_投稿のUI設定.md](./69_投稿のUI設定.md)（フリー投稿のキャプチャ整合）

---

## 概要

- **出力物**: 壁紙・装飾・（画面上に見えている）吹き出し・Hossii キャラ・QR などが **1 枚に合成**された **PNG / JPEG / PDF（ラスタ 1 ページ）**。**現行実装は PNG のみ**；JPEG・PDF・高 DPI 明示・背景縮小・フッタ「一言」・管理画面からの設定／履歴は **次項以降のロードマップ**で定義する。
- **操作**: 画面 **右上クラスタに「書き出し」ボタン**（投稿数バッジ等と同じ右上エリア）。条件を満たすときのみ表示。**形式・品質・解像度プリセット**は書き出し **モーダル内**で選べるようにする（既定は PNG・端末追従の `devicePixelRatio`）。確定で生成し、ブラウザ **ダウンロード**（履歴は別節）。
- **権限**: **誰でも不可**。**コミュニティ管理者**（`isAdmin`）、またはスペース単位 Feature Flag **`space_canvas_export_enabled` が ON** のときのみボタンが表示・利用可能（詳細は「権限」節）。**書き出し履歴の閲覧・「一言」編集**は **コミュニティ管理者**（および将来のロール拡張）に限定する。

## 実装概要（現在のプロダクト）

| 項目 | 内容 |
|------|------|
| キャプチャ手段 | メイン領域相当の **DOM ルート**（書き出し専用コンテナ）を `html-to-image` でラスタ化し、その上に **細い枠・ヘッダ（スペース名・日付）・フッタ（QR）** を canvas で合成した **1 枚の PNG**。 |
| 共有 URL（QR） | 既存の QR パネルと同じ規則（コミュニティスラッグ付き URL 等）。 |
| 吹き出し上限 | 表示中の投稿数 **80 件超**のときは書き出し不可（ツールチップで案内。エラー時も同趣旨の文言）。 |
| 解像度 | 端末の **`devicePixelRatio` を `pixelRatio` に反映**（**高 DPI の明示プリセット**はロードマップで追加）。 |
| 操作中 UI | **モーダル表示**と **キャンセル**ボタン。`AbortSignal` により完了後のダウンロードを抑止（ライブラリ都合で途中打ち切しは保証しない）。 |
| ロードマップ（未実装） | **JPEG / PDF**、**解像度プリセット**、**壁紙等の縮小キャプチャ**、**フッタ「Hossii からの一言」**、**スペース設定の「書き出し」タブ**（一言編集・**書き出し履歴**）、コミュニティ横断ビュー（任意）。詳細は下記各節。 |

## キャンバス・アスペクト・余白

| 項目 | 仕様 |
|------|------|
| 縦横比・サイズ感 | **現在表示されているメインスペース領域**に合わせる。**16:9 相当**を目安とする（実装はビューポート／レイアウト・表示倍率に追従）。 |
| 余白（フレーム） | **細いフレーム**を付与する。**スペース名・書き出し日付**をフレーム内の余白（ヘッダ）に表示する。**現行実装ではロゴは未配置**（将来、同じ余白帯に追加可能）。**装飾トーンは下記「フレーム周りのデザイン方針」に従い、本体 UI と統一する。** |
| QR の位置 | **フレームのフッタ余白・右下**に配置（コンテンツ本体と重ならないレイアウト）。見出し「スペースに参加」を付与。**QR まわりの色・余白・タイポも同じ方針で揃える。** QR の **左**のコピー領域（「Hossii からの一言」）は **下記「フッタ左…」節**。 |

## フレーム周りのデザイン方針（スペース名・日付・QR）

書き出し画像の **枠線・背景地・ヘッダ（スペース名／日付）・フッタ（QR とその見出し）** は、中身のスペース画面だけでなく **「Hossii らしい一枚」としての顔**になる。次の方針で **デザインと色味を統一**し、**より可愛く・親しみやすく**仕上げること（実装は canvas 合成などでも、見た目のルールは共通とする）。

- **トーン**: 堅すぎない **柔らかさ・あたたかさ**を優先する。コントラストは読みやすさを落とさない範囲で確保する。
- **色味の統一**: 枠・ヘッダ帯・フッタ帯・主テキスト・補助テキスト・QR の白余白まわりを **一つのパレット**にまとめる（単色の事務的グレー一辺倒を避け、**ブランドやスペース UI と調和するアクセント**を検討してよい）。
- **タイポグラフィ**: スペース名・日付・「スペースに参加」等の **字重・サイズ階層・字間**をそろえ、**同じフォントファミリ（またはアプリ既定の UI フォント）**で揃える。
- **QR ブロック**: QR 本体は読み取り優先で **十分な白余白（クワイエットゾーン）**を保ちつつ、**角丸の台紙・ごく薄い背景・細い境界**などでヘッダ／フッタと **同じ言語**で装飾する。
- **フッタの横並び（一言ブロック）**: QR 台紙の **左**に短文ブロックを置く場合の **視覚ルール**は、下記 **「フッタ左「Hossii からの一言」」** 節と一体で設計する（未設定時は領域を空けすぎない等）。
- **余白とリズム**: ヘッダ・本文キャプチャ・フッタの **余白の取り方をルール化**し、詰まりすき・離れすぎの両方を避ける。
- **本体 UI との一貫性**: アプリの **トップバー・ボタン・グラデーション**など既存コンポーネントのトークン（色・角丸・影）があれば **流用または準拠**し、画面内 UI と書き出し PNG で **別アプリ感**が出ないようにする。
- **将来**: ロゴを余白に置く場合も、**同じパレットと余白ルール**の上に載せる。

※ 上記は **仕様上のデザイン要件**（「こうしたい」）であり、ピクセル固定値はデザイン／実装の反復で決める。

## フッタ左「Hossii からの一言」（空間レビュー短文）

**現行実装では未対応。** フレーム合成（`exportSpaceCanvasWithFrame` 相当）の **フッタ帯**に、QR の左へ **任意で**表示するブロック。

| 項目 | 方針 |
|------|------|
| 位置・レイアウト | **QR ブロック（台紙＋コード）の左**に、**見出し＋本文**の縦スタックまたは短い段落。QR の **読み取りに必要な幅は常に確保**。書き出し幅が狭いときは **改行・文字数上限**（例: 本文 〜120 文字目安、実装で調整）で省略・折り返し。 |
| 見出し | 既定ラベル例: **「Hossii からの一言」**（プロダクト文案で変更可）。**スペース設定で非表示／カスタム見出し**に拡張してよい。 |
| 本文の性質 | スペースの **雰囲気・活動の要約・励まし** など。**誤解を招く断定**・**個人特定**・**他者を貶める内容**は不可。ガイドラインは運用ドキュメントで補足。 |
| **本文のソース（優先の目安）** | 1) **スペース設定で管理者が手入力**（最優先で確実にコントロール可能）。2) **未入力時**に限り、ルールベースの短文案内や将来の **AI 要約**を **下書きとして提案**し、管理者が承認・編集してから保存、など。**保存された値が書き出しに使われる**ことを原則とする（現場の責任範囲を明確にする）。 |
| データ保持 | **スペース設定**（例: `space_settings` の JSON カラム、または専用カラム）に **`export_footer_blurb`（本文）**、任意で **`export_footer_heading`** を保持。未設定・空文字は **左ブロックを描画しない**（プレースホルダなし。フッタの高さは QR ブロック基準でよい）。 |
| 権限 | **編集**: コミュニティ管理者（`isAdmin`）または将来の「スペース編集」ロール。**一般メンバーからの編集は不要**（読み取りも設定画面外では不可でよい）。 |

デザインは **「フレーム周りのデザイン方針」** のパレット・タイポに **必ず揃える**。

## 出力形式の詳細（PNG / JPEG / PDF）

| 形式 | 内容 |
|------|------|
| **PNG** | **現行どおり**。透過が必要な要素との相性がよい。ファイルサイズは JPEG より大きくなりがち。 |
| **JPEG** | 合成済み **1 枚のラスタ**を `image/jpeg` で保存。**品質スライダーまたは Low / Standard / High** の段階でも可（既定は Standard）。**アルファ通道は持たない**ため、フレーム内の透過要素は **マット色**（既存 `CANVAS_BG` 等）上に焼き付けた見た目とする。 |
| **PDF** | **第 1 段階**: 上記と同じ **1 枚のラスタ**を **1 ページに貼った PDF**（解像度は PNG/JPEG と同じ生成パイプラインの結果を利用）。**第 2 段階（長期）**: テキスト選択可能・ベクター中心は任意目標。 |

- 書き出しモーダルで **形式を切り替え**、拡張子・MIME・`toBlob` / PDF 生成ライブラリを切り替える。
- **ファイル名**: 既存ルール（スペース名・日付・スラッグ）に **`.png` / `.jpg` / `.pdf`** を付与（サニタイズは共通関数で拡張）。

## 解像度・ピクセル密度（高 DPI 明示オプション）

| 項目 | 方針 |
|------|------|
| **既定** | **画面追従**: 現行どおり `devicePixelRatio` を `html-to-image` の `pixelRatio` に反映。 |
| **明示プリセット** | モーダルで **「標準（端末に合わせる）」** に加え、例: **1x / 1.5x / 2x / 3x** または **「高解像度（最大 2x）」** など **上限付き**の選択肢を提供。**無制限に上げない**（メモリ・タイムアウト・ブラウザクラッシュ防止）。 |
| **上限** | 生成画像の **長辺ピクセル上限**（例: 4096 または 8192）を仕様値として定め、プリセットと組み合わせて **クランプ**する。 |
| **表示** | 履歴・設定に **実効ピクセル寸法またはプリセット名**を記録し、サポート・再現に使う。 |

## 背景画像・メモリ・時間対策（キャプチャ前処理）

| 項目 | 方針 |
|------|------|
| **目的** | 高解像度壁紙・多数 DOM による **メモリ超過・タブ落ち・極端な待ち時間**を抑える。 |
| **壁紙の縮小** | キャプチャ用パイプライン内で、背景画像を **長辺上限**（例: 2048px、プリセット連動で変更）に **縮小したオフスクリーン／一時レイヤ**から取り込む、または `html-to-image` 前に **data URL 化する際に縮小**する、等。実装は **見た目の劣化が最小になる方式**を選ぶ。 |
| **段階的フォールバック** | プリセット「高解像度」失敗時は **自動で 1 段階低い解像度で再試行**する、またはユーザーに **簡易モード**を提示してよい。 |
| **ユーザー向け文言** | 失敗時は **「投稿数・壁紙・解像度を下げて再試行」** など、仕様 68 の大量データ節と整合するメッセージを出す。 |

## コミュニティ管理・スペース設定（書き出しタブ／履歴）

管理者が **書き出しまわりを把握・設定**できる UI を **スペース設定画面**（実装: `SpaceSettingsScreen`）に **「書き出し」タブ**として追加する方針とする（タブ名・分割は実装で調整可。既存 **シェア / QR** と近いが、**履歴・品質・一言**は本タブに集約しやすい）。

### 設定（タブ内）

| 領域 | 内容 |
|------|------|
| **一言（フッタ左）** | 見出し（任意）・本文の **編集・保存**。文字数カウンタ・ガイドラインリンク。空なら書き出しに **ブロック非表示**（上節どおり）。 |
| **既定の書き出しオプション（任意）** | スペースごとに **既定形式**（PNG 推奨）・**既定 JPEG 品質**・**既定解像度プリセット**を保存し、スペース画面のモーダル初期値に反映してよい。 |
| **Feature Flag との関係** | `space_canvas_export_enabled` が OFF のときは **スペース上の書き出しボタンは非表示**のまま。**設定タブは管理者のみ閲覧可**とし、一言の下書き編集を許すかはプロダクト判断（通常は **フラグ ON 時のみ編集意味がある**旨を UI で示す）。 |

### 書き出し履歴（監査・参照）

| 項目 | 方針 |
|------|------|
| **記録タイミング** | ブラウザで **書き出しが成功しダウンロードが開始されたとき**（またはサーバ生成に切り替えた場合は **ファイル準備完了時**）に 1 件追加。 |
| **記録項目（最低限）** | **日時**、**スペース ID**、**実行者ユーザー ID**（匿名運用時は方針要定義）、**形式**（png/jpeg/pdf）、**解像度プリセット名または概算ピクセル寸法**、**成否**（失敗は任意でエラーコードのみ）。 |
| **ファイル本体** | **フェーズ A（推奨）**: **メタデータのみ**。ファイルはユーザーのローカルダウンロードに任せ、サーバに残さない（プライバシー・コスト・容量の観点）。**フェーズ B（任意）**: Supabase Storage 等に **暗号化または限定公開**で保存し、管理画面から **再ダウンロード**（保持期間・削除ポリシー必須）。 |
| **閲覧権限** | **コミュニティ管理者**が **自コミュニティのスペース**の履歴を見られる。コミュニティ横断の **一覧画面**（例: コミュニティ管理ダッシュボード）は **フェーズ 2** として任意。 |
| **保持期間** | 既定 **90 日**など上限を仕様で定め、超過分は自動削除（実装はバッチまたは DB TTL）。 |
| **プライバシー** | 履歴は **個人の行動ログ**にあたるため、利用規約／プライバシーポリシーでの **説明対象**とする。 |

## スペース画面上の書き出しモーダル（拡張 UI）

| 項目 | 方針 |
|------|------|
| 形式 | **PNG / JPEG / PDF** の選択（ラジオまたはセレクト）。 |
| 品質 | JPEG のみ **品質** UI を表示。 |
| 解像度 | **端末に合わせる**＋ **明示プリセット**（上節）。 |
| プレビュー | **必須としない**（コストが高い）。任意で **縮小プレビュー**は後続。 |
| 確定 | 既存どおり **処理中表示・キャンセル**を維持。 |

## 背景・スライドショー（方針 A）

| 状態 | 書き出し |
|------|----------|
| **通常のスペース表示** | **壁紙背景・スペース装飾を含めて**書き出す。 |
| **スライドショー専用画面**、または **スライドショー表示中** | **書き出し不可**（**書き出しボタン自体を表示しない**）。ユーザーがスライド画面のまま画像を得られないこと。 |

## 写す範囲に含めるもの

- 通常スペース表示における **壁紙・装飾**（キャプチャルート内に含まれる範囲）。
- **画面上の一覧に含まれる吹き出し**（期間・件数・表示モードで絞り込まれた `displayHossiis` に整合。**スクロール領域が別途ある将来仕様**では「見えている範囲」の解釈を再確認する）。
- **フリー投稿**（`post_kind === canvas`・ツリー上のラスタカード。旧称キャンバス投稿）。**吹き出しと同じく表示中一覧に載るもののみ**がキャプチャ対象（[69](./69_投稿のUI設定.md) と整合）。透過 PNG の部分は、合成フレームの **内側マット**（`exportSpaceCanvasWithFrame` 内の `CANVAS_BG` 塗りつぶし後に `drawImage` するため）上に乗る。
- **Hossii キャラ**（定位置で吹き出しの邪魔にならない配置）。**複数パターン（目安 5 種）**を将来的に選べる拡張を想定し、**MVP は 1 パターン**でよい。
- **QR**（**合成フレームのフッタ**に描画。画面上の QR フローティングパネルは写さない）。**フッタ左の「一言」**は設定・データがある場合に **合成へ含める**（詳細は **「フッタ左「Hossii からの一言」」** 節）。

### データ上の除外

- **非表示**の投稿、**モデレーションにより非表示**の投稿は **書き出しに含めない**（画面上に出ていないものと整合）。

### 書き出し直前の挙動（実装）

- 吹き出し **選択中**（編集 UI 表示中）の場合、実行時に **選択を解除**してからキャプチャする（ハンドル等が残らないようにする）。

## 写さない UI・オーバーレイ（一覧）

次の要素は **書き出し画像に含めない**。実装は **写すルート（DOM）の分離**を主とし、ルート内に残るノードは **`data-space-export="exclude"`** や **`data-resize-handle` の filter 除外**で二重に防ぐ。

| 対象 | 出力に含める（○＝含む／×＝含まない） |
|------|:----------------------------------:|
| 左コントロールバー | × |
| トップバー | × |
| ナビゲーション（メニュー類） | × |
| 音声パネル | × |
| 投稿パネル | × |
| ログ一覧などのパネル | × |
| QR フローティングパネル（画面オーバーレイ） | ×（※合成 PNG 内の QR は別途フレームに含める） |
| メッセージボトル | × |
| スピーチの文字起こしパネル等、オーバーレイ系（原則すべて） | × |
| 訪問バナー | × |
| 左上のスペースタイトル（情報バッジ） | × |
| 右上の投稿数バッジ・投稿順ツールバー・**書き出しボタン** | × |
| Listening インジケーター | × |
| 吹き出しの **編集 UI**（選択ハイライト、ハンドル等） | × |
| トースト | × |
| モーダル | ×（下記「モーダル時の制限」） |

### トーストとは

短時間表示される **通知メッセージ**（成功・エラー等）。書き出しに写さない。

### モーダルとは

画面中央などに重なる **ダイアログ／詳細画面のオーバーレイ**（例: 投稿詳細、同意確認など）。

### モーダル時の制限

**いずれかのモーダルが開いている間は書き出し不可**（ボタン無効、ツールチップで案内）。トーストのみの表示は、**モーダル表示中かどうか**の判定に含めない。

**実装上、次を開いている間も書き出し不可**（いずれもオーバーレイ／パネルに相当）:

- 投稿詳細モーダル（モバイル）、Listen 同意モーダル
- クイック投稿パネル、ログ一覧パネル、音声パネル（開いているとき）
- 装飾の **ポップアップが開いているとき**（装飾ウィジェット自体はキャプチャに含める。ポップアップはモーダル相当としてブロック）

## 訪問モード（隣人・訪問中）

- **訪問モード中の書き出しはスコープに含めない**。
- **隣人のスペースを訪問している状態では、当該スペースを書き出せない**（**書き出しボタンを表示しない**。自スペースでのみ、かつ権限あるユーザーに限定）。

|------|------|------|

## ファイル名

**スペース名・日付・スラッグ**を **`_` 連結**し、OS 禁止文字等を除去して **最大長を制限**したうえで、選択した形式に応じて **`.png` / `.jpg`（または `.jpeg`）/ `.pdf`** を付与（実装でサニタイズ。拡張子は `buildSpaceExportFilename` 等で統一）。

---

## 操作中・キャンセル

- 書き出しボタン押下後、**処理中であることが分かる表示**（オーバーレイ／モーダル）を出す。
- **キャンセル**可能にする。キャンセル後は **ダウンロードしない**。
- **書き出し処理中モーダル**（および形式・解像度を選ぶ **拡張モーダル**）は **キャプチャルート外**であり、**「モーダル時の制限」でブロックする対象のモーダルではない**（投稿詳細など **別機能のオーバーレイ**のみブロック）。

---

## 大量データ・失敗時

次のいずれかまたは組み合わせで **仕様として許容**する。

- **吹き出し枚数の上限**（**現行: 80 件**。超過時はボタン無効＋ツールチップ、またはエラー文言）。
- **背景画像等の縮小・解像度クランプ**（メモリ・時間超過の回避）— **「背景画像・メモリ・時間対策」** および **「解像度・ピクセル密度」** 節（**未実装**、ロードマップ）。
- **エラーメッセージ**を明示（原因が分かる文言）。**CORS 等で壁紙画像が canvas に取り込めない場合**は、単色・パターン背景での再試行を促す旨を含めてよい。

## 権限

- **全ユーザーが書き出せるわけではない**。
- **コミュニティ管理者**（承認済み管理者ユーザー）、またはスペースに対して Feature Flag **`space_canvas_export_enabled` を ON** にしたとき（スペース設定の Feature Flags タブ）のみ、**書き出しボタンが表示され、利用できる**。
- **`zine_export_enabled`（ZINE 出力）とは別キー**とする。
- 将来、メンバー単位の許可や DB ロールに拡張してよい。

---

## 実装メモ（参考・仕様外の自由度）

- 「写す範囲」と「UI を隠す」は、**専用の書き出し用コンテナ**（キャプチャルート）に相当する DOM 境界を定義すると保守しやすい（**現行: `SpaceScreen` 内の書き出しルート**）。
- スライドショー中は **UI レベルでボタン非表示**とするのが要件と一致しやすい。
- 履歴テーブルは **`space_id` + `community_id` インデックス**を貼り、RLS で **自コミュニティの管理者のみ SELECT** とすると [10](./10_ログインユーザー管理設計.md) と整合しやすい。
