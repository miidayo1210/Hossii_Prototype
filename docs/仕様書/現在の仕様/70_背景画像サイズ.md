# 70 背景画像サイズ（スペース壁紙・フリー編集の背景写真）

## この文書の目的

**画像を背景として敷いたとき**、ビューポートやコンテナの縦横比と写真の縦横比が一致しない場合の **切り抜き・余白** の扱いを整理する。実装の正は CSS（`background-size`・二層レイヤー・`filter: blur`）および `CanvasPostEditor` のステージ背景。

---

## 現状の挙動（実装）

| 箇所 | 実装の目安 | 結果 |
|------|------------|------|
| **スペース全体の背景画像** | [`spaceBackgrounds.module.css`](../../../src/styles/spaceBackgrounds.module.css) の **`.bgImageStack`** 内、**下層 `.bgImageBlurFill`**（`background-size: cover` ＋ **`filter: blur`** ＋ わずかな **`scale`**）、**上層 `.bgImageSharp`**（**`contain`**・中央） | 写真は **まるごと（シャープ）** 表示。余白は **同じ画像を拡大・ぼかした色味** で埋まり、単色レターボックスより違和感が少ない。ルートには読み込み失敗用に **暗色フォールバック**（[`SpaceScreen.tsx`](../../../src/components/SpaceScreen/SpaceScreen.tsx) の `SPACE_IMAGE_LETTERBOX_COLOR`）を指定。 |
| **フリー編集ステージの背景写真** | [`CanvasPostEditor.module.css`](../../../src/components/PostScreen/CanvasPostEditor.module.css) の **`.stageBgStack`**／**`.stageBgBlur`**／**`.stageBgSharp`**（スペース壁紙と同じ発想） | **壁紙透過オフ**かつ写真ありのとき **ぼかし下層＋ contain 上層**。**壁紙透過オン**のときは従来どおり **単層・contain**（レターボックスは透明でスペース側が見える）。**色**ピッカーは、ぼかし併用時は **スタック下のフォールバック色**（画像周りのベース）として効く。 |

---

## 課題（プロダクト）

- 利用者が **「写真まるごと見せたい」** とき、`cover` だけでは不満が出る。
- **`contain` だけ＋単色余白** だと、写真の色味と余白の差が **目立つ** ことがある → **ぼかし拡大レイヤー**で緩和。

---

## 方針の選択肢（推奨の整理）

| 方針 | 内容 | メリット | デメリット |
|------|------|----------|------------|
| **A. cover のまま** | 画面いっぱいに切り抜き | 常に「埋まる」。 | 比率が合わないと **必ず切れる**。 |
| **B. contain ＋単色余白** | 画像全体を枠内に収め、余白は一色 | **切れない**、実装が単純。 | 色味の **違和感** が出やすい。 |
| **B′. contain ＋ぼかし拡大（採用）** | 下層に同一画像を cover＋blur、上層に contain | **切れない**、余白が **写真に近い色** になる。 | `filter` の **GPU 負荷**、極端に弱い端末では要検討。 |
| **C. ユーザー選択** | 「全体表示／画面いっぱい」等のトグル | 用途に合わせられる。 | UI・保存項目の増加。 |
| **D. スペースサイズを画像に合わせる** | レイアウト基準を画像比率に依存 | 理論上は切れにくい | 吹き出し・書き出し・UI と **衝突しやすい**。基本 **非推奨**。 |

### 本書の推奨（段階的）

1. **短期（現状）**: **B′** をスペース壁紙・フリー編集（透過オフ時）に適用。  
2. **中期**: 需要があれば **C（トグル）** や **ぼかし強度** の調整を検討。  
3. **D** はデフォルト案にしない。

---

## 実装メモ

- **スペース**: 画像 URL は `SpaceScreen` の **`imageWallpaperUrl`** で分岐し、**二層 div** を `spaceExportRoot` 直下（`StarLayer` より手前）に挿入。書き出し（[`spaceCanvasExport.ts`](../../../src/core/utils/spaceCanvasExport.ts) の `toPng`）も **DOM 上の見た方がそのまま焼かれる**。
- **フリー**: **壁紙透過オン**ではぼかし層を付けない（スペース壁紙を阻害しない）。
- **パフォーマンス**: `blur` は 1 枚あたり 1 層。値は CSS で調整可能（現状おおよそ **28px**、`scale`／**`inset: -12%`** で端の滲みを抑える）。
- 詳細は [69 投稿の UI 設定](./69_投稿のUI設定.md)（フリー背景）、[68 スペース書き出し機能](./68_スペース書き出し機能%20.md) と突き合わせる。
