# 75 スマホ横持ちレイアウト

> **分類:** `[Core / UX]` モバイルランドスケープ時のスペース画面レイアウト  
> **関連:** [11_レスポンシブ設計](./11_レスポンシブ設計.md) / [62_スマホのスペース画面](./62_スマホのスペース画面.md) / [71_スマホマイク機能](./71_スマホマイク機能.md) / [08_スペース（HOME）の仕様書](./08_スペース（HOME）の仕様書.md)

> 最終更新: 2026-06-02

---

## 1. 目的

スマートフォンを **横持ち（ランドスケープ）** にしたとき、現状のモバイル UI（星表示＋大きなプレビュー＋縦型左バー等）が **文字過大・情報過多** に感じられる問題を解消する。

**ゴール:** PC 画面（769px 以上）に近い **情報密度・余白・コンポーネント比率** でスペースを見せる。縦持ちモバイル UX（コンパクト・投稿しやすさ）は維持する。

---

## 2. 判定条件

| 概念 | 条件（実装目安） | 用途 |
|------|------------------|------|
| **モバイル** | `max-width: 768px` | 既存 `useMediaQuery` と同一 |
| **縦持ちモバイル** | モバイル **かつ** `orientation: portrait` | StarView・横長左バー（[62](./62_スマホのスペース画面.md)） |
| **横持ちモバイル** | モバイル **かつ** `orientation: landscape` | 本仕様の対象 |

`screen.orientation.lock` 等の強制回転 API は **使わない**（[62](./62_スマホのスペース画面.md)・[71](./71_スマホマイク機能.md) と整合）。

---

## 3. レイアウト方針（縦 vs 横）

| 項目 | 縦持ちモバイル | 横持ちモバイル | PC（769px+） |
|------|----------------|----------------|--------------|
| 投稿の見た目 | **StarView**（点＋一部プレビュー） | **Bubble に近い表示**（または Star＋プレビュー OFF） | **Bubble**（フル／バブル／画像モード） |
| 星プレビュー（3件ローテ） | あり（現状） | **OFF** または縮小 | — |
| 左コントロールバー | **左上・横長**（[62](./62_スマホのスペース画面.md)） | **左縦・コンパクト**（PC に近い） | 左縦・フル |
| スペース名（`.spaceTitle`） | 現状 | `clamp` で小さめ、安全域内 | 現状 |
| 下ナビ（BottomNavBar） | 表示 | 表示（高さは現状） | 非表示 |
| クイック投稿パネル | **上 16:9 + 下固定ドック**（[51](./51_簡単投稿.md)・[76](./76_スマホ投稿ナビとクイック投稿.md)） | **左スペース + 右固定ドック**（幅 `clamp(240px, 36vw, 320px)`・ドラッグ不可） | 右サイド（`FloatingPanelShell`） |
| 大画面（没入）モード | CSS 優先 | PC 寄せの余白（既存 `containerImmersive` + 拡充） | CSS + Fullscreen 補助 |
| 表示倍率・表示モード切替 | 非表示（現状） | 非表示（現状） | 表示 |

---

## 4. タイポグラフィ・UI 密度

横持ちモバイルでは **PC 比率を上限** とし、以下を抑える。

| 対象 | 方針 |
|------|------|
| `StarView` プレビューバブル | 横持ちでは **非表示**（タップで `PostDetailModal`） |
| 吹き出し内テキスト（Bubble 使用時） | PC と同系の `font-size` / `line-height`（過大な `rem` を避ける） |
| 装飾ウィジェット | フォントサイズを 1 段階縮小 |
| HossiiLive | 下端余白を横持ち用に再計算（BottomNav + 安全域） |
| 横持ち案内バナー（71） | 縦持ち＋Listen 時のみ。横持ちでは **出さない** |

---

## 5. CSS・実装の所在（予定）

| ファイル | 変更内容 |
|----------|----------|
| `SpaceScreen.tsx` | `isPortrait` / `isMobileLandscape` で Star/Bubble 切替、プレビュー OFF |
| `SpaceScreen.module.css` | `containerMobilePostLandscapeSplit` / `mobilePostSideDock`、landscape 用メディアクエリ |
| `StarView.module.css` | 横持ち時プレビュー非表示 |
| `LeftControlBar.module.css` | 横持ち時は左縦コンパクト（縦持ちの横長バーと分岐） |
| `Tree.tsx` / `Bubble` | 横持ちモバイルでも Bubble パスを通せるよう分岐 |

---

## 6. 受け入れ条件

- [ ] 横持ち iPhone / Android Chrome で、スペース画面が **PC より過剰に大きい文字・プレビューだらけに見えない**
- [ ] 横持ちでも投稿のタップ → 詳細（投稿者・画像・気持ち等）は [11](./11_レスポンシブ設計.md) の方針どおり確認できる
- [ ] 縦持ちモバイルの StarView・横長左バー（62）の挙動は **退化しない**
- [ ] 横持ち案内バナーが横持ち画面で **重複表示されない**
- [ ] 訪問モード・スライドショー中の既存制約は維持

---

## 7. 実装フェーズ（参考）

1. **P0:** `orientation: landscape` 用 CSS（余白・タイトル・左バー）  
2. **P1:** 横持ちで Star プレビュー OFF  
3. **P2:** 横持ちで Bubble 表示（または Star 密度のみ PC 寄せ）  
4. **P3:** 実機 QA（Safari / Chrome）と微調整
