# 76 スマホ投稿ナビとクイック投稿

> **分類:** `[Core / UX]` モバイル下部ナビ「投稿」とクイック投稿パネルの連携  
> **関連:** [06_ナビゲーション・UI構成](./06_ナビゲーション・UI構成.md) / [51_簡単投稿](./51_簡単投稿.md) / [62_スマホのスペース画面](./62_スマホのスペース画面.md) / [56_クリックフィードバックとフローティングパネル](./56_クリックフィードバックとフローティングパネル.md)

> 最終更新: 2026-06-02  
> **ステータス:** ✅ 実装済み

---

## 1. 目的

スマートフォンで下部ナビの **「投稿」** を押したとき、別ページ（`#post`）へ遷移せず、スペース画面（`#screen`）のまま **下半分にクイック投稿パネル（ボトムシート）** がスライドアップする UX に変更する。

**意図:** 背景の **ダブルタップ** と同系統の「その場で置く」体験を、ナビからも一発で呼び出せるようにする（[51](./51_簡単投稿.md)）。

---

## 2. 適用範囲

| 条件 | 挙動 |
|------|------|
| **PC（769px 以上）** | 右上メニュー「投稿する」→ クイック投稿パネル（`#post` へは遷移しない） |
| **モバイル + `#screen` 表示中** | 本仕様（クイック投稿パネル） |
| **モバイル + 他画面**（`#comments` / `#account` 等） | 先に `#screen` へ遷移し、続けてパネルを **開く** |
| **訪問モード**（隣人スペース閲覧中） | **開かない**（[44_隣人システム](./44_隣人システム.md)・[51](./51_簡単投稿.md) と同一） |
| **スライドショー表示中** | 開かない |

### 2.1 パネル形状（向き別）

| 条件 | パネル | 既定位置 | storageKey |
|------|--------|----------|------------|
| PC | 右サイド（幅 25%） | 画面右端 | `quickPost.desktop` |
| モバイル **縦** | **ボトムシート**（高さ 55%） | 画面下半分・下ナビ上 | `quickPost.mobile` |
| モバイル **横** | 右サイド（PC 同系） | 画面右端 | `quickPost.mobileLandscape` |

## 3. タブ操作の詳細

### 3.1 `#screen` 表示中に「投稿」タブを押す

| 状態 | 結果 |
|------|------|
| クイック投稿パネル **未表示** | URL は `#screen` のまま。パネルを **開く** |
| クイック投稿パネル **表示中** | パネルを **閉じる**（トグル） |
| 下ナビのアクティブ表示 | `#screen` のままのときは **「スペース」タブをアクティブ**とする（「投稿」タブを `#post` 専用アクティブにしない） |

### 3.2 投稿位置

下ナビから開く場合は、ダブルタップ座標がないため **中央既定 `{ x: 50, y: 50 }`**（[51](./51_簡単投稿.md) のキューブドックと同じ）。

### 3.3 他画面から「投稿」タブを押す

1. `navigate('screen')`  
2. マウント完了後（または同一 tick で state 連携後）クイック投稿パネルを **開く**  
3. 訪問モード等で開けない条件のときは `#screen` への遷移のみ

### 3.4 ダブルタップとの関係

- 背景ダブルタップ: 従来どおり **タップ位置** で開く（320ms 遅延・トリプル競合回避は [51](./51_簡単投稿.md)・[63_ダブルクリック感知](./63_ダブルクリック感知.md) のまま）
- 下ナビとダブルタップは **同じ `quickPostPos` state** を共有（排他ではない。両方トグル）

### 3.5 `#post` フルページへの直接アクセス

- ブックマーク・外部リンクで `#post` に来た場合: **現状どおりフルページ**（本仕様では変更しない）
- 将来、モバイルでは `#post` を `#screen` + パネルへリダイレクトする案は **別チケット**

## 4. パネル UI

[51_簡単投稿](./51_簡単投稿.md) の **モバイルボトムシート** 定義に従う。

- `FloatingPanelShell` + `PostScreen`（パネルモード）
- 下端は `BottomNavBar` より上（`MOBILE_BOTTOM_NAV_RESERVE_PX`）
- 閉じる: パネル内 ✕、再度「投稿」タブ、背景ダブルタップ（開いているとき）

---

## 5. 実装アーキテクチャ

`quickPostPos` は `SpaceScreen` 内 state。下ナビ・PC 右上メニューから操作するため **App ↔ SpaceScreen の連携** を使う。

| 層 | 役割 |
|----|------|
| `SpaceScreen` | `forwardRef` + `useImperativeHandle` で `openQuickPost()` / `toggleQuickPost()` / `closeQuickPost()` を公開 |
| `App.tsx` | `screen === 'screen'` 時に ref を保持。`BottomNavBar` に `onPostPress` を渡す。訪問中は Toast |
| `BottomNavBar.tsx` | モバイルかつ `onPostPress` あり → 遷移せずコールバック |
| `TopRightMenu.tsx` | `onPostClick` prop で PC「投稿する」→ クイック投稿 |

## 7. 受け入れ条件

- [x] モバイル・`#screen` 中に「投稿」タブ → **ページ遷移なし**で下半分に投稿フォーム

- [x] `#comments` 等から「投稿」→ **`#screen` + パネル開**

- [x] PC でも `#post` へ遷移せずクイック投稿パネルを開く
