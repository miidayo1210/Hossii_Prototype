> **分類:** `[Core / UX]` モバイルで星（StarView）・吹き出し座標をシャープ壁紙内に揃える

## 1. 目的

スマホ（StarView）で投稿が **レターボックス（余白）側にランダムに散らばる** 問題を解消する。  
**16:9 等のシャープ壁紙（`.bgImageSharp` / `contain`）の上** に、星・座標付き投稿が載るようにする。

---

## 2. 現状の問題

```text
┌─────────────────────────┐  ← bubbleArea（全面 0〜100%）
│  letterbox（ぼかし）     │
├─────────────────────────┤
│                         │
│   16:9 シャープ画像      │  ← 実際に見せたい領域
│                         │
├─────────────────────────┤
│  letterbox              │
└─────────────────────────┘
     ★  ← positionY=15% だと上余白に落ちる
```

| 項目 | 現状 |
|------|------|
| 座標系 | `positionX/Y` は **bubbleArea 全面** の 0〜100%（[45](./45_投稿の位置を定める機能.md)） |
| ランダム配置 | `createBubblePosition` が 8〜92% × 12〜78%（[bubblePosition.ts](../../../src/core/utils/bubblePosition.ts)） |
| モバイル表示 | StarView が `%` を bubbleArea 基準で配置（[StarView.tsx](../../../src/components/SpaceScreen/StarView.tsx)） |
| 壁紙 | `contain` 中央（[70](./70_背景画像サイズ.md)） |

**結果:** 縦長スマホでは星が **画像の外（上下の余白）** に多く出る。ユーザーからは「ランダムに散らばる」ように見える。

---

## 3. 用語

| 用語 | 説明 |
|------|------|
| **コンテナ矩形** | `spaceExportRoot` / `bubbleArea` の表示サイズ（px） |
| **シャープ矩形** | `background-size: contain` で **画像本体が描画される矩形**（レターボックス除く） |
| **論理座標** | DB に保存される `positionX/Y`（0〜100、現行＝コンテナ基準） |
| **表示座標** | 画面上の `%`（モバイル壁紙時はシャープ矩形基準に変換した値） |

---

## 4. 仕様（確定方針）

### 4.1 適用条件

| 条件 | 座標変換 |
|------|----------|
| モバイル **かつ** 画像壁紙あり（`imageWallpaperUrl != null`） | **シャープ矩形** 基準で表示 |
| 上記以外（PC、単色/パターン背景） | **現状どおり** コンテナ全面基準 |

### 4.2 シャープ矩形の算出

- コンテナ幅 `W`、高さ `H`、画像アスペクト `R`（**既定 16:9 = 16/9**。将来は画像メタデータで上書き可）
- `contain` と同式:

```text
if (W / H > R):
  sharpH = H, sharpW = H * R, sharpX = (W - sharpW) / 2, sharpY = 0
else:
  sharpW = W, sharpH = W / R, sharpX = 0, sharpY = (H - sharpH) / 2
```

- リサイズ・向き変更・没入モードで **再計算**（`ResizeObserver`）

### 4.3 論理座標 → 表示座標（モバイル壁紙）

コンテナ基準の論理 `(lx, ly)` を表示 `(dx, dy)` に:

```text
dx = (sharpX + (lx / 100) * sharpW) / W * 100
dy = (sharpY + (ly / 100) * sharpH) / H * 100
```

- `isPositionFixed` の投稿・クイック投稿の `%` も **同じ変換**
- **DB の値は変更しない**（表示側のみ変換）

### 4.4 新規ランダム配置（モバイル壁紙）

- `createBubblePosition` の乱数範囲を、論理座標で **シャープ矩形に相当する内側**（例: 5〜95% × 5〜95% をコンテナ換算でシャープ内に収める）に限定
- 投稿順格子（`createOrderedBubblePosition`）も **シャープ内の格子** に再マッピング

### 4.5 PC

- 変更なし（コンテナ全面）。PC でも contain 壁紙はあるが、Bubble 表示では従来 UX を維持。

## 6. 受け入れ条件

- [x] iPhone 縦 + 16:9 壁紙: 星が **上下レターボックスにほぼ出ない**
- [x] 既存の固定座標投稿も **画像上の相対位置が保たれる**（大きくずれない）
- [x] 単色背景・PC では **挙動変化なし**
- [x] 画面回転・リサイズ後も **星が画像から外れにくい**

### 6.1 追加受け入れ条件（[85](./85_スマホ吹き出し配置の中央寄せ調査.md) 実装済み）

- [x] シャープ矩形 **内** で投稿が中央一極集中しない（一様分布）
- [x] ダブルタップ投稿がタップ位置付近に表示される
- [x] ドラッグ保存座標がモバイル壁紙でも論理 % で一貫する
- [x] 下ナビ・TopRightMenu 経路の投稿が **(50,50) 固定にならない**（85 §4.6）

## 7. 非対象（今回）

- 画像アスペクトの個別取得（EXIF 等）— 初版は **16:9 固定** でよい（[70](./70_背景画像サイズ.md) の典型）
- 書き出し PNG の座標系変更（表示のみ変換）

### 補足: クイック投稿パネル（縦）

クイック投稿を開いても **スペース canvas のレイアウトは変えない**。投稿 UI は `mobilePostSheet` としてスペースの上に重ねる（[51](./51_簡単投稿.md)）。星のシャープ座標変換（§4）は **通常のスペース表示矩形** を基準に継続する。
