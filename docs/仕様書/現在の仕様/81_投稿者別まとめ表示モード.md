## 1. 目的

スペース上に投稿が増えると、**1投稿＝1吹き出し** の表示では次の問題が起きやすい。

- 同じ人の連続投稿がバラバラの位置に散らばり、**その人の流れ**が読み取りにくい
- 吹き出しが重なり、背景や他者の投稿が見えにくくなる（特にスマホ縦）

本仕様は、**投稿者ごとに1つの吹き出しにまとめる** 表示モードを追加し、  
**同じ人がいつ・何を投稿したか** を時系列で把握しやすくする。

---

## 2. 用語

| 用語 | 説明 |
|------|------|
| **個別投稿** | 既存どおり `Hossii` 1件＝画面上の要素1つ（`random` / `ordered` モード） |
| **投稿者グループ** | 同一投稿者に属する個別投稿の集合。画面上では **1吹き出し** として表現 |
| **まとめ吹き出し** | 投稿者グループを表す UI 1個。内部に複数投稿を時系列で持つ |
| **投稿者キー** | グループ化の識別子。後述 §4.1 |

既存の **表示モード**（`viewMode`: フル / バブル / 画像）とは独立。  
本機能は **並びモード**（`layoutMode`）の第3選択肢として追加する。

---

## 3. 既存モードとの関係

### 3.1 並びモード（`layoutMode`）— 本仕様の追加先

| 値 | 名称 | 画面上の単位 | 状態 |
|----|------|--------------|------|
| `random` | ランダム | 個別投稿 | ✅ 実装済み |
| `ordered` | 投稿順 | 個別投稿（格子） | ✅ 実装済み |
| `byAuthor` | 投稿者まとめ | **投稿者グループ** | ✅ 実装済み |

保存先: `localStorage['hossii.layoutMode']`（既存キーを拡張）

### 3.1.1 投稿者グループの並び（`authorGroupSort`）

`byAuthor` 選択時のみ有効。スペース右上ツールバー（§6.3）で切替。

| 値 | ボタンラベル | 意味 |
|----|-------------|------|
| `firstPostAsc` | 初投稿順 | **既定**。その人の **最初の1投稿** が早いグループほど左上（index 小） |
| `latestDesc` | 最新順 | 最新投稿が新しいグループほど左上 |
| `postCountDesc` | 投稿数順 | 投稿数が多いグループほど左上（同数は最新順） |

保存先: `localStorage['hossii.authorGroupSort']`

### 3.2 表示モード（`viewMode`）— 併用

まとめ吹き出しの **中身の描画** は、選択中の `viewMode` に従う。

| viewMode | まとめ吹き出し内の表示方針 |
|----------|---------------------------|
| `full` | 各投稿のメッセージ・感情・画像サムネ・時刻を縦に積む（推奨デフォルト） |
| `bubble` | 投稿者名・件数・最新1件の感情アイコン・最新時刻などコンパクト表示 |
| `image` | 画像付き投稿のみをサムネ一覧。画像なし投稿は件数バッジのみ |

`viewMode === 'image'` のとき、画像のない投稿者グループは **一覧から除外するか**、グレーアウトで件数のみ表示するかは実装時に決定（初期案: 除外しない・件数表示）。

### 3.3 スライドショー・訪問モード

| 条件 | 挙動 |
|------|------|
| `viewMode === 'slideshow'` | 左バー非表示のため並び切替不可。まとめモードは選択不可 |
| 訪問中スペース | 閲覧専用。まとめ表示・詳細閲覧は可。投稿・編集は不可（既存どおり） |

## 4. グループ化ルール

### 4.1 投稿者キー

表示対象の `displayHossiis` を、次の優先順でグループ化する。

1. `authorId` が両方とも存在し、かつ一致 → 同一グループ
2. `authorId` が無く `authorName` が非空で一致 → 同一グループ（ゲスト等）
3. それ以外は **投稿者不明** として1投稿1グループ、または `authorName || '名無し'` ごとに分離

> **注意:** 表示名だけが同じ別人は同一グループに入る可能性がある。  
> 将来 `authorId` の普及で解消する。仕様上は「表示名フォールバックはベストエフォート」と明記する。

### 4.2 グループ内の並び

| 項目 | ルール |
|------|--------|
| 時系列 | `createdAt` **昇順**（古い → 新しい）。読み物として自然な順 |
| 件数上限 | グループ内の表示件数は **直近 N 件**（初期案: N = 20）。超過分は「他 k 件」リンク |
| 代表投稿 | 感情カラー・イニシャルバッジ等の見た目は **グループ内で最新の投稿** を代表とする（座標は §5.4 参照） |

### 4.3 フィルタとの整合

期間・件数・表示モード・非表示フラグで絞り込まれた **のち** にグループ化する。  
つまり `displayHossiis` と同じ母集団を、投稿者単位に再集約する。

## 5. 画面表現

### 5.1 まとめ吹き出し（`AuthorClusterBubble`）

1投稿者あたり **1つのカード型吹き出し** をキャンバス上に配置する。  
**初期状態は折りたたみ（collapsed）**。タップ/クリックで本文を展開する。

#### 5.1.1 折りたたみ状態（デフォルト）

キャンバスの密度を保つため、常にヘッダー行のみ表示する。

```
┌──────────────────────────────────┐
│ [👤] Yuki  3件  12:04  💜 🌟 💛  │  ← 高さ 40px
└──────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| **イニシャルバッジ** | 投稿者名の先頭1文字。16×16px 円形。背景色は §5.3 の感情カラー（最新投稿準拠）|
| **投稿者名** | `font-size: 0.75rem; font-weight: 600; color: #1f2937` |
| **件数バッジ** | 件数 + 「件」。`font-size: 0.6rem; color: #6b7280` |
| **最新時刻** | 相対時刻（例: `12:04`）。`font-size: 0.6rem; color: #9ca3af` |
| **感情アイコン列** | 最新3件の感情絵文字を横並び。`font-size: 0.75rem`。3件未満はその数だけ表示 |
| **展開インジケータ** | `ChevronDown`（Lucide）アイコン。展開時は `ChevronUp` |

**カード外観（折りたたみ時）**

| プロパティ | 値 |
|------------|----|
| 背景 | `rgba(255, 255, 255, 0.88)` |
| backdrop-filter | `blur(12px)` |
| border | `1px solid rgba(255, 255, 255, 0.55)` |
| border-radius | `0.875rem` |
| box-shadow | `0 4px 16px rgba(0, 0, 0, 0.10)` |
| padding | `0.5rem 0.75rem` |
| min-width | `160px` |
| max-width | `220px` |
| height（折りたたみ時） | `40px` |

左端には最新投稿の感情カラーを **3px 幅の縦ライン**（`border-left`）でアクセントとして付与する。

#### 5.1.2 展開状態

ヘッダー行 + 投稿タイムラインリストを表示する。

```
┌──────────────────────────────────┐
│ [👤] Yuki  3件  12:04  💜 🌟 💛  │ ← ヘッダー（常時表示）
├──────────────────────────────────┤
│  12:00  💜  "今日はすごく..."     │ ← タイムライン行
│  12:02  🌟  "ランチがおいしかっ"  │
│  12:04  💛  [🖼] "写真撮った"     │
│                        他 2件 ›   │ ← N=20 超過時のみ
└──────────────────────────────────┘
```

| プロパティ | 値 |
|------------|----|
| max-height（本文部分） | `200px`（CSSで制限）|
| overflow-y | `auto`（スクロール可）|
| タイムライン行の高さ | `min-height: 28px`（テキスト折り返し時は拡張）|

**タイムライン行レイアウト（横並び 1行）**

```
[時刻]  [感情アイコン]  [テキスト（最大2行で切り捨て）]  [画像サムネ（任意）]
 0.6rem    0.8rem         0.7rem / color: #374151          32×32px
```

| 要素 | 仕様 |
|------|------|
| 時刻 | `font-size: 0.6rem; color: #9ca3af; width: 2.4rem; flex-shrink: 0` |
| 感情アイコン | `font-size: 0.8rem; flex-shrink: 0` |
| テキスト | `font-size: 0.7rem; color: #374151; line-clamp: 2; flex: 1` |
| 画像サムネ | `32×32px; object-fit: cover; border-radius: 4px; flex-shrink: 0`。画像なし時は非表示 |
| 「他 k件 ›」 | `font-size: 0.65rem; color: #7c3aed; text-align: right`。クリックで §5.4 モーダルへ |

既存の StarView・Bubble と一貫したカラーパレットを使用する。  
まとめ吹き出しは **代表投稿（最新投稿）の感情** に基づいて色付けする。

| emotion | カラー | 用途（まとめ吹き出し） |
|---------|--------|------------------------|
| `joy` | `#fbbf24` | border-left / イニシャルバッジ背景 |
| `wow` | `#60a5fa` | 〃 |
| `empathy` | `#f472b6` | 〃 |
| `inspire` | `#a78bfa` | 〃（デフォルト・感情なし時） |
| `think` | `#10b981` | 〃 |
| `laugh` | `#f59e0b` | 〃 |
| `moved` | `#ec4899` | 〃 |
| `fun` | `#8b5cf6` | 〃 |
| 感情なし | `#9ca3af` | 〃 |

イニシャルバッジ背景は上記カラーを `opacity: 0.2` で薄め、文字色は同カラー `opacity: 1`。

### 5.3 アニメーション

キャンバスの既存スター同様、**存在感を主張しすぎない控えめなアニメーション** にとどめる。

| 状態 | アニメーション | 詳細 |
|------|----------------|------|
| 通常 | `cardFloat` | 上下に ±2px、6s ease-in-out infinite。個別スターの float と同系 |
| 新着あり（最新投稿が5分以内） | `cardGlow` | box-shadow に感情カラーの glow を 2.2s で脈動。`starRecentGlow` と同系 |
| 展開開始 | `clusterOpenPop` | scale 0.97→1.02→1.0 を 0.2s。タップフィードバック |
| 並び替え時 | `left` / `top` transition | 0.3s ease。ツールバー切替でスロット間をスライド |
| `prefers-reduced-motion` | すべて無効化 | glow のみ静的 outline に変換 |

```css
/* cardFloat（参考） */
@keyframes cardFloat {
  0%, 100% { transform: translate(0, 0); }
  50%       { transform: translate(0, -2px); }
}

/* cardGlow（参考）— --glow-color は emotion カラーを JS でインライン注入 */
@keyframes cardGlow {
  0%, 100% { box-shadow: 0 4px 16px rgba(0,0,0,0.10), 0 0 0 0 transparent; }
  50%       { box-shadow: 0 4px 16px rgba(0,0,0,0.10), 0 0 0 5px var(--glow-color, rgba(167,139,250,0.35)); }
}
```

### 5.4 配置（座標）

投稿者まとめモードでは **投稿順モード（`ordered`）と同様、計算座標のみ** を使用する。  
`random` モードで保存された `isPositionFixed` / `positionX/Y` は **無視** する（並び替えと競合するため）。

#### 5.4.1 折り返し格子（`createAuthorWrapGrid`）

左→右に並べ、行末で折り返して上→下へ。画面外にはみ出さない。

```
[0] [1] [2] [3]     ← 1行目（最大 cols 列）
[4] [5] [6]         ← 2行目
```

| パラメータ（PC） | 値 |
|------------------|-----|
| 配置関数 | `createAuthorRowPosition` → `createAuthorWrapGrid`（[bubblePosition.ts](../../src/core/utils/bubblePosition.ts)） |
| 原点 | 左上基準（`AuthorClusterBubble` は `left` / `top` %） |
| 横範囲 | `startX=8%` 〜 `spanX=84%`（右端 92%） |
| 縦範囲 | `startY=16%` 〜 `spanY=62%`（下端 78% 付近） |
| 1セル最小幅 | `minCellW=20%` → **1行最大 4 列**（`floor(84/20)`） |
| 列数 | `cols = min(total, floor(spanX / minCellW))` |
| 行数 | `rows = ceil(total / cols)` |
| セル幅 | `cellW = spanX / cols` |
| セル高 | `cellH = spanY / rows`（行数に応じて均等分割） |
| index → 座標 | `col = index % cols`, `row = floor(index / cols)` |

**モバイル（16:9 シャープ矩形）:** `createAuthorRowPositionInSharp`。`minCellW=22%`、矩形内 5–95% に収める。

#### 5.4.2 並び index と表示順

1. `groupHossiisByAuthor(displayHossiis)` でグループ化
2. `sortAuthorGroups(groups, authorGroupSort)` で並び替え（§3.1.1）
3. 並び後の `index` を `createAuthorRowPosition(index, total)` に渡す
4. **index 0 = 格子の左上**、右端で改行

#### 5.4.3 ドラッグ・固定座標

| 項目 | 方針 |
|------|------|
| ドラッグ | **無効**（`canEdit={false}`）。並び替えツールバーが配置の唯一の操作 |
| `random` の固定座標 | `byAuthor` 中は **参照しない** |
| モード切替 | `byAuthor` 解除後、`random` モードでは個別投稿の固定座標が復活（53 と同様） |

## 6. UI 仕様（切替）

### 6.1 配置

`LeftControlBar` の「並び」セクション（[53](./53_スペース投稿表示並び替え.md)）に第3ボタンを追加。

```
[ 並び ]
[ 🔀 ] [ ↔️ ] [ 👥 ]
  ランダム  投稿順  投稿者まとめ
```

| ボタン | アイコン | ラベル（title） | `layoutMode` |
|--------|----------|-----------------|--------------|
| 🔀 | Lucide `Shuffle` | ランダム | `random` |
| ↔️ | Lucide `AlignJustify` | 投稿順に整列 | `ordered` |
| 👥 | Lucide `Users` | 同じ人の投稿を1つにまとめて表示 | `byAuthor` |

> **アイコン選定理由:** 既存の `LayoutMode` ボタンは絵文字でなく Lucide アイコンに統一する方針（[LeftControlBar.tsx](../../src/components/Navigation/LeftControlBar.tsx) の `VIEW_MODE_OPTIONS` 参照）。`Users` アイコンは「複数人をグループ化する」意図を直感的に表現する。

- ツールチップ / `title`: 「同じ人の投稿を1つにまとめて表示」
- `byAuthor` 選択時は **降順・昇順トグルは非表示**（個別投稿の格子順ではないため）
- アクティブ状態のスタイル: 既存 `.miniActive`（`linear-gradient(135deg, #a855f7 0%, #ec4899 100%)`）を流用

### 6.2 モバイル

`...` オーバーフローメニュー内の「並び」に同じ3択を載せる（[33](./33_スペース左サイドのバー.md)）。

### 6.3 並び替えツールバー（`byAuthor` 時）

`layoutMode === 'byAuthor'` かつ `viewMode !== 'slideshow'` のとき、スペース右上（投稿順の昇降順トグルと同位置）に表示。

```
[ 初投稿順 ] [ 最新順 ] [ 投稿数順 ]
```

| ボタン | `authorGroupSort` | 意味 |
|--------|-------------------|------|
| 初投稿順 | `firstPostAsc` | **既定**。初投稿が早いグループほど左上 |
| 最新順 | `latestDesc` | 最新投稿が新しいグループほど左上 |
| 投稿数順 | `postCountDesc` | 投稿数が多いグループほど左上 |

- スタイル: 既存 `.orderedSortToolbar` / `.orderedSortButton` を流用
- `ordered` モードの昇降順トグルと **同時には出さない**
- 切替時: `sortAuthorGroups` 再実行 → 格子 index が変わり、カードがアニメーション付きで移動

## 7. データ・状態管理

### 7.1 型の拡張

```ts
// displayPrefsStorage.ts
export type LayoutMode = 'random' | 'ordered' | 'byAuthor';
export type AuthorGroupSort = 'firstPostAsc' | 'latestDesc' | 'postCountDesc';

/** 画面描画用。DB には保存しない */
export type AuthorPostGroup = {
  groupKey: string;
  authorId?: string;
  authorName: string;
  posts: Hossii[]; // createdAt 昇順
  latestPost: Hossii;
  /** 新着判定: latestPost.createdAt が現在時刻から 5分以内 */
  isRecent: boolean;
};
```

### 7.2 集約・並び替え

```ts
// groupHossiisByAuthor.ts
function groupHossiisByAuthor(hossiis: Hossii[]): AuthorPostGroup[] {
  // 1. groupKey で Map に集約（§4.1）
  // 2. 各グループ posts を createdAt 昇順ソート
  // 3. latestPost = posts.at(-1)
  // 4. isRecent = Date.now() - latestPost.createdAt < 5 * 60 * 1000
}

function sortAuthorGroups(groups: AuthorPostGroup[], sort: AuthorGroupSort): AuthorPostGroup[] {
  // firstPostAsc: posts[0].createdAt 昇順
  // latestDesc:   latestPost.createdAt 降順
  // postCountDesc: posts.length 降順 → 同数は latestPost 降順
}
```

`SpaceScreen` では `sortAuthorGroups(groupHossiisByAuthor(displayHossiis), authorGroupSort)` の結果を描画する。

### 7.3 座標

| 項目 | 方針 |
|------|------|
| 配置の決定 | `createAuthorRowPosition(index, total)`（§5.4.1）。`index` は並び替え後の配列順 |
| 固定座標 | **`byAuthor` 中は使用しない**（`ordered` と同様） |
| ドラッグ | 無効。将来、手動整列を復活させる場合は別仕様 |
| モード切替 | `byAuthor` 解除後、`random` では個別投稿の `isPositionFixed` が復活 |

## 9. 受け入れ条件

- [x] 左バー（およびモバイル `...`）から **投稿者まとめ** を選べる
- [x] 同一 `authorId` の投稿が **1吹き出し** にまとまり、中身が **時系列** で読める
- [x] まとめ吹き出しは **初期状態で折りたたみ**（ヘッダー行のみ表示）
- [x] まとめ吹き出しタップで展開 or `AuthorTimelineModal` を開ける
- [x] まとめ吹き出しの **左端ライン・イニシャルバッジ** が最新投稿の感情カラーを反映している
- [x] 新着グループ（最新投稿 5分以内）に **cardGlow アニメーション** が適用される
- [x] `viewMode` 切替と矛盾なく動作する（少なくとも `full` で実用可能）
- [x] `random` / `ordered` との切替で、既存の個別投稿表示が壊れない
- [x] 期間・件数フィルタ後の集合に対して正しくグループ化される
- [x] スマホ縦で、1投稿者のみのときに **背景の大部分が1吹き出しで隠れない**（折りたたみ + ボトムシート方式）
- [x] `prefers-reduced-motion` 有効時、すべてのアニメーションが無効化または静的代替に切り替わる
- [x] 右上ツールバーで **初投稿順 / 最新順 / 投稿数順** を切替でき、格子位置が更新される
- [x] 投稿者カードが **左→右・折り返し格子** に収まり、右端が切れない
- [x] `random` モードの固定座標が `byAuthor` の並びを阻害しない
