## 1. 目的

スペース URL から **ゲストとしてニックネームを付けて参加** した参加者は、同じブラウザ・端末で投稿を続ける。  
現状、下ナビ／メニューの **「ログ」**（`#comments`）で開ける一覧は **スペース全員の全投稿** のみで、自分が残した投稿だけを手軽に振り返る導線がない。

本仕様は、ログ一覧の上部に **「全体 ↔ 私」のスライドセグメント** を設け、**画面遷移なし** で自分の投稿履歴を閲覧できるようにする。  
「私のログ」モードでは視覚的に **「自分の記録を見ている」感覚** を演出するため、ヘッダー・カード・空状態を差別化する。

---

## 2. 背景（現状とギャップ）

| 項目 | 現状 |
|------|------|
| 下ナビ「ログ」 | 常に `#comments` → `CommentsScreen` → 全投稿 |
| `LogListBody` | `getActiveSpaceHossiis()` をそのまま表示。スコープ切替なし |
| `#mylogs` / `MyLogsScreen` | `authorId === profile.id` で全スペース横断の自分の投稿を表示する画面が**存在するが、ナビからは到達できない** |
| ゲストの本人識別 | 初回投稿時に `hossii.profile.id`（端末固有 ID）を生成し投稿の `authorId` に付与 |
| 表示名 | `hossii.spaceNicknames[spaceId]` → `defaultNickname` の優先順 |

**ギャップ:** ゲストが「このスペースで自分が何を投稿したか」を、いつも使う **ログボタン** から見られない。

---

## 3. 用語

| 用語 | 説明 |
|------|------|
| **全体のログ** | アクティブ（または訪問中）スペースの全参加者の投稿一覧。既存 `#comments` の表示 |
| **私のログ** | 上記スペースのうち、現在の端末の自分（`profile.id`）が `authorId` として付いた投稿のみ |
| **ログスコープ** | `all`（全体）または `mine`（私）。一覧の表示対象を決める UI 状態 |
| **スライドセグメント** | 2 択を横一列に並べ、選択中の pill（背景ブロック）がスライドするトグル UI |

---

## 4. 対象ユーザー

| ユーザー種別 | 全体のログ | 私のログ |
|-------------|-----------|---------|
| **ゲスト**（ニックネームで入室） | ✅ | ✅（`profile.id` がある場合） |
| **アカウントログイン済み参加者** | ✅ | ✅ |
| **訪問モード**（他スペース閲覧） | ✅ | ✅（訪問先スペース内の自分の投稿） |
| **管理者** | ✅ | ✅（非表示ボタンは両スコープとも既存どおり） |

> 端末をまたいだ同一人物判定は行わない（[02](./02_匿名利用機能.md) どおり）。

### 5.2 スライドセグメント

#### 外観・寸法

```
┌────────────────────────────────────────────────┐
│ ████████████████████  │                        │  ← pill が左を覆う（all 選択時）
│  全体のログ  (42)          私のログ  (3)        │
└────────────────────────────────────────────────┘

┌────────────────────────────────────────────────┐
│                        │ ████████████████████  │  ← pill が右へ（mine 選択時）
│  全体のログ  (42)          私のログ  (3)        │
└────────────────────────────────────────────────┘
```

| 属性 | 値 |
|------|-----|
| **トラック** | `width: 100%`、`height: 2.5rem`、`border-radius: 999px`、背景 `#f3f4f6`（gray-100）、`border: 1px solid #e5e7eb` |
| **pill** | `width: 50%`、高さはトラックと同じ、`border-radius: 999px`、背景 `#7c3aed`（紫）、`box-shadow: 0 2px 6px rgba(124,58,237,0.35)` |
| **ラベル** | `font-size: 0.8125rem`、`font-weight: 600`、pill 上は `#fff`、非アクティブは `#6b7280` |
| **件数バッジ** | ラベルの後ろに `(N)` 形式。フォントサイズ `0.6875rem`。pill 上では `rgba(255,255,255,0.75)`、非アクティブは `#9ca3af` |

#### アニメーション

| 状態 | 挙動 |
|------|------|
| 切替 | pill が `transform: translateX(0)` → `translateX(100%)` へ移動。`transition: transform 0.22s cubic-bezier(0.34, 1.0, 0.64, 1)` |
| ラベル色 | `transition: color 0.18s ease` |
| `prefers-reduced-motion: reduce` | **トランジションなし**（即時切替） |

#### インタラクション

| 操作 | 動作 |
|------|------|
| タップ / クリック | 対応スコープへ切替 |
| `role="tablist"` + `role="tab"` | 各セグメントに割り当て |
| 左右矢印キー | フォーカスを移動し `Enter` / `Space` で選択 |
| `aria-selected="true/false"` | 選択状態を通知 |

### 5.3 スコープ切替時のリスト遷移

ログ一覧部分（カードリスト）は **スコープ切替と同時にフェードイン** させ、唐突な差し替えを緩和する。

```
切替直後: opacity 0 → opacity 1  (0.18s ease-out)
          transform: translateY(6px) → translateY(0)
```

`prefers-reduced-motion: reduce` では **opacity 切替のみ**（translateY なし）。

---

### 5.4 ヘッダー文言（スコープ連動）

スコープ切替時にタイトル・サブタイトルもフェードで変化させる（`opacity 0.15s ease`）。

| スコープ | タイトル（h1） | サブタイトル |
|---------|---------------|-------------|
| `all` | コメント一覧 | みんなの声が流れてくるよ |
| `mine` | 私のログ | あなたが残したログ |

件数表示:

| スコープ | 表示 |
|---------|------|
| `all` | `{N} 件の投稿` |
| `mine` | `{N} 件` |

### 5.5 アイデンティティバナー（mine モード専用）

「私のログ」選択中、ヘッダーの件数行の直下（FilterBar より上）に **現在のニックネームを示すバナー** を表示する。  
「これは自分のログを見ている」という文脈を視覚的に伝え、誤認を防ぐ。

```
┌──────────────────────────────────────────┐
│  🙂  もぐみ として参加中                  │
└──────────────────────────────────────────┘
```

| 属性 | 値 |
|------|-----|
| 背景 | `linear-gradient(135deg, #e9d5ff 0%, #fbcfe8 100%)`（`MyLogsScreen.module.css` の `.filterBtnActive` と同系） |
| 枠線 | `1px solid rgba(167,139,250,0.4)` |
| 角丸 | `border-radius: 0.625rem` |
| パディング | `0.5rem 0.875rem` |
| テキスト | `font-size: 0.8125rem`・`font-weight: 600`・`color: #7c3aed` |
| アイコン | 先頭に `🙂`（固定絵文字）または Lucide の `User` アイコン（16px、同色）|
| ニックネーム | `state.spaceNicknames[activeSpaceId]` → `profile.defaultNickname` の優先順。未設定なら非表示（バナー自体を隠す） |
| 表示アニメーション | `all` → `mine` 切替時に `opacity 0 → 1`・`translateY(-4px) → 0`（0.2s ease-out） |
| 非表示アニメーション | `mine` → `all` で逆方向（0.15s ease-in） |

> スコープが `all` のときはバナーを DOM から**非表示**にする（`display: none` または条件レンダリング）。

### 5.6 「私のログ」カードの差別化

「全体のログ」と同じ `LogListBody` を共有しつつ、`mine` スコープのときに **自分の投稿を見ている感覚** を演出する。

#### カードのビジュアル差分

| 要素 | `all` モード | `mine` モード |
|------|-------------|--------------|
| カード背景 | `#faf5ff`（既存） | 同上（維持） |
| **カード左ボーダー** | なし | `border-left: 3px solid #7c3aed`（紫のアクセントライン） |
| **タイムスタンプ形式** | `HH:MM`（既存） | **相対時刻**（「今」「3分前」「2時間前」「6月20日」） |
| **投稿者名行** | 表示 | **省略**（自分の投稿なので冗長。`mine` スコープ時は非表示） |

> タイムスタンプの相対時刻は `MyLogsScreen.tsx` の `getRelativeTime()` と同じロジックを共有する。

#### 全体との比較イメージ

```
【全体のログ — カード】
┌────────────────────────────────────┐
│ もぐみ                             │
│  ✏ 今日のワークショップ楽しかった！  │
│  #まちあるき                        │
│                      16:32  🤍 2   │
└────────────────────────────────────┘

【私のログ — カード（mine モード）】
┌▌───────────────────────────────────┐  ← 紫の左ボーダー
│  ✏ 今日のワークショップ楽しかった！  │  ← 投稿者名なし
│  #まちあるき                        │
│                    2時間前  🤍 2    │  ← 相対時刻
└────────────────────────────────────┘
```

### 5.7 空状態（mine モード）

素っ気ないテキストではなく、**文脈に合った誘導 UI** を表示する。

```
┌──────────────────────────────────────────┐
│                                          │
│              📭                          │  ← 大きめのアイコン
│                                          │
│    まだ投稿がありません                   │  ← メインメッセージ
│    ログ画面で自分の声を残してみよう！      │  ← サブメッセージ
│                                          │
│    ┌────────────────────────────────┐   │
│    │ 💬 投稿してみる                │   │  ← アクションボタン
│    └────────────────────────────────┘   │
│                                          │
└──────────────────────────────────────────┘
```

| 条件 | アイコン | メインメッセージ | サブメッセージ | ボタン |
|------|---------|----------------|--------------|--------|
| `profile` なし（未投稿） | 📭 | まだ投稿がありません | 気持ちを投稿すると、ここに記録が残ります | 投稿してみる（`#post` or クイック投稿を開く） |
| `profile` あり・該当 0 件 | 📭 | このスペースへの投稿はまだありません | 最初の投稿をして記録を残してみよう！ | 投稿してみる |
| `all` で 0 件 | — | まだ反応がありません（現状維持） | — | なし |

ボタンのスタイル: `background: linear-gradient(135deg, #e9d5ff 0%, #fbcfe8 100%)`、`color: #7c3aed`、`border: 1px solid rgba(167,139,250,0.4)`、`border-radius: 999px`、`padding: 0.5rem 1.25rem`。

ボタン押下時:
- フル画面（`CommentsScreen`）: `#post` へ遷移
- クイックログパネル（`panelMode`）: パネルを閉じてクイック投稿パネルを開く（`onClose()` + `onOpenQuickPost?.()`）

### 5.8 クイックログパネル内（`panelMode`）

狭い幅（約 25vw〜）に対応するため、ラベルを短縮する。

```
【パネルクローズバー直下】

┌──────────────────────────────┐
│  全体 (42)  │  私 (3)        │  ← 短縮ラベル + 件数
└──────────────────────────────┘
```

| 属性 | 値 |
|------|-----|
| ラベル | **「全体」「私」**（ツールチップで「全体のログ」「私のログ」を表示） |
| トラック高さ | `2rem`（フル画面より 0.5rem 低い） |
| フォントサイズ | `0.75rem` |

パネルモード時も **アイデンティティバナー** を表示する（`mine` 選択時のみ）。  
ただし幅が狭いため `padding: 0.375rem 0.625rem`・`font-size: 0.75rem` に縮小する。

### 5.9 永続化

| localStorage キー | 型 | デフォルト |
|------------------|----|----------|
| `hossii.logScope` | `'all' \| 'mine'` | `'all'` |

- 不正値・未保存は `'all'` にフォールバック。

## 6. データ・フィルタ仕様

### 6.1 データソース（変更なし）

| 画面／モード | 取得元 |
|-------------|--------|
| 自スペース | `getActiveSpaceHossiis()` |
| 訪問モード | `visitingHossiis`（既存 [57](./57_簡単にログ一覧.md) 相当） |

スコープ `mine` では取得済み配列をクライアント側でさらに絞り込む。Supabase への追加クエリは初版では行わない。

### 6.2 「私」の判定

```text
isMine(hossii) :=
  profile?.id が存在
  AND hossii.authorId === profile.id
  AND coerceIsHidden(hossii.isHidden) === false
```

| ケース | 含むか |
|--------|--------|
| 通常の手動投稿（ゲスト / 参加者） | ✅ |
| Listen 等で `authorId` が `undefined` | ❌ |
| `authorNameOverride` で `authorId` なし | ❌ |
| 管理者が他人名義で投稿 | ❌ |

> ニックネームを変更しても `authorId` は不変のため、改名前の投稿も「私のログ」に残る（正しい挙動）。

### 6.3 フィルタ・ソートの適用順

```
1. isHidden 除外
2. ログスコープ（mine なら authorId === profile.id）
3. ソート（新着 / いいね）
4. FilterBar（コメント / 気持ち）
5. タグ OR 絞り込み
```

### 6.4 `profile` 未作成時

`profile` は初回投稿時に生成。一度も投稿していない参加者は `profile === null` の可能性がある。  
→ セグメント自体は無効化せず、「私のログ」を選ぶと空状態（§5.7）を表示。

## 7. ナビゲーション・既存画面との関係

### 7.1 `#comments`（本仕様の主導線）

- 下ナビ「ログ」・`TopRightMenu`「ログ一覧」は **引き続き `#comments` のみ**。
- 新しいハッシュは増やさない（`#comments?scope=mine` 等は初版では不要）。

### 7.2 `#mylogs` / `MyLogsScreen` との役割分担

| 画面 | スコープ | 主な用途 |
|------|---------|---------|
| `#comments`（本仕様） | 現在のスペース内の自分 / 全員 | **スペース参加中の振り返り** |
| `#mylogs` | 全スペース横断の自分 | **全活動履歴の確認**（既存） |

初版では `#mylogs` への変更なし。ナビへの追加も行わない。

## 12. テスト観点（手動）

1. ゲストでニックネーム入室 → 2 件投稿 → ログ → **私のログ** に 2 件のみ表示
2. 件数バッジが正しく「全体 (全件数)」「私 (自分の件数)」を示している
3. **全体のログ** に戻すと他参加者の投稿も見える
4. スコープ切替時にカードリストがフェードインする
5. 「私のログ」切替後、ヘッダー下にニックネームバナーが出る
6. リロード後も最後のスコープが復元される
7. 未投稿のゲストで **私のログ** → 空状態 UI（アイコン・メッセージ・ボタン）
8. 「投稿してみる」ボタンで投稿フローへ遷移する
9. 訪問モードで **私のログ** → 訪問先スペース内の自分の投稿のみ
10. Listen 自動投稿（`authorId` なし）が**私のログに含まれない**
11. クイックログパネルで同様に切替・バナー表示できる
12. モバイル幅でセグメントが折り返さず、タップ領域が十分（高さ `2.5rem`）
13. `prefers-reduced-motion: reduce` 環境でアニメーションなしの即時切替
14. キーボードの矢印キーでセグメントを操作でき、スクリーンリーダーが切替を読み上げる
