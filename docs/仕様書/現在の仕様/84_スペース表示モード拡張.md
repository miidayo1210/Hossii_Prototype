## 1. 目的

スペース画面（`#screen`）の表示を、用途に応じて拡張する。

| # | 機能 | ゴール |
|---|------|--------|
| **A** | **PC 星表示モード** | スマホ縦の「星＋ランダムプレビュー」体験を PC でも選べるようにする。PC は画面が広いため **同時プレビュー 6 件**（§14。初版は 5 件） |
| **B** | **タグ絞り込み** | スペース上に載せる投稿をタグで絞り込む。右上「タグ」ボタンから即時適用 |

## 2. 現状（実装ベースライン）

### 2.1 星表示（StarView）

| 項目 | 現状 |
|------|------|
| 有効条件 | `(isMobile && isPortrait) \|\| (!isMobile && presentationMode === 'stars')` |
| PC | デフォルト **星モード**（`presentationMode === 'stars'`）。右上 pill で **カスタムモード**（吹き出し配置）へ切替 |
| 用語 | **星モード** = StarView。**カスタムモード** = Bubble 等の従来配置（旧「吹き出し表示」/`bubbles` ストレージ値） |
| プレビュー件数 | **PC 星 ON:** 最大 **6** スロット（§14 改定。初版実装は 5）。**モバイル縦:** 従来どおり 1〜3（≤3件→1スロット） |
| ローテ間隔 | 6 秒ごとにランダム選出 |
| 対象 | `message` または `imageUrl` がある投稿 |
| 除外 | `layoutMode === 'byAuthor'` では星ローテ無効 |

### 2.2 表示モード（viewMode）

左バー `LeftControlBar` の「表示」: `full` / `bubble` / `image` / `slideshow`（[`displayPrefsStorage.ts`](../../../src/core/utils/displayPrefsStorage.ts)）。

星表示は **viewMode とは独立** のプレゼンテーション層として追加する（既存 viewMode の意味を壊さない）。

### 2.3 タグ

| 項目 | 現状 |
|------|------|
| スペース設定 | `presetTags`（管理者登録） |
| 投稿 | `tags` / `hashtags` |
| スペース画面 | **タグ絞り込みあり**（右上「タグ」/ モバイル `...` メニュー、`spaceTagFilterStorage` per spaceId） |
| ログ一覧 | `LogListBody` でタグ OR フィルタ（[26](./26_タグとフィルタリング機能.md)） |

### 2.4 右上クラスタ（`spaceTopRightCluster`）の現状

PC では `top: calc(4rem + 10px); right: 12px; z-index: 95` に `flex-direction: column; align-items: flex-end; gap: 8px` で積み上げ。

既存要素:
1. 書き出しボタン（権限あり時）
2. 投稿数バッジ（`postCountBadge`）— ガラス風半透明 pill
3. 投稿順ツールバー（降順・昇順、`ordered` / `byAuthor` 時）

本仕様で **タグボタン** と **星表示トグル** を追加する。

## 3. 機能 A — PC 星表示モード

### 3.1 ユーザーストーリー

> 「スペースを眺めているとき、吹き出しではなく星だけ漂う静かなモードにして、
> 気になる星にだけカーソルを合わせてプレビューを読みたい。」

### 3.2 トグルボタン（右上クラスタ）

#### 外観

切替 pill は **遷移先モード名** を表示する（現在モード名ではない）。

```
星モード中（アクティブ pill）
┌─────────────────┐
│  カスタム        │  ← タップでカスタムモード（吹き出し）へ
└─────────────────┘

カスタムモード中（非アクティブ pill）
┌─────────────────┐
│ ☆  星           │  ← タップで星モードへ
└─────────────────┘
```

| 項目 | 値 |
|------|-----|
| ラベル | 星 ON 時 **`カスタム`** / カスタム時 **`☆ 星`** |
| サイズ | `padding: 5px 12px; border-radius: 1.5rem`（`postCountBadge` と同等） |
| 星モード時スタイル | `clusterPillActive`（紫グラデーション + 白テキスト） |
| カスタムモード時スタイル | ガラス風 pill（非アクティブ） |
| ホバー | OFF 時: `border-color: rgba(255,255,255,0.40)`. ON 時: `box-shadow` 広がる |
| フォント | `font-size: 0.8125rem; font-weight: 600; letter-spacing: 0.02em` |
| ツールチップ | `"カスタムモード（吹き出し表示）に切り替え"` / `"星モードに切り替え"` |
| aria | 星 ON: `"カスタムモードに切り替え"` / OFF: `"星モードに切り替え"` |

#### 表示条件

| 条件 | 表示 |
|------|------|
| PC（`min-width: 769px`）+ slideshow 以外 | ✅ 表示 |
| モバイル縦 | ❌ 非表示（常に StarView、トグル不要） |
| モバイル横 | ❌ 非表示（現状どおり StarView） |
| `viewMode === 'slideshow'` | ❌ 非表示 |

#### 永続化

```typescript
// src/core/utils/presentationModeStorage.ts
type PresentationMode = 'custom' | 'stars';
const KEY = 'hossii.presentationMode';
// 旧値 'bubbles' は load 時に 'custom' へ読み替え
export function loadPresentationMode(): PresentationMode { ... }
export function savePresentationMode(mode: PresentationMode): void { ... }
```

PC でのみ読み込み・保存（モバイル縦は常に StarView、トグル非表示）。**デフォルト: `'stars'`**（星モード）。

### 3.3 吹き出し ↔ 星 モード切替アニメーション

ユーザーがトグルを押したとき、表示中の投稿が **滑らかに変形** する。

| フェーズ | 内容 | 時間 |
|---------|------|------|
| **1. フェードアウト** | 現モードの全要素が `opacity: 0; transform: scale(0.95)` へ | `200ms ease-out` |
| **2. スイッチ** | `useStarView` が反転、StarView / Bubble が切替 | フェーズ 1 完了後即時 |
| **3. フェードイン** | 新モードの全要素が `opacity: 0 → 1; transform: scale(0.95) → 1` で現れる | `250ms ease-in` |
| **合計** | 約 450ms | |

実装: `bubbleArea` に `.modeTransitionOut` / `.modeTransitionIn` クラスを toggle し、`transitionend` で後片付け。`prefers-reduced-motion` 時はアニメーション省略（即時切替）。

### 3.4 星表示 ON 時の挙動（PC）

| 項目 | スマホ縦（現状） | PC（本仕様） |
|------|----------------|-------------|
| コンポーネント | `StarView` | 同左 |
| 同時プレビュー数 | 最大 3 件 | **最大 6 件**（§14 実装済み） |
| ローテ間隔 | 6 秒 | **6 秒**（同一） |
| プレビュー対象条件 | `message \|\| imageUrl` あり | 同左 |
| 並びモード | `layoutMode` に従う | 同左 |
| `layoutMode === 'byAuthor'` | 星ローテ無効 | 同左 |
| 星クリック | 投稿詳細モーダル | 同左 |

**スロット数の計算（PC）:**

```text
postsWithContent = displayHossiis.filter(h => h.message || h.imageUrl)
maxSlots = isMobile ? 3 : 5
previewSlotCount = Math.min(postsWithContent.length, maxSlots)
```

### 3.5 PC プレビューカード（星ホバー）

PC では **星にカーソルを乗せたとき** に短いプレビューが吹き出しとして表示される（ローテとは別途）。

| 項目 | 仕様 |
|------|------|
| トリガー | `mouseenter`（50ms debounce で誤作動防止） |
| 内容 | `message` 最大 2 行 + `imageUrl` があればサムネ + **気持ち絵文字（選択時のみ・名前の左・同フォントサイズ）** + 投稿者名 |
| 位置 | 星の右（右端に寄っていれば左）。`position: fixed` で overflow を避ける |
| スタイル | 既存 `Bubble` の `backdrop-filter` スタイルを流用。`border-radius: 12px; padding: 10px 14px; max-width: 220px` |
| 出現 | `opacity: 0 → 1; transform: translateX(-4px) → 0` で `150ms` |
| 消滅 | `mouseleave` → `100ms` フェードアウト |
| 競合 | ローテプレビュー中の星にホバーしても **ホバーカードが優先**（ローテが一時停止するわけではない） |
| **全文** | 省略されている長文は [99_吹き出し全文表示](./99_吹き出し全文表示.md) に従い **ホバー時に切り詰めなし**（現状 80 文字切りは改定対象） |

> スマホでは **タップ = 詳細モーダル** のまま（ホバーカードはPC専用）。

### 3.6 星表示モードでの `byAuthor`

`layoutMode === 'byAuthor'` では星ローテを行わず、`AuthorClusterBubble` が前面に出る（現状スマホと同じ）。トグルを ON にしても **クラスタは Bubble 形式のまま**。ユーザーへの注意: 特になし（既存どおりサイレント）。

### 3.7 空状態

投稿が 0 件または `message` / `imageUrl` を持つ投稿が 0 件の場合:

- 星もプレビューも表示されない
- トグル自体は表示・操作可能（切り替えは機能するが見た目に変化はない）

## 4. 機能 B — スペース画面のタグ絞り込み

### 4.1 ユーザーストーリー

> 「スペースに『#質問』タグの投稿が混在しているとき、
> タグボタンから `#質問` を選ぶと、その投稿だけが星・吹き出しとして残り、
> 他は一時的に消えて画面がすっきりする。」

### 4.2 タグボタン（右上クラスタ）

#### 外観

```
未選択
┌──────────────┐
│ 🏷  タグ  ▾  │   ← ガラス風 pill
└──────────────┘

選択中（#質問）
┌───────────────────┐
│ 🏷  #質問  ×      │   ← 紫アクセント + 選択タグ名 + クリアボタン
└───────────────────┘

候補 0 件（disabled）
┌──────────────┐
│ 🏷  タグ  ▾  │   ← opacity: 0.4, pointer-events: none
└──────────────┘
```

| 項目 | 値 |
|------|-----|
| アイコン | タグアイコン（`🏷` またはインライン SVG） |
| ラベル未選択 | `タグ ▾`（▾ は CSS border triangle か svg chevron） |
| ラベル選択中 | `#<タグ名>` + `×`（クリアボタン）。タグ名が 8 文字超は省略（`max-width: 80px; text-overflow: ellipsis`） |
| OFF スタイル | `postCountBadge` と同じガラス風 |
| ON スタイル | `background: linear-gradient(135deg, #a855f7 0%, #ec4899 100%); color: #fff; box-shadow: 0 0 12px rgba(168,85,247,0.45)` |
| disabled | `opacity: 0.4; cursor: default; pointer-events: none` |
| aria | `aria-haspopup="listbox" aria-expanded={open} aria-label="タグで絞り込む"` |
| トランジション | `all 0.2s ease` |

#### `×` クリアボタン（選択中のみ表示）

- pill 内右端に配置
- `padding: 2px 4px; margin-left: 4px`
- クリックでフィルタ解除（パネルは開かない）
- `aria-label="タグフィルタを解除"`

### 4.3 タグ選択パネル（ドロップダウン）

#### レイアウト

```
タグボタン直下（またはやや内側）にポップオーバーとして出現
                         ┌────────────────────┐
                         │ ╔══════════════╗   │
                         │ ║  すべて表示  ║   │ ← 太め・現在のフィルタが null なら選択状態
                         │ ╚══════════════╝   │
                         │  ──────────────    │
                         │  #質問             │ ← 選択中のタグは ✓ アイコン
                         │  #感想             │
                         │  #アイデア         │
                         │  #つぶやき         │
                         └────────────────────┘
```

| 項目 | 値 |
|------|-----|
| 幅 | `min-width: 160px; max-width: 240px` |
| 最大高さ | `max-height: 280px; overflow-y: auto` |
| スタイル | `background: rgba(15,10,30,0.88); backdrop-filter: blur(16px); border: 1px solid rgba(255,255,255,0.14); border-radius: 12px; box-shadow: 0 8px 32px rgba(0,0,0,0.4); padding: 6px` |
| 出現 | `opacity: 0→1; transform: translateY(-6px)→translateY(0); 180ms ease-out` |
| 消滅 | `opacity 1→0; transform translateY(0)→translateY(-4px); 140ms ease-in` |
| 閉じ方 | ① 選択（アイテムクリック） ② パネル外クリック ③ `Escape` キー |
| z-index | `200`（`spaceTopRightCluster` の z-index 95 より上） |

#### 「すべて表示」行

| 項目 | 値 |
|------|-----|
| スタイル（現在未フィルタ） | `color: rgba(255,255,255,0.95); font-weight: 700` + 左に `✓` アイコン |
| スタイル（フィルタ中） | `color: rgba(255,255,255,0.65)` |
| クリック | フィルタ解除 + パネル閉じる |
| 区切り線 | 行の下に `border-bottom: 1px solid rgba(255,255,255,0.10); margin: 4px 0` |

#### タグ行

| 項目 | 値 |
|------|-----|
| 表示順 | `presetTags` の順 → `presetTags` にないが投稿に付いているタグを出現件数降順で追加 |
| 各行スタイル | `padding: 8px 12px; border-radius: 8px; cursor: pointer; color: rgba(255,255,255,0.85); font-size: 0.875rem` |
| ホバー | `background: rgba(255,255,255,0.10)` |
| 選択中 | 左に `✓` + `color: #e879f9; font-weight: 600` |
| ラベル | `#<タグ名>`（`#` は常に表示） |
| 件数バッジ | 行右端に `(n件)`（薄いテキスト `font-size: 0.75rem; color: rgba(255,255,255,0.45)`） |
| クリック | 同タグを選択 → フィルタ適用 → パネル閉じる |

#### キーボード操作

| キー | 動作 |
|------|------|
| `Enter` / `Space` | ボタン押下（パネル開閉） |
| `↓` / `↑` | パネル内アイテムをフォーカス移動 |
| `Enter` | アイテム選択 |
| `Escape` | パネルを閉じてボタンにフォーカス戻す |

### 4.4 フィルタ適用アニメーション

ユーザーがタグを選択・解除したとき:

| フェーズ | 内容 | 時間 |
|---------|------|------|
| **1. フィルタアウト** | 非該当の Bubble / Star が `opacity: 1 → 0; transform: scale(0.92)` | `200ms ease-out` |
| **2. フィルタイン** | 該当投稿が `opacity: 0 → 1; transform: scale(0.92) → 1` で安定 | `250ms ease-in` |
| **3. 再配置** | `layoutMode === 'random'` の場合は座標そのまま（非該当が消えて残る）。`ordered` / `byAuthor` は再計算後にフェードイン | |

解除（すべて表示）は逆: 非表示だった投稿が `opacity: 0 → 1; transform: scale(0.92) → 1`。

`prefers-reduced-motion` ではアニメーション省略。

### 4.5 フィルタルール

```typescript
// state
const [activeSpaceTagFilter, setActiveSpaceTagFilter] = useState<string | null>(null);
// 例: "質問"（# なしで保持）

// displayHossiis 算出後に適用
const filteredHossiis = useMemo(() => {
  if (!activeSpaceTagFilter) return displayHossiis;
  return displayHossiis.filter(h => {
    const combined = [...(h.tags ?? []), ...(h.hashtags ?? [])];
    return combined.includes(activeSpaceTagFilter);
  });
}, [displayHossiis, activeSpaceTagFilter]);
```

| 項目 | 仕様 |
|------|------|
| 候補タグ | `presetTags` ∪ 現 `displayHossiis`（期間・件数・viewMode 適用後）に存在する `tags` / `hashtags` |
| **タグ初版スコープ（87 連携）** | **取得済み `displayIds` 内のみ** client filter。DB 側タグ filter / 51 件目以降のタグ探索は [87 §4.5](./87_スペース表示パフォーマンス最適化.md) Phase 2（U3） |
| 候補 0 件 | タグボタンを `disabled` |
| 件数バッジ | フィルタ後の件数を表示 |
| 星ローテ | フィルタ後の集合のみ対象 |
| `layoutMode === 'byAuthor'` | フィルタ後の投稿でグループ再計算。**該当投稿が 0 の作者グループは非表示** |
| `layoutMode === 'ordered'` | フィルタ後の件数で格子を再計算（列数が変わる可能性あり） |

### 4.6 空状態（フィルタ後 0 件）

スペース上にすべての投稿が消え、空になる場合:

```
┌──────────────────────────────────────────┐
│                                          │
│          🏷                              │
│    「#○○」の投稿はまだありません          │
│                                          │
│    [すべての投稿を表示する]              │
│                                          │
└──────────────────────────────────────────┘
```

| 項目 | 値 |
|------|-----|
| 位置 | `bubbleArea` 中央（`display: flex; align-items: center; justify-content: center`） |
| スタイル | アイコン `font-size: 2.5rem` + テキスト `font-size: 0.9rem; color: rgba(255,255,255,0.55)` |
| ボタン | `[すべての投稿を表示する]` — クリックで `activeSpaceTagFilter = null` |
| アニメーション | `opacity: 0 → 1; 300ms` |

### 4.7 永続化

```typescript
// src/core/utils/spaceTagFilterStorage.ts
const KEY = (spaceId: string) => `hossii.spaceTagFilter.${spaceId}`;
export function loadSpaceTagFilter(spaceId: string): string | null { ... }
export function saveSpaceTagFilter(spaceId: string, tag: string | null): void { ... }
```

- スペースごとに保存（スペース切替で独立）
- `null` = すべて表示（保存値なし or `null` で同等）
- 訪問モードでも同じキーで保存・復元

### 4.8 ログ一覧との関係

| 画面 | タグフィルタ | 保存キー |
|------|-------------|---------|
| スペース `#screen` | 本機能 | `hossii.spaceTagFilter.<spaceId>` |
| ログ `#comments` | 既存 `LogListBody` | セッション内 state |

**連動しない**（スペースで `#質問` を選んでも、ログ一覧のタグ選択は自動では変わらない）。

## 5. 右上クラスタ — レイアウト詳細

### 5.1 並び順（左→右・上→下）

`spaceTopRightCluster` は `flex-direction: column; align-items: flex-end; gap: 8px`（既存）。
各行は `display: flex; align-items: center; gap: 6px` の **row**。

```
┌────────────────────────────────────────────────────────────┐
│ [row 1]  [書き出し（権限あり）]                            │
│ [row 2]  [#42件]                                           │
│ [row 3]  [降順 | 昇順]（ordered / byAuthor 時）            │
│ [row 4]  [🏷 タグ ▾]  [★ 星]                              │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

「タグ」と「星」は **同じ row（横並び）** に置く（row 4）。

`gap: 6px` で並べ、クラスタ全体の `gap: 8px` で各行が縦に積まれる。

### 5.2 pill の高さ統一

`postCountBadge` / タグボタン / 星トグルはすべて `line-height` 含め **高さ 30px** を目安に揃える（`padding: 5px 10px〜12px`）。フォント `0.8125rem`、`font-weight: 600`。

### 5.3 モバイル（768px 以下）

| 要素 | モバイル対応 |
|------|-------------|
| 星トグル | **非表示**（モバイルは常に StarView） |

## 7. 状態管理

```typescript
// SpaceScreen.tsx 内 state

// 機能 A
const [presentationMode, setPresentationMode] =
  useState<'custom' | 'stars'>(() => loadPresentationMode());

  (isMobile && isPortrait) ||
  (!isMobile && presentationMode === 'stars');

// 機能 B
const [activeTagFilter, setActiveTagFilter] =
  useState<string | null>(() => loadSpaceTagFilter(space.id));

const [tagPanelOpen, setTagPanelOpen] = useState(false);

// フィルタ適用
const filteredHossiis = useMemo(() => {
  if (!activeTagFilter) return displayHossiis;
  return displayHossiis.filter(h =>
    [...(h.tags ?? []), ...(h.hashtags ?? [])].includes(activeTagFilter)
  );
}, [displayHossiis, activeTagFilter]);

// タグ候補
const tagCandidates = useMemo(() => {
  const fromPosts = [...new Set(
    displayHossiis.flatMap(h => [...(h.tags ?? []), ...(h.hashtags ?? [])])
  )];
  const presetOrder = space.presetTags ?? [];
  return [
    ...presetOrder.filter(t => fromPosts.includes(t)),
    ...fromPosts.filter(t => !presetOrder.includes(t)),
  ];
}, [displayHossiis, space.presetTags]);

// プレビュースロット数
const previewSlotCount = useMemo(() => {
  if (layoutMode === 'byAuthor' || !useStarView) return 0;
  const withContent = filteredHossiis.filter(h => h.message || h.imageUrl);
  return Math.min(withContent.length, isMobile ? 3 : 5);
}, [filteredHossiis, layoutMode, useStarView, isMobile]);
```

### 14.6 プレビュー対象星の回転アニメ

**意図:** ローテで「次に（現在）プレビューが付く星」だけが **ゆっくり回転** し、どの星に注目すべきか直感的にわかるようにする。**画面上の他の星は回転しない**。

#### 付与条件

```typescript
const isPreviewCandidate = previewHossiiIds.has(hossii.id);
// StarView: className に isPreviewCandidate && isPcStarMode なら styles.starPreviewRotate
```

| 項目 | 仕様 |
|------|------|
| 対象 | `previewHossiiIds` に含まれる星 **最大 6 件**（PC 星モードのみ） |
| 非対象 | プレビュー非表示の星、モバイル、Bubble モード、byAuthor |
| ローテ切替 | 6 秒ごとに選出集合が変わる → **回転クラスも付け替え** |
| ホバー | 回転は **継続**（停止しない）。`StarHoverPreview` と併用可 |

#### アニメーション定義

```css
@keyframes starPreviewSpin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

.starPreviewRotate .starDot {
  animation:
    starPreviewSpin 14s linear infinite,
    /* 既存 pulse / float / twinkle は animationLevel に従い併用 or 回転時は twinkle のみ */
    twinkle 4s ease-in-out infinite;
  transform-origin: center center;
}
```

| パラメータ | 値 | 理由 |
|-----------|-----|------|
| 1 周 | **14s** | 目立ちすぎない「可愛い」速度 |
| easing | `linear` | 一定速度で回転感を出す |
| 同時 CSS Animation | 回転 **1** + twinkle **1** = **2 本/星** × 6 = **12 本**（87 の tier 予算内） |

**`animationLevel === 'none'` の星**がプレビュー候補に選ばれた場合: 回転 + twinkle（light 相当）のみ付与。full tier の pulse/float は付けない。

**`prefers-reduced-motion`:** 回転・previewBob とも **無効**。吹き出しはフェードインのみ（即時表示可）。
