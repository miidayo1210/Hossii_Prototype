## 1. 目的

投稿パネル（ダブルタップ・「投稿」ボタンから開くパネル）に表示する入力項目を、**スペースごとに管理者が制御**できるようにする。

| 設定 | 内容 |
|------|------|
| **表示 ON/OFF** | OFF にした項目は投稿パネルに表示されなくなる |
| **必須 ON/OFF** | ON にした項目を空欄のまま投稿ボタンを押せなくなる |

**デフォルト:** 全項目 **表示 ON / 必須 OFF**（現状の動作を維持）。

---

## 2. 管理対象の投稿フォーム項目

| ID | ラベル | 表示条件（実装） | 説明 |
|----|--------|----------------|------|
| `message` | 💬 メッセージ | `resolvePostFields(...).message.enabled` | テキスト入力（メッセージ本文） |
| `emotion` | 😊 気持ち | `pf.emotion.enabled` | 感情ボタン 8 種 |
| `tags` | 🏷 タグ | `pf.tags.enabled`（`presetTags` 有無に関わらず） | プリセットタグ選択 + 自由入力 |
| `photo` | 📷 写真 | `pf.photo.enabled` | 画像アップロード |
| `bubbleColor` | 🎨 吹き出し色 | `pf.bubbleColor.enabled` | カラーパレット選択 |
| `bubbleShape` | 🔷 吹き出し形状 | `pf.bubbleShape.enabled && bubble_shapes_extended` | 吹き出し形状プリセット選択 |
| `numberPost` | 🔢 数値入力 | `pf.numberPost.enabled`（デフォルト OFF） | 数値投稿 |

> **スコープ外（初版）:** フリー投稿タブ（キャンバス）、位置選択グリッド（[45](./45_投稿の位置を定める機能.md)）— これらは引き続き Feature Flags で管理。

## 3. データ設計

### 3.1 新規型 `PostFieldConfig` / `PostFieldSettings`

`src/core/types/settings.ts` に追加:

```typescript
export type PostFieldConfig = {
  enabled: boolean;  // 表示するか（false = 投稿フォームに出ない）
  required: boolean; // 必須入力か（enabled が false のとき無効）
};

export type PostFieldSettings = {
  message:     PostFieldConfig;
  emotion:     PostFieldConfig;
  tags:        PostFieldConfig;
  photo:       PostFieldConfig;
  bubbleColor: PostFieldConfig;
  bubbleShape: PostFieldConfig;
  numberPost:  PostFieldConfig;
};

export const DEFAULT_POST_FIELD_SETTINGS: PostFieldSettings = {
  message:     { enabled: true,  required: false },
  emotion:     { enabled: true,  required: false },
  tags:        { enabled: true,  required: false },
  photo:       { enabled: true,  required: false },
  bubbleColor: { enabled: true,  required: false },
  bubbleShape: { enabled: true,  required: false }, // feature flag との AND が最終判定
  numberPost:  { enabled: false, required: false }, // 既存デフォルトが OFF のため
};
```

### 3.2 `SpaceSettings` への追加

```typescript
export type SpaceSettings = {
  // ... 既存フィールド ...
  postFields?: PostFieldSettings; // optional — 旧データ互換
};
```

PostScreen は **`resolvePostFields(settings)`** のみ参照する（`src/core/utils/postFieldSettings.ts`）。

### 3.3 既存 `SpaceFeatures` との関係（移行方針）

`SpaceFeatures`（`commentPost` / `emotionPost` / `photoPost` / `numberPost`）は **GeneralTab に並存**。PostScreen は `postFields` のみ参照。

| 既存フィールド | `postFields` の対応 | 読み方 |
|--------------|---------------------|--------|
| `features.commentPost` | `postFields.message.enabled` | `postFields` **あり** → merge して使用 |
| `features.emotionPost` | `postFields.emotion.enabled` | `postFields` **なし** → `features.*` から enabled のみ導出（required は全 false） |
| `features.photoPost` | `postFields.photo.enabled` | `tags` / `bubbleColor` / `bubbleShape` → enabled true |
| `features.numberPost` | `postFields.numberPost.enabled` | |

```typescript
// 実装: src/core/utils/postFieldSettings.ts
export function resolvePostFields(settings: SpaceSettings | null | undefined): PostFieldSettings
export function mergePostFieldSettings(partial): PostFieldSettings  // DEFAULT 補完 + enabled OFF 時 required リセット
export function applyPostFieldChange(settings, field, key, value): SpaceSettings  // PostFormTab 用
```

### 3.4 保存先

| 層 | 保存先 | 備考 |
|----|--------|------|
| 端末 | `localStorage` `space_settings_<spaceId>` | `saveSpaceSettings` — 即時 |
| サーバー | Supabase `space_settings.post_fields`（**jsonb 列**） | `upsertSpaceSettings` — 3 段階 upsert（下記） |

**マイグレーション:**

- `supabase/migrations/20260623000000_add_post_fields_to_space_settings.sql` — `post_fields jsonb`
- `supabase/migrations/20260623000001_add_bubble_edit_permission_to_space_settings.sql` — `bubble_edit_permission`（GeneralTab 用・upsert 分割のため）

**`upsertSpaceSettings` の保存順序**（`spaceSettingsApi.ts`）:

1. 基本列のみ upsert（`feature_*`, `card_type`, `hossii_color`, `bottle_frequency` 等）
2. `bubble_edit_permission` を update（列がなければスキップ）
3. `post_fields` を update（列がなければ `PostFieldsColumnMissingError`）

PostFormTab は Supabase 失敗時に **楽観的更新をロールバック** し、赤トーストを表示する。

### 4.1 新規タブ「投稿フォーム」

`SpaceSettingsScreen` の既存タブリストに追加:

```typescript
type Tab = 'general' | 'postForm' | 'hossii' | 'background' | 'share'
```

### 4.3 各行の UI 詳細

| 要素 | 仕様 |
|------|------|
| 行 | `padding: 14px 16px; border-bottom: 1px solid rgba(0,0,0,0.07)` |
| 項目名 | アイコン + テキスト `font-size: 0.9375rem; font-weight: 500` |
| 説明文 | 項目名の下に `font-size: 0.8125rem; color: #6b7280` で 1 行（後述） |
| 「表示」トグル | `FeatureFlagsTab` と同じデザインのトグルスイッチ |

| ※ 注釈 | `bubble_shapes_extended` flag が OFF の場合、行末に `※ Feature Flag 要` バッジ（`font-size: 0.75rem; color: #9ca3af; border: 1px solid currentColor; border-radius: 4px; padding: 1px 6px`） |

#### 各項目の説明文

| 項目 | 説明文 |
|------|--------|
| メッセージ | 投稿本文のテキスト入力です |
| 気持ち | Wow・刺さった など 8 種類の感情ボタンです |
| タグ | プリセットタグの選択と自由タグ入力です |
| 写真 | 画像ファイルのアップロードです |
| 吹き出し色 | 吹き出しの背景カラーパレットです |
| 吹き出し形状 | ハート・雲・風船など 14 種類の形状選択です |
| 数値入力 | 数値だけを投稿する専用フィールドです |

### 4.4 保存タイミング

**即時保存**（`FeatureFlagsTab` と同方式）。トグル操作ごとに Supabase へ upsert し、3 秒後消えるトーストで完了通知。楽観的更新（失敗時は元に戻す）。

```
● ON にしました        → トースト「「メッセージ」の入力必須を ON にしました」
● OFF にしました       → トースト「「気持ち」の表示を OFF にしました」
● Supabase 失敗時     → トースト「設定の保存に失敗しました」（赤）＋ 元の値に戻す
● post_fields 列なし  → トースト「Supabase に post_fields 列がありません。マイグレーション…」（赤）
```

### 4.5 バリデーション（管理 UI 側）

| チェック | 動作 |
|---------|------|
| 表示 OFF → 必須 ON は不可 | 必須トグルを disabled 状態にする（操作不能） |
| 表示を OFF にしたとき必須が ON だった | 自動で必須を OFF にリセットしてから保存 |
| すべての項目を表示 OFF | 許可するが「投稿パネルが空になります」警告メッセージを赤文字で表示 |

```
⚠️ すべての項目が非表示になっています。
   投稿パネルを開いても何も入力できなくなります。
```

## 5. 投稿パネル（PostScreen）の動作

### 5.1 項目の表示制御

`PostScreen` は `SpaceSettings.postFields`（なければ `DEFAULT_POST_FIELD_SETTINGS`）を読み込み、各項目の `enabled` に応じてセクションを表示・非表示にする。

```typescript
const pf = resolvePostFields(spaceSettings);

// メッセージ欄
{pf.message.enabled && (
  <section>
    <textarea ... />
    {pf.message.required && (
      <span className={styles.requiredBadge}>必須</span>
    )}
  </section>
)}

// 気持ちボタン
{pf.emotion.enabled && (
  <section>
    {/* 感情ボタン 8 種 */}
    {pf.emotion.required && (
      <span className={styles.requiredBadge}>必須</span>
    )}
  </section>
)}
// ... 他項目も同様
```

**`bubbleShape` の最終判定:**

```typescript
const showBubbleShape =
  pf.bubbleShape.enabled &&
  featureFlags.bubble_shapes_extended;
```

Feature Flag が OFF なら、`postFields.bubbleShape.enabled === true` であっても非表示。

### 5.2 必須バッジの表示

各セクションの右上（またはラベル横）に「必須」バッジを表示する。

```
┌─────────────────────────────────────┐
│  💬 メッセージ                [必須] │
│ ┌───────────────────────────────┐   │
│ │ 気持ちやメモをどうぞ...        │   │
│ └───────────────────────────────┘   │
└─────────────────────────────────────┘
```

| 項目 | 値 |
|------|-----|
| スタイル | `font-size: 0.6875rem; font-weight: 700; color: #ef4444; background: #fee2e2; border-radius: 4px; padding: 1px 6px` |
| 表示条件 | `postFields[field].required === true` |

### 5.3 バリデーション（投稿ボタンの活性制御）

```typescript
// 各必須フィールドが充足されているか（enabled でない項目は常に充足）
const isMessageSatisfied   = !pf.message.enabled   || !pf.message.required   || message.trim() !== '';
const isEmotionSatisfied   = !pf.emotion.enabled   || !pf.emotion.required   || selectedEmotion !== null;
const isTagsSatisfied      = !pf.tags.enabled       || !pf.tags.required      ||
                             hashtags.length > 0 || selectedPresetTags.length > 0;
const isPhotoSatisfied     = !pf.photo.enabled      || !pf.photo.required     || imagePreview !== null;
const isNumberSatisfied    = !pf.numberPost.enabled || !pf.numberPost.required || numberInput.trim() !== '';
// bubbleColor / bubbleShape は「選択なし = デフォルト」で実質常に充足

const allRequiredSatisfied = /* 上記の AND */;

// enabled な項目のみカウント
const hasAnyInput = /* emotion/message/photo/number/tags の enabled 項目に入力がある */;

const canSubmit = allRequiredSatisfied && hasAnyInput && !sending;
```

**投稿ボタン UI:**

| 状態 | 挙動 |
|------|------|
| `!hasAnyInput` | `disabled` |
| `hasAnyInput && !allRequiredSatisfied` | クリック可（`submitButtonInactive` で視覚的に非活性）。押下で赤枠エラー + 先頭フィールドへスクロール |
| `canSubmit` | 通常の投稿処理 |

`submitAttempted` state は投稿ボタン押下時に `true`。入力中は赤枠を出さない。

### 5.4 未充足時のフィードバック

投稿ボタンを押して必須フィールドが未入力の場合、ボタンを押した瞬間に各未充足フィールドを **赤枠 + エラーメッセージ** でハイライトする（フォームをスクロール先頭へ戻す）。

```
┌─────────────────────────────────────┐  ← border: 1px solid #ef4444
│  💬 メッセージ                [必須] │
│ ┌───────────────────────────────┐   │
│ │                               │   │  ← 赤枠
│ └───────────────────────────────┘   │
│  ⚠️ メッセージを入力してください    │  ← エラーテキスト
└─────────────────────────────────────┘
```

| 要素 | 値 |
|------|-----|
| エラーボーダー | `border: 1.5px solid #ef4444; border-radius: 8px` |
| エラーテキスト | `font-size: 0.8125rem; color: #ef4444; margin-top: 4px` |
| 表示タイミング | 投稿ボタン押下後に `touched` state を `true` にして表示（入力中に赤くならない） |

## 9. テスト観点

1. **デフォルト確認:** 新規スペースでは全項目 ON / 全任意。投稿フォームが現状と同じ表示
2. **表示 OFF:** メッセージを OFF → 投稿パネルにテキストエリアが出ない
3. **必須 ON:** 気持ちを必須 → 感情未選択で投稿ボタン押下 → 赤枠 + エラー
4. **必須 ON → 解消:** 気持ちを選択 → 投稿可能になる
5. **表示 OFF → 必須 OFF 自動化:** 写真 必須 ON → 写真 表示 OFF → 必須も自動で OFF になる
6. **全 OFF 警告:** 全項目を非表示 → 警告文が出る
7. **bubble_shapes_extended が OFF:** 吹き出し形状の「表示」を ON にしても PostScreen に出ない
8. **旧データ補完:** `postFields` なしの設定データをロード → DEFAULT で補完される
9. **リロード後復元:** 設定変更 → リロード → 設定が維持されている
10. **他スペースへの非波及:** スペース A の設定がスペース B に影響しない
