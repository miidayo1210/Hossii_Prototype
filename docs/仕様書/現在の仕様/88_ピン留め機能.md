# 88 ピン留め機能

> **ステータス**: ✅ 実装済み（2026-06-24）

---

## 1. 概要・目的

スペース上の投稿（Hossii）を「ピン留め」することで、どの表示モードでも常に目立つ場所に表示し続けられる機能。

| ユーザーニーズ | 解決方法 |
|---------------|---------|
| 「この投稿をいつでも見えるようにしたい」 | ピン留めした投稿は常時トレイに表示 |
| 「PC 星モードでも常に特定の吹き出しを出したい」 | 6 枠のローテーションとは独立して常時表示 |
| 操作が簡単・直感的 | 吹き出しにホバーするだけでピンアイコンが現れる |

---

## 2. 用語定義

| 用語 | 意味 |
|------|------|
| ピン | 特定の Hossii を「固定表示対象」として記録すること |
| ピン留めトレイ | スペース画面上端に固定表示される帯状エリア |
| ローテートプレビュー | 既存の PC 星モードで 6 件が自動切り替わる吹き出し（84 §14） |

---

## 3. 機能スコープ（初版）

### 3.1 対象画面・モード

| 画面/モード | ピン操作 | ピン表示 |
|------------|---------|---------|
| PC 星モード（`showPcStarHover`） | ✅ 吹き出しへのホバーで操作可 | ✅ ローテ吹き出しの外に常時表示 ＋ トレイ |
| 通常表示（バブルモード等、PC） | ✅ 吹き出しへのホバーで操作可 | ✅ ピン留めトレイに表示 |
| モバイル（全モード） | ❌ 初版スコープ外（後回し） | ✅ トレイには表示（閲覧のみ） |

### 3.2 初版スコープ外

- モバイルでのピン操作 UI（タップ長押し等）
- ピン留め状態のサーバー同期（初版はクライアントローカルのみ）
- 他ユーザーとのピン共有
- 複数スペース間でのピン引き継ぎ

## 4. ピン状態管理

### 4.1 ストレージ

`localStorage` に保存する。スペースを閉じてもセッションをまたいでも保持。

```typescript
// キー: "hossii_pins_{spaceId}"
// 値: JSON.stringify(string[])  // hossii.id の配列（順序=ピン留め順）
```

### 4.2 上限

**上限なし**。ピン件数に制限を設けない。トレイが多くなった場合はスクロール対応（§7.1）。

### 4.3 状態フック

```typescript
// src/core/hooks/usePinnedHossiis.ts
usePinnedHossiis(spaceId: string): {
  pinnedIds: Set<string>;
  pin: (id: string) => void;
  unpin: (id: string) => void;
  toggle: (id: string) => void;
}
```

## 5. UI 仕様：ピンボタン

### 5.1 ボタンの出現条件

- `.previewBubble`（吹き出し）に**マウスカーソルが乗っている**とき（`:hover`）、吹き出し右上にピンアイコンが現れる
- すでにピン留め済みの吹き出しは**常時**ピンアイコンを表示（ホバー不要）
- **StarHoverPreview**（ホバーポップアップ）右上にもピンボタンを表示（ローテ対象外の星からもピン留め可能）

### 5.2 ボタンのデザイン

```
┌─────────────────────┐ ← 吹き出し
│  投稿テキスト…      📌 ← ピンボタン（右上コーナー）
│  投稿者名          │
└─────────────────────┘
```

| 状態 | アイコン | 背景 | サイズ |
|------|---------|------|-------|
| ホバー時（未ピン） | 📌（薄い） | `rgba(255,255,255,0.5)` | 22×22px |
| ピン済み（常時） | 📌（濃い・紫系） | `rgba(139,92,246,0.15)` | 22×22px |
| ホバー中（ピン済み） | 📌 + ✕ で解除示唆 | `rgba(239,68,68,0.12)` | 22×22px |

- ボタン要素は `pointer-events: all`（吹き出し自体は `pointer-events: none` のため個別に設定）
- クリック時は `e.stopPropagation()` で星クリックへの伝播を防ぐ
- ツールチップ（`aria-label`）: 未ピン時「ピン留めする」、ピン済み「ピン留めを解除する」

## 6. PC 星モードでの表示動作

### 6.1 ローテートプレビューとの関係

```
ローテートプレビュー（最大 6 件）: 既存ロジックそのまま維持
                         +
ピン留め吹き出し（上限なし）    : 常時表示（ローテに依らない）
```

- ピン済み Hossii は `previewHossiiIds`（6 枠のローテート）から**除外**してローテ枠を有効活用（実装済み）
- ピンとローテが被っても問題なし（ピン済みは `showPreview` 常時 true で別 DOM）

### 6.2 ピン吹き出しの配置

PC 星モードのピン吹き出しは、**各星の真上**（§84 §14.5 と同じスタイル）に常時表示。`showPreview` フラグをピン済みの場合は常に `true` に設定することで実現。

## 7. ピン留めトレイ（全モード共通）

### 7.1 配置・デザイン

スペースエリアの**上端に固定**された帯状のトレイ。ピン済み投稿がない場合は非表示。

```
┌──────────────── スペース上端 ───────────────────────────────┐
│ 📌 ○○さん「今日も楽しかった…」  📌 △△さん「感動した！」       │  ← ピン留めトレイ
└──────────────────────────────────────────────────────────────┘
│                                                              │
│              スペース本体（星 / 吹き出し）                     │
```

| プロパティ | 値 |
|-----------|-----|
| `position` | `absolute; top: 0; left: 0; right: 0` （スペース内上端） |
| `z-index` | スペース要素より高い（例: `z-index: 50`） |
| 背景 | `linear-gradient(160deg, rgba(255,255,255,0.22) 0%, rgba(220,210,255,0.16) 100%)` + `backdrop-filter: blur(12px)` |
| `padding` | `6px 12px` |
| `display` | `flex; flex-wrap: wrap; gap: 8px; align-items: center` |
| 多数時 | `overflow-x: auto` でスクロール対応（上限なしのため） |
| ピンなし時 | `display: none` |

### 7.2 トレイ内の各カード

```
┌──────────────────────────────────┐
│ 📌  ○○さん「テキスト…」           │
└──────────────────────────────────┘
```

| プロパティ | 値 |
|-----------|-----|
| 背景 | ガラス風（`rgba(255,255,255,0.28)` + `backdrop-filter: blur(12px)`） |
| `border-radius` | `12px` |
| `padding` | `4px 10px` |
| テキスト最大幅 | `180px`（`text-overflow: ellipsis`） |
| ピンアイコン | 左端に 📌（固定のシンボル） |
| クリック | 対応する星を `isRecentHighlight` 相当でハイライト表示（視覚的に点滅して場所を示す） |
| ✕ ボタン | カード右端。クリックでアンピン |

## 9. 受け入れ条件

| # | 条件 |
|---|------|
| AC1 | 吹き出しにホバーするとピンアイコンが右上に現れる |
| AC2 | アイコンをクリックするとピン留めされ、アイコンが常時表示になる |
| AC3 | もう一度クリックするとピンが解除され、アイコンが非表示になる（ホバー時のみに戻る） |
| AC4 | ページリロード後もピン状態が維持される（localStorage） |
| AC5 | PC 星モードでピン済み投稿の吹き出しが常時表示される |
| AC6 | ピン済みが 1 件以上のとき、スペース上端にトレイが表示される |
| AC7 | トレイの ✕ でもアンピンできる |
| AC8 | ピン件数に上限はなく何件でもピン留め可能 |
| AC9 | トレイのカードをクリックすると対応する星がハイライトされる |
| AC10 | `prefers-reduced-motion` 時にアニメーションが抑制される |
| AC11 | モバイルではピン操作 UI は表示されないが、トレイは表示される |
