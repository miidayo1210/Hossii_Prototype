# 89 スターマーカー種別設定

> **ステータス**: ✅ 実装済み（2026-06-24）

---

## 1. 概要・目的

PC 星モード（および将来の全表示モード）で使用される「星マーク」を、管理者がスペース設定から別のマーカーに変更できる機能。

- 初版は **4 種類（A〜D）** を提供
- 差分は小さく、パフォーマンスへの影響はほぼゼロ

---

## 2. マーカー種別定義

| ID | 名称 | 表示文字 | 説明 |
|----|------|---------|------|
| `star` | ★ 星（デフォルト） | `★` | 現行の星マーク（後方互換） |
| `circle` | ● まる | `●` | シンプルな塗り円 |
| `pin` | 📍 ピン | CSS で描画 | 逆涙滴形。位置・場所を示すピン |
| `person` | 人影 | CSS で描画 | 建築模型のような簡素な人型 |

> **実装方針**: 文字で表現できるもの（`star`, `circle`）は Unicode 1 文字を `starDot` に入れる。  
> CSS で描画するもの（`pin`, `person`）は `starDot::before / ::after` の組み合わせで純 CSS 描画し、余分な DOM 要素を追加しない。

---

## 3. データ設計

### 3.1 型追加（差分 3 行）

**`src/core/types/settings.ts`**

```typescript
// 追加
export type StarMarkerType = 'star' | 'circle' | 'pin' | 'person';

// SpaceSettings に 1 フィールド追加（省略時は 'star'）
export type SpaceSettings = {
  ...
  starMarkerType?: StarMarkerType;  // 省略 = 'star'
};
```

### 3.2 ストレージ

- **localStorage**: `SpaceSettings` JSON に `starMarkerType` を含めて保存（`saveSpaceSettings` 経由）
- **Supabase**: 初版は **列追加なし**。`fetchSpaceSettings` 後に localStorage の値を merge して設定が消えないようにする（[SpaceSettingsScreen.tsx](src/components/SpaceSettingsScreen/SpaceSettingsScreen.tsx)）
- 追加フィールドはオプショナルのため、**既存データの破壊なし**

---

## 4. 表示ロジック

### 4.1 StarView への伝搬（差分 2〜3 行）

`SpaceScreen.tsx` → `StarView` に既存の `isPcStarMode` と同様に `markerType` prop を追加するだけ。

```typescript
// StarView.tsx の Props に追加
markerType?: StarMarkerType;  // 省略 = 'star'
```

`starDot` 要素のレンダリング:

```typescript
const MARKER_CHAR: Record<StarMarkerType, string> = {
  star:   '★',
  circle: '●',
  pin:    '',   // CSS ::before で描画
  person: '',   // CSS ::before / ::after で描画
};

// JSX
<span
  className={`${styles.starDot} ${styles[`marker_${markerType}`]}`}
  aria-hidden="true"
>
  {MARKER_CHAR[markerType]}
</span>
```

### 4.2 CSS（`StarView.module.css` 追加分のみ）

- `star` / `circle` は文字が入るため CSS 変更なし
- `pin` / `person` は純 CSS で描画（`::before` / `::after`）
- 既存アニメーション（`twinkle`, `pulse`, `starPreviewSpin` 等）はそのまま適用

#### pin マーカー CSS（概要）

```css
.marker_pin {
  font-size: 0;          /* 文字非表示 */
  display: inline-block;
  width: 14px;
  height: 14px;
  position: relative;
}
.marker_pin::before {
  /* 円（頭部） */
  content: '';
  position: absolute;
  top: 0; left: 50%;
  transform: translateX(-50%);
  width: 10px; height: 10px;
  border-radius: 50% 50% 50% 0;
  rotate: -45deg;
  background: currentColor;
}
/* 先端の点は ::after で補完可 */
```

#### person マーカー CSS（概要）

```css
.marker_person {
  font-size: 0;
  display: inline-block;
  width: 12px; height: 16px;
  position: relative;
}
.marker_person::before {
  /* 頭：小さな円 */
  content: '';
  position: absolute;
  top: 0; left: 50%;
  transform: translateX(-50%);
  width: 6px; height: 6px;
  border-radius: 50%;
  background: currentColor;
}
.marker_person::after {
  /* 胴体：台形 */
  content: '';
  position: absolute;
  bottom: 0; left: 50%;
  transform: translateX(-50%);
  width: 10px; height: 9px;
  border-radius: 3px 3px 0 0;
  background: currentColor;
}
```

## 5. 管理者設定 UI

**配置**: [GeneralTab.tsx](src/components/SpaceSettingsScreen/GeneralTab.tsx) —「吹き出しの形」セクションと同じ `.shapeList` / `.shapeOption` UI を流用。

### 5.1 UI 構成

```
星モードのマーカー形状
  [★ 星]  [● まる]  [📍 ピン]  [人 人型]
   ↑ 選択中は枠ハイライト
```

- ボタン 4 つ並べたラジオ風セレクター
- 各ボタンには実際のマーカーをプレビュー表示
- 「保存」ボタンで `saveSpaceSettings` + `upsertSpaceSettings` に含めて永続化

### 5.2 実装方針

`FeatureFlagsTab` は Supabase Feature Flags 専用のため使用しない。`GeneralTab` の既存 shape セレクターパターンに合わせる。

## 7. パフォーマンス考慮

| 懸念 | 評価 |
|------|------|
| 余分な DOM | なし（CSS `::before/::after` のみ） |
| JS 処理 | `MARKER_CHAR` マップ参照のみ（O(1)） |
| CSS 再計算 | `.marker_pin` / `.marker_person` は `.starDot` のサブクラスのため既存セレクターに干渉しない |
| アニメーション | `starPreviewSpin` 等は `starDot` 要素の `transform` に作用するため、内容が変わっても影響なし |
| バンドルサイズ増加 | ≒ 0（Unicode 文字 + 純 CSS） |

---

## 8. 受け入れ条件

| # | 条件 |
|---|------|
| AC1 | デフォルト（設定なし）では `★` が表示される（後方互換） |
| AC2 | 管理者が設定画面でマーカーを変更すると即時反映される |
| AC3 | `circle` 選択時は `●` が星の場所に表示される |
| AC4 | `pin` 選択時はピン形状（CSS 描画）が表示される |
| AC5 | `person` 選択時は人型（CSS 描画）が表示される |
| AC6 | どのマーカーでも既存アニメーション（pulse / twinkle / spin）が正常に動作する |
| AC7 | マーカー設定はページリロード後も保持される |
