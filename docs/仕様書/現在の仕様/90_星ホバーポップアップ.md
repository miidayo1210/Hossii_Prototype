# 90 星ホバーポップアップ

> **ステータス**: ✅ 実装済み（2026-06-24）

---

## 1. 概要・目的

星表示モードでマーカー（星・まる・ピン・人型）にカーソルを重ねたとき、その投稿の吹き出し（テキスト＋著者名）をマーカーの**真上**にポップアップ表示する。カーソルを外すと消える。

| ユーザーニーズ | 解決方法 |
|---------------|---------|
| マーカーの中身をすぐ確認したい | ホバーで吹き出しが出る |
| 気持ちよくポップアップしてほしい | スプリングスケールアニメ（ムクっと） |
| ガラス風の統一感を保ちたい | §84 §14.5 と同じ frosted-glass スタイル |

---

## 2. 現状との差分

| 項目 | 現状（`StarHoverPreview`） | 本仕様（改定後） |
|------|--------------------------|----------------|
| 表示条件 | `showPcStarHover`（PC 星モードのみ） | `useStarView`（PC + モバイル星モード両方） |
| 配置 | マーカーの左右（portal + `position: fixed`） | マーカーの**真上**（portal + `position: fixed`、中央揃え） |
| アニメーション | `translateX(-4px)` の横スライド 150ms | スプリングスケール＋上昇（`scale(0.7) translateY(8px)` → `scale(1) translateY(0)`） |
| スタイル | `rgba(255,252,248,0.96)` 白ベース | §84 §14.5 と同じ frosted-glass |
| テキスト量 | `line-clamp: 2` | `line-clamp: 3`（最大 80 文字切り） |

> **注意**: モバイルは hover イベントが発火しないため実質 PC のみ動作する。モバイルでタップ時のポップアップは別仕様（88 ピン留めと合わせて後回し）。

## 3. 表示仕様

### 3.1 表示トリガー

| イベント | 動作 |
|---------|------|
| `onMouseEnter` (マーカー要素) | 50ms debounce 後にポップアップ表示 |
| `onMouseLeave` (マーカー要素) | 100ms debounce 後に非表示 |

50ms / 100ms は現状と同じ値を踏襲する。

### 3.2 位置計算

ポップアップは `document.body` への portal（`position: fixed`）で実装。

```
anchorRect = マーカー要素の getBoundingClientRect()  // ※ setTimeout 前に同期取得必須（§9）
centerX    = anchorRect.left + anchorRect.width / 2
gap        = 12px

left   = clamp( centerX - popupWidth/2, 8, window.innerWidth - popupWidth - 8 )
bottom = window.innerHeight - anchorRect.top + gap  // カード下端を星の上端 + gap に合わせる
```

推定高さは使わない。`bottom` 指定によりコンテンツ量に関わらず星の真上に下端が揃う。

### 3.3 表示内容

```
┌──────────────────────────────────┐
│  テキスト本文（最大 3 行）          │
│  📸 画像（あれば先頭に表示）        │
│  ── 著者名                         │
└──────────────────────────────────┘
         ↕ gap 12px
         ★ マーカー
```

| 要素 | 仕様 |
|------|------|
| 幅 | `min(300px, 28vw)`（PC）、`min(280px, 80vw)`（小画面フォールバック） |
| テキスト | 80 文字まで切り詰め + `…`、`line-clamp: 3` |
| 著者名 | 常に表示（なければ `—`） |
| 画像 | あれば表示（`max-height: 80px`） |
| 下向きしっぽ | `::after` CSS 三角（§84 §14.5 の previewBubble と同系統） |

### 3.4 スタイル（frosted-glass）

§84 §14.5 の `.starPc .previewBubble` と同じトークンを使用する。

| プロパティ | 値 |
|-----------|-----|
| 背景 | `linear-gradient(160deg, rgba(255,255,255,0.48) 0%, rgba(235,220,255,0.36) 100%)` |
| `backdrop-filter` | `blur(18px) saturate(180%)` |
| `border` | `1px solid rgba(255,255,255,0.60)` |
| `box-shadow` | `0 8px 32px rgba(139,92,246,0.18), 0 2px 10px rgba(0,0,0,0.08), inset 0 1px 0 rgba(255,255,255,0.72)` |
| `border-radius` | `18px` |
| テキスト色 | `rgba(30,20,50,0.88)` |
| 著者名色 | `rgba(109,40,217,0.90)` |

### 3.5 アニメーション（「ムクっと」）

```css
@keyframes hoverPopIn {
  from {
    opacity: 0;
    transform: translateY(8px) scale(0.72);
  }
  to {
    opacity: 1;
    transform: translateY(0) scale(1);
  }
}
/* timing: cubic-bezier(0.34, 1.56, 0.64, 1) — spring overshoot */
/* duration: 220ms */
```

`prefers-reduced-motion: reduce` のとき → アニメーション無効（`opacity` のみ 120ms fade）。

## 4. 既存コンポーネントとの関係

### 4.1 `StarHoverPreview` の改修

既存 `StarHoverPreview` を **そのまま改修**する（新コンポーネント不要）。

変更点:

| 箇所 | 変更内容 |
|------|---------|
| 位置計算 | 左右 → **真上**（`left = centerX - maxW/2`、`bottom = innerHeight - anchorRect.top + gap`） |
| アニメーション CSS | `translateX(-4px)` → **`hoverPopIn`**（スプリング） |
| スタイル CSS | 既存白ベース → **frosted-glass**（§84 §14.5 統一） |
| しっぽ | `::after` CSS 三角追加 |
| Props | `previewSide` は不要になるため削除（位置計算が上固定になるため） |
| テキスト | `line-clamp: 2` → `line-clamp: 3`、最大文字数 80 |

### 4.2 `SpaceScreen.tsx` の変更

- `showPcStarHover` → **`useStarView`** に条件変更（モバイル星モードでも有効化）

```tsx
// 変更前
{showPcStarHover && hoveredHossiiId && hoverAnchorRect && ...}
// 変更後
{useStarView && hoveredHossiiId && hoverAnchorRect && ...}
```

- `handleStarMouseEnter` / `handleStarMouseLeave` の `if (!showPcStarHover) return;` ガードを `if (!useStarView) return;` に変更
- **`getBoundingClientRect()` は 50ms debounce の setTimeout 内では呼ばない**（React が `currentTarget` を null 化するため）。ハンドラ内で同期取得し、クロージャで渡す（§9）

## 9. 実装上の注意（バグ修正記録）

### 9.1 `currentTarget` と debounce

50ms debounce 内で `e.currentTarget.getBoundingClientRect()` を呼ぶと、React 合成イベントの `currentTarget` が既に `null` になり `hoverAnchorRect` が設定されない。

```typescript
// NG: setTimeout 内で currentTarget 参照
setTimeout(() => {
  setHoverAnchorRect(e.currentTarget.getBoundingClientRect());
}, 50);

// OK: 同期取得してから debounce
const rect = e.currentTarget.getBoundingClientRect();
setTimeout(() => {
  setHoveredHossiiId(hossiiId);
  setHoverAnchorRect(rect);
}, 50);
```

表示条件は `hoveredHossiiId && hoverAnchorRect` のため、rect 未設定だとポップアップは一切出ない。

### 9.2 位置は `bottom` 基準

推定高さ `160px` による `top` 計算は、コンテンツ量によって星から離れすぎる。実装では `position: fixed; bottom: innerHeight - anchorRect.top + gap` でカード下端を星の直上に揃える。

| AC1 | PC 星モードでマーカーにホバーすると真上に吹き出しがポップアップする |
