## 1. 概要・目的

スペース画面（`#screen`）で「このスペースは何のための場所か」を参加者に伝える、**一行の短い説明文**を表示する。

| ユーザーニーズ | 解決方法 |
|---------------|---------|
| 初めて来た人がすぐスペースの目的を知りたい | 管理者が登録した説明文を画面で確認できる |
| PC では邪魔にならない位置に常時出したい | 右上にコンパクトなガラス pill として表示 |
| 説明が邪魔なら隠したい（PC） | 左コントロールバーのトグルで表示 / 非表示を切替 |
| スマホでは画面を占有させたくない | pill 横の `(i)` ボタンを押したときだけ底部シートで表示 |

---

## 2. 既存機能との関係

本機能は以下と**用途が異なる別フィールド**。ただし管理画面の**操作感は入室セリフに準じる**。

| 既存機能 | 用途 | 本機能との違い |
|---------|------|---------------|
| `welcomeMessage`（入室セリフ） | 入室時に Hossii キャラが話す一言 | キャラのセリフ。常時表示パネルではない |
| `decorations`（掲示板装飾） | canvas 上の任意位置に置くウィジェット | 自由配置。右上固定の pill ではない |
| `space.name` | スペース名（左上 pill / TopBar） | 名称のみ。補足文は含まない |
| QR パネル | 参加用 QR（右下・ドラッグ可） | 参加導線。説明内容とは無関係 |

## 3. データ設計

### 3.1 新規フィールド `Space.description`

`src/core/types/space.ts` の `Space` 型に追加:

```typescript
export type Space = {
  // ...既存フィールド
  description?: string;  // スペースの一行説明（未設定時は説明 UI を一切出さない）
};
```

| 項目 | 仕様 |
|------|------|
| 型 | `string \| undefined` |
| 最大文字数 | **50 文字**（管理画面 input で `maxLength={50}` 制限） |
| 改行 | **不可**（1 行のみ。`<input type="text">` で入力） |
| 未設定時 | 説明 UI 全体を出さない（左バートグル・PC パネル・モバイル (i) ともに非表示） |
| 空白のみ | 未設定と同扱い（`trim()` 後が空なら `undefined` として保存） |

### 3.2 永続化（Supabase）

`spaces` テーブルにカラムを追加。

| 層 | 対応 |
|----|------|
| DB | `text NULL`（カラム名例: `description`） |
| `spacesApi.ts` | `row.description ?? undefined` ↔ `space.description` |
| `spacesApi.ts` patch | `if (patch.description !== undefined) obj.description = patch.description ?? null` |
| ローカル demo | `updateSpace` / mock データで同フィールドを保持 |

> 既存スペースは `NULL`（説明なし）。マイグレーション SQL のみ追加すればよい。

### 3.3 表示 / 非表示のユーザー設定（PC）

QR パネルと同様、**セッション内 state** で管理（初版）。

```typescript
// SpaceScreen.tsx
const [descriptionPanelVisible, setDescriptionPanelVisible] = useState(true);
```

| 項目 | 仕様 |
|------|------|
| 初期値 | `true`（ページ読込時は表示） |
| 永続化 | 初版は **なし**（QR `qrPanelVisible` と同じ。リロードで再表示） |
| 将来拡張 | `localStorage['hossii.descriptionPanelVisible']` に保存する形に移行可 |

## 4. 管理画面（GeneralTab）

### 4.1 配置

`GeneralTab`（スペース設定 → 一般）の **ウェルカムメッセージセクションの直後** に追加する。

```
[ ウェルカムメッセージ セクション ] ← 既存
   入室時にHossiiキャラが話すメッセージ…
   [textarea 3行]
   xx / 100

[ スペース説明 セクション ]          ← 新規（ここに追加）
   スペース画面に表示する、このスペースの目的や使い方の一言説明です。
   [input type="text"                   ]
   xx / 50

[ 星モードのマーカー形状 セクション ] ← 既存（これ以降は変わらない）
```

### 4.2 入力 UI

| 要素 | 仕様 |
|------|------|
| タグ | `<input type="text">` |
| `maxLength` | `50` |
| プレースホルダ | `例: チームの日々の気持ちを共有するスペースです。` |
| 文字数カウント | `{length} / 50`（`welcomeMessage` の `.charCount` スタイルと同じ） |
| 保存タイミング | `onChange` ごとに即時 `onUpdateSpace({ description: e.target.value || undefined })` — 他項目と同じ |
| 消去 | テキストを全削除して保存 → `undefined` 扱いとなり説明 UI 全消え |

> `onUpdateSpace` が提供されない（閲覧専用参加者）場合は `GeneralTab` ごと非表示のため、このセクションは実質管理者専用。

### 4.3 管理画面での見せ方（ライブプレビュー、任意拡張）

初版スコープ外。将来的に管理画面のプレビュー領域に説明文の表示イメージを添えてもよい。

## 5. PC / 大画面の表示仕様

**対象:** `min-width: 769px`（`.spaceTitle` 非表示 breakpoint と同一）

### 5.1 説明 pill（右上固定）

`space.description` が非空かつ `descriptionPanelVisible === true` のとき表示。

```
[ TopBar ────────────────────────────────── ]

                [書き出し][#xx][☆ 星][🏷 タグ]  ← 既存 spaceTopRightCluster（top: 4rem + 10px）
                ╔══════════════════════════════╗
                ║ ℹ チームの気持ちを共有する場所 ║  ← 新規 SpaceDescriptionPanel pill
                ╚══════════════════════════════╝

[ 左バー ]                         [ 投稿エリア ... ]
                                   [ QR（右下）      ]
```

**位置・サイズ**

| プロパティ | 値 |
|-----------|-----|
| `position` | `fixed` |
| `top` | `calc(4rem + 56px)`（TopBar 高さ + spaceTopRightCluster 分の余白。クラスタが増えたら調整可） |
| `right` | `12px` |
| `max-width` | `min(360px, 32vw)` |
| `z-index` | `95`（`spaceTopRightCluster` と同水準） |
| `pointer-events` | `none`（クリック不要・下のコンテンツをブロックしない） |

> `spaceTopRightCluster` の高さが変動するケースは多くないため、初版は CSS 固定 offset で十分。将来的に JS で動的 offset 計算してもよい。

**スタイル — 既存ガラス pill 統一**

| プロパティ | 値 |
|-----------|-----|
| 背景 | `rgba(255, 255, 255, 0.18)` |
| `backdrop-filter` | `blur(12px)` |
| `-webkit-backdrop-filter` | `blur(12px)` |
| 枠線 | `1px solid rgba(255, 255, 255, 0.30)` |
| 角丸 | `9999px`（完全 pill）または `10px`（小判型。`.spaceTitle` と統一） |
| 影 | `0 2px 12px rgba(0, 0, 0, 0.12)` |
| padding | `6px 14px 6px 10px` |
| `display` | `inline-flex`, `align-items: center`, `gap: 8px` |

**テキストコントラスト対策**

背景が白系（明るいスペース）でも読めるよう:
```css
color: rgba(255, 255, 255, 0.95);
text-shadow: 0 1px 4px rgba(0, 0, 0, 0.5);
```
- `text-shadow` で暗い背景 / 明るい背景の両方でコントラストを確保する。
- 将来的に背景色に応じた自動ダーク化（`color-mix` など）に移行してもよい。

**内部構成**

```
[ ℹ アイコン（14px） ]  [ 説明テキスト（0.8125rem） ]
```

| 要素 | 仕様 |
|------|------|
| アイコン | Lucide `Info` size 14、`flex-shrink: 0` |
| テキスト | `space.description`（最大 50 文字）、`white-space: nowrap`、`overflow: hidden`、`text-overflow: ellipsis` |
| フォントサイズ | `0.8125rem`（`13px`）— `.spaceTitle` と同じ |
| フォントウェイト | `500`（やや太め。スペース名の `700` より控えめ） |

> 50 文字上限のためスクロールは不要。幅が狭い場合（`32vw` 制限）は末尾 `…` で切る。

### 5.2 表示 / 非表示アニメーション

左バーのトグルでパネルを切り替える際、唐突に消えないよう `opacity` + `scale` でフェードアウト / インする。

```css
/* SpaceDescriptionPanel.module.css */
.panel {
  transition: opacity 0.18s ease, transform 0.18s ease;
  transform-origin: top right;
}
.panelHidden {
  opacity: 0;
  transform: scale(0.94);
  pointer-events: none;
}
```

React 側は `descriptionPanelVisible` に応じて `.panelHidden` を付け外す（DOM から外すのではなく class 切替）。または条件付きレンダリングのまま `React.memo` で最適化してもよい（初版は単純な条件分岐で可）。

### 5.3 表示しない条件

| 条件 | 理由 |
|------|------|
| `space.description` が空 / 未設定 | 表示するものがない |
| `descriptionPanelVisible === false` | ユーザーが隠した |
| `viewMode === 'slideshow'` | QR と同様、没入表示優先 |
| `isMobile === true` | モバイルは §6 の (i) ボタン方式を使う |
| `isMobileLandscape === true` | スマホ横の狭い領域には出さない（初版。将来再検討可） |
| スペース書き出し（PNG）中 | `data-space-export="exclude"` で除外 |

## 6. 左コントロールバー — 表示 / 非表示トグル（PC のみ）

[QR トグル](./50_QRの表示.md/非表示切り替え.md) と**同系のボタン**を QR トグルの直上に追加する。

### 6.1 ボタン仕様

| 項目 | 仕様 |
|------|------|
| 表示条件 | `space.description` が非空 **かつ** `!isMobile`（`onDescriptionToggle` が渡された場合のみ描画） |
| アイコン | Lucide `Info`（size 18）|
| active 時 | `descriptionPanelVisible === true` のとき `.active` クラス（QR と同じ） |
| ツールチップ | 表示中: `スペース説明を隠す` / 非表示: `スペース説明を表示` |
| `aria-label` | 上記と同文 |
| `aria-pressed` | `descriptionPanelVisible` を反映 |
| className | `${styles.controlButton} ${descriptionPanelVisible ? styles.active : ''}` |

### 6.2 LeftControlBar Props 追加

```typescript
// LeftControlBar.tsx の Props 型に追加
descriptionPanelVisible?: boolean;
onDescriptionToggle?: () => void;
```

### 6.3 SpaceScreen からの配線

```typescript
// SpaceScreen.tsx
onDescriptionToggle={
  hasDescription && !isMobile
    ? () => setDescriptionPanelVisible(v => !v)
    : undefined
}
descriptionPanelVisible={descriptionPanelVisible}
```

`onDescriptionToggle` が `undefined` の場合、`LeftControlBar` 内でボタンを描画しない（QR トグルと同一パターン）。

### 6.4 左バー内の配置順

QR トグルの**直上**（`onShowPostCountBadgeToggle` と `onQrToggle` の間のあたり）:

```
…
[ # 投稿数トグル ]
[ ℹ スペース説明トグル ]  ← 追加
[ QR トグル ]
[ 🏝 隣スペース ]
---（区切り線）
[ ··· オーバーフロー ]
```

## 7. スマホの表示仕様

**対象:** `max-width: 768px`

### 7.1 方針

常時表示はせず、スペース名 pill の**右隣の `(i)` ボタン**を押したときだけ開く。
画面の最小面積で「開けば読める」体験を提供する。

### 7.2 `(i)` トリガーボタン

| 項目 | 仕様 |
|------|------|
| DOM 位置 | `.spaceTitle` pill と同じ `fixed` 親コンテナ内、または pill の`right`側に隣接 |
| 実装方針 | `.spaceTitle` のラッパー `<div>` に pill + `(i)` ボタンをまとめる |
| 表示条件 | `space.description` が非空 かつ `isMobile` |
| サイズ | `32px × 32px`（safe-area-inset も考慮） |
| アイコン | Lucide `Info`（size 16）または文字 `ⓘ` |
| スタイル | pill と**同系のガラスボタン**（`background: rgba(255,255,255,0.18)`, `backdrop-filter: blur(12px)`, `border: 1px solid rgba(255,255,255,0.30)`, `border-radius: 9999px`） |
| タップ領域 | `min-width: 32px; min-height: 32px; padding: 0;`（最低 44px のタップエリアを `padding` で確保してもよい） |
| 左バー / `...` メニュー | **出さない**（QR と同じ方針。pill 横が最も発見しやすい） |

```
┌──────────────────────────────────────┐
│  ●  わたしのスペース  [ℹ]            │  ← 左上: pill + (i)
│                                      │
│         ★  ★   ★                    │
│    ★          ★                     │
└──────────────────────────────────────┘
```

- `(i)` は pill と `gap: 6px` 程度の間隔で横並び。
- pill 最大幅 `min(220px, 55vw)` はそのまま維持。`(i)` ボタン分は pill 幅を削らない（親コンテナが横に伸びる）。

### 7.3 説明ボトムシート

| 項目 | 仕様 |
|------|------|
| 形式 | ボトムシート（`position: fixed`, `bottom: 0`, `left: 0`, `right: 0`） |
| portal | `document.body` に `createPortal`（z-index の影響を受けないようにする） |
| 開き方 | `(i)` ボタンタップ |
| 閉じ方 | ① シート外タップ（backdrop クリック） ② ヘッダー「閉じる」 ③ 上から下へスワイプ（初版は実装省略可） |
| 最大高 | `min(200px, 40dvh)`（説明は 50 文字以内のため高さは不要。コンパクトに収める） |
| 角丸 | `border-radius: 16px 16px 0 0` |
| 背景 | `rgba(30, 20, 50, 0.92)` + `backdrop-filter: blur(24px)`（他モーダルと統一） |
| アニメーション | `transform: translateY(100%)` → `translateY(0)` 200ms ease |

**シート内部構成**

```
╔══════════════════════════════╗
║  このスペースについて   [閉じる] ║  ← ヘッダー行
║ ─────────────────────────── ║
║  チームの日々の気持ちを         ║
║  共有するスペースです。         ║  ← 説明本文（white-space: pre-wrap）
╚══════════════════════════════╝
```

| 要素 | 仕様 |
|------|------|
| ヘッダーラベル | `このスペースについて`（固定） |
| 閉じるボタン | 右端に `×` または `閉じる` テキストボタン |
| 説明本文 | `space.description` をそのまま表示。`font-size: 0.9375rem`（15px）、`line-height: 1.6` |
| スペース名 | 初版は表示しなくてよい（説明文だけで完結する） |
| padding | `16px 20px` |
| `role` | `"dialog"` |
| `aria-modal` | `true` |
| `aria-labelledby` | ヘッダーラベルの `id` |

### 7.4 スマホ横持ち

左上 pill + `(i)` ボタンは縦持ちと同じ位置に表示。シートは同じボトムシート形式。横持ち時は高さが窮屈なため `max-height: 40dvh` が自然な制約になる。

## 10. SpaceScreen 配線（概要）

```typescript
// state
const [descriptionPanelVisible, setDescriptionPanelVisible] = useState(true);
const [descriptionSheetOpen, setDescriptionSheetOpen] = useState(false);

// derived
const spaceDescription = spaceForVisual?.description?.trim() ?? '';
const hasDescription = spaceDescription.length > 0;

// PC パネル
{hasDescription && descriptionPanelVisible && !isMobile && !isMobileLandscape && viewMode !== 'slideshow' && (
  <SpaceDescriptionPanel description={spaceDescription} data-space-export="exclude" />
)}

// モバイル (i) ボタン + シート
{hasDescription && isMobile && (
  <>
    {/* pill ラッパーに (i) を追加 — spaceTitle 近傍 */}
    <button
      className={styles.descriptionInfoButton}
      onClick={() => setDescriptionSheetOpen(true)}
      aria-label="スペースの説明を表示"
      data-space-export="exclude"
    >
      <Info size={16} />
    </button>
    <SpaceDescriptionSheet
      open={descriptionSheetOpen}
      onClose={() => setDescriptionSheetOpen(false)}
      description={spaceDescription}
    />
  </>
)}

// LeftControlBar に渡す props
descriptionPanelVisible={descriptionPanelVisible}
onDescriptionToggle={hasDescription && !isMobile ? () => setDescriptionPanelVisible(v => !v) : undefined}
```
