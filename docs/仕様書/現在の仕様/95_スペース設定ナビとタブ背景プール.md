> **分類:** `[Core / Space / Navigation / Admin]`
> **機能名:** スペース設定ショートカット、タブ別背景画像プール、**タブ背景ボード**

## 1. 概要

本機能は次の **3 本柱** で構成する。

| # | 機能 | 概要 |
|---|------|------|
| A | **スペース設定ナビ** | グローバルナビに管理者専用の設定ボタンを追加し、現在のスペースの `#settings` へ直行する |
| B | **タブ背景プール** | スペース共通の背景画像 **最大 3 枚** をプールとして保持。追加タブはプール内 URL のみ割り当て可能 |
| C | **タブ背景ボード** | 背景タブ内で **全タブを一覧表示**し、**ボタン 1 タップ**で「このタブはどの背景か」を切り替え。一括適用も同画面から行う |

**ユーザー体験のゴール:**

```text
タブが 3 つある
→ 背景タブを開く
→ プールのサムネイルを見ながら
→ 「今日の問い」行の [画像2] ボタンを 1 回押す
→ そのタブだけ背景が変わる
→ 必要なら [すべて画像1に] を 1 回押して一括適用
```

**非目的（初期スコープ外）:**

- 追加タブごとに色・パターン・テーマを個別設定する
- 追加タブごとに画像を個別アップロードする
- プール枚数を 3 枚超に拡張する UI（定数変更のみで将来拡張可）
- タブ背景ボード以外の画面（スペース画面内フローティング等）からの背景割り当て

## 2. 背景と課題

### 2.1 現状（2026-06-29 時点）

| 項目 | 現状 |
|------|------|
| 設定画面への入口 | 主に **スペース管理一覧**（`SpacesScreen`）のカード操作。スペース画面 `#screen` 内の設定ショートカットは **未実装**（§3） |
| 共通画像プール | `spaces.saved_background_images` に最大 3 枚。メインタブの `BackgroundTab` / 管理一覧の `BackgroundSelector` でアップロード・ギャラリー表示 |
| 追加タブの背景 | Phase 7 により `space_panes.background` / `saved_background_images` で **フル override** 可能。Pane セレクタ（`<select>`）でタブを切り替えて **1 タブずつ** 編集する UI |
| ギャラリー追加 | 連続 `setDraft` の競合により新規画像がギャラリーに載らない不具合があった → **`onImageUploaded` による原子的更新で修正済み**（実装メモ） |

### 2.2 解決したいこと

1. 管理者がスペースを見ながらすぐ設定を開きたい（§3）
2. 追加タブの背景差し替えは **軽量** にしたい — 共通 3 枚から選ぶだけ（§4）
3. 運用ミス（プール画像削除）でも壊れた URL のまま表示したくない（§4.4）
4. **タブが増えたとき、背景の割り当てを直感的にしたい** — ドロップダウンでタブを切り替えて個別保存する操作は重い（§5）
5. **「全部同じ背景にしたい」「1 タブだけ変えたい」** を、ボタン 1〜2 回で完結させたい（§5）

## 3. スペース設定ナビ

（変更なし — 初版仕様を維持）

### 3.1 表示条件

以下 **すべて** を満たすときのみ表示する。

| 条件 | 説明 |
|------|------|
| ログイン済み管理者 | `currentUser.isAdmin === true`（コミュニティ管理者 **および** スーパー管理者 — `isSuperAdmin` も `isAdmin: true`） |
| 非ゲスト | ゲストモード・未ログインでは非表示 |
| 非訪問モード | `visitingSpaceId !== null` の隣人スペース閲覧中は非表示 |
| アクティブスペースあり | `activeSpaceId` が解決されている |

一般参加者・ゲストには **一切表示しない**。

### 3.2 配置

**PC（769px 以上）— `TopRightMenu`**

```text
[ スペース ] [ 投稿する ] [ ログ一覧 ] [ アカウント ] [ スペース設定 ]
```

**スマートフォン（768px 以下）— `BottomNavBar`**

5 番目のタブ「**設定**」（歯車アイコン）→ `#settings`

### 3.3 動作

1. タップ / クリックで `navigate('settings')`
2. **`activeSpaceId` は変更しない**
3. スーパー管理者の override 表示中も **そのスペース** の設定を開く
4. 戻るは既存 `SpaceSettingsScreen` に従う（`isAdmin ? #spaces : #screen`）

### 3.4 既存入口との関係

スペース管理一覧のカード入口は **維持・併存**。

## 4. タブ背景プール — データモデル

### 4.1 基本方針

```text
spaces.saved_background_images  … スペース共通の画像プール（最大 3 枚）
spaces.background               … メイン（デフォルト）タブの背景正本
space_panes.background          … 追加タブのみ。NULL = メインに従う
```

**追加タブに `space_panes.saved_background_images` 列は使わない**（画像に限り pane 別ギャラリー override は廃止方向）。

### 4.2 メインタブ

| 項目 | 仕様 |
|------|------|
| 正本 | `spaces.background` / `spaces.saved_background_images` |
| 背景種別 | 色・パターン・テーマ・画像 — **制限なし**（[24](./24_スペース管理_各スペースの設定_背景設定.md) 維持） |
| プール | 画像タブのギャラリー = 共通プール 3 枚 |

### 4.3 追加タブ

| 選択（UI） | 保存値 | 表示 |
|------------|--------|------|
| メインと同じ | `NULL` | `resolvePaneBackground` → メインと同一 |
| プール画像 N | `{ kind: 'image', value: プール[N], source: 'cloud' }` | その画像 |
| （空スロット） | 選択不可 | グレーアウト |

### 4.4 解決ロジック（表示時）

`resolvePaneBackground(pane, space)` — 初版 §4.4 と同じ。

- プール外 URL・削除済み URL → **メインタブの現在の背景** にフォールバック
- 削除済み画像そのものは **表示しない**

### 4.5 プール画像の削除

Storage 削除 + 推奨: 当該 URL を指す `space_panes.background` を **バッチ NULL 正規化**。

## 5. 背景タブ UI — タブ背景ボード（新規）

### 5.1 設計方針

| 原則 | 内容 |
|------|------|
| **一覧性** | 全タブの現在背景を **1 画面** で見える |
| **1 タップ割り当て** | タブ行のボタン 1 回で割り当て完了（保存バーは画面下部 1 つ） |
| **一括操作** | よくある操作（全タブメイン合わせ / 全タブ同じ画像）は **上部クイックアクション** |
| **メインは特別** | 色・パターン・テーマ・プール管理は **メインタブ行** のみフル編集 |
| **追加タブは軽量** | 追加タブ行は **メイン / プール画像ボタンのみ** |

**廃止方向:** 背景タブで `SettingsPaneSelector` の `<select>` だけに依存し、タブを切り替えないと他タブの背景が見えない UX。

**移行:** Pane セレクタは **投稿フォーム・装飾など他設定タブ** では維持。背景タブでは **タブ背景ボードが主 UI** とし、セレクタは非表示。

### 5.2 画面構成（ワイヤー）

```text
┌─ 背景 ─────────────────────────────────────────────────────┐
│ スペースの背景と、タブごとの背景割り当てを設定します。          │
├─ 共通プール（最大 3 枚）─────────────────────────────────────┤
│ [thumb1✓] [thumb2] [thumb3] [＋追加]     2 / 3 枚            │
│ ※ 画像の追加・削除はここ。メインタブの「画像」と同一プール。    │
├─ クイック操作 ────────────────────────────────────────────────┤
│ [ すべてメインに合わせる ]  [ すべて画像1 ] [ すべて画像2 ] …  │
│   （空スロットのボタンは disabled）                           │
├─ タブごとの背景 ──────────────────────────────────────────────┤
│ ┌─ メイン（はじめに）──────────────────────────────────────┐ │
│ │ 現在: パターン「霧」  [プレビュー]                          │ │
│ │ [ 色 ] [ パターン ] [ テーマ ] [ 画像 ]  ← フル BackgroundSelector │
│ └──────────────────────────────────────────────────────────┘ │
│ ┌─ 今日の問い ─────────────────────────────────────────────┐ │
│ │ 現在: 画像2  [mini preview]                               │ │
│ │ ( ● メイン ) ( ○ 画像1 ) ( ● 画像2 ) ( ○ 画像3 )          │ │
│ └──────────────────────────────────────────────────────────┘ │
│ ┌─ アイデア ────────────────────────────────────────────────┐ │
│ │ 現在: メインと同じ  [mini preview]                          │ │
│ │ ( ● メイン ) ( ○ 画像1 ) ( ○ 画像2 ) ( ○ 画像3 )          │ │
│ └──────────────────────────────────────────────────────────┘ │
├─ 保存バー ───────────────────────────────────────────────────┤
│ [ 破棄 ]                                    [ 保存 ]          │
└──────────────────────────────────────────────────────────────┘
```

### 5.3 操作定義

#### 5.3.1 追加タブ行 — セグメントボタン（1 タップ割り当て）

各追加タブ行に **排他ボタン群** を表示する。

| ボタン | ラベル | 押下時の draft 更新 |
|--------|--------|---------------------|
| `main` | **メイン** | 当該 pane の `background` → `NULL`（未 override） |
| `pool-0` | **画像1** またはサムネイル | `background` → プール[0] の image JSON |
| `pool-1` | **画像2** | プール[1] |
| `pool-2` | **画像3** | プール[2] |

| ルール | 内容 |
|--------|------|
| 視覚 | 選択中ボタンは filled / チェック。非選択は outline |
| プレビュー | 行左または右に **解決後背景** のミニプレビュー（draft から `resolvePaneBackground` 相当を計算） |
| 即時反映 | ボタン押下で **draft のみ** 更新。DB 保存は「保存」ボタンまで遅延 |
| 空スロット | プールに URL が無いスロットのボタンは **disabled** + ツールチップ「メインタブの背景設定で画像を追加」 |
| タップ数 | **1 回** で割り当て完了（確認ダイアログなし） |

#### 5.3.2 クイック操作（一括）

画面上部（プールの直下）に **横並びボタン** を配置する。

| ボタン | 対象 | 動作 |
|--------|------|------|
| **すべてメインに合わせる** | **追加タブすべて** | 各追加 pane の `background` → `NULL` |
| **すべて画像1に** | 追加タブすべて | 各追加 pane → プール[0]（存在時のみ active） |
| **すべて画像2に** | 同上 | プール[1] |
| **すべて画像3に** | 同上 | プール[2] |

| ルール | 内容 |
|--------|------|
| メインタブ | 一括操作の **対象外**（メインの色/パターン/テーマは変更しない） |
| 確認 | 初版は **確認なし**（保存前 draft 変更のみ。破棄で戻せる） |
| 無効 | プール空スロットに対応する一括ボタンは disabled |

**将来拡張（スコープ外）:** タブ行にチェックボックスを付け「選択タブのみに適用」。初版は **全追加タブ一括** のみ。

#### 5.3.3 メインタブ行 — フル背景編集

| 項目 | 仕様 |
|------|------|
| 位置 | タブ背景ボードの **先頭行**（`pane.isDefault === true`） |
| UI | 現行 `BackgroundSelector` 全文（色・パターン・テーマ・画像ギャラリー） |
| プール | ギャラリー = §4.1 の共通プール。アップロード・削除は **ここでのみ** |
| 他タブへの影響 | メインの **色/パターン/テーマ** 変更は、「メインと同じ」設定の追加タブの **見た目** に即反映（preview） |
| 他タブへの影響 | メインが **画像** を選び直した場合、追加タブが **特定プール URL を指している限り** そのタブの見た目は変わらない |

#### 5.3.4 プールサムネイルとタブ行の関係

| 操作 | 仕様 |
|------|------|
| プール上部サムネイルクリック | **メインタブ** の背景画像として選択（現行 `BackgroundSelector` と同じ） |
| 追加タブ行のボタン | 当該 **追加タブのみ** に割り当て |
| プール `＋追加` | メインのプールに 1 枚追加。追加後も **保存** まで DB 未反映 |

**意図:** プール管理（アップロード）と、タブへの **割り当て** を視覚的に分離しつつ、同一画面に集約する。

### 5.4 保存モデル

背景タブの draft は **スペース全体 + 全 pane 分** を 1 つの dirty 状態として扱う。

```typescript
type BackgroundBoardDraft = {
  /** メイン（spaces 正本） */
  main: {
    background?: SpaceBackground;
    savedBackgroundImages?: string[];
  };
  /** 追加 pane の override（paneId → background override or null） */
  paneOverrides: Record<SpacePaneId, SpaceBackground | null>;
};
```

| 操作 | 保存先 | API |
|------|--------|-----|
| メインの background / プール | `spaces` | `updateSpaceInDb` + `onUpdateSpace` |
| 追加タブ各 `background` | `space_panes.background` | pane ごと `updateSpacePane` または **batch 保存（推奨）** |
| 一括保存 | 上記を **1 回の保存ボタン** で実行 | 部分成功時はエラー toast + 再試行可能に |

**保存成功後:** draft を確定し、`SpacePaneProvider` を reload。スペース画面のタブ切替で背景が整合すること。

### 5.5 タブが 1 つのみのスペース

| 条件 | UI |
|------|-----|
| `visiblePanes.length === 1` | **タブ背景ボードの追加タブ行・クイック操作は非表示**。メインのフル `BackgroundSelector` のみ |

### 5.6 モバイル

| 項目 | 仕様 |
|------|------|
| タブ行 | 縦積み。ボタン群は **横スクロール** または 2 行折り返し |
| クイック操作 | 横スクロール 1 行 |
| プレビュー | 高さ固定の mini preview（16:9 クロップ） |

### 5.7 アクセシビリティ

| 項目 | 仕様 |
|------|------|
| ボタン | `aria-pressed` で選択状態を明示 |
| ラベル | `aria-label="{タブ名}の背景: 画像2"` |
| キーボード | セグメントボタンは roving tabindex |

## 6. 設定画面 UI — 旧構成からの変更

### 6.1 廃止する UX（背景タブに限る）

| 旧 UX | 新 UX |
|-------|-------|

|------|----------------|
| 背景 | **非表示**（ボードが代替） |

## 9. 移行・互換

### 9.1 既存スペース

- `spaces.saved_background_images` / `spaces.background` — 変更なし
- 追加タブで `background IS NULL` — 従来どおりメイン継承

### 9.2 既存の pane 別フル override データ

| データ | 移行 |
|------|------|
| `background.value` が現プールに含まれる | そのまま有効。ボード上の対応ボタンが selected |
| プール外 URL / pane 別ギャラリー | resolver でメインへフォールバック。ボード上は「メイン」表示 |
| `saved_background_images` pane 列 | 読取無視 |

### 9.3 実装済みコードとの関係

| 項目 | 方針 |
|------|------|
| `BackgroundTab` + `useScreenDraft` | ボード用 draft 構造へ拡張 |
| `onImageUploaded` | メイン行のプール追加で **引き続き使用** |
| `savePaneBackgroundOverride` | pane 単位 → **board 一括保存** にラップ |

### 9.4 [93](./93_Hossiiタブ機能追加.md) との差分

| 93 現行 | 本仕様後 |
|---------|----------|
| 追加 Pane で background + savedBackgroundImages フル override | 画像は **プール選択のみ** |
| 背景タブで Pane セレクタ切替 | 背景タブは **タブ背景ボード** |

- [ ] タブ 1 つのスペースではボード省略 UI
