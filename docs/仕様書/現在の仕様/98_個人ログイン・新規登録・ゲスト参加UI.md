# 0. 目的

Phase 1 で整備した **個人認証 API・Identity 基盤**（[97](./97_個人アカウント・所属・個人データ基盤.md)）を、利用者が実際に操作できる **UI と導線** として完成させる。

本仕様書は、次を定義する。

* トップ画面（`StartScreen`）からの認証導線
* ゲスト入室画面（`GuestEntryScreen`）からの認証導線
* 非公開スペース（`PrivateSpaceScreen`）からの認証導線
* pathname ベースのログイン・新規登録画面（既存 `LoginScreen`）
* 認証成功後の戻り先
* エラー・状態表示
* スマホ表示

**新規 LoginScreen の作り直しは行わない。** 既存 `LoginScreen` と `AuthEntryShell` を再利用する。

# 5. StartScreen（トップ入口）

## 5.1 表示条件

`App.tsx` において、**未認証**かつ**ゲストモードでない**かつ**認証ルート（`/login` / `/signup`）でない**ときに表示する（[65](./65_トップページ.md) の表示条件を本書で拡張）。

## 5.2 画面構成

スペース参加 UI を主役として維持する。

```text
✨ Hossii

気持ちを、つながる空間に置いていこう

スペースのID、または共有URLを入力して参加できます。

スペースID または URL
[ スペースに参加 ]

──────── または ────────

[ アカウントでログイン ]

はじめての方はこちら
新規登録
```

## 5.3 スペース参加（既存維持）

* 入力値を `parseSpaceSlugFromInput` → `validateSpaceURL` で検証
* 成功時は `window.location.assign(`${origin}/s/${slug}`)` でスペース URL へ遷移（[65](./65_トップページ.md) と同様。`pushState` のみでは slug 解決 effect が再実行されないためフル遷移を維持）

## 5.4 認証導線

| 操作 | 遷移 |
|------|------|
| アカウントでログイン | `/login` |
| 新規登録 | `/signup` |

pathname 遷移により `LoginScreen` を表示する。トップ上にフォームを埋め込まない。

## 5.5 削除する表示

次を **表示しない**。

```text
ログインしてスペースを作る
Coming soon
```

**削除理由:**

* 個人ユーザーには現状スペース作成権限がない（[05](./05_管理者ログイン画面.md) — 管理者のみ作成可）
* スペース作成を約束する文言は誤解を招く
* 個人認証機能は Coming soon ではない（Phase 1 で API 有効化済み）

StartScreen では **「スペースを作る」という文言を使用しない**。

# 6. GuestEntryScreen

## 6.1 表示条件

`/s/{slug}` または `/c/{community}/s/{slug}` に未ログインでアクセスし、当該スペースのニックネーム未設定かつゲストモード前のとき（[16](./16_ゲスト入室フロー.md)）。

## 6.2 画面構成

```text
スペースへ参加

ニックネーム
[ ゲストとして参加 ]

──────── または ────────

[ アカウントでログイン ]

はじめての方はこちら
新規登録
```

（実装上は既存の 2 ステップ UI（選択 → ニックネーム入力）を維持してよい。上記は概念上の主要アクション一覧。）

## 6.3 ゲスト参加

* 個人登録を **強制しない**
* 現在の `profile.id`、ニックネーム、`isGuestMode` の挙動を **維持**する（[97](./97_個人アカウント・所属・個人データ基盤.md) §19.1）
* ゲスト参加を **最初の主要アクション** として表示する

## 6.4 認証導線

| 操作 | 処理 |
|------|------|
| アカウントでログイン | return path（`pendingReturnPath` / `pendingLoginSlug`）を保存 → `/login` |
| 新規登録 | 同上 → `/signup` |

認証成功後は **元のスペース** へ戻る（§13）。

## 6.5 解消すべき不具合

認証開始後、`pendingLoginSlug` がセットされても `GuestEntryScreen` が再表示され続ける問題を解消する（§12）。

# 10. URL と pathname

## 10.1 正式 URL（pathname）

| pathname | 画面 |
|----------|------|
| `/login` | `LoginScreen`（login モード） |
| `/signup` | `LoginScreen`（signup モード） |
| `/admin/login` | `AdminLoginScreen`（変更なし） |

アプリ内画面遷移（ログイン後）は引き続き Hash（例: `#account`、`#mylogs`）を使用する（[06_ナビゲーション・UI構成](./06_ナビゲーション・UI構成.md)）。

## 10.2 pathname 正規化

次を **同一ルート** として扱う。

```text
/login   ≡ /login/
/signup  ≡ /signup/
```

query や hash が付いていても **pathname を正しく判定**する。

例: `/login?foo=1#bar` → login ルートとして扱う（hash の `#bar` はアプリ hash ルーターと独立）。

## 10.3 appRoute 同期

`appRoute`（または同等の pathname 派生 state）を **初回マウント時だけ** で固定しない。

最低限、次を同期対象とする。

* `history.pushState`
* `history.replaceState`
* ブラウザ戻る・進む（`popstate`）

新しい Router ライブラリは導入しない。既存 `App.tsx` 構造に合わせ、**最小限の route helper または state 同期**を利用する。

# 11. App.tsx の表示優先順位

未ログイン時、概念上の評価順序は次とする。

```text
1. /admin/login          → AdminLoginScreen
2. /login または /signup → LoginScreen
3. pendingLoginSlug / pendingAuthMode による LoginScreen 要求
4. PrivateSpaceScreen
5. GuestEntryScreen
6. StartScreen または通常画面（ログイン後 UI）
```

## 11.1 解消すべき不具合（必須）

```text
GuestEntryScreen
  ↓ アカウントでログイン
pendingLoginSlug がセット
  ↓
GuestEntryScreen が再表示され続ける   ← 現状バグ
```

**仕様:** 認証開始後は `GuestEntryScreen` ではなく **`LoginScreen` を表示**する。

実装上は、`pendingLoginSlug` / pathname 認証ルートによる `LoginScreen` 表示を **`guestSpaceId` による `GuestEntryScreen` より先**に評価する。

# 12. 認証成功後の遷移

## 12.1 優先順位

```text
1. pendingReturnPath
2. pendingLoginSlug から生成した space path（`/s/{slug}`）
3. #account
```

## 12.2 pendingReturnPath

**pathname 全体**を保持する。community slug を失ってはならない。

例:

```text
/s/example
/c/community-a/s/example
```

## 12.3 pendingLoginSlug

過去コードとの **互換用** として維持する。`pendingReturnPath` が無い場合の fallback として `/s/{slug}` を生成する。

## 12.4 戻り先なし

`/login` または `/signup` から直接認証し、return path が無い場合は **`#account`** へ遷移する（既存 hash 処理と一致）。

---

# 13. OAuth をまたぐ戻り先

OAuth は画面リロード・外部 redirect を伴うため、React state だけでは戻り先が失われる可能性がある。

## 13.1 方針

実装前に **現在の保存方法**（`App.tsx` の state、`localStorage` 等）を確認し、既存の永続化処理があれば **それを優先して再利用**する。

不足する場合のみ、次を **sessionStorage** へ一時保存する最小構成を採用する。

```text
pendingReturnPath
pendingAuthMode
```

## 13.2 sessionStorage 利用時の条件

* **localStorage へ恒久保存しない**
* 認証成功後に **削除**する
* 戻る・キャンセル時に不要な値を **残さない**
* 古い戻り先を **次回ログインで利用しない**

---

# 14. ログイン済みユーザー

ログイン済みユーザーが `/login` または `/signup` を開いた場合:

| 条件 | 挙動 |
|------|------|
| `pendingReturnPath` あり | `pendingReturnPath` へ移動 |
| `pendingReturnPath` なし | `#account` へ移動 |

**ログインフォームを再表示しない。**

管理者アカウントも個人ユーザーとして **同じ挙動**を利用する。無条件に管理画面（`#spaces` / `#communities`）へ転送 **しない**。

---

# 15. 戻る操作

`LoginScreen`（login / signup 両モード）に **戻る操作**を用意する。

## 15.1 優先順位

```text
1. 認証開始元がある → 元の画面へ戻る
2. ブラウザ履歴に利用可能な画面がある → history.back()
3. それ以外 → /
```

## 15.2 戻る際に消去する state

* 一時的な `pendingAuthMode`
* 不要になった `pendingReturnPath`（キャンセル時）
* 送信中 state
* エラー state

sessionStorage に保存している場合も同様にクリアする。

## 15.3 Private Space キャンセル

Private Space の認証をキャンセルした場合、**元の `PrivateSpaceScreen` へ戻って構わない**。

# 16. エラー・状態表示

## 16.1 仕様化する状態

| 状態 | 説明 |
|------|------|
| 未入力 | 初期表示 |
| 入力中 | フィールド編集可能 |
| 送信中 | ボタン disabled、二重送信禁止 |
| 認証成功 | `App.tsx` が遷移処理を担当 |
| 登録成功 | 同上 |
| メール形式エラー | クライアントまたは API 由来 |
| パスワードエラー | 長さ・不一致等 |
| 認証失敗 | API エラーをフォーム内表示 |
| プロフィール補完失敗 | §16.3 |
| OAuth 開始 | redirect 前 loading |
| OAuth 失敗 | フォーム内エラー |
| ログイン済み | §14 — フォーム非表示 |
| 戻る | §15 |

## 16.2 エラー表示原則

* フォーム **内** に表示する
* `alert` だけに **依存しない**
* 入力値を必要以上に **消さない**
* 送信中は **二重送信を禁止**する

## 16.3 プロフィール補完失敗

`ensureUserProfile` 失敗時（[97](./97_個人アカウント・所属・個人データ基盤.md) §12.3）:

* Auth セッションが成立している場合、**無限ロードにしない**
* 最低限の `AppUser` で **継続可能**とする
* `console.error` で記録する
* 詳細な retry UI は **本 Phase では作らない**

# 22. テスト計画

既存 vitest 基盤を利用する。新しい大型 E2E 基盤は導入しない。

## 22.1 最低限のテスト観点

| 観点 | 内容 |
|------|------|
| `/login` route | pathname → login モード |
| `/signup` route | pathname → signup モード |
| trailing slash | `/login/` / `/signup/` |
| popstate | 戻る・進むで appRoute 同期 |
| StartScreen login | 導線が `/login` へ |
| StartScreen signup | 導線が `/signup` へ |
| GuestEntry login | GuestEntry を離れ LoginScreen 表示 |
| GuestEntry signup | 同上 |
| pendingReturnPath | community 付き pathname 保持 |
| community 付き return path | `/c/.../s/...` 復帰 |
| ログイン済み route | フォーム非表示、`#account` 等 |
| `/admin/login` 回帰 | 管理者導線 |
| ゲスト参加回帰 | ニックネーム入室 |

route helper を切り出す場合は **ユニットテスト**を優先する。
