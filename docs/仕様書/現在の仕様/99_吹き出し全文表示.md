## 1. 背景・課題

スペース画面では、表示領域の都合で投稿本文が **切り詰め** または **行数クランプ** され、ユーザーが **全文を読めない** 状態がある。

| モード | 現状の切り詰め | 問題 |
|--------|----------------|------|
| **カスタムモード**（Bubble） | `Tree.tsx` で最大 **120 文字** + CSS `line-clamp` | 長文が `…` で途切れる。リサイズで拡大しても隠れた行は見えないまま |
| **星モード** — ホバー | `StarHoverPreview` 最大 **80 文字** + `line-clamp: 3` | 星にカーソルを合わせても全文読めない |
| **星モード** — ローテ 6 件 | `StarView` 最大 **60 文字** + `line-clamp: 3` | ランダム表示の吹き出しも途中まで |

**ゴール:** 省略されている本文は、**ホバー** または **リサイズ操作の前段** で **必ず全文にアクセスできる** ようにする。省略されていない短文は現状どおり（ホバーで余計な UI を出さない）。

## 2. 用語

| 用語 | 定義 |
|------|------|
| **カスタムモード** | PC 右上 pill で切り替える吹き出し配置モード（`presentationMode === 'custom'`）— [84](./84_スペース表示モード拡張.md) |
| **星モード** | StarView 表示（`presentationMode === 'stars'` またはモバイル縦） |
| **省略状態** | 表示上、本文の一部が見えていない状態（文字数切り **または** `line-clamp` / `overflow: hidden` による行省略） |
| **全文表示** | 投稿の `message`（および音声ログの `renderHossiiText` 結果）を **切り詰めなし** で提示すること。改行は保持 |
| **ローテプレビュー** | 星モードで最大 6 件（PC）が同時に付く `.previewBubble`（[84 §14.5](./84_スペース表示モード拡張.md)） |

## 3. 共通原則

### 3.1 省略の判定

```typescript
type FullTextContext = {
  rawMessage: string;           // trim 前後の正本
  displayedPlainText: string;   // 画面上に描画している文字列（切り詰め後）
  isLineClamped: boolean;       // DOM または layout 計測で scrollHeight > clientHeight
};

function isTextTruncated(ctx: FullTextContext): boolean {
  if (!ctx.rawMessage.trim()) return false;
  if (ctx.rawMessage.trim().length > ctx.displayedPlainText.replace(/…$/, '').length) return true;
  return ctx.isLineClamped;
}
```

| ルール | 内容 |
|--------|------|
| 判定タイミング | 描画後（`ResizeObserver` または ref 計測）。静的な文字数だけに依存しない |
| 短文 | `isTextTruncated === false` のとき **全文用 UI は出さない**（ホバー拡張・ポップアップなし） |
| 対象フィールド | 通常投稿: `message`。笑い・音声ログ: 既存 `renderHossiiText(hossii)` の結果 |
| 画像のみ投稿 | テキスト省略なし → 本仕様の対象外（現状どおり） |
| 最大高さ | 全文ポップアップは **`max-height: min(60vh, 420px)`** + **`overflow-y: auto`**（極端な長文でも画面外に溢れない） |
| モバイル | タップ = 既存 `PostDetailModal` で全文可。**本仕様のホバー全文は PC マウス操作が主対象**（星ホバーと同様） |

### 3.2 全文表示 UI の共通スタイル

[90_星ホバーポップアップ](./90_星ホバーポップアップ.md) / [84 §14.5](./84_スペース表示モード拡張.md) の frosted-glass トークンを流用する。

| 項目 | 値 |
|------|-----|
| 背景 | §84 §14.5 と同系のグラデ + `backdrop-filter: blur(18px)` |
| 幅 | `min(360px, 32vw)`（カスタム Bubble ホバー）/ `min(380px, 34vw)`（星モード） |
| テキスト | **`white-space: pre-wrap`**、`word-break: break-word`、**切り詰めなし** |
| 出現 | `opacity` + 軽い `translateY(6px→0)`、150〜220ms |
| 消滅 | ホバー解除後 **100ms** debounce（90 と同じ） |
| `prefers-reduced-motion` | 位置アニメ省略、フェードのみ |

## 4. カスタムモード（Bubble）

### 4.1 ユーザーストーリー

> カスタムモードで長文の吹き出しを見ている。省略されているので、**カーソルを乗せれば全文** が読める。  
> さらに大きくしたいとき、**隠れている行が先に見えるようになってから** 全体が拡大する。

### 4.2 ホバーで全文表示

| 項目 | 仕様 |
|------|------|
| トリガー | `.bubble`（吹き出し本体）に `mouseenter` — **省略状態のときのみ** |
| 表示 | 吹き出し **真上または真下**（画面端で flip）。portal + `position: fixed` 可 |
| 内容 | 全文 + 投稿者名 + 気持ち絵文字（既存 Bubble メタと同順） |
| 非トリガー | いいねバッジ・リサイズハンドル・画像ライトボックスボタン上では **ポップアップを出さない**（操作優先） |
| 選択モード | F14 選択中（紫枠・ハンドル表示）でもホバー全文は **有効** |
| 共存 | カラーパレットポップアップと競合する場合、**全文ポップアップを優先**（パレットはホバー外れで閉じる） |

**受け入れ条件:**

- AC-C1: 120 文字超の投稿で、省略されている吹き出しにホバー → 全文が読める
- AC-C2: 省略されていない短文にホバー → 追加ポップアップは出ない
- AC-C3: ホバー解除 → 100ms 以内にポップアップが消える

### 4.3 リサイズ時 — 「全文を見せてから拡大」

[08 §F05](./08_スペース（HOME）の仕様書.md) のスケールリサイズを拡張する。

#### 問題（現状）

`scale` のみが増え、**切り詰められたテキスト行数は変わらない**。拡大しても隠れた部分は読めない。

#### 改定後の 2 フェーズ

```text
Phase A（コンテンツ展開）  … 省略されていた本文を吹き出し内で全文表示
Phase B（スケール適用）    … 既存どおり hossii.scale を更新（0.5〜2.5）
```

| フェーズ | タイミング | 動作 |
|---------|-----------|------|
| **A** | リサイズハンドル `pointerdown` 時点で `isTextTruncated === true` | 当該 Bubble の **表示用切り詰めを解除**（120 文字 cap オフ、`line-clamp` 解除）。必要なら `max-width` を一時拡張して折り返し表示。**`scale` はまだ変えない** |
| **B** | ユーザーがドラッグして拡大方向に動かしたとき | 既存ロジックで `scale` をリアルタイム更新 → PointerUp で DB 保存 |

| ルール | 内容 |
|--------|------|
| 縮小のみ | Phase A 済みの Bubble を縮小するだけなら Phase A を繰り返さない |
| 省略なし | Phase A を **スキップ** し、従来どおり Phase B のみ |
| 保存 | `scale` のみ永続化。Phase A の「展開」は **表示上の一時状態**（リロード後は再び省略表示。ホバーで全文可） |
| 管理者 `displayScale`（125% 等） | 左バーの表示倍率とは独立。個別 Bubble の `hossii.scale` リサイズのみ本節の対象 |

**受け入れ条件:**

- AC-C4: 長文 Bubble を選択 → 右下ハンドルで拡大開始 → **拡大アニメの前に** 隠れていた行が Bubble 内に現れる
- AC-C5: その後ドラッグを続けると全体がスケールアップする（F05 従来どおり保存）
- AC-C6: 短文 Bubble のリサイズは従来どおり（Phase A なし）

### 4.4 実装ファイル（予定）

| ファイル | 変更 |
|---------|------|
| `src/components/SpaceScreen/Tree.tsx` | 省略判定、`BubbleFullTextPopover` 連携、リサイズ Phase A |
| `src/components/SpaceScreen/BubbleFullTextPopover.tsx` | **新規** — 全文 portal |
| `src/components/SpaceScreen/SpaceScreen.module.css` | `.bubbleTextExpanded` 等 |
| `src/core/utils/bubbleTextTruncation.ts` | **新規** — 切り詰め定数・判定ヘルパー |

## 5. 星モード（StarView）

### 5.1 ユーザーストーリー

> 星モードで星にカーソルを合わせると内容がわかるが、**長文は途中まで** しか見えない。  
> ローテで出ている 6 つの吹き出しも同様。**星にも吹き出しにもホバーすれば全文** が読めるようにしたい。

### 5.2 星マーカーへのホバー（`StarHoverPreview` 改定）

[90_星ホバーポップアップ](./90_星ホバーポップアップ.md) を **全文対応** に改定する。

| 項目 | 現状 | 改定後 |
|------|------|--------|
| 文字数上限 | 80 文字 + `…` | **なし**（`message` 全文） |
| 行クランプ | `line-clamp: 3` | **解除**（§3.2 の `max-height` + scroll） |
| 省略なし投稿 | 同左 | 現状と同サイズで表示（無駄に巨大化しない） |
| 省略あり投稿 | 80 文字で切れる | **全文** + 必要時スクロール |

**受け入れ条件:**

- AC-S1: 200 文字以上の投稿の星にホバー → ポップアップ内で **全文** が読める（スクロール可）
- AC-S2: 40 文字の短文 → 従来どおりコンパクト表示

### 5.3 ローテプレビュー吹き出し（最大 6 件）

[84 §14.5](./84_スペース表示モード拡張.md) の `.previewBubble` を拡張する。

#### 通常表示（変更なし部分）

- ローテ中は引き続き **コンパクトプレビュー**（60 文字 + 3 行クランプ）でよい — 6 件同時表示のレイアウト維持

#### ホバー時全文

| ホバー対象 | 動作 |
|-----------|------|
| **星マーカー**（§5.2） | `StarHoverPreview` で全文 |
| **ローテ吹き出し本体**（`.previewBubble`） | 省略状態なら **吹き出し上に全文レイヤー** を表示（星ホバーと同内容・同スタイル）。`pointer-events: none` の吹き出しは **子 overlay のみ `pointer-events: auto`** |
| 両方にまたがるホバー | **1 つの全文ポップアップ** に統合（二重表示しない） |

| ルール | 内容 |
|--------|------|
| トリガー | `.previewBubble` または親 `.star` に `mouseenter`、かつ `isTextTruncated` |
| 位置 | 吹き出し真上（ローテ吹き出しと星ホバーで z-index 調整。`z-index ≥ 200`） |
| ピン留め吹き出し | [88_ピン留め機能](./88_ピン留め機能.md) の常時表示プレビューにも **同一ルール** を適用 |
| モバイル | タップは `PostDetailModal`（変更なし）。ローテ吹き出しへの long-press 全文は **対象外** |

**受け入れ条件:**

- AC-S3: ローテ中の長文プレビュー吹き出しにホバー → 全文表示
- AC-S4: ローテ中の星（吹き出し付き）にホバー → 全文表示（AC-S3 と重複しない）
- AC-S5: 省略のない短いプレビューにホバー → 追加 UI なし

### 5.4 定数の整理（実装時）

| 定数 | 現状 | 改定 |
|------|------|------|
| `MAX_BUBBLE_TEXT_LENGTH`（Tree） | 120 | **通常表示用として維持**。全文はホバー / Phase A で別経路 |
| `MAX_PREVIEW_TEXT`（StarView） | 60 | **通常表示用として維持** |
| `MAX_TEXT`（StarHoverPreview） | 80 | **廃止** → 全文 |

## 6. モード横断

| シナリオ | 期待 |
|---------|------|
| カスタム ↔ 星 切替 | 各モードの全文ルールが独立に動作 |
| `layoutMode === 'byAuthor'` | `AuthorClusterBubble` 内の省略テキストにも **§4.2 と同様のホバー全文** を適用（将来。初版は Bubble + StarView を優先） |
| `viewMode === 'bubble'` / `'image'` | テキスト省略ルールは `viewMode` に従う。全文ホバーは **テキストが省略されている場合のみ** |
| タグ絞り込み | フィルタ後の投稿に対して同样に全文ルール適用 |

## 8. パフォーマンス

| 項目 | 方針 |
|------|------|
| portal 数 | **ホバー中 1 件のみ**（全 Bubble 常時 portal しない） |
| 計測 | `isLineClamped` はホバー直前または `ResizeObserver` で **対象 1 件** のみ |
| 星モード | 全文ポップアップは既存 `StarHoverPreview` を **拡張**（コンポーネント増殖を最小化） |

[87_スペース表示パフォーマンス最適化](./87_スペース表示パフォーマンス最適化.md) の animation tier とは独立。全文ポップアップに CSS Animation は **付けない**（フェードのみ）。
