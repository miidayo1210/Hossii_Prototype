# 112 スペースのアーカイブ機能

## 目的

終了したイベントや活動のスペースを削除せず、
記録として閲覧できる状態で保存する。

アーカイブ中のスペースは閲覧専用とし、
新しい投稿や既存投稿の変更はできない。

## 管理画面

スペース設定の「運営」または「基本設定」に、
次の設定を追加する。

### スペースをアーカイブ

- OFF：通常のスペース
- ON：閲覧専用のアーカイブ状態

説明文：

「アーカイブすると、このスペースは閲覧専用になります。
過去の投稿は引き続き見ることができますが、
新しい投稿や投稿内容の変更はできません。
あとからアーカイブを解除できます。」

## アーカイブ中にできること

- スペースを開く
- 背景を見る
- Pane・タブを切り替える
- 過去の投稿を見る
- My Hossiiを見る
- 投稿者名や投稿日時を見る
- public / invite_onlyなど、既存の閲覧権限に従って閲覧する
- 管理者がアーカイブを解除する

## アーカイブ中にできないこと

- 新しい投稿
- 既存投稿の編集
- 公開／自分だけの切り替え
- 投稿の削除
- いいねやリアクション
- コメント
- 投稿位置の変更
- 投稿の並べ替えなど、投稿データを書き換える操作

スペース設定そのものは管理者が変更できる。
アーカイブ解除操作まで禁止しない。

## 画面表示

アーカイブ中のスペース上部に、
次のような案内を表示する。

「このスペースはアーカイブされています」
「過去の記録を閲覧できます」

投稿フォームは非表示またはdisabledにする。

disabledにする場合は、次の説明を表示する。

「アーカイブ中のため、新しい投稿はできません」

既存投稿の編集・削除などの操作ボタンも表示しない。

スペース一覧では、対象スペースに
「アーカイブ」バッジを表示する。

## 権限

アーカイブのON／OFFを変更できる人：

- 対象スペースの管理者
- 対象コミュニティの管理者
- super_admin

一般ユーザー・参加ID・guestは変更できない。

## 閲覧権限との関係

アーカイブは、公開範囲を変更しない。

### public

アーカイブ前に閲覧できた人は、
アーカイブ後も閲覧できる。

### invite_only

アーカイブ前に閲覧権限を持っていた
space member・管理者だけが閲覧できる。

アーカイブしたことを理由に、
非memberへ内容を公開しない。

### personal

owner本人・許可された管理者など、
既存のpersonal space権限を維持する。

## データ

spacesにアーカイブ状態を保存する。

候補：

- `is_archived boolean not null default false`
- `archived_at timestamptz null`
- `archived_by uuid null`

最低限必要なのは `is_archived`。

既存スペースはすべて `false` とする。

アーカイブON：

- is_archived = true
- archived_at = 現在時刻
- archived_by = 操作したユーザー

アーカイブOFF：

- is_archived = false
- archived_at = null
- archived_by = null

既存データを削除しない。

## セキュリティ

画面上で投稿フォームを隠すだけでは不十分。

アーカイブ中は、DB・RPC・API側でも
投稿データへの書き込みを拒否する。

対象：

- 投稿作成
- 投稿編集
- visibility変更
- soft delete
- いいね
- コメント
- 投稿位置変更

クライアントから直接APIを呼んでも書き込めないこと。

エラー文：

「このスペースはアーカイブされているため変更できません」

管理者も、アーカイブを解除するまでは投稿を書き換えられない。

## Realtime

アーカイブ中も閲覧用のRealtime購読は維持してよい。

ただし、アーカイブ切り替え後は画面へ即時反映し、

- 投稿フォームを閉じる
- 編集中なら保存を停止する
- 操作ボタンを非表示にする

## 受入条件

1. 管理者が設定からアーカイブをONにできる
2. ONにするとスペース上にアーカイブ表示が出る
3. 過去の投稿は閲覧できる
4. 新しい投稿ができない
5. 既存投稿を編集・削除できない
6. いいね・コメントなどの書き込みができない
7. URLへ直接アクセスしても閲覧できる
8. public / invite_only / personalの閲覧権限は維持される
9. APIを直接呼んでも書き込みを拒否する
10. 管理者がアーカイブを解除できる
11. 解除後は通常どおり投稿できる
12. 既存スペースはアーカイブされない
13. スペース一覧にアーカイブ表示が出る
