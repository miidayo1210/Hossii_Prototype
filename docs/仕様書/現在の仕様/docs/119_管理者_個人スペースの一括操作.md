# 119 管理者：個人スペースの一括操作

> **ステータス:** 調査・仕様ドラフト（**未実装**）  
> **最終更新:** 2026-07-16  
> **想定 branch:** `feat/admin-personal-space-bulk`（仮）  
> **上位仕様:** [112_スペースのアーカイブ機能](./112_スペースのアーカイブ機能.md) / [118 個人スペース一覧の検索](./118_管理者_個人スペース一覧の検索.md)  
> **関連画面:** `#spaces` 個人スペースセクション

## 背景

管理者は個人スペースを 1 件ずつ「開く」「設定」できるが、メンバー数が増えると同じ操作（とくにアーカイブ）の繰り返しが負担になる。

一方、投稿・削除・テンプレート一括適用は **データ破壊や権限逸脱** のリスクが高い。v1 は **安全で既存 RPC を再利用できる操作だけ** に限定する。

## 現状調査（main `e8a7da8`）

### 管理者が個人スペースに対してできること（1 件単位）

| 操作 | 経路 | 備考 |
|------|------|------|
| 閲覧（開く） | `#spaces` → `/c/.../s/...` | 管理者は community 管理者として personal を閲覧可 |
| 設定画面 | `setActiveSpace` + settings | 各種タブ |
| アーカイブ ON/OFF | `SpaceArchiveTab` → `set_space_archived` | `can_manage_space_archive`（community 管理者 / super_admin） |
| テンプレート編集 | `PersonalSpaceTemplateEditor` | **新規作成時のみ** 適用。既存スペースは変更しない |
| 投稿の編集・削除 | 本人 authorship 前提 | 管理者は他人の本人投稿を編集不可（109 / authorship 基盤） |
| スペース物理削除 | 共有スペース用 `removeSpace` | personal 行には **削除ボタンなし** |

### 候補操作の安全性評価

| 候補 | v1 | 理由 |
|------|-----|------|
| 一括アーカイブ | **○ 採用** | 既存 `set_space_archived(true)` を N 回。閲覧専用化のみ |
| 一括アーカイブ解除 | **○ 採用** | `set_space_archived(false)`。112 仕様どおり可逆 |
| 背景テンプレート一括適用 | **×** | 現行テンプレートは **新規作成専用**。既存へ一括適用は仕様外・意図しない上書き |
| 表示設定の一括適用 | **×** | space_settings / pane 単位で差分が大きく、migration または新 RPC が必要 |
| 一括削除 | **× 禁止** | personal はユーザー資産。誤削除は復旧困難 |
| 一括投稿操作（編集・削除・非表示） | **× 禁止** | 投稿所有権は本人。管理者の一括変更は 109 と矛盾 |
| 一括 suspend（`spaces.status`） | **×** | Phase 3 lifecycle。UI・RPC が未整備。archive とは別軸 |
| 一括メンバー停止 | **×** | community_membership 操作。個人スペース一覧の責務外 |

## v1 仕様（推奨）

### v1 で実装する操作

1. **選択した個人スペースをアーカイブする**
2. **選択した個人スペースのアーカイブを解除する**

いずれも既存 `set_space_archived` を **1 スペースずつ順次呼び出し**（新 RPC 不要）。

### v1 で実装しない操作

- 一括削除（物理・論理）
- 投稿の一括編集 / 削除 / 公開設定変更
- テンプレート・背景・pane 設定の一括適用
- community membership の一括変更
- 共有スペースへの一括操作（スコープ外）

### 選択 UI

- 個人スペース各行の先頭に **checkbox**
- ツールバー（検索欄横またはリスト上）に:
  - `アーカイブする`
  - `アーカイブを解除する`
- 操作ボタンは **1 件以上選択時のみ** enabled

### 全選択

- リストヘッダに **「すべて選択」** checkbox
- 対象: **現在表示中の行のみ**（検索フィルタ後の結果。→ [118](./118_管理者_個人スペース一覧の検索.md) と連携）

### 検索結果だけを選択するか

- **はい。** 全選択はフィルタ後の表示行だけ。意図しない全 community 一括を防ぐ。

### 確認ダイアログ

| 操作 | 確認 |
|------|------|
| アーカイブ | `N 件の個人スペースをアーカイブします。閲覧専用になり、新しい投稿はできなくなります。` |
| 解除 | `N 件の個人スペースのアーカイブを解除します。` |

- キャンセルで何もしない
- 確認後に処理開始

### 部分失敗

- 1 件ずつ RPC。失敗した space はスキップし **最後にまとめて報告**
- 例: `3 件中 2 件をアーカイブしました。1 件は権限エラーのためスキップしました。`
- 成功分は store / 再 fetch で UI 反映
- **全体をロールバックしない**（archive は可逆のため）

### 権限

- UI 表示: `currentUser.isAdmin`（community 管理者）または super_admin（現行 `#spaces` と同じ）
- 各 RPC: `can_manage_space_archive` が最終判定
- 一般メンバー・ゲスト: 一括 UI 非表示

### audit log

- v1: **専用 audit log は追加しない**（112 と同様、既存 `set_space_archived` に追記が無い）
- v2 検討: `community_audit_log` への bulk 記録（要 migration / RPC 拡張）

---

## 非 active / archived 既存データへの影響

- アーカイブは `is_archived` フラグのみ（112）。`spaces.status` は変更しない
- 既にアーカイブ済みの行を再度アーカイブ → no-op 相当（RPC 内で同値 UPDATE）
- 未アーカイブ行の解除 → 同様 no-op

---

## 受入条件

- [ ] 複数 personal を選択してアーカイブ / 解除できる
- [ ] 確認ダイアログがある
- [ ] 検索フィルタ後の「全選択」がフィルタ結果のみを対象にする
- [ ] 部分失敗時に成功件数と失敗件数が分かる
- [ ] 一括削除・投稿操作 UI が **存在しない**
- [ ] 権限のないユーザーに一括 UI が出ない
- [ ] 追加 migration なし（既存 RPC のみ）
- [ ] 共有スペース一覧の削除・編集挙動に影響しない
