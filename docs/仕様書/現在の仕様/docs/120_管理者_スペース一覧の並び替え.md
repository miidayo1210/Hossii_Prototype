## 背景

管理者の `#spaces` では共有スペース・個人スペースとも、現状は **作成順（`created_at` 昇順）固定** で表示される。

管理者が「名前順で探したい」「アーカイブを下に寄せたい」という需要がある一方、**カードの並べ替え（drag & drop）や DB 永続 sort** は現行に存在しない。

v1 は **表示時ソートのみ** とし、DB の並びを壊さない。

## 現状調査（main `e8a7da8`）

### 共有スペース

| 項目 | 現状 |
|------|------|
| 取得 | `fetchSpaces` → `order('created_at', { ascending: true })` |
| `spaces.sort_order` 列 | **存在しない** |
| 表示 | `sharedSpaces.map` — 配列順そのまま |
| カード操作 | 名前・slug 編集、背景、削除、設定、アーカイブバッジ |
| drag & drop | **なし**（space 一覧単位） |

### 個人スペース

| 項目 | 現状 |
|------|------|
| 取得 | 同上（`fetchSpaces` 経由） |
| 分区後 | `partitionAdminCommunitySpaces` — filter のみ、再ソートなし |
| 所有者名 | `ownerLabels` 非同期取得後に表示（ソートキーに使うなら labels 読込後） |
| `sort_order` | **なし** |

### 他箇所の `sort_order`（混同注意）

| 対象 | 用途 |
|------|------|
| `space_panes.sort_order` | Pane タブ並び（SpacePaneBar DnD） |
| `tab_folders.sort_order` | タブフォルダ並び |
| personal space template JSON | 新規作成時 pane 初期並び |

→ **管理者スペース一覧とは無関係**。v1 でこれらと連動しない。

### drag and drop との競合

- スペース一覧に DnD は **未実装** → v1 表示ソートを入れても競合なし
- 将来 DnD + 永続 sort を入れる場合は v2 以降で設計し直す

## v1 仕様（推奨）

### 方針

- **クライアント側の表示ソートのみ**（`useMemo` でソート済み配列）
- 選択状態は **localStorage 非保存**（リロードで既定に戻る）
- migration **不要**

### 共有スペース — ソート選択肢

| 値 | ラベル | ロジック |
|----|--------|----------|
| `created_asc` | 作成が古い順 | **既定**（現行どおり） |
| `created_desc` | 作成が新しい順 | `createdAt` 降順 |
| `name_asc` | 名前順（A→Z） | `space.name` localeCompare `ja` |
| `archived_last` | アーカイブを下へ | `isArchived` false 先、同グループ内は `created_asc` |

**v1 対象外:** `sort_order`（列なし）、更新順（`updated_at` 列なし）、手動 DnD

### 個人スペース — ソート選択肢

| 値 | ラベル | ロジック |
|----|--------|----------|
| `created_asc` | 作成が古い順 | **既定** |
| `created_desc` | 作成が新しい順 | `createdAt` 降順 |
| `owner_asc` | 所有者名順 | `resolvePersonalSpaceOwnerDisplay(...).displayName` localeCompare |
| `archived_last` | アーカイブを下へ | `isArchived` false 先、同グループ内は `owner_asc` |

**owner 名ソート:** `ownerLabels` 未取得時は `名前未設定` としてソート（ちらつき許容）。

### UI 配置

- 共有: グリッド上または見出し横に `<select>` またはセグメント（コンパクト）
- 個人: 折りたたみセクション内、検索欄（118）の横または下
- ラベル: `並び替え`

### mobile

- ネイティブ `<select>` で十分
- ソート変更は即時反映（追加ボタン不要）

### DB 保存が必要な並び替え（v2 以降）

以下は v1 **対象外**:

- 管理者が決めた順序の永続化（`spaces.display_order` 新設 + migration）
- 共有スペースカードの drag & drop 並べ替え
- チーム全体での既定順序共有
