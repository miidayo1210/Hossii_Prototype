# 121 マイスペースの説明と導線改善

> **ステータス:** 調査・仕様ドラフト（**未実装**）  
> **最終更新:** 2026-07-16  
> **想定 branch:** `feat/my-space-copy-navigation`（仮）  
> **上位仕様:** [109](../109_コミュニティ所属個人アカウント・個人スペース・招待制スペース.md) / [111_個人スペースをタブとして表示](./111_個人スペースをタブとして表示.md) / [112_スペースのアーカイブ機能](./112_スペースのアーカイブ機能.md)

## 背景

「マイスペース」（コミュニティ内の個人スペース）の入口が複数あり、用語・導線・説明が画面ごとにばらついている。

ユーザーが「どこから開くか」「コミュニティごとに別か」「タブが見えない理由」を迷わないよう、**短くやさしい案内** を足す。ロジック変更は最小にする。

## 現状調査（main `e8a7da8`）

### 用語のばらつき

| 画面 | 表記 |
|------|------|
| アカウント `CommunityPersonalSpacesSection` | **マイスペース** |
| 共有スペース Pane タブ | **マイスペース**（旧「わたし」） |
| コミュニティ HOME | **個人スペース** |
| 管理者 `#spaces` | **個人スペース** |

v1 方針: **メンバー向け UI は「マイスペース」に統一**。管理者画面は「個人スペース」のまま（対象が他人のスペース一覧のため）。

### 1. アカウント画面

**実装:** `CommunityPersonalSpacesSection`

| 状態 | 現状 |
|------|------|
| 未ログイン | 「ログインして…マイスペースをここから作成できます」 |
| membership 0 | 「参加中のコミュニティがありません…」 |
| 未作成 | 「マイスペース未作成」+ 作成ボタン |
| 作成済 | 「マイスペースあり」+ 「共有スペースの『マイスペース』タブから利用できます」 |
| プライバシー | Lock 付き 1 行説明あり |

**ギャップ**

- 作成後に **直接開く** 導線がない（タブ経由のみ）
- コミュニティごとに **別スペース** である説明が弱い（リストは community 名付きだが一文説明なし）
- **アーカイブ表示バグ:** `personalSpaceStatus === 'archived'` で判定しているが、112 のアーカイブ正本は `spaces.is_archived`。`status` は `active` のままアーカイブされている可能性がある → バッジが出ない

### 2. コミュニティ HOME

**実装:** `CommunityHomeScreen`

| 項目 | 現状 |
|------|------|
| 個人スペースカード | 「個人スペースを開く」/「作成して開く」 |
| 遷移 | `window.location.href` で **personal space URL へフル遷移**（共有 shell のタブではない） |
| suspended/removed | バナー文言あり（個人スペースに入れない旨） |
| 用語 | 「個人スペース」（マイスペースと不一致） |

**ギャップ**

- 「マイスペースとは」の一文なし
- HOME から開くと共有スペースのタブ UX と経路が異なる（ユーザー混乱）

### 3. 共有スペースの「マイスペース」タブ

**実装:** `SpacePaneBar` + `personalSpaceTabView` + `canShowPersonalShortcut`

| 項目 | 現状 |
|------|------|
| 表示位置 | Pane タブ列の **右端**（横スクロール内） |
| 表示条件 | ログイン + 当該 community で membership `active` + shared shell |
| 非表示 | guest / visiting / suspended / removed / community_id なし |
| 未作成時 | タップで `ensureMyPersonalSpace` → in-place タブ表示 |
| エラー | `window.alert('個人スペースを開けませんでした')` |
| tooltip | `マイスペースを開く` / `表示中` |

**ギャップ**

- タブが **見えない理由** の案内なし
- 右端にあり気づきにくい（初回ヒントなし）
- 「コミュニティごとに別」の説明なし

### 4. personal space 未作成時

- アカウント: 作成ボタンあり ○
- HOME: 作成して開く ○
- 共有タブ: 初回タップで暗黙作成 ○
- **共通の一文説明**（自分専用・管理者のみ閲覧可）はアカウントのみ

### 5. archived personal space

| 経路 | 現状 |
|------|------|
| 112 アーカイブ | `is_archived` → 閲覧専用バナー（SpaceScreen） |
| アカウント | `status === 'archived'` 判定（**112 と不整合の疑い**） |
| マイスペースタブ | archived でも開ける想定（読み取り専用）— 明示説明なし |

### 6. membership 不足時

| status | タブ | HOME | アカウント |
|--------|------|------|------------|
| suspended | 非表示 | バナー | 行は出ない（active のみ RPC） |
| removed | 非表示 | バナー | 同上 |
| invited | 非表示 | バナー | 同上 |
| 非メンバー | 非表示 | — | 参加コミュニティなし |

タブ非表示時の **インライン説明はどこにもない**。

### 7. personal space 直リンク

- URL 直開き → personal `SpaceScreen`（shared shell ではない）
- 「マイスペース」タブは **shared 上での in-place 表示**（111）— 直リンク UX とは別経路
- 直リンク時の「共有スペースからも開けます」導線なし

## v1 仕様（推奨）

### 共通コピー（Hossii トーン・短文）

**マイスペースとは（標準 1 行）**

> コミュニティごとにひとつ、あなただけのスペースです。ほかのメンバーには見えません。

**コミュニティが違えば別（補足・必要な画面のみ）**

> コミュニティが変わると、マイスペースも別になります。

### アカウント画面

| 変更 | 内容 |
|------|------|
| セクション冒頭 | 上記「マイスペースとは」1 行を `privacyNote` の上に追加 |
| 作成済み行 | 既存のタブ案内は維持。任意で「アカウント」バッジはそのまま |
| アーカイブ | **修正:** `list_my_community_personal_spaces` に `is_archived` を返す拡張 **または** 作成済み行で `fetchSpaceArchiveFlags` を併用し `isArchived` でバッジ表示 |
| 作成済み + 共有スペースが分かる場合 | v1 は **開くボタン追加しない**（タブ導線の説明強化に留める） |

**migration:** アーカイブ表示修正のみ RPC 拡張が必要なら **小さな migration 1 本**（`personal_space_is_archived boolean` を RPC に追加）。クライアントだけで `spaces` SELECT が可能なら migration 不要 — 要実装時確認。

### コミュニティ HOME

| 変更 | 内容 |
|------|------|
| 見出し | `個人スペース` → **`マイスペース`** |
| ボタン | `マイスペースを開く` / `マイスペースを作る` |
| 説明 | カード内に「マイスペースとは」1 行 |
| 遷移 | v1 は **現行の URL 遷移を維持**（111 の in-place は shared 上のみ。HOME 変更はスコープ外） |

### 共有スペース「マイスペース」タブ

| 変更 | 内容 |
|------|------|
| 初回のみ | localStorage `hossii:my-space-tab-hint-seen` — タブ付近に小さな吹き出し「右端のマイスペースは、あなただけの場所です」→ 閉じたら再表示しない |
| 非表示時 | v1 は **専用 UI を出さない**（条件が複雑なため）。アカウント・HOME 側の説明で補う |
| エラー | `window.alert` → 画面内 toast またはインライン文言（既存パターンに合わせる） |

### archived 時

| 場所 | 文案 |
|------|------|
| in-place タブ / personal 直開き | 既存アーカイブバナーに加え不要（112 済） |
| アカウント行 | `アーカイブ中 — 見ることはできます`（短く） |

### mobile 右端の分かりやすさ

- 初回ヒント（上記）で対応
- v1 でタブの固定表示・スクロール自動は **しない**（111 の範囲外）
