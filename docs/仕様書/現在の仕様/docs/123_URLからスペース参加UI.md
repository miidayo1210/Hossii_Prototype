## 1. 開発背景

現在の共有URL参加導線には、以下の課題がある。

* URLを開いた人が、Hossiiが何をするサービスなのか理解しにくい
* 参加するスペースの目的や使い方が、入室前には十分伝わらない
* ゲスト参加とアカウント参加の違いが分かりにくい
* 「ログインして参加」が、参加IDログインなのかメールログインなのか分かりにくい
* ゲストには入室後のチュートリアルが表示されず、最初に何をすべきか分かりにくい
* ニックネーム、アカウント名、スペース内の表示名の役割が整理されていない
* 入室後にも独立した「ようこそ画面」を挟むと、参加操作を繰り返しているように感じる
* 既存の仕様書が複数に分散し、実装と異なる古い記述も存在する

本仕様では、共有URLを開いてからスペースを利用し始めるまでを、**Hossiiが案内する一続きの体験**として再設計する。

---

# 2. 体験の基本方針

## 2.1 Hossiiがスペースへ案内する

オンボーディングは単なるログイン手続きではなく、Hossiiがユーザーをスペースへ迎え入れる体験とする。

Hossiiは各画面で次の役割を持つ。

* スペースへ歓迎する
* 参加方法を説明する
* 次に行う操作を案内する
* 入力結果に反応する
* エラー時に次の行動を伝える
* 入室後にスペースの使い方を簡潔に伝える

---

## 2.2 参加までをできるだけ短くする

基本フローは、最大でも次の3操作程度に収める。

```text
共有URLを開く
→ 参加方法を選ぶ
→ 必要な情報を入力して参加する
→ スペースを利用する
```

独立した参加完了画面や、同じ意味の確認操作を繰り返さない。

---

## 2.3 スペース名を最初に伝える

URLを開いた直後から、必ずスペース名を表示する。

ユーザーが「どこに参加しようとしているか」を理解してから、参加方法を選べるようにする。

---

## 2.4 Hossiiの吹き出しにスペース説明を統合する

スペースの説明文とHossiiの吹き出しは分けない。

参加画面では、Hossiiが吹き出し内で次を伝える。

* 短い挨拶
* スペースの短い説明
* 次に行う操作

例：

```text
こんにちは！
ここは、みんなの気づきや想いを
自由に残せるスペースだよ
```

特定の公園、授業、イベントなどに限定した固定文言は使用しない。

管理者がスペースごとの説明を設定した場合は、その内容をHossiiが話している形で表示する。

---

## 2.5 参加画面では共通Hossiiを使用する

参加前の全画面では、スペース固有キャラクターではなく、Hossii共通の黄色い星形キャラクターを使用する。

これにより、以下を実現する。

* Hossiiブランドとしての一貫性
* 画面状態に応じた表情変化
* スペース固有画像の品質や形式に左右されない表示
* 初めて利用する人が案内役を理解しやすい

スペースへ入った後は、既存どおりスペースまたはPaneに設定されたキャラクターを表示できる。

---

# 3. 対象ユーザー

## 3.1 ゲスト

アカウントを使用せず、そのスペースで使用するニックネームだけを登録して参加するユーザー。

## 3.2 アカウント参加者

管理者から発行された参加IDとパスワードを使用して参加するユーザー。

将来的には、メールアドレスと参加IDのどちらでも同じアカウントへログインできる設計を想定する。

## 3.3 再訪ユーザー

同じ端末または同じアカウントで、過去に同じスペースへ参加したことがあるユーザー。

---

| 用語     | 定義                              |
| ------ | ------------------------------- |
| ゲスト    | アカウントを使用せず、ニックネームのみで参加する人       |

# 6. スペース管理者による参加方法設定

## 6.1 設定の種類

スペース管理者は、スペースごとに以下の3種類から参加方法を選択できる。

| 設定値候補               | 管理画面での表記  | 参加画面       |
| ------------------- | --------- | ---------- |
| `guest_only`        | ゲストのみ     | ゲストとして参加のみ |
| `guest_and_account` | ゲストとアカウント | 両方を表示      |
| `account_only`      | アカウントのみ   | アカウントで参加のみ |

DBカラムは **`spaces.participation_mode`** として実装済み（migration `20260721100000`）。

```text
spaces.participation_mode  -- 'guest_only' | 'guest_and_account' | 'account_only'
                           -- DEFAULT 'guest_and_account'
```

`is_private`（未ログイン停止）・ `access_mode`（招待制など）とは別軸。personal スペースではアプリ側で無視する。

---

## 6.2 各設定の表示

### ゲストのみ

```text
［ゲストとして参加］
```

アカウント参加は表示しない。

選択肢が1つでも、参加意思を確認するため、必ず「ゲストとして参加」を一度押してからニックネーム入力へ進む。

---

### ゲストとアカウント

```text
［ゲストとして参加］
すぐに参加できます
ニックネームだけで参加できます

［アカウントで参加］
参加したスペースや情報を引き継げます
```

補足：

```text
ゲストで参加しても、あとから同じ投稿を
アカウントへ引き継ぐことはできません
```

この注意は参加方法カード上で長く表示せず、「参加方法について」などの補足領域で伝えてもよい。

---

### アカウントのみ

```text
このスペースへの参加には
アカウントが必要だよ

［アカウントで参加］
```

ゲスト参加はdisabledで見せず、完全に非表示にする。

---

# 7. ゲストとアカウントの違い

| 機能               |   ゲスト | アカウント |
| ---------------- | ----: | ----: |
| スペースを閲覧する        |     ○ |     ○ |
| 投稿する             |     ○ |     ○ |
| 同じ端末で再訪する        |     ○ |     ○ |
| 同じ端末で過去の自分の投稿を見る |     ○ |     ○ |
| 別端末から利用を継続する     |     × |     ○ |
| 参加スペースを一覧で保存する   |     × |     ○ |
| ゲスト投稿をアカウントへ引き継ぐ |     × |   対象外 |
| マイHossii         | 表示しない |  表示する |
| アカウント情報を編集する     |     × |     ○ |

## 7.1 ゲスト投稿の扱い

ゲストとして投稿した内容は、後からアカウントへ引き継がない。

同じ端末で同じゲスト情報を利用している場合は、過去の投稿を確認できるようにする。

ブラウザのデータ削除、端末変更、シークレットモードなどにより識別情報が失われた場合は復元できない。

---

## 7.2 ゲストのマイHossii

ゲストにはマイHossii機能を表示しない。

必要な場合はアカウント画面内に、次のような未解放案内を表示できる。

```text
マイHossii

アカウントで参加すると、
自分のHossiiを使えるようになるよ
```

スペースの主要ナビゲーションでは、ゲストに不要な機能を積極的に見せない。

# 8.2 参加方法選択画面

## 目的

* Hossiiであることを伝える
* スペース名を伝える
* スペースの概要を伝える
* 許可された参加方法を選んでもらう

1. `✨ Hossii`
2. スペース名
3. 共通Hossii
4. Hossiiの吹き出し
5. 参加方法
6. 必要に応じて補足

## 表示例


「どの方法で参加する？」も、吹き出し内に含めてよい。

* 省略せず全文表示
* 行数制限を設けない
* 極端に文字を小さくしない
* 長い場合はカード全体を縦に広げる
* 中央揃えを基本とする

## トップへ戻る

ブラウザ標準の戻る操作は妨げない。

# 8.3 ゲストのニックネーム入力画面

## 目的

スペース内で表示する名前を登録する。

## 表示例


## 入力仕様

| 項目         | 仕様     |
| ---------- | ------ |
| 表示名        | ニックネーム |
| 必須         | 必須     |
| 最大文字数      | 10文字   |
| 最小文字数      | 1文字    |

* 薄い紫
* 押下不可

* active
* 紫系グラデーション
* 押下可能

```text

```

## 戻る

推奨位置：

* カード左上
* または入力カード上部

ゲストのみで、前の選択画面を省略していない場合も、参加方法画面へ戻れるようにしてよい。

# 8.4 アカウントログイン画面

## 初期実装

初期段階では、管理者から案内された参加IDとパスワードのみを受け付ける。

表示例：

```text
アカウントで参加

[Hossii]

案内された参加IDと
パスワードを入力してね

［参加ID］
［パスワード］

［参加する］

← 戻る
```

## 新規登録

以下は表示しない。

* 新しくアカウントを作る
* メールアドレスで登録
* Googleなどの外部サービスで登録

アカウントは、管理者などから発行された場合のみ利用できる状態とする。

---

## 8.5 将来のアカウントログイン

参加IDとメールアドレスが同一アカウントに紐づいた後は、一つの入力窓口へ統合する。

```text
メールアドレスまたは参加ID
パスワード
```

ユーザーが参加IDとメールアドレスのアカウント種別を意識しなくても、同じ画面からログインできるようにする。

認証側で入力値から対象アカウントを解決する。

単純に `@` の有無だけで認証処理を分岐しない。

# 8.6 再訪確認画面

## 表示条件

* 同一スペースのニックネームが端末に保存されている
* ゲスト参加が現在も許可されている
* 必要なゲスト識別情報が残っている

## 表示例

```text
[Hossii・にこにこ]

おかえり！
みいさんとして参加する？

［このまま参加］

別の方法で参加
```

## 「このまま参加」

保存済みのゲスト情報を利用してSpaceScreenへ進む。

## 「別の方法で参加」

通常の参加方法選択画面へ進む。

そこから以下を選べる。

* ゲストとして別のニックネームを入力
* アカウントで参加

別のニックネームで参加する場合、過去のゲスト情報との扱いは実装前に確認する。原則として既存ゲストの名前変更として扱い、同じ端末プロフィールを不必要に増やさない。

# 9. 入室処理

## 9.1 「参加する」押下後

「参加する」を押した後は、独立した参加完了画面を挟まない。

```text

↓
SpaceScreenを表示

```

# 10. 入室直後の初回ガイド

## 10.1 基本仕様

* SpaceScreen上に1枚だけ表示
* 全画面を覆わない
* 「わかった」で閉じる
* 自動では消えない
* ユーザーとスペースの組み合わせごとに初回のみ表示
* ヘルプから再表示できる
* 管理者がスペース固有部分を編集できる
* 共通操作案内はシステム側が追加する
* 旧TutorialOverlayと二重表示しない

## 10.2 表示例

```text
[Hossii・嬉しい表情]

みいさん、ようこそ！

このスペースでは、
感じたことや気づいたことを
自由に投稿してみてね。

画面をダブルタップするか、
下の「投稿」から投稿できるよ。

［わかった］
```

特定の用途に限定しない汎用的なデフォルト文言を用意する。

管理者が設定した場合は、スペース固有の説明へ差し替える。

## 10.3 配置

# 12. Hossiiの役割・人格・話し方

## 12.1 役割

Hossiiは以下の役割を持つ。

* そのスペースの案内役
* ユーザーを評価しない
* 投稿を強制しない
* 迷ったときに次の行動を伝える
* ときどき歓迎や応援をする
* エラー時にも責めず、やり直し方を案内する

---

## 12.2 話し方

* 「〜だよ」調
* ユーザーを「名前＋さん」で呼ぶ
* 「！」は比較的多め
* 絵文字は1メッセージにつき0〜1個程度
* 音声は一旦使用しない
* 強い命令形を避ける
* 行動や入力内容を評価しすぎない
* 短く、親しみやすく話す

---

## 12.3 推奨文言

### 読み込み

```text
スペースを確認しているよ
少しだけ待っててね
```

### 参加画面

```text
こんにちは！
ここは、みんなの気づきや想いを
自由に残せるスペースだよ
```

### 参加方法

```text
どの方法で参加する？
```

### ニックネーム

```text
このスペースで、
なんて呼ばれたい？
```

### 入力後

```text
みいさんだね！
準備できたよ
```

「いい名前だね」など、入力内容を評価する表現は原則使用しない。

### 再訪

```text
おかえり！
みいさんとして参加する？
```

### 入室直後

```text
みいさん、ようこそ！
まずはみんなの投稿を見てみてね
```

### エラー

```text
うまくいかなかったみたい
もう一度試してみてね
```

* `SET_SPACE_NICKNAMES` reducer で local + server を merge して保存
* 重複する `saveSpaceNicknames(nicknames)` を削除
* local-only guest nickname が reload / auth session 復元後も保持される

Production smoke test（2026-07-21）で guest reload 後の再訪 UI を確認済み（`/s/machizukuri`、`addInitScript` 不使用）。
