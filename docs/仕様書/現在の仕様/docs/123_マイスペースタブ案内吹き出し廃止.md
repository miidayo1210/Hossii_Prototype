## 1. 背景

PR #19（仕様 121）で、マイスペースの用語統一・導線改善・初回タブヒントが追加された。スマホ利用では、Pane タブ列上の **初回吹き出し**（`MY_SPACE_TAB_HINT`）が左上 UI と重なり視認性が悪い。また、マイスペース未作成・タブ非表示・通常タブ 1 件の共有スペースなどでも「右端にあります」系の案内が出ると、利用者が混乱する。

**ユーザー方針（確定）:** タブ上の位置案内吹き出しは **不要**。マイスペース機能の説明そのものは、アカウント・HOME 等で維持する。

---

## 2. 現状の問題

### 2.1 現象（現行）

スマホで共有スペースを開いたとき、次のような状態でも

- マイスペースが **未作成**
- マイスペースタブが **概念的に不要**（未ログイン・非メンバー等は別 UI）
- 通常 Pane タブが **1 件だけ**
- マイスペースと無関係な公開共有スペース

次の案内が表示されることがある。

| 種別 | 文言例 | 見た目 |
|------|--------|--------|
| **タブ初回吹き出し** | 「右端のマイスペースは、あなただけの場所です」 | `SpacePaneBar` 上の `.mySpaceTabHint` |
| **タブ非表示理由バナー**（別 UI） | 「ログインすると、タブ列の右端にマイスペースが表示されます。」等 | `SpaceScreen` 上部固定 `.mySpaceHiddenNote` |

吹き出しが **左上バー・タイトル周辺と重なり**、文字が読みにくい（特にモバイル）。

### 2.2 原因（現行・調査結果）

#### A. 初回吹き出し（`mySpaceTabHint`）

| 項目 | 内容 |
|------|------|
| 表示コンポーネント | `SpacePaneBar.tsx`（`personalShortcut` ラップ内） |
| 制御 | `SpaceScreen.tsx` の `showMySpaceTabHint` |
| 文言定数 | `MY_SPACE_TAB_HINT`（`mySpaceCopy.ts`） |
| 初回管理 | `localStorage` キー `hossii:my-space-tab-hint-seen`（`mySpaceTabHint.ts`） |

**表示条件（現行）:**

```text
isSharedSpaceShell(activeSpace)
AND canShowPersonalShortcut({
  isAuthenticated,
  !isVisiting,
  spaceCommunityId あり,
  membershipStatus === 'active'
})
AND NOT hasSeenMySpaceTabHint()
```

**重要:** 個人スペースの **作成有無は見ていない**。membership が `active` なら「マイスペース」ショートカットタブは常に出る（未作成時はタップで `ensureMyPersonalSpace`）。

**単一タブでも出る理由:**

```text
showPaneBar =
  shouldShowSpacePaneBar(...)   // 一般ユーザーは visiblePaneCount >= 2 のみ true
  OR (personalShortcutEligible && !isVisiting && !panesLoading)
```

PR #19 以降、一般ユーザーでも Pane が 1 件でも **マイスペースショートカットのためタブ列を強制表示** する。吹き出しはそのタブ列上に描画される。

**mobile / desktop:** 同一ロジック。`SpacePaneBar` に `mySpaceTabHint` を渡す（`variant` は mobile / desktop 両方）。

#### B. タブ非表示理由バナー（本吹き出しとは別・混同注意）

| 項目 | 内容 |
|------|------|
| 関数 | `getPersonalShortcutHiddenReason()`（`personalSpaceShortcut.ts`） |
| 表示 | `SpaceScreen.tsx` — モバイルは常時、デスクトップは `!isMobile` 時 |
| CSS | `.mySpaceHiddenNote` / `.mySpaceHiddenNoteDesktop`（画面上部 fixed） |

`personalShortcutEligible === false` のとき表示。ゲスト向け文言に **「右端」** が含まれる。

**本仕様の廃止対象は A（吹き出し）が主。** B は **維持** するが、受入条件に合わせ **「右端」位置表現の除去** を推奨する（6章）。

### 2.3 121・PR #19 との関係（現行）

PR #19 で追加された機能のうち、本仕様が **取り下げる** のは初回タブ吹き出しのみ。

| PR #19 機能 | 本仕様 |
|-------------|--------|
| `mySpaceCopy` 共通コピー | 維持（`MY_SPACE_TAB_HINT` 定数のみ削除候補） |
| アカウント `MY_SPACE_INTRO` 等 | **維持** |
| コミュニティ HOME コピー | **維持** |
| `getPersonalShortcutHiddenReason` バナー | **維持**（文言調整は可） |
| `mySpaceTabHint` + localStorage | **廃止** |
| アーカイブ表示修正 | **維持**（触らない） |

### 2.4 Hossiiガイド吹き出し（117）との競合

`origin/main` @ `38c44a6` に **117（guideBubble）は未マージ**。実装・表示チャネルともに **競合なし**。本仕様実装時も 117 と独立。

## 6. localStorage の扱い（確定）

| キー | 方針 |
|------|------|
| `hossii:my-space-tab-hint-seen` | **書き込み・参照を廃止**。`mySpaceTabHint.ts` を削除 |

他の localStorage（voice consent、community 選択等）には **触れない**。

### 6.1 非表示理由バナーの文言調整（確定・最小）

受入条件「『右端にマイスペースがあります』系の文言が表示されない」に合わせ、**維持する** `getPersonalShortcutHiddenReason` のゲスト向け文言から位置案内を除く。

| 現行 | 推奨（実装時） |
|------|---------------|
| ログインすると、タブ列の右端にマイスペースが表示されます。 | ログインすると、マイスペースタブが使えるようになります。 |

他 status（suspended / removed / invited / default）は **現行維持**（「右端」なし）。
