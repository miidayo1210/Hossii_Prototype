## 1. 背景と目的

Hossiiに投稿された回答を、活動の振り返り、授業・プログラムの分析、研究、記録保存などに活用できるようにする。

Googleフォームの回答スプレッドシートのように、

- 1つの回答を1行として表示する
- 回答項目を列として並べる
- 投稿日時や対象タブ等の基本情報を付加する

形式で出力する。

管理者と個人では取得可能な範囲を明確に分離し、サーバー側で権限を確認する。

---

## 2. 用語

### 2.1 回答

Hossiiの投稿フォームから利用者が送信し、`hossiis` に保存された投稿1件を「1回答」とする。

Phase 1では次を対象とする。

- 利用者が投稿操作によって作成した投稿
- `message` / `emotion` / `hashtags` / `number_value` / `image_url` 等を持つ投稿
- canvas投稿も、利用者の投稿操作によるものなら含む

除外する。

- システムが自動生成した記録
- 管理操作ログ
- いいね
- コメント
- 削除済み投稿（`deleted_at IS NOT NULL`）
- 非表示投稿（`is_hidden = true`）

**判定条件について:** 本仕様書上、`origin` / `logType` 等の列の有無を断定しない。現行DBで自動生成投稿を正確に判別できない場合は、実装前に正確な判定条件を確認する。

### 2.2 マイスペース

- `spaces.space_type = 'personal'` の実スペース
- `owner_user_id` が所有者
- `personalShortcut` はUI導線であり、export単位ではない（[121](121_マイスペースの説明と導線改善.md)）
- export対象は personal space の `space_id`

### 2.3 タブ

- 正本テーブル: `space_panes`
- CSVではタブ名を列として出力する
- 特定タブ指定時はその Pane の投稿のみ対象とする

## 3. 利用者・権限・対象投稿

### 3.1 権限表

| export種別 | space_type | 実行者 | 対象投稿 |
|---|---|---|---|
| 管理者 | shared | community admin | `visibility = public`、非hidden、非deleted |
| 個人 | shared | 参加中の本人 | authorshipに紐づく本人投稿、非hidden、非deleted |
| マイスペース | personal | owner本人 | authorshipに紐づく本人投稿、`public` および `owner_only`、非hidden、非deleted |

### 3.2 管理者エクスポート（Phase 1A）

Phase 1Aでは次に限定する。

| 項目 | 内容 |
|---|---|
| 対象スペース | コミュニティに所属する **shared space** のみ |
| 実行者 | **community admin** のみ |
| space admin / owner のみ | 不可（[68](../68_スペース書き出し機能%20.md) と同様） |
| community なし space | 対象外 |
| personal space | 管理者エクスポート対象外 |
| super admin | RPC上は許可可能。通常UIには super admin 専用導線を出さない |
| 権限確認 | 実行時点でサーバー側で再確認する |

投稿者は全員（ゲスト・通常アカウント・participant account を含む）。他コミュニティのスペースは取得不可。

### 3.3 個人エクスポート（Phase 1B）

| 項目 | 内容 |
|---|---|
| 対象スペース | 自分が **参加中** の shared space |
| 本人判定 | **`hossii_authorships` のみ**（[109](../109_コミュニティ所属個人アカウント・個人スペース・招待制スペース.md)） |
| nickname 一致 | 禁止 |
| raw `author_id` fallback | 禁止 |
| クライアントから `auth_user_id` を渡して本人判定 | 禁止 |
| participant account | authorship があれば含む（[101](../101_管理者発行参加者アカウント.md)） |
| legacy authorship 欠損投稿 | 含めない |
| 未紐づけ guest 投稿 | 含めない |
| membership / アクセス権失効後 | 取得不可 |

### 3.4 マイスペースエクスポート（Phase 1C）

個人エクスポートと **同一RPC** を共用する。対象は `space_type = 'personal'` かつ `owner_user_id` が本人の space のみ。他人のマイスペースは指定不可。

## 4. 投稿状態の扱い

### 4.1 管理者CSV

**含める**

- `visibility = 'public'`
- `deleted_at IS NULL`
- `is_hidden = false`（または NULL）

**含めない**

- `owner_only`
- hidden（`is_hidden = true`）
- deleted

### 4.2 個人CSV（shared space）

- authorship に紐づく本人投稿
- 本人がアクセス可能なスペース内の投稿
- 非hidden、非deleted

`visibility` による追加除外は Phase 1B では行わない（本人が閲覧できる public 投稿を対象とする）。

### 4.3 個人CSV（personal space / マイスペース）

- authorship に紐づく本人投稿
- `visibility = 'public'` および `visibility = 'owner_only'`
- 非hidden、非deleted

## 5. participation_mode

[123](123_URLからスペース参加UI.md) の `spaces.participation_mode` と整合させる。

- `guest_only`
- `account_only`
- `guest_and_account`

export時に participation_mode による **新たな除外処理は行わない**。DB上に存在し、権限・投稿状態条件を満たす回答を出力する。

例:

- `account_only` では通常 guest 投稿は存在しない
- `guest_only` では個人 authorship export の対象が 0 件の場合がある

## 6. エクスポートの基本単位

初期実装では **1スペース単位** とする。

| 指定項目 | Phase 1 |
|---|---|
| 対象スペース | 必須 |
| 対象タブ | すべて／特定タブ |
| 対象期間 | 全期間のみ |
| 投稿者 | 権限により自動決定 |
| 出力形式 | CSVのみ |

期間指定は Phase 2。Phase 1 では UI に期間選択を表示しないか、変更不可表示とする。

## 7. 対象タブとCSVの関係

### 7.1 特定タブ

選択した `space_panes` に投稿された回答のみ出力する。

### 7.2 すべてのタブ

すべての表示対象タブの回答を **1つのCSV** に統合する。各行にタブ名列を付ける。

CSV形式には複数シートを持たせられないため、Googleスプレッドシート上のシート分割は行わない。将来的に複数CSVのZIP出力を Phase 2 で検討する。

## 8. CSVの行と列

### 8.1 基本構造

Googleフォームの回答一覧と同様、**1回答 = 1行** とする。

- 1行目: 列名
- 2行目以降: 回答

### 8.2 Phase 1A 標準列（管理者）

| 順序 | 列名 | 内容 |
|---|---|---|
| 1 | 投稿日時 | ISO 8601 |
| 2 | タブ名 | 対象タブ表示名 |
| 3 | スペース内匿名ID | 同一人物の回答をまとめる匿名識別子（§9） |
| 4 | 本文 | `message` |
| 5 | 気持ち | `emotion` |
| 6 | タグ | `hashtags` |
| 7 | 数値 | `number_value` |
| 8 | 画像あり | true / false（`image_url` の有無） |

**CSV列に含めないメタデータ（ファイル名または RPC 内部で扱う）:**

- 投稿ID（`hossii_id`）
- スペース名（ファイル名に含む）
- 投稿者種別（回答分析に不要）
- 投稿種別（Phase 1A では実質 `bubble` のみ）
- エクスポート日時（ファイル名に含む）

現行DBでは投稿項目が固定列に保存される。実装時の対応関係の正本:

| フォーム項目（[86](../86_投稿フォーム項目設定.md)） | DB列 |
|---|---|
| message | `message` |
| emotion | `emotion` |
| tags | `hashtags` |
| photo | `image_url` |
| numberPost | `number_value` |
| bubbleColor | `bubble_color` |

### 8.3 任意列（管理者のみ・初期OFF）

| 列名 | 初期値 | 注意 |
|---|---|---|
| 投稿者表示名 | OFF | ON時: 「表示名には、投稿者が入力した個人情報が含まれる可能性があります。」 |
| 画像URL | OFF | ON時: 「画像URLはCSVの共有先から閲覧される可能性があります。」 |

### 8.4 個人CSV（Phase 1B / 1C）

- §8.2 と同型の **回答中心列**（投稿日時・タブ名・本文・気持ち・タグ・数値・画像あり）を基本とする
- 投稿者表示名は不要
- スペース内匿名IDは **省略可**（本人 export のため）
- 投稿ID・スペース名・投稿者種別・投稿種別・エクスポート日時は §8.2 と同様 CSV 列に含めない

### 8.5 絶対に含めない列

- email
- `auth_user_id`
- raw `author_id`
- `login_id`
- participant slot
- password
- `like_count`
- `deleted_at`
- raw `visibility`
- `hidden_at`
- `hidden_by`

投稿ID（`hossii_id`）は Phase 1A 管理者 CSV の **列には含めない**（ファイル名でスペース・出力日を識別する）。

### 8.6 全タブ時の列構造

- 1回答 = 1行、回答項目 = 列（Googleフォーム風）
- **全タブ export:** space の基本投稿項目と、全 Pane の postFields override（[93](../93_Hossiiタブ機能追加.md)）の **和集合** を列とする
- 特定タブ専用の項目は、他タブの回答行では **空欄**
- **特定タブ export:** その space + Pane で有効な項目のみを列とする

### 8.7 複数値の区切り

タグ等の複数値はセル内で `｜` 区切りとする。カンマ区切りはCSVの区切り文字と混同しやすいため使用しない。

例: `挑戦｜気づき｜次の一歩`

## 9. 投稿者の匿名化

### 9.1 管理者エクスポート

同一人物の回答をまとめて分析できるよう、**スペース内で一貫する匿名識別子** を付与する。

例（表示形式は実装で決定）: `R-001`, `R-002`

### 9.2 匿名IDの条件

- 同一 space 内では同じ投稿者に同じ ID
- 別 space では同一人物でも異なる ID でよい
- raw ID から復元・推測できない値とする
- **RPC 内で生成** する。クライアント側生成は禁止
- raw identifier（`auth_user_id`, raw `author_id`, `login_id`, slot 等）をレスポンスへ返さない

具体的な salt / HMAC / SQL は技術実装計画側で決定する。本仕様書では実装詳細を定めない。

### 9.3 投稿者表示名

管理者CSVでは **opt-in（初期OFF）** の任意列。本人確認の根拠には使用しない。

## 10. 出力形式・ファイル名

### 10.1 CSV技術仕様

| 項目 | 内容 |
|---|---|
| 形式 | CSV のみ |
| 文字コード | UTF-8 with BOM |
| 準拠 | RFC 4180 |
| エスケープ | カンマ・引用符・改行を適切に escape。セル全体をダブルクォートで囲み、内部 `"` は二重化 |
| 生成 | 文字列連結のみでCSVを作らず、共通関数またはライブラリを使用 |
| 日時 | ISO 8601（例: `2026-07-21T22:35:10+09:00`）。UTC保存時もタイムゾーンを曖昧にしない |
| 0件 | CSVを生成せず「エクスポートできる回答がありません」を表示 |
| 保存 | CSV本体をサーバーへ保存しない |


### 10.1.1 スプレッドシート向け Formula Injection 対策

Excel / Google スプレッドシート等がセル内容を数式として解釈しないよう、**ユーザー入力由来の文字列列** に対して export 前処理を行う。

**対象列（Phase 1A）**

- タブ名
- 本文
- 気持ち
- タグ
- 投稿者表示名（opt-in ON 時）
- 画像URL（opt-in ON 時）
- スペース内匿名ID

**判定**

- セル値の先頭から、空白・改行・NBSP・zero-width 文字・BOM を無視したうえで、次のいずれかで始まる場合を formula 相当とみなす
  - `=`
  - `+`
  - `-`
  - `@`

**処理**

- 該当セルの先頭に `'`（シングルクォート）を付与してから RFC 4180 escape する
- これによりスプレッドシート上では文字列として扱われる

**改変しない型**

- ISO 8601 日時（投稿日時）
- number（数値）
- boolean（画像あり）

**実装正規表現（参考）**

```regex
/^[\s\uFEFF\u200B-\u200D\u2060]*[=+\-@]/
```

### 10.2 ファイル名

| 種別 | 形式 |
|---|---|
| 管理者 | `hossii_{スペース名}_{タブ名}_全回答_{出力日}.csv` |
| 管理者（全タブ） | `hossii_{スペース名}_全タブ_全回答_{出力日}.csv` |
| 個人 | `hossii_{スペース名}_{タブ名}_自分の回答_{出力日}.csv` |
| マイスペース | `hossii_マイスペース_{タブ名}_自分の回答_{出力日}.csv` |

ファイル名に使用できない記号（`/ \ : * ? " < > |`）は削除または `_` に置換する。

## 12. 権限と安全性

エクスポート対象の抽出と権限確認は **必ずサーバー側** で行う。ブラウザで全投稿を取得してからクライアント側で絞り込む方式は禁止する。

**管理者:** 認証済み、community admin、対象 shared space が当該 community に所属、権限が実行時点で有効。

**個人:** 認証済み、active な space membership、投稿が本人 authorship に紐づく。クライアントから渡された user ID を信用しない。

**マイスペース:** `owner_user_id` が本人。URLや ID を改変しても他人の回答を取得できない。

大量データのタイムアウト回避のため、サーバー側でページングして取得する。Phase 1 では同期出力とし、上限値は実装前に既存投稿件数と処理性能を確認して決める。

## 13. 投稿時の説明とプライバシー

### 13.1 管理者エクスポートの説明（本仕様の範囲）

スペース内の投稿前または投稿フォーム付近に、管理者が匿名性に配慮して回答をエクスポートする場合がある旨を表示できるようにする。

短縮例:

> 投稿内容は、匿名性に配慮したうえで、運営者が振り返りや分析に利用する場合があります。

### 13.2 投稿時の個人情報注意

「投稿に個人情報を書かない」注意はエクスポートの有無にかかわらず Hossii 全体に必要。配置・文言・頻度は [126](126_投稿時のプライバシー・個人情報注意表示.md) で定める。本仕様書では管理者エクスポート可能性の説明要件のみを扱う。

## 14. エクスポート前の確認

管理者がエクスポートする際、確認画面を表示する（Phase 1 では毎回表示してもよい）。

```text
回答履歴をエクスポートします

このCSVには、スペース内で投稿された回答が含まれます。
回答者のメールアドレスや認証情報は含まれません。

取得したデータは、スペースの運営・振り返り・分析など、
参加者に説明した目的の範囲内で取り扱ってください。

［キャンセル］  ［CSVをエクスポート］
```

投稿者表示名または画像URLを ON にする場合は、§8.3 の注意を追加表示する。

## 19. 受入条件

**管理者（Phase 1A）**

- 管理対象 shared space の public 回答のみ出力できる
- personal space / community なし space / 他 community は取得できない
- space admin / owner のみでは取得できない
- ゲストを含む全回答が対象（DB上存在する分）
- メールアドレス等・raw identifier が含まれない
- 指定タブに絞れる。1回答 = 1行
- 表示名・画像URLは初期OFF
- 標準CSVは8列。メタデータ列（投稿ID・スペース名・投稿者種別・投稿種別・エクスポート日時）は含めない

**個人（Phase 1B）**

- authorship 本人投稿のみ。nickname 一致・author_id fallback なし
- 未紐づけ guest・legacy authorship 欠損は含まれない
- アクセス権失効後は取得不可

**マイスペース（Phase 1C）**

- owner 本人のみ。`owner_only` 本人投稿を含む

**CSV**

- UTF-8 BOM、RFC 4180、ISO 8601
- 0件時はCSV生成しない
- Googleスプレッドシートで行単位に並ぶ
- **標準列は8列**（§8.2）
- 標準列に **含めない:** 投稿ID、スペース名、投稿者種別、投稿種別、エクスポート日時
- スペース名と出力日は **ファイル名**（§10.2）で識別する
- **任意列（opt-in・初期OFF）:** 投稿者表示名、画像URL（§8.3）
- Formula Injection 対策を §10.1.1 に従い適用する

- 自動・定期送信
