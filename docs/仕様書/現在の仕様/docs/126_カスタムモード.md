カスタムモード｜Hossii同士を「糸」でつなぐ機能

分類: [Core / Space / Custom Mode / Connection UI]
機能名: Hossiiの関連づけ・糸のつながり
ステータス: 📋 仕様策定中（Phase 0 調査済み）
対象画面: PC版 `PresentationMode === 'custom'`（Bubble 表示）
初期対象権限: community admin または super admin（糸の作成・編集・解除）。一般参加者は閲覧のみ
関連仕様: [84_スペース表示モード拡張](../84_スペース表示モード拡張.md)
目的: つながりを常時表示して空間を複雑にするのではなく、Hossiiを選び、覗き、辿るときだけ関係が浮かび上がる体験をつくる

1. 背景

スペースに投稿が増えてくると、投稿同士の間に、

なんとなく近そう
同じテーマに関係していそう
一方がもう一方のきっかけになっていそう
今は説明できないが、一緒に見ておきたい

といった、明確に分類しきれない関係が見えてくる。

これらをタグやフォルダで整理するのではなく、カスタムモード上で投稿同士を自由に動かしながら、仮の関係として糸でつないでおける機能を追加する。

投稿が蓄積された空間を触りながら、パズルの一部が少しずつ組み合わさるように、新しい関係やまとまりを発見できる体験を目指す。

2. コンセプト
ふわっと、つながる。

Hossiiは、宇宙や水面のような空間に、ぷかぷか浮かぶ小さなかけらとして存在する。

気になる2つのHossiiを糸でつなぐと、互いを少しだけ意識するようになる。

強く固定したり、ひとつのグループに閉じ込めたりするのではなく、

仮につなぐ
後から強さを変える
違うと思ったらほどく
つながりを見ながら配置を変える

ことができる。

本機能は、情報を正確に分類するためではなく、関係を試しながら考えるための機能とする。

3. 用語
用語	意味
糸	2つのHossiiの間に設定された、方向を持たないつながり
つないでみる	新しい糸を作成する操作
つながりを見る	「つながり N」メニューから 1 階層先のつながりを表示する操作（§10.3）
糸の強さ	2つのHossiiがどの程度近いと感じるかを示す3段階
糸をほどく	設定済みの糸を解除する操作
結び目	糸とHossiiが接続する部分に表示する小さな装飾
接続点	Hossiiまたは吹き出しの輪郭上に設ける、糸の始点・終点
connection編集状態	糸の作成・編集操作中の内部状態（画面上に「つながりモード」とは表示しない）

ユーザー向けの主な文言
場面	文言
操作開始	つないでみる
相手選択	近そうなHossiiを選んでね
強さ選択	糸の強さをえらぼう
保存	つなぐ
解除	糸をほどく
つながり一覧	つながり N（Nは接続数）

糸の作成は、カスタムモード内でHossiiを選択した際に表示される操作のひとつとして扱う。独立した「つながりモード」という表示・名称は使用しない。

内部的にはconnection編集状態を持ってよいが、画面上に「つながりモード」とは表示しない。

内部の実装名はconnectionまたはrelationとしてよいが、ユーザーに表示する文言は、原則として「糸」「つなぐ」「ほどく」に統一する。

3.1 用語の区別（実装前提）

本仕様でいう「カスタムモード」は、スペース画面の **`PresentationMode === 'custom'`**（Bubble / `Tree.tsx` による吹き出し表示）を指す。

これは管理画面の **`SpaceModeId: 'custom'`**（スペースプリセット・投稿項目設定）とは **別概念** である。混同しないこと。

| 用語 | 意味 |
|------|------|
| `PresentationMode 'custom'` | 星モード（StarView）ではなく Bubble を描画する表示モード |
| `SpaceModeId 'custom'` | スペース設定のカスタムプリセット。糸機能とは無関係 |

4. 基本方針
4.1 方向を持たない関係

糸は、親子関係や因果関係を表す矢印ではない。

AからB、BからAという方向は持たず、2つのHossiiが対等につながっている状態として扱う。

4.2 関係を確定しすぎない

糸は、正しい分類や確定済みの関係ではない。

「今のところ近そう」という関係を、あとから自由に変更・解除できるものとする。

4.3 必要なときだけ、関係が浮かび上がる

糸を常時すべて表示すると、投稿、吹き出し、操作メニュー、糸、結び目が重なり、空間が複雑になる。

通常時は糸そのものを原則として表示せず、Hossiiを選び、覗き、辿るときだけ関係を浮かび上がらせる。

4.4 可愛さと軽さを優先する

糸は硬い直線ではなく、少し曲がり、たるみ、伸びる表現とする。

本格的な物理演算は初期実装に含めず、軽量なSVGまたはCanvas表現を優先する。

5. 対象デバイスと有効条件

本機能の初期実装は、**Bubble が実際に描画されている画面** のみを対象とする。

画面幅だけで判定せず、次の **有効条件（`connectionsEnabled`）をすべて満たすとき** のみ糸 UI を表示・操作可能とする。

```
connectionsEnabled =
  !renderAsStar                    // PresentationMode === 'custom' かつモバイル常時星を除く
  && presentationMode === 'custom'
  && layoutMode !== 'byAuthor'    // 投稿者別まとめは MVP 対象外（§7）
  && viewMode !== 'slideshow'
```

| 条件 | 理由 |
|------|------|
| `presentationMode === 'custom'` | 星モード（StarView）では個別 Bubble が存在しない |
| `!renderAsStar` | 狭幅モバイル縦など、幅に関わらず StarView 固定の環境を除外 |
| `layoutMode !== 'byAuthor'` | クラスター表示では個別 Hossii DOM がない（§7） |
| `viewMode !== 'slideshow'` | スライドショー中は対象外 |

初期対象
- PC 相当環境で Bubble が描画されていること
- `layoutMode === 'random'` または `'ordered'`
- マウスまたはトラックパッド操作

初期対象外
- スマートフォン（`renderAsStar === true` の環境）
- 星モード（`presentationMode === 'stars'`）
- `layoutMode === 'byAuthor'`（投稿者別まとめ表示）
- タッチ操作を前提とした糸編集
- 横持ち等、画面幅だけ広く Bubble が描画されない環境

`connectionsEnabled === false` のとき、糸の描画・操作メニュー・結び目印は **一切表示しない**（取得 API も呼ばない）。

6. UIの基本思想

本機能は、全体の関係図を一度に閲覧するためのものではない。

Hossiiを起点として、小さな星座をひとつずつ覗いていく体験を目指す。

何かがつながっていることに気づく
気になるHossiiを選ぶ
その周囲のつながりだけを見る
必要に応じて、さらに先へ辿る

という段階的な体験を採用する。

7. 接続可能な範囲

MVPでは、以下の条件を **すべて** 満たす Hossii 同士のみ接続できる。

- `connectionsEnabled === true`（§5）
- `layoutMode === 'random'` または `'ordered'`（`'byAuthor'` は対象外）
- 同じスペースに属している
- 同じ Pane に属している
- 現在のカスタム空間に表示されている（`filteredHossiis` に含まれる）
- 削除されていない（`deleted_at IS NULL`）
- 非表示状態ではない（`is_hidden`）
- archived スペースではない
- 自分自身ではない

別 Pane 間のつながりは初期実装に含めない。

**Pane 切替時**（§10.2, §12, §32）: 現在 Pane に属する糸のみを取得・表示する。あわせて選択状態・connection 編集状態・仮糸・操作メニューを **すべて解除** する。

8. 糸の強さ

糸の強さは3段階とする。

数値ではなく、感覚的な名前と説明を表示する。

8.1 ほのか

表示文言:

なんとなく近いかも

見た目:

細い
半透明
たるみが大きい
必要に応じて部分的な点線
結び目は小さい

動き:

一方のHossiiを移動しても、もう一方は基本的に動かない
糸だけが伸びて追従する
引っ張り操作中は、接続先はほとんど動かず淡く光る程度

内部値: soft

8.2 やわらか

表示文言:

関係がありそう

見た目:

標準的な太さ
なめらかな曲線
適度なたるみ
結び目が少し見える

動き:

一方を移動した際、もう一方が数pxだけ引かれるように揺れる
手を離すと元の位置へ戻る
引っ張り操作中は、接続先が数px揺れ、少しだけ引かれる

内部値: medium

8.3 しっかり

表示文言:

かなりつながっている

見た目:

やや太い
たるみが小さい
少し張りのある曲線
結び目が少し大きい
選択時に淡く発光する

動き:

一方を移動した際、もう一方も少し引かれる
完全には固定せず、独立した位置を保つ
引っ張り操作中は、接続先がやや明確に反応し、結び目と糸が少し強く光る

内部値: strong

9. 糸のビジュアル
9.1 基本表現

糸は、次の特徴を持つ。

完全な直線にはしない
Hossii同士の距離に応じて緩やかにカーブする
少しだけたるみを持つ
半透明で、背景を大きく遮らない
接続部分に小さな結び目を表示する
選択時・hover時だけ少し明るくなる

イメージは、毛糸そのものよりも、光の糸と水中を漂う繊維の間とする。

9.2 結び目

糸の両端には、小さな結び目を設ける。

候補：

小さな丸
星の粒
ぷくっとした光
小さなリボン状の形

初期実装では、小さな丸または星粒程度のシンプルな表現とする。

結び目は通常時には控えめに表示し、次の場合に明確にする。

Hossii選択時
糸hover時
つながり閲覧時
引っ張り操作中

9.3 色

MVPでは、ユーザーによる色選択は行わない。

スペースの背景やHossiiの色に対して見える、淡い共通色を使用する。

ただし、糸の強さを色だけで区別しない。太さ、たるみ、透明度を組み合わせて表現する。

9.4 重なり順と SVG overlay 配置

表示の重なり順は、原則として以下とする。

```
背景（壁紙・StarLayer）
  ↓
SVG overlay（`bubbleArea` 内、`data-bubble-area`）
  ↓
Hossii・吹き出し（Bubble）
  ↓
既存編集 UI（リサイズハンドル、カラーパレット、本人操作バー、editToolbar）
  ↓
コンパクト操作メニュー・強さ選択ポップオーバー（portal 可）
  ↓
モーダル・ポップオーバー
```

**SVG overlay の配置（MVP 確定）**

- 親: `bubbleArea`（`SpaceScreen` 内、`data-bubble-area` 付き div）
- スタイル: `position: absolute; inset: 0; z-index: 0; pointer-events: none`（糸 hit area のみ `pointer-events: stroke`）
- Bubble より **後ろ**、背景より **前**
- `ScaledContent` の `displayScale` zoom 配下に置き、座標追従を DOMRect ベースで行う

糸が本文や操作ボタンの上に表示されないようにする。

9.5 星座をつなぐような描画表現

糸の作成操作は、メニューやフォームで完結するだけではなく、Hossii同士を星座の点と点のようにつないでいく感覚を重視する。

基本イメージ
最初のHossiiを選択すると、接続点が小さく光る
相手を選ぶまで、接続点から仮の糸がポインターへ伸びる
別のHossiiに近づくと、相手側の接続点が反応して光る
相手をクリックすると、糸が「ピッ」と結ばれる
結ばれた瞬間に小さな星粒が弾ける
操作は重くせず、軽く、短く、気持ちよく完了する

糸を引く動作は、本格的な線描画ではなく、点から点へピッピッと星座を結ぶ操作として設計する。

仮の糸

相手を選ぶ前の糸は仮表示とする。

少し薄い
点線または光の粒を含む
ポインターの動きに追従する
保存前であることが見た目で分かる
キャンセルすると消える

相手のHossiiに近づいた際は、仮の糸が軽く吸い寄せられるように接続点へスナップしてよい。

9.6 接続点の表現

糸は Hossii の中心から直接伸ばさず、Bubble 矩形の辺上に接続点を設ける。

**MVP の接続点計算（確定）**

1. 各 Bubble の `getBoundingClientRect()` で DOMRect を取得
2. 2 点の **矩形中心** を結ぶ方向ベクトルを算出
3. 中心から相手方向へ伸ばした半直線と、Bubble 矩形の辺（padding 適用後）との **交点** を接続点とする
4. `bubbleArea` 基準の SVG 座標へ変換して描画

**padding（UI 回避）**

接続点が次の UI と重ならないよう、矩形に方向別 padding を設ける。

| 回避対象 | 目安 |
|----------|------|
| いいね badge（右下外側） | 右・下方向に 12〜16px |
| 本人操作バー（選択時・直下） | 下方向に 24〜32px |
| リサイズハンドル・カラーパレット | 四隅から 8〜12px |

padding 適用後の矩形で交点を求める。交点が取れない場合は、padding なし矩形の最近傍辺に fallback する。

Hossii を移動した場合、接続位置を毎フレーム再計算する（`requestAnimationFrame` 推奨）。

9.7 糸を結ぶ瞬間

糸の作成時に、短いアニメーションを入れる。

推奨表現
接続点が一度だけぷくっと膨らむ
糸が端から端へ短時間で描かれる
小さな星粒が1〜3個だけ散る
糸が一度だけ軽く揺れる
0.2〜0.5秒程度で完了する

長い演出や、操作を待たせる演出は避ける。

糸をほどく瞬間
糸が中央または端からふっと消える
小さな光の粒にほどける
Hossii自体は動かさない

10. 糸を常時見せない3段階の表示
10.1 通常状態

通常時は、糸そのものを原則として表示しない。

つながりを持つHossiiには、小さく控えめな印だけを表示する。

表示候補
小さな結び目
小さな星粒
ごく淡い光
接続数を示す小さな数字
結び目と接続数の組み合わせ

例：

✦ 3

この表示によって、

このHossiiには、何かつながりがある

ことだけが分かる。

表示ルール
本文やリアクションを隠さない
常時アニメーションしない
接続数が0件の場合は表示しない
数字を表示する場合も、視線を奪わない大きさにする
同じHossiiに複数の印を重ねない

10.2 Hossii選択状態

Hossii または吹き出しを短くクリックすると、その Hossii を **選択状態**（既存 `selectedBubbleId`）にする。

**既存選択 UX は置き換えない**（§11）。選択と同時に、従来どおり次を表示する。

- 4 角リサイズハンドル・カラーパレット（編集権限あり時）
- 本人操作バー（`canManageOwn` 時）
- 画面下部 `editToolbar`
- **新規** コンパクト操作メニュー（Bubble 近傍）

選択された Hossii に直接つながる糸のみを、ふわっと表示する。

表示内容
- 選択した Hossii につながる糸
- 1 階層先の接続先 Hossii
- 接続先 Hossii の淡い発光
- 糸の強さに応じた形状
- 接続先ごとの小さな結び目

選択した Hossii に関係しない糸は表示しない。

解除

以下の操作で **選択・connection 編集状態・仮糸・操作メニューをすべて解除** する。

- 背景をクリック
- Esc キー
- 別の Hossii を選択
- カスタムモード（Bubble 表示）を終了
- **Pane を切り替える**

10.3 つながりを広げて見る状態

コンパクト操作メニューの **「つながり N」** を選択すると、対象 Hossii から 1 階層先のつながりを明示的に表示する（N は直接接続数。0 件のときは項目非表示または空状態）。

さらに、接続先の Hossii を選ぶことで、その Hossii を新しい起点として、次の 1 階層を表示できる。

基本ルール
- 一度に表示するのは、現在の起点から 1 階層先まで
- 接続先を選ぶと、起点が切り替わる
- 以前の起点の糸は薄く消える
- 関係を全件一括展開しない
- パンくずや複雑なツリーナビゲーションは初期実装に含めない

「広げる」というより、ひとつずつ辿る操作とする。

11. Hossii選択時の UI 構成

### 11.1 既存 UX（維持・置き換え禁止）

カスタムモードの Bubble 選択フローは **現行実装を維持** する。

| 操作 | 挙動 |
|------|------|
| 1 回目クリック（未選択） | `selectedBubbleId` を設定（選択） |
| 選択済み + 距離閾値超ドラッグ | 位置移動（§12.1） |
| 選択済み | リサイズハンドル、カラーパレット（`canEdit` 時） |
| 選択済み + `canManageOwn` | 本人操作バー（編集・公開範囲・削除） |
| 選択済み | 画面下部 `editToolbar` |

投稿の編集・削除・配置変更は、従来の UI から行う。コンパクト操作メニューに「編集する」項目は **MVP では置かない**。

### 11.2 コンパクト操作メニュー（新規追加）

選択中 Hossii の近傍に、**既存 chrome と併存** する小さなカード型メニューを追加する。

**MVP 表示項目（この 3 つのみ）**

| 項目 | 挙動 |
|------|------|
| くわしく見る | `PostDetailModal` を開く（星クリックと同等の詳細閲覧） |
| つないでみる | 糸作成フロー開始（§13）。**community admin / super admin のみ表示** |
| つながり N | 直接接続一覧ポップオーバー（§14）。N=0 のとき非表示 |

表示例（つながり 3 件あり・管理者）:

```
くわしく見る
つながり 3
つないでみる
```

表示例（つながりなし・一般参加者）:

```
くわしく見る
```

（「つないでみる」は権限なしのため非表示）

見た目
- ぷくっとした小さなカード、角丸、軽い影
- 選択 Hossii を隠しすぎない（portal または absolute 配置）
- 「つないでみる」は小さな糸または接続点アイコン付き
- `HossiiFullTextPopover` と同様、`createPortal` で zoom 外配置してよい

「つないでみる」は独立したモード切替ではなく、このメニュー内のひとつの操作とする。

12. ポインター操作の優先順位

カスタムモード内で、操作同士が競合しないよう、以下の優先順位を採用する。

12.1 通常状態

| 操作 | 挙動 |
|------|------|
| 短いクリック（未選択） | Hossii を選択。既存 chrome + コンパクト操作メニューを表示 |
| ドラッグ（選択済み・閾値超） | Hossii を移動（配置保存） |
| 長押し | 現行仕様を維持。Bubble 上には新操作を割り当てない |
| ダブルクリック | Bubble 上では使用しない（背景 dblclick はクイック投稿） |
| 背景クリック | 選択・connection 状態・メニューを解除 |
| Esc | 選択・connection 状態・メニューを解除 |

**クリックとドラッグの分離（Bubble 移動）**

- 現行の「未選択クリック＝選択のみ、選択済み pointerdown＝ドラッグ開始」フローは維持する
- 選択済み Bubble の移動開始に **5〜6px の距離閾値**（`DRAG_THRESHOLD_PX`、既存 `AuthorClusterBubble` / `SpacePaneBar` と同程度）を追加する
- 閾値未満の pointer up はクリック扱いとし、誤ドラッグを防ぐ
- いいね・Pin・owner bar・画像等の `stopPropagation` ルールは維持

16. 引っ張ると、つながりが視覚的に釣れる
16.1 目的

結び目を引っ張ることで、

このHossiiには何がつながっているんだろう？

という関係を、空間の中から視覚的に引き出せるようにする。

これはHossiiや投稿の保存位置を変更する操作ではない。

関係を覗くための、一時的な視覚表現である。

16.2 操作

Hossiiに表示された小さな結び目を、ポインターでドラッグする。

ドラッグ中は、対象Hossiiからつながる糸が引き出される。

見た目の流れ
結び目が少し膨らむ
糸がポインター方向へ伸びる
直接つながるHossiiが少し揺れる
接続先Hossiiが淡く浮かび上がる
その先にもつながりがある場合、小さな星粒が現れる
ポインターを離すと、すべて元の表示へ戻る

16.3 接続先Hossiiの反応

接続先 Hossii の **保存座標（`positionX` / `positionY`）および DOM の `left` / `top` は変更しない**。

ドラッグ中のみ、次の **一時的な視覚効果** を適用する。

- CSS カスタムプロパティ（例: `--connection-pull-x`, `--connection-pull-y`）と **`transform: translate(...)` のみ**
- 数 px 程度の揺れ、淡い発光、軽い浮遊
- 糸の方向へ少し引かれる表現

`requestAnimationFrame` で SVG パスを更新してよいが、**React state の毎フレーム更新は避ける**。

pointer up またはキャンセル時に、transform と CSS 変数をクリアし、元の表示へ戻す。

**DB 更新・`handlePositionSave`・配置保存 API は呼び出さない。**

16.4 その先のつながり

直接接続されたHossiiが、さらに別の糸を持っている場合、その存在だけを小さな星粒で示す。

表現
接続先Hossiiの輪郭付近に小さな星粒
接続数に応じて最大1〜3粒
2階層目の糸そのものは表示しない
星粒をクリックした場合の展開は初期実装に含めない

これにより、画面を複雑にせず、

この先にも何かある

ことだけを伝える。

17. ごちゃつかせないための表示ルール

以下を必須ルールとする。

- 通常時は、すべての糸を常時表示しない
- 選択中の Hossii に関係する糸だけを表示する
- 一度に表示するのは、原則として 1 階層先までとする
- 2 階層目は、糸ではなく星粒のみで示す
- コンパクト操作メニューは MVP で最大 3 項目
- 糸の強さは、色だけでなく、太さ・張り・たるみ・透明度で表現する
- 結び目と接続数は、小さく控えめにする
- 糸を表示しても、投稿本文や操作 UI を隠さない
- 糸作成中以外は、Hossii の通常操作（選択・移動・いいね等）を優先する
- 保存済みの Hossii 配置を、糸によって自動変更しない
- 関係は「一覧で管理する」より、「覗く・辿る・引き出す」体験として扱う
- 全体のネットワーク図を初期画面に表示しない
- 同時に複数の Hossii を選択状態にしない
- 選択対象以外の糸は非表示、または十分に薄くする
- 引っ張り中でも、2 階層目の糸は描画しない
- **`connectionsEnabled === false` の環境では糸 UI を一切出さない**

18.1 基本

Hossiiを移動すると、接続されている糸の始点・終点も追従する。

糸の描画位置は、移動中も更新する。

18.2 MVP

初期実装では、Hossiiの位置そのものを糸によって自動変更しない。

次の表現のみ実装対象とする。

糸が伸びる
糸のカーブが変わる
接続先のHossiiが数px揺れる（引っ張り操作時のみ一時的）
手を離した時に糸がぷるんと戻る

18.3 将来拡張

将来的には、強さに応じて接続されたHossiiが少し引かれる物理表現を検討する。

ただし、ユーザーが保存した配置を勝手に変更しないことを優先する。

19. 権限

19.1 MVP（Phase 1）

| 操作 | 対象 |
|------|------|
| 糸の作成・編集・解除 | **community admin**（`AppUser.isAdmin`）または **super admin**（`AppUser.isSuperAdmin`）のみ |
| 糸の閲覧（✦N、選択時表示、一覧、引っ張り覗き） | スペースにアクセスできる全参加者 |
| archived スペース | 作成・編集・解除不可（閲覧はスペースポリシーに従う） |

クライアント判定: `currentUser?.isAdmin ?? false`（super admin 含む）。  
RLS: `is_space_community_admin(space_id)` または super_admin JWT（`space_panes` と同型）。

19.2 将来拡張

将来的に、スペース設定で以下を選択できるようにする。

- 管理者のみ（MVP 既定）
- 投稿者は自分の投稿に関係する糸のみ操作可能
- 参加者全員
- 糸機能を使用しない

初期実装では権限設定 UI は追加しない。

20. データ仕様

20.1 テーブル定義

現行 schema に合わせた定義とする。

```sql
create table public.hossii_connections (
  id                 uuid primary key default gen_random_uuid(),
  space_id           text not null references public.spaces(id) on delete cascade,
  pane_id            text not null references public.space_panes(id) on delete cascade,
  source_hossii_id   text not null references public.hossiis(id) on delete cascade,
  target_hossii_id   text not null references public.hossiis(id) on delete cascade,
  strength           text not null check (strength in ('soft', 'medium', 'strong')),
  created_by         uuid references auth.users(id) on delete set null,
  created_at         timestamptz not null default now(),
  updated_at         timestamptz not null default now(),
  constraint hossii_connections_no_self check (source_hossii_id <> target_hossii_id),
  constraint hossii_connections_pair_unique unique (
    space_id,
    pane_id,
    source_hossii_id,
    target_hossii_id
  )
);
```

**DB 側で担保する制約**

| 制約 | 実装 |
|------|------|
| 自己接続禁止 | `CHECK (source_hossii_id <> target_hossii_id)` |
| A–B / B–A 重複禁止 | INSERT 前に ID を lexicographic sort して正規化 + `UNIQUE (space_id, pane_id, source, target)` |
| 同一 space | `space_id` FK + trigger で両 hossii の `space_id` 一致を検証 |
| 同一 pane | `pane_id` FK + trigger で両 hossii の `space_pane_id` が `pane_id` と一致を検証 |
| archived write 禁止 | `BEFORE INSERT/UPDATE/DELETE` trigger で `space_is_archived(space_id)` を拒否 |
| cascade delete | hossii / space / pane 削除時に FK `ON DELETE CASCADE` |
| `created_by` | trigger または RPC で `auth.uid()` を強制設定 |

20.2 データモデル（TypeScript）

```typescript
type HossiiConnectionStrength = 'soft' | 'medium' | 'strong'

type HossiiConnection = {
  id: string
  spaceId: string
  paneId: string
  sourceHossiiId: string
  targetHossiiId: string
  strength: HossiiConnectionStrength
  createdBy: string | null  // uuid
  createdAt: string
  updatedAt: string
}
```

20.3 アプリケーション制約

- 同じ 2 投稿間に複数の糸を作成不可
- `source` と `target` の順序に意味を持たせない（正規化して保存）
- 接続先 Hossii が削除・非表示の場合、取得結果から除外（FK cascade で DB からも削除）
- 異なるスペース / Pane の Hossii 同士はつなげない
- archived スペースでは作成・編集・解除不可
- 閲覧権限のない Hossii を含む糸は取得・描画しない

20.4 重複防止

A–B と B–A を同じ組み合わせとして扱う。

```typescript
const [sourceId, targetId] = [firstId, secondId].sort()
```

DB の unique 制約と INSERT trigger の両方で担保する。

21. RLS・セキュリティ

- **SELECT**: `can_access_space(space_id)`。必要に応じ両端 hossii の visibility / deleted を JOIN 検証
- **INSERT / UPDATE / DELETE**: `is_space_community_admin(space_id)` OR super_admin JWT
- archived スペース: write trigger で拒否（`assert_space_not_archived_for_write` 相当）
- クライアント側の権限制御だけに依存しない
- 異なる space / pane の Hossii ID を送信しても登録できない（trigger 検証）
- `created_by` は `auth.uid()` から自動設定（クライアント値を信頼しない）
- service role 以外への不要な direct grant を避ける

詳細な RLS は、既存 `space_panes` の実装パターンに合わせて設計する。

22. フロントエンド状態管理

**`HossiiStore`（global reducer）には connections データも connection 編集状態も入れない。**

| データ / 状態 | 置き場所 |
|---------------|----------|
| `connections[]` | `SpaceScreen` local state + `useHossiiConnections(spaceId, paneId)` hook |
| `connectionCountsByHossiiId` | 上記から `useMemo` で derived |
| `selectedBubbleId` | 既存 `SpaceScreen` state をそのまま利用 |
| `connectionEditPhase` | `SpaceScreen` local enum（`'idle' \| 'pickingTarget' \| 'pickingStrength' \| 'viewing'`） |
| `draftConnection` | local `{ sourceId, targetId?, strength? }` |
| `hoveredConnectionId` | local |
| `draggingConnectionHandle` | **ref 中心** + rAF（毎フレーム React setState 禁止） |

URL / hash には一時編集状態を載せない。  
糸 fetch 失敗時もスペース全体を loading にしない（§32）。

25. パフォーマンス

- 糸は表示中の Hossii に関係するものだけ描画する
- 画面外・非表示 Pane の糸は描画しない
- Hossii 移動中の描画更新は `requestAnimationFrame` で制御する
- React state を毎フレーム大量更新しない（§16, §22）
- 糸の本数が多い場合でも、投稿操作を妨げない
- SVG overlay は `bubbleArea` 内（§9.4）
- 大量表示で問題が出た場合は Canvas 化を検討する

26. PC版UI詳細

`connectionsEnabled === true` の環境で、選択中 Hossii の近傍にコンパクト操作メニューと強さ選択ポップオーバーを表示する。

操作の流れ
1. Hossii をクリック → 既存選択 chrome + コンパクト操作メニュー
2. 「つないでみる」を選択（管理者のみ）
3. ポインターに仮の糸が追従
4. 接続先 Hossii をクリック
5. 選択元または接続先の近くに強さ選択
6. 小さな「つなぐ」ボタンで確定

配置

- コンパクト操作メニュー: 選択 Bubble の近傍（portal 可）
- 強さ選択: 吹き出し直下 / 2 点中間 / 接続先近く
- 画面下部固定は不要（既存 `editToolbar` とは別）

画面外にはみ出す場合は、自動で上下左右を切り替える。

27. アクセシビリティ

- 色だけで糸の強さを区別しない
- 太さ・たるみ・透明度を併用する
- 糸と結び目に十分なクリック領域を設ける
- ボタンに明確なラベルを設定する
- エラーは `role="alert"` で通知する
- 動きを減らす設定が有効な場合、揺れやぷるんとしたアニメーションを抑える
- キーボード操作で、Hossii 選択・相手選択・保存・解除が可能な構成を目指す

30. 受入条件

30.1 通常表示
- `connectionsEnabled === false` では糸 UI が一切出ない
- 糸が常時すべて表示されない
- つながりがある Hossii だけに小さな印が表示される
- 印が本文や既存操作を妨げない
- 接続数が正しく表示される

30.2 選択表示
- Hossii 選択時、既存 chrome（ハンドル・パレット・owner bar・editToolbar）が維持される
- コンパクト操作メニューが表示される
- 直接接続の糸だけが表示される
- 背景クリック / Esc / Pane 切替で選択・connection 状態・メニューがすべて解除される

30.3 つながり一覧
- 「つながり N」で直接接続一覧を開ける
- 一覧から接続先を選択できる

30.4 引っ張り操作
- 結び目ドラッグで糸が伸び、接続先が CSS transform のみで揺れる
- pointer up 後に元へ戻る
- **`left`/`top` および DB 座標が変更されない**
- 配置保存 API が呼ばれない

30.5 操作開始
- 管理者の選択時に「つないでみる」が表示される
- 一般参加者には「くわしく見る」「つながり N」のみ
- 「つながりモード」見出しは表示しない

30.6 接続操作
- 「つないでみる」→ 仮糸 → 強さ → 保存
- 同一 Pane / `filteredHossiis` 内のみ
- キャンセル時 DB 未登録

30.7 作成・編集・解除
- community admin / super admin のみ CRUD 可能
- reload 後も維持。重複不可

30.8 権限・制約
- `bubbleEditPermission` と独立（参加者が Bubble 編集可でも糸 CRUD は不可）
- 異 space / 異 Pane / archived / 自己接続は不可
- DB trigger + RLS で担保

30.9 エラー
- 保存 / 更新 / 解除失敗時に UI と DB の不整合が残らない

30.10 非回帰
- 1 回目クリックで選択、5〜6px 超ドラッグで移動
- リサイズ・カラーパレット・owner bar・editToolbar・いいね・Pin が壊れない
- `layoutMode === 'byAuthor'` では糸 UI 非表示
- guest 直リンク・Pane 切替・星モード非回帰

32. 実装時の注意

- 最初から物理演算を導入しない
- 糸の描画と DB 保存を分けて実装する
- 1 つの PR に DB・描画・編集をすべて含めない
- **既存選択 UX（§11.1）を置き換えない**
- **`HossiiStore` に connections を入れない**（§22）
- 糸 fetch 失敗でスペース全体を loading にしない
- 引っ張り操作で配置保存 API を呼び出さない
- **Pane 切替時に選択・connection 状態・仮糸・メニューを全解除**
- Production migration は UI 実装と別工程

33. 仕様の中心となる一文

Hossii同士の、まだ言葉にならない「近さ」を、強さの異なる糸で仮につないでおける機能。

糸は、投稿を整理しきるためのものではない。
投稿を触り、動かし、つなぎ直す中で、まだ見えていなかった関係を発見するためのものである。

つながりは常に見せるのではなく、Hossiiを選び、糸を引き、ひとつずつ辿るときだけ、小さな星座として浮かび上がる。
