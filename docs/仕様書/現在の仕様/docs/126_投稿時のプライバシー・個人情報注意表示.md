## 1. 背景・目的

Hossii の投稿は、ワークショップ・授業・内省・地域活動など **参加者の言葉が残る場** である。
参加者が無意識に **氏名・住所・電話番号・メールアドレス・学籍番号・社員番号** などを書き込むと、本人や第三者のプライバシーが侵害されるリスクがある。

本仕様は次を目的とする。

| 目的 | 内容 |
|------|------|
| **予防** | 投稿前・投稿中に、やさしく PII を書かないよう案内する |
| **共通ルール** | エクスポート機能だけでなく、**すべての投稿導線** で一貫した注意を定義する |
| **UX** | 強すぎる警告・毎回のモーダル・投稿操作の妨害を避ける |
| **分離** | 管理者がエクスポート時に匿名配慮する説明（別機能）と **混同しない** |

---

## 2. 対象となる情報（案）

「個人情報」とは、本仕様上 **個人を特定しうる情報** を指す。厳密な法的定義の代替ではなく、**参加者向けの実用的な案内** とする。

| カテゴリ | 例 | 案内の強度 |
|----------|-----|------------|
| 氏名・フルネーム | 本名、他人のフルネーム | 書かないよう促す |
| 住所・位置 | 番地、マンション名、自宅近くの目印 | 書かないよう促す |
| 連絡先 | 電話、メール、SNS アカウント、LINE ID | 書かないよう促す |
| 識別番号 | 学籍番号、社員番号、マイナンバー | 書かないよう促す |
| 画像内の PII | 名札、書類、看板、送り状 | Phase 2 以降で検討（§8） |

**ニックネーム**（スペース内表示名）は参加フローで入力する想定であり、本注意の「書かないでね」の主対象ではない。ただし **本名の入力は避ける** 旨を初回ガイド等で補足してよい。

## 3. 基本文言

トーンは **Hossii らしいやわらかさ** を優先する。同じ趣旨でも **用途に応じて強さを変える**（混在させない）。

| 用途 | 推奨文言 | トーン | 例の配置 |
|------|----------|--------|----------|
| **通常表示**（常時・短文） | §3.1 やさしい表示 | 「**できるだけ** … しないでね」 — やわらかく、投稿のハードルを上げない | 投稿フォーム下部、GuestEntry 下部（任意） |
| **初回ガイド・詳細説明** | §3.2 明確な説明 | 「**書かないでください**」 — 初回理解のためやや明確に | `SpaceWelcomeGuide`、スペース注意事項 |

> **使い分けの原則:** 毎回見る場所（投稿フォーム）は §3.1。初回だけ読む場所（初回ガイド）は §3.2。**同一画面に両方を並べない**（強さのブレを避ける）。

### 3.1 やさしい表示（短い・日常向け / 通常表示）

**推奨デフォルト（Phase 1・投稿フォーム下部など）:**

```text
安心して使うために、名前・住所・連絡先などの個人情報は、できるだけ投稿しないでね。
```

### 3.2 より明確な説明（初回ガイド・詳細説明）

**推奨（初回参加・初回ガイド・スペース説明の補足）:**

```text
投稿には、氏名・住所・電話番号・メールアドレスなど、個人を特定できる情報を書かないでください。
```

### 3.3 補足（任意・Phase 1 では省略可）

```text
ニックネーム以外に、自分やほかの人を特定できる内容は書かないでね。
```

### 3.4 文言トーンの禁止事項

| 避ける | 理由 |
|--------|------|
| 「禁止」「違反します」などの強い命令 | 参加・投稿の心理的ハードルを上げる |
| 毎投稿前の確認モーダル | 操作を妨げ、離脱を招く |
| 赤い警告バナーの常時表示 | 不安を煽りすぎる |
| 長文の利用規約全文 | 読まれず、投稿導線から離れる |

## 5. 配置候補と推奨

### 5.1 配置候補一覧

| # | 配置 | 想定ユーザー | 文言の長さ | Phase |
|---|------|--------------|------------|-------|
| A | **投稿フォーム下部** | 投稿する全員 | 短い（§3.1） | **Phase 1 必須** |
| B | **初回ガイド（SpaceWelcomeGuide）** | 初回入室者 | 明確（§3.2）+ 任意で §3.3 | **Phase 1 推奨** |
| C | 初回参加（GuestEntry 下部） | 初参加ゲスト | 短い | Phase 1 任意 |
| D | スペース説明・注意事項 | 入室前後 | 管理者編集可なら中〜長 | Phase 2 |
| E | アカウントの「利用上の注意」 | ログイン利用者 | 中 | Phase 2 |
| F | 画像投稿 UI 付近 | 画像投稿者 | 短い + 画像向け一文 | Phase 2 |

### 5.2 Phase 1 推奨（最小実装）

| 原則 | 内容 |
|------|------|
| 必須 | **投稿フォーム下部** に §3.1 の短い注意文 |
| 推奨 | **初回ガイド** に §3.2 の詳細説明を 1 段落追加（[123](./123_URLからスペース参加UI.md) §10 と整合） |
| UX | 強すぎる警告にしない。投稿ボタンを disabled にしない |
| 頻度 | **毎回モーダルは出さない**。初回ガイドは既存の「わかった」1 回表示に含める |
| 任意 | GuestEntry 下部に 1 行（スペース説明と重複しない位置） |

### 5.3 表示イメージ（投稿フォーム下部・Phase 1）

```text
┌──────────────────────────────────────┐
│  [メッセージ入力・感情・画像…]         │
│                                      │
│  [投稿する]                          │
│                                      │
│  安心して使うために、名前・住所・       │
│  連絡先などの個人情報は、できるだけ   │
│  投稿しないでね。                     │  ← 補助テキスト（小さめ・くすみ赤）
└──────────────────────────────────────┘
```

### 5.4 色・視覚方針（Phase 1 確定）

| 項目 | 方針 |
|------|------|
| **採用色** | `#b84a4a`（muted red / くすみ赤の補助テキスト） |
| **適用範囲** | `PostPrivacyNotice` / `SpeechPanel.privacyNotice` / `SpaceWelcomeGuide.privacyNotice` の **3 画面で同一色** |
| **文言** | §3.1 / §3.2 の固定文言。管理者・利用者とも変更不可（Phase 1） |
| **背景・枠** | 付けない（注意文はテキスト色のみで強調） |
| **アイコン** | 付けない（Lock / 警告アイコン等は Phase 1 では使わない） |
| **モーダル化** | しない（§3.4 の禁止事項に従う） |
| **警告バナー化** | しない。赤い全面バナー・エラー toast 風 UI にしない |
| **エラー表示との関係** | エラー表示より **控えめ**。投稿失敗 toast 等の強い赤とは視覚的に区別する |
| **可読性** | 白背景・glass 背景（`SpaceWelcomeGuide` カード等）で読めること。通常テキストとして **WCAG AA**（4.5:1 以上）を満たす |

> **Phase 1 以前の案:** 補助テキストをグレー系（例: `#6b7280`）とする案は、注意の視認性不足のため **`#b84a4a` のくすみ赤** に更新した（PR #48）。

## 8. 検討事項（保留）

| # | 論点 | 現時点の方針案 |
|---|------|----------------|
| 1 | **全スペース共通**か | Phase 1 は **全スペース共通固定文言**。Phase 2 でスペース別上書きを検討 |
| 2 | **管理者が文言変更**可能か | Phase 1 は不可。Phase 2 で `spaces` の注意事項 or 専用フィールド |
| 3 | **子ども向けスペース**で強調するか | Phase 3。テンプレート `personal` / 用途タグ連動を検討 |
| 4 | **画像投稿**にも注意を出すか | Phase 2。§3.1 に加え「写真に名前や住所が写っていないか確認してね」等 |
| 5 | **外部公開スペース**で追加注意するか | Phase 2。public + 共有 URL スペースで §3.2 を初回ガイドに必ず含める案 |
| 6 | **通報・削除**導線との関係 | 本仕様は **予防**。削除・通報は別仕様。注意文に「書いた場合は管理者が削除することがある」とは Phase 1 では書かない |
| 7 | **法的表現**（個人情報保護法等） | 参加者向け実用案内に留め、法的助言は利用規約側 |
| 8 | **多言語** | 現行は日本語のみ。i18n は将来 |
